# 02 - 安装配置

## 环境要求

- Node.js >= 18
- npm / pnpm / yarn / bun 之一
- Claude Code CLI v2.1.0+ 已安装

## 安装方式

### 方式一：插件安装（推荐）

#### 1. 添加插件市场

```bash
/plugin marketplace add https://github.com/affaan-m/ECC
```

#### 2. 安装插件

```bash
/plugin install ecc@ecc
```

#### 3. 安装 Rules（插件不会自动安装）

```bash
# 克隆仓库
git clone https://github.com/affaan-m/ECC.git
cd ECC

# 复制规则到用户目录
mkdir -p ~/.claude/rules/ecc
cp -r rules/common ~/.claude/rules/ecc/
# 选择你的技术栈
cp -r rules/typescript ~/.claude/rules/ecc/   # 或 python, golang, swift, php
```

### 方式二：手动安装

如果你想更精细地控制安装：

```bash
# 克隆仓库
git clone https://github.com/affaan-m/ECC.git
cd ECC

# 安装依赖
npm install        # 或 pnpm install | yarn install | bun install

# 安装 agents
cp agents/*.md ~/.claude/agents/

# 安装 skills（主要工作流表面）
mkdir -p ~/.claude/skills/ecc
cp -r .agents/skills/* ~/.claude/skills/ecc/

# 安装 rules
mkdir -p ~/.claude/rules/ecc
cp -r rules/common ~/.claude/rules/ecc/
cp -r rules/typescript ~/.claude/rules/ecc/

# 安装 commands（可选，ECC 正在迁移到 skills）
mkdir -p ~/.claude/commands
cp commands/*.md ~/.claude/commands/
```

## 验证安装

```bash
# 检查插件是否安装成功
/plugin list ecc@ecc

# 检查可用命令
/ecc:plan
/ecc:code-review
```

## 基础配置

### 设置包管理器

ECC 自动检测你喜欢的包管理器，也可以手动设置：

```bash
# 环境变量方式
export CLAUDE_PACKAGE_MANAGER=pnpm

# 或使用命令
/setup-pm
```

优先级：环境变量 > 项目配置 > package.json > lock 文件 > 全局配置

### Hook 运行时控制

```bash
# Hook 严格程度（默认: standard）
export ECC_HOOK_PROFILE=standard

# 禁用特定 Hooks
export ECC_DISABLED_HOOKS="pre:bash:tmux-reminder,post:edit:typecheck"

# SessionStart 上下文上限（默认: 8000 字符）
export ECC_SESSION_START_MAX_CHARS=4000
```

### 模型设置

推荐在 `~/.claude/settings.json` 中设置：

```json
{
  "model": "sonnet",
  "env": {
    "MAX_THINKING_TOKENS": "10000",
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "50"
  }
}
```
