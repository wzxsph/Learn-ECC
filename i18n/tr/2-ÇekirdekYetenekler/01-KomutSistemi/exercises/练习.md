# Alistirma: Komut Sistemi Ustaligi

## Alistirma 1: Tum Komut Siniflandirmalarini Keşfet

### Hedef
Claude Code'un tum komut siniflandirmalarini ogrenin ve her siniflandirmadaki ana komutlari tanimlayin.

### Adimlar

1. **Tum Kullanilabilir Komutlari Listele**
   - Claude Code'a `/` girerek tum komutlari goruntuleyin
   - Komutlarin siniflandirma bicimine dikkat edin

2. **Her Siniflandirmayi Keşfet**
   - Her siniflandirma icin `/help` veya `?` ile yardim almayi deneyin
   - Her siniflandirmanin temel komutlarini ve kullanim amaclarini kaydedin

3. **Komut Listesi Olustur**
   Kendi komut hizli rehberinizi olusturun:

   | Siniflandirma | Komut | Amac | Kullanim Sikligi |
   |------|------|------|----------|
   | Gelistirme akisi | /plan | Uygulama plani olustur | ★★★ |
   | ... | ... | ... | ... |

### Kabul Kriterleri
- [ ] En az 5 siniflandirmanin komutlarini listeleyebilme
- [ ] 20+ sik kullanilan komutu kaydetme
- [ ] Komutlarin siniflandirma mantigini anlama

## Alistirma 2: Birden Fazla Komutu Birlikte Kullanma

### Hedef
Birden fazla komutu birlikte kullanarak tutarli gelistirme is akislari olusturmayi ogrenin.

### Senaryo
Bir Go projesinde build hatasi oldugunu ve kod incelemesi yapilmasi gerektigini farz edin.

### Adimlar

1. **Senaryoyu Simule Et**
   Bir Go projesinde build hatasi oldugunu farz edin:

   ```
   # Senaryo kurulumu (gercek calistirma gerektirmez)
   Tamir edilmesi gereken su build hatalari oldugunu farz edin:
   - import cycle detected
   - undefined: someFunction
   ```

2. **Komut Kombinasyonu Alistirmasi**
   Sirayla su komutlari dusunun ve kaydedin:

   a. Oncelikle hatayi analiz etmek icin `/plan` kullanin
   b. Build etmeye calismak icin `/go-build` kullanin
   c. Hatayi tamir etmek icin `/build-fix` kullanin
   d. Inceleme yapmak icin `/code-review` kullanin
   e. Tamirleri dogrulamak icin `/verify` kullanin

3. **Is Akisini Kaydet**
   Tamir is akisi dokumaninizi olusturun:

   ```markdown
   ## Build Tamir Is Akisi

   1. /plan - Sorunu analiz et, tamir plani olustur
   2. /build-fix - Tamiri calistir
   3. /code-review - Tamir kalitesini kontrol et
   4. /verify - Islevselligin dogru oldugunu dogrula
   ```

### Kabul Kriterleri
- [ ] Komut kombinasyonunun mantigini anlama
- [ ] En az 3 ozel is akisi olusturma
- [ ] Komut kombinasyonunun en iyi uygulamalarini kaydetme

## Alistirma 3: Ozel Is Akisi Olusturma

### Hedef
Proje gereksinimlerine dayali otomasyon is akisi komut kombinasyonlari olusturun.

### Senaryo
Bir Flutter projesi icin gelistirme is akisi olusturun.

### Adimlar

1. **Is Akisi Hedeflerini Tanimla**
   Flutter projeniz icin su is akislarina ihtiyaciniz var:

   - Gunluk gelistirme: Kodlama → Test → Build
   - Kod inceleme: Oz-inceleme → Inceleme → Tamir
   - Yayin hazirlama: Test → Build → Dogrulama

2. **Komut Kombinasyonlari Tasarimi**

   **Gunluk Gelistirme Is Akisi**:
   ```
   /flutter-build → /flutter-test → /verify
   ```

   **Kod Inceleme Is Akisi**:
   ```
   /flutter-review → /build-fix → /code-review
   ```

3. **Is Akisi Dokumani Olustur**
   Projede `CLAUDE_WORKFLOWS.md` olusturun:

   ```markdown
   # Claude Code Is Akislari

   ## Flutter Gelistirme Is Akislari

   ### Gunluk Gelistirme
   1. Gorevi planlamak icin `/plan` kullanin
   2. Build etmek icin `/flutter-build` kullanin
   3. Test etmek icin `/flutter-test` kullanin
   4. Dogrulamak icin `/verify` kullanin

   ### Kod Inceleme
   1. Oz-inceleme icin `/flutter-review` kullanin
   2. Sorunlari tamir etmek icin `/build-fix` kullanin
   3. Inceleme istemek icin `/code-review` kullanin

   ### Yayin Hazirlama
   1. Tam test paketini calistirin
   2. Build dogrulamasini calistirin
   3. Son onay icin `/verify` kullanin
   ```

4. **Pratik Dogrulamasi**
   Eger gercek bir projeniz varsa, bu is akisini calistirmay deneyin.

### Kabul Kriterleri
- [ ] 3 tam is akisi tasarladi
- [ ] Is akisi dokumani olusturdu
- [ ] En az 1 is akisini gercek projede test etti

## Alistirma Cevaplari Referansi

### Alistirma 1 Komut Listesi Ornegi

| Siniflandirma | Komut | Amac |
|------|------|------|
| Gelistirme akisi | /plan | Uygulama planlamasi |
| Gelistirme akisi | /tdd | Test-driven gelistirme |
| Gelistirme akisi | /build-fix | Build tamiri |
| Gelistirme akisi | /verify | Degisiklik dogrulama |
| Kod inceleme | /code-review | Genel kod inceleme |
| Kod inceleme | /security-review | Guvenlik inceleme |
| Build | /go-build | Go Build |
| Build | /rust-build | Rust Build |
| Proje | /project-init | Proje baslatma |
| Proje | /projects | Proje listesi |
| Beceri | /skill-create | Beceri olusturma |
| Beceri | /learn | Ogrenme cikarimi |
| Ogrenme | /deep-research | Derin arastirma |

### Alistirma 3 Is Akisi Sablonu

```markdown
# [Proje Adi] Claude Code Is Akislari

## Gelistirme Akislari
- Gunluk: /plan → [Kodlama] → /verify
- Tamir: /build-fix → /code-review → /verify
- Yayin: /test → /build → /verify

## Kullanim Talimati
1. Her gorev icin `/plan` ile baslayin
2. Kodlama bittiginde `/verify` calistirin
3. Onemli degisiklikler icin `/code-review` kullanin
```
