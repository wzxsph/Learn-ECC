# 01 Komut Sistemi Ustaligi

ECC'nin komut sistemini derinlemesine ogrenin, tum komut turlerini ve kullanim senaryolarini kavrayin.

## Genel Bakis

ECC **75 [komut](../../Referans-belgeleri/commands/01-Cekirdek-Is-Akis.md)** saglar, uce ayrilir: satir komutlari (Slash Commands), script komutlari ve Agent komutlari. Bu komutlar standartlastirilmis is akislariyla gelistirme verimliligini onemli olcude artirir.

## Komut Dosya Formati

Her komut YAML frontmatter + Markdown govde iceren bir `.md` dosyasidir:

```yaml
---
description: "Komutun kisa aciklamasi"
argument-hint: "[Istege bagli parametre ipucu]"
name: command-name          # Istege bagli, dosya adini kullanir
command: true              # Istege bagli, CLI pass-through komutu
allowed_tools: ["Bash"]    # Istege bagli, kullanilabilir araclari sinirla
---
```

### Frontmatter Alan Aciklamalari

| Alan | Zorunlu | Aciklama |
|------|--------|----------|
| description | Evet | / komut listesinde gosterilir |
| argument-hint | Hayir | Parametre ipucu |
| name | Hayir | Komut adi olarak dosya adini ezert |
| command | Hayir | CLI pass-through komut olarak isaretle |
| allowed_tools | Hayir | Kullanilabilir araclari sinirla |

## 2. Komut Zincirleme ve Kombinasyon

Birden fazla komut birlestirilerek karmaşik is akislari olusturulabilir:

```markdown
# Tipik gelistirme akisi
/plan → /tdd → /code-review → /verify → /pr

# Sorun duzeltme akisi
/build-fix → /code-review → /test-coverage

# Cogul model isbirligi
/multi-plan → /multi-execute → /pr
```

## 3. Ozel Komut Olusturma

### Komut Saklama Konumlari

| Tur | Yol |
|------|------|
| Komut dosyasi | `commands/` |
| Kayit mekanizmasi | `.claude-plugin/plugin.json` |

### Olusturma Adimlari

1. `commands/` dizininde bir `.md` dosyasi olustur
2. YAML frontmatter ekle
3. Komut icerigini yaz (Markdown formati)
4. Komut ECC'ye otomatik olarak kaydolur

---

## 4. En Iyi Uygulamalar

### Komut Secim Kilavuzu

| Senaryo | Tavsiye Edilen Komutlar |
|------|----------|
| Yeni ozellik gelistirme | `/plan` → `/tdd` |
| Kod inceleme | `/code-review` |
| Bug duzeltme | `/build-fix` → `/verify` |
| Test-driven gelistirme | `/tdd` → `/test-coverage` |
| PR olusturma | `/pr` |
| Kalip ogrenme | `/learn` → `/evolve` |

### Komut Calistirma Siralamasi

Is akisi ilkelerine uyun:
1. Planlama: `/plan` veya `/prp-plan`
2. Gerceklestirme: `/tdd` veya ozellik komutlari
3. Inceleme: `/code-review`
4. Dogrulama: `/verify` veya `/quality-gate`
5. Commit: `/pr`
