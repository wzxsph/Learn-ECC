# Alıştırma：Ajans İşbirliği

## Alıştırma 1: 使用Kod İnceleme Agent

### Hedef
学习调用Kod İnceleme Agent，并理解其反馈机制。

### Senaryo
假设你编写了以下 Python 代码（不Gerekli实际执行，只是在脑海中）：

```python
def get_user_data(user_id, include_password=False):
    user = db.query("SELECT * FROM users WHERE id = " + user_id)
    if include_password:
        return user
    return {"name": user.name, "email": user.email}
```

### Adımlar

1. **Bu Kodu Analiz Et**
   思考Olası Sorunlar：
   - SQL Enjeksiyonu漏洞
   - 密码泄露风险
   - Hata Yönetimi缺失

2. **Uygun Ajanı Seç**
   对于这段代码，Olmalı使用：
   - `security-reviewer` - 检查Güvenlik Açıkları
   - `python-reviewer` - 检查 Python En İyi Uygulamalar

3. **模拟 Ajan 反馈**
   记录你认为 Ajan 会给出的反馈：

   ```markdown
   ## Kod İnceleme反馈

   ### Güvenlik问题 (CRITICAL)
   - SQL Enjeksiyonu：直接拼接 user_id 到 SQL 语句
   - Şifre Sızıntısı: include_password Seçeneği Şifreyi Açığa Çıkarabilir

   ### Kalite Sorunu (YÜKSEK)
   - 缺少Hata Yönetimi
   - 缺少输入Doğrulama

   ### İyileştirme Önerileri
   - Parametreli Sorgu Kullan
   - 不要Geri Dön密码字段
   - İstisna İşleme Ekle
   ```

4. **Onarım Planı Oluştur**
   使用 `/plan` 的思路，Onarım Planı Oluştur：

   ```markdown
   # Onarım Planı

   1. 修复 SQL Enjeksiyonu：Parametreli Sorgu Kullan
   2. 移除密码Geri Dön：永远不要Geri Dön密码
   3. 添加输入Doğrulama：Doğrulama user_id 格式
   4. 添加Hata Yönetimi：Kullanıcı Yok Durumunu İşle
   ```

### Kabul Kriterleri
- [ ] 能识别代码中的Güvenlik问题
- [ ] 理解审查 Ajan 的反馈结构
- [ ] 能Onarım Planı Oluştur

## Alıştırma 2: 使用Yapı Onarımı Agent

### Hedef
Nasıl使用构建 Ajan 诊断和Yapı Hatalarını Onarma。

### Senaryo
假设 Go 项目有以下构建错误：

```
# github.com/myproject/pkg
./main.go:15: tanımsız: " fmt"
./main.go:23: tanımsız: calculateTotal
./main.go:31: too many arguments to function call
```

### Adımlar

1. **分析错误Tür**
   将错误分类：

   | 错误 | Tür | 严重程度 |
   |------|------|----------|
   | tanımsız: " fmt" | İçe Aktarma Hatası | HIGH |
   | tanımsız: calculateTotal | Fonksiyon Tanımsız | HIGH |
   | too many arguments | Parametre Hatası | MEDIUM |

2. **Tasarım修复流程**
   使用 Ajans İşbirliği：

   ```markdown
   # Yapı Onarımı流程

   1. /go-build - Tam Hata Almak İçin Yapılandırmayı Çalıştır
   2. /build-fix - 调用Yapı Onarımı Agent
   3. Hata Önceliğini Analiz Et：Yüksek Öncelikliler Önce Onarılır
   4. 逐个修复并Doğrulama
   5. /verify - Yapılandırmanın Başarılı Olduğunu Doğrula
   ```

3. **为每个错误Tasarım修复方案**

   **错误 1: tanımsız: " fmt"**
   ```go
   // 问题：Fazladan Tırnak İşareti
   import "fmt"  // Doğru Yazım
   
   // 修复：Tırnak İşaretini Kaldır
   ```

   **错误 2: tanımsız: calculateTotal**
   ```go
   // 问题：Fonksiyon Tanımsız或命名错误
   // 修复：Fonksiyon Tanımını Kontrol Et，可能Gerekli添加或重命名
   ```

   **错误 3: too many arguments**
   ```go
   // 问题：Fonksiyon Çağrısı Parametre Sayısı Uyuşmuyor
   // 修复：检查Fonksiyon İmzası，Çağrı Parametrelerini Düzelt
   ```

4. **记录修复经验**
   创建错误模式总结：

   ```markdown
   # Go Yapı Hata Kalıpları

   ## Yaygın Hatalar
   1. tanımsız: İçe Aktarma/Fonksiyon Sorunu
   2. too many/few arguments: Parametre Uyuşmazlığı
   3. tanımsız: " xxx": İçe Aktarma Sözdizimi Hatası

   ## Onarım Stratejisi
   - Önce Hata Sayısına Bak，Sonra Hata Konumuna Bak
   - Yüksek Öncelikli Hatalar Düşük Öncelikli Hatalara Neden Olabilir
   - 修复后重新构建Doğrulama
   ```

### Kabul Kriterleri
- [ ] 能分析构建错误Tür和优先级
- [ ] Tasarım了完整的修复流程
- [ ]  yaygın Hata Kalıplarını Belgeleme

## Alıştırma 3: Ajan 组合工作流

### Hedef
Tasarım一个复杂Görev，使用多个 Ajans İşbirliğiBitir。

### Senaryo
开发一个新的用户认证模块，包含以下功能：
- Kullanıcı Kayıt（含Şifre GüçlülüğüDoğrulama）
- Kullanıcı Girişi（Brute Force Koruması ile）
- JWT token 生成和Doğrulama

### Adımlar

1. **Görev Ayrıştırma**
   将Görev Ayrıştırma为子Görev：

   ```markdown
   # Kimlik Doğrulama Modülü Geliştirme Görevi

   ## 子Görev
   1. 数据库Tasarım (Schema)
   2. Kullanıcı Kayıt API
   3. Kullanıcı Girişi API
   4. JWT Token İşleme
   5. Güvenlik机制（Şifre Güçlülüğü、Brute Force Koruması）
   ```

2. **Ajan Ataması**
   Her Alt Göreve Ajan Ata：

   | 子Görev | Ajan | Gerekçe |
   |--------|-------|------|
   | 数据库Tasarım | architect | Gerekli架构视角 |
   | 注册 API | tdd-guide + security-reviewer | TDD + Güvenlik |
   | 登录 API | tdd-guide + security-reviewer | TDD + Güvenlik |
   | JWT 处理 | security-reviewer | Güvenlik敏感 |
   | Güvenlik机制 | security-reviewer (优先) | Güvenlik优先 |

3. **Tasarım工作流**

   ```
   [Başla]
      ↓
   /plan (规划Genel)
      ↓
   ┌─────────────────────────────────────┐
   │  Paralel Yürütme:                            │
   │  - architect: 数据库Tasarım               │
   │  - tdd-guide: TasarımTest策略            │
   └─────────────────────────────────────┘
      ↓
   [Uygulama Aşaması]
      ↓
   ┌─────────────────────────────────────┐
   │  Seri + Paralel Karışık:                      │
   │  1. tdd-guide: 编写注册Test            │
   │  2. Gerçekleştir注册功能                       │
   │  3. security-reviewer: Güvenlik审查        │
   │  ──────────────────────────────────   │
   │  1. tdd-guide: 编写登录Test            │
   │  2. Gerçekleştir登录功能                       │
   │  3. security-reviewer: Güvenlik审查        │
   └─────────────────────────────────────┘
      ↓
   [Entegrasyon Aşaması]
      ↓
   ┌─────────────────────────────────────┐
   │  - architect: JWT 架构Tasarım            │
   │  - security-reviewer: GenelGüvenlik审查     │
   │  - code-reviewer: Kod Kalitesi审查        │
   │  - /verify: GenelDoğrulama                  │
   └─────────────────────────────────────┘
      ↓
   [Bitir]
   ```

4. **İş Akışı Belgesi Yaz**

   ```markdown
   # Kullanıcı Kimlik Doğrulama Modülü Geliştirme İş Akışı

   ## Aşama 1: Planlama
   - 使用 /plan 制定Genel Plan
   - 使用 architect TasarımVeritabanı ve JWT Mimari

   ## Aşama 2: TDD Geliştirme
   - 使用 tdd-guide 指导Test编写
   - 先写Test，再Gerçekleştir功能
   - 每个子功能单独Doğrulama

   ## 阶段 3: Güvenlik审查
   - 使用 security-reviewer Her API'yi İncele
   - Özellikle Dikkat：Şifre İşleme、Token Güvenlik、Enjeksiyon Koruması

   ## 阶段 4: 整合Doğrulama
   - 使用 code-reviewer Genel审查
   - 使用 /verify Doğrulama功能完整性

   ## Ajans İşbirliği总结
   - architect: 架构Tasarım (1次)
   - tdd-guide: Test指导 (2次)
   - security-reviewer: Güvenlik审查 (3次)
   - code-reviewer: Kalite İncelemesi (1次)
   ```

### Kabul Kriterleri
- [ ] Görev Ayrıştırma合理，包含 5+ 个子Görev
- [ ] Ajan 分配策略明确
- [ ] 工作流Tasarım清晰
- [ ] Belgelenmiş İş Akışı Yeniden Kullanımı Kolaylaştırır
