# Alıştırma：Hooks Özelleştirme

## Alıştırma 1: 创建一个 PreToolUse Hook

### Hedef
创建第一个 PreToolUse Hook，Gerçekleştir危险命令Uyarı。

### Senaryo
当用户尝试执行危险命令（如 `rm -rf`）时，显示Uyarıİpucu。

### Adımlar

1. **Tasarım Hook 配置**

   ```json
   {
     "name": "dangerous-command-warning",
     "matcher": {
       "tool": "Bash"
     },
     "condition": "command.includes('rm -rf') || command.includes('drop table')",
     "hooks": [
       {
         "type": "notification",
         "when": "before",
         "message": "Uyarı: 检测到危险命令！请确认操作Güvenlik。",
         "continue": true
       }
     ]
   }
   ```

2. **创建 Hook 文件**
   在项目中创建 `hooks/dangerous-warning.json`

3. **Test Hook**
   TasarımTestSenaryo：

   | 命令 | 预期行为 |
   |------|----------|
   | `rm -rf /tmp/test` | 显示Uyarı |
   | `rm -rf ./project` | 显示Uyarı |
   | `ls -la` | 无Uyarı |
   | `git commit` | 无Uyarı |

4. **扩展 Hook 功能**
   尝试添加更多危险命令检测：

   ```json
   "condition": "command.includes('rm -rf') || 
                 command.includes('drop table') || 
                 command.includes('truncate') ||
                 command.includes('shutdown')"
   ```

5. **添加白名单**
   某些情况下Gerekli允许危险命令，添加白名单机制：

   ```json
   "condition": "dangerousCommand && !whitelisted"
   ```

### Kabul Kriterleri
- [ ] 创建了 PreToolUse Hook 配置文件
- [ ] 定义了危险命令条件
- [ ] Tasarım了TestSenaryo
- [ ] 添加了至少 5 种危险命令检测

## Alıştırma 2: 创建一个 PostToolUse Hook

### Hedef
创建 PostToolUse Hook，GerçekleştirTestBitir后的自动通知。

### Senaryo
当Test执行Bitir后，自动检查Test结果并发送通知。

### Adımlar

1. **Tasarım Hook 配置**

   ```json
   {
     "name": "test-result-notifier",
     "matcher": {
       "tool": "Bash"
     },
     "condition": "command.includes('npm test') || command.includes('pytest')",
     "hooks": [
       {
         "type": "notification",
         "when": "after",
         "message": "Test执行Bitir",
         "continue": true
       }
     ]
   }
   ```

2. **增强功能：解析Test结果**
   创建一个 JavaScript 处理脚本 `scripts/parse-test-result.js`:

   ```javascript
   module.exports = function parseTestResult(output) {
     // 解析Test输出
     const passed = output.match(/(\d+) passed/);
     const failed = output.match(/(\d+) failed/);
     
     return {
       passed: passed ? parseInt(passed[1]) : 0,
       failed: failed ? parseInt(failed[1]) : 0,
       status: failed ? 'FAILED' : 'PASSED'
     };
   };
   ```

3. **创建完整的 Hook 配置**
   结合脚本创建完整配置：

   ```json
   {
     "name": "test-result-checker",
     "matcher": {
       "tool": "Bash"
     },
     "condition": "command.includes('npm test')",
     "hooks": [
       {
         "type": "command",
         "when": "after",
         "command": "node scripts/parse-test-result.js",
         "continue": true
       }
     ]
   }
   ```

4. **Tasarım多种TestÇerçeveler适配**
   Destekle不同的TestÇerçeveler：

   | Çerçeveler | 输出关键词 |
   |------|----------|
   | Jest | `Tests:` |
   | Pytest | `passed` |
   | Go test | `ok` |
   | RSpec | `examples` |

5. **添加持续集成Senaryo**
   Tasarım CI 环境下的行为：

   ```json
   {
     "condition": "isCI && testCommand",
     "hooks": [
       {
         "type": "command",
         "when": "after",
         "command": "node scripts/ci-reporter.js",
         "continue": true
       }
     ]
   }
   ```

### Kabul Kriterleri
- [ ] 创建了 PostToolUse Hook
- [ ] Gerçekleştir了Test结果解析
- [ ] Destekle多种TestÇerçeveler
- [ ] Tasarım了 CI Senaryo处理

## Alıştırma 3: 配置质量门禁 Hook

### Hedef
创建一个完整的质量门禁系统，在代码提交前进行多维度检查。

### Senaryo
在 `git commit` 前自动运行一系列质量检查，确保Kod Kalitesi。

### Adımlar

1. **定义质量门禁检查项**

   | 检查项 | 检查内容 | 失败行为 |
   |--------|----------|----------|
   | Test覆盖率 | >= 80% | 阻止提交 |
   | Lint 检查 | 无错误 | 阻止提交 |
   | Güvenlik扫描 | 无漏洞 | Uyarı+阻止 |
   | 格式检查 | 符合规范 | 自动修复 |

2. **Tasarım Hook Yürütme Akışı**

   ```
   git commit
      ↓
   [PreToolUse Hook]
      ↓
   运行质量检查
      ├── Test覆盖率检查
      ├── Lint 检查  
      ├── Güvenlik扫描
      └── 格式检查
      ↓
   [结果判断]
      ↓
   ├── 全部Vasıtasıyla → 允许提交
   ├── 有Uyarı → 显示Uyarı，继续提交
   └── 有错误 → 阻止提交
   ```

3. **创建 Hook 配置文件**

   ```json
   {
     "name": "quality-gate",
     "matcher": {
       "tool": "Bash"
     },
     "condition": "command.includes('git commit')",
     "hooks": [
       {
         "type": "command",
         "when": "before",
         "command": "npm test && npm run lint && npm run security-scan",
         "continue": false
       }
     ]
   }
   ```

4. **创建检查脚本**

   `scripts/quality-gate.js`:

   ```javascript
   const { execSync } = require('child_process');
   
   function runChecks() {
     const checks = [
       { name: 'Test', command: 'npm test', required: true },
       { name: 'Lint', command: 'npm run lint', required: true },
       { name: '覆盖率', command: 'npm run coverage', required: true },
       { name: 'Güvenlik', command: 'npm run security-scan', required: false }
     ];
   
     let allPassed = true;
     for (const check of checks) {
       try {
         execSync(check.command, { stdio: 'inherit' });
         console.log(`✓ ${check.name} Vasıtasıyla`);
       } catch (error) {
         console.error(`✗ ${check.name} 失败`);
         if (check.required) allPassed = false;
       }
     }
     return allPassed;
   }
   
   process.exit(runChecks() ? 0 : 1);
   ```

5. **添加渐进式质量门禁**

   ```json
   {
     "name": "progressive-quality-gate",
     "stages": [
       { "name": "快速检查", "commands": ["npm run lint"], "blockOnFail": true },
       { "name": "Test", "commands": ["npm test"], "blockOnFail": true },
       { "name": "Güvenlik扫描", "commands": ["npm run security-scan"], "blockOnFail": false },
       { "name": "覆盖率", "commands": ["npm run coverage"], "blockOnFail": true }
     ]
   }
   ```

6. **配置跳过机制**

   ```json
   {
     "condition": "command.includes('git commit') && !command.includes('--no-verify')",
     "message": "质量门禁已启用。使用 --no-verify 跳过检查（不推荐）。"
   }
   ```

### Kabul Kriterleri
- [ ] 定义了完整检查项清单
- [ ] 创建了 Hook 配置文件
- [ ] Gerçekleştir了检查脚本
- [ ] Destekle跳过机制
- [ ] 文档化了使用方式
