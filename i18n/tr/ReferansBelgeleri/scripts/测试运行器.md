# Test Çalıştırıcıları

本文档介绍 ECC 项目的Test结构和运行方式。

## Test入口

**路径**: `tests/run-all.js`

集中式Test Çalıştırıcıları,自动发现并运行所有Test文件。

### Kullanım Yöntemi

```bash
# 运行所有Test
node tests/run-all.js

# 运行单个Test文件
node tests/lib/utils.test.js
node tests/hooks/hooks.test.js
```

### Test文件发现

Test ÇalıştırıcılarıVasıtasıyla glob 模式 `tests/**/*.test.js` 自动发现Test文件。

规则:
- 文件Zorunlu位于 `tests/` Dizin下
- 文件名Zorunlu以 `.test.js` 结尾
- Dizin结构会被保留Kullanılan显示

Test Çalıştırıcıları会解析每个Test文件的输出来汇总统计:
- `Passed: <数量>`

## Test结构

### Dizin布局

```
tests/
├── run-all.js              # 主Test Çalıştırıcıları
├── lib/                    # 库工具Test
│   ├── utils.test.js
│   ├── package-manager.test.js
│   └── ...
├── hooks/                  # Hook 集成Test
│   └── hooks.test.js
├── integration/            # 集成Test
├── scripts/                # 脚本Test
├── commands/               # 命令Test
├── docs/                   # 文档Test
├── opencode-config.test.js
├── plugin-manifest.test.js
├── test_*.py               # Python Test
└── conftest.py             # pytest 配置
```

### Test型

1. **单元Test** (`*.test.js`)
   - Test独立函数和工具
   - 使用 Node.js 原生 assert 或 Jest 风格

2. **集成Test** (`tests/integration/`)
   - Test多个组件协作
   - Test钩子完整流程

3. **Python Test** (`test_*.py`)
   - pytest 格式的 Python Test
   - 使用 `conftest.py` 配置

## Test配置

### Node.js Test

Test文件使用标准的 Node.js assert 模块:

```javascript
const assert = require('assert');

test('example test', () => {
  assert.strictEqual(actual, expected, 'Optional message');
});
```

### Python Test (pytest)

位置: `tests/conftest.py`

pytest 配置文件,Sağla共享的 fixtures 和配置。

```bash
# 运行 Python Test
python -m pytest tests/

# 运行特定Test
python -m pytest tests/test_builder.py
```

## TestEn İyi Uygulamalar

### AAA 模式

```
test('TestTanım', () => {
  // Arrange - 准备Test数据
  const input = 'test data';

  // Act - 执行被测操作
  const result = myFunction(input);

  // Assert - Doğrulama结果
  assert.strictEqual(result, expected);
});
```

### Test命名

使用Tanım性Ad,清晰AçıklamaTest的行为:

```javascript
test('returns empty array when no markets match query', () => {});
test('throws error when API key is missing', () => {});
test('falls back to substring search when Redis is unavailable', () => {});
```

### Test隔离

每个TestOlmalı:
- 独立运行,不依赖其他Test
- 清理自己的Test数据
- 使用 mock 隔离外部依赖

## CI 集成

在 `package.json` 中配置Test脚本:

```json
{
  "scripts": {
    "test": "validate-commands.js && tests/run-all.js"
  }
}
```

Test ÇalıştırıcılarıTasarım为可在 CI 环境中工作:
- 非零退出码表示失败
- JSON 输出Kullanılan自动化解析
- 清晰的错误消息Kullanılan调试
