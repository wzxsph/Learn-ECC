# Planlama ve Mimari命令

## Genel Bakış

Planlama ve Mimari命令Kullanılan产品规划、功能Tasarım和系统架构决策。这些命令确保在写代码之前有清晰的计划，减少返工和提高Kod Kalitesi。

## 命令列表

### /plan

**Kullanım**: 通用Uygulama Planlaması - 复述需求、评估风险、创建分Adımlar实施计划

**Tanım**: 在触碰任何代码之前，等待用户确认后创建全面的实施计划。接受自由形式需求或 PRD markdown 文件。

**输入模式**:

| 输入 | 模式 | 行为 |
|---|---|---|
| `path/to/name.prd.md` | PRD artifact 模式 | 读取 PRD，选择下一个待交付里程碑，生成 `.claude/plans/{name}.plan.md` |
| 其他 markdown 路径 | 引用模式 | 读取文件作为上下文，产生内联计划 |
| 自由形式文本 | 对话模式 | 产生内联计划 |
| 空输入 | 澄清模式 | 询问要规划什么 |

**关键参数**:
- `$ARGUMENTS`: `[feature description | path/to/*.prd.md]` - 功能Tanım或 PRD 文件路径

**工作流**:
1. **复述需求** - 澄清Gerekli构建什么
2. **识别风险** - 列出潜在问题和阻碍
3. **创建Adımlar计划** - 分解为阶段性实施
4. **等待确认** - Zorunlu收到用户批准后才能继续

**输出格式** (PRD artifact 模式):
```markdown
# Plan: {Feature Name}

**Source PRD**: {path}
**Selected Milestone**: {milestone or phase name}
**Complexity**: {Small | Medium | Large}

## Summary
{2-3 sentences}

## Patterns to Mirror
| Category | Source | Pattern |
|---|---|---|
| Naming | `path:line` | {short description} |
| Errors | `path:line` | {short description} |
| Tests | `path:line` | {short description} |

## Files to Change
| File | Action | Why |
|---|---|---|
| `path` | CREATE / UPDATE / DELETE | {reason} |

## Tasks
### Task 1: {name}
- **Action**: {what to do}
- **Mirror**: {pattern to follow}
- **Validate**: {command that proves correctness}

## Validation
```bash
{project-specific validation commands}
```

## Risks
| Risk | Likelihood | Mitigation |
|---|---|---|
```

**En İyi Uygulamalar**:
- 使用前先研究代码库中的现有模式
- 在 PRD artifact 模式中创建 `.claude/plans/` Dizin（如不存在）
- 如果 PRD 包含 `Delivery Milestones` 表格，只更新所选行状态

**常见陷阱**:
- 空输入时不询问澄清就尝试规划
- 不等待用户确认就Başla写代码
- 跳过 Pattern Grounding Adımlar

**与Diğer Komutlar集成**:
- 规划后使用 `tdd-workflow` skill 进行Test Güdümlü Geliştirme
- 使用 `/build-fix` Yapı Hatalarını Onarma
- 使用 `/code-review` 审查BitirGerçekleştir
- 使用 `/pr` 或 `/prp-pr` 打开 Pull Request

### /plan-prd

**Kullanım**: 交互式 PRD 生成器

**Tanım**: 生成以问题为中心的产品需求文档，包含问题定义、用户画像、成功指标和 MVP 范围。

**Kullanım Senaryosu**: 
- 确定要构建的功能
- Gerekli明确产品需求
- 从零Başla新功能规划

**工作流**:
1. FRAME - 了解用户和问题
2. GROUND - 收集证据
3. DECIDE - 确定范围和假设
4. GENERATE - 生成 PRD 文件

---

### /prp-plan

**Kullanım**: 全面的功能规划

**Tanım**: 完整的项目规划，涵盖代码库分析、风险评估和分Adımlar实施计划。

**Kullanım Senaryosu**:
- 复杂功能的详细规划
- Gerekli代码库上下文分析
- 多阶段实施项目

---

### /prp-prd

**Kullanım**: PRP 工作流 PRD 生成器

**Tanım**: 使用 PRP（Plan-Record-Produce）工作流生成产品需求文档。

---

### /prp-implement

**Kullanım**: 执行 PRP 计划+Doğrulama循环

**Tanım**: 按照 PRP 工作流执行计划，包含持续的Doğrulama和检查。

---

### /prp-pr

**Kullanım**: 从 PRP 工作流创建 PR

**Tanım**: 从 PRP 工作流的结果创建 GitHub Pull Request。

---

### /prp-commit

**Kullanım**: PRP Doğrulama提交

**Tanım**: 在 PRP 工作流中执行Doğrulama并提交代码。

---

### /multi-plan

**Kullanım**: 多模型协作规划

**Tanım**: 结合 Codex 和 Gemini 的双模型分析，生成综合实施计划。

**Kullanım Senaryosu**:
- Gerekli多角度分析
- 复杂跨领域项目
- Gerekli平衡前端和后端考量
