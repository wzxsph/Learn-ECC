# Diğer Komutlar

## Genel Bakış

其他杂项命令，İçerir Jira 集成、GAN 操作、Güvenlik扫描等功能。

## 命令列表

### /jira

**Kullanım**: 与 Jira 工单交互 - 获取、分析、评论、转换状态、搜索

**Tanım**: 与 Jira Proje Yönetimi工具集成，Destekle直接工作流中使用 Jira tickets。

**Kullanım**:

| 命令 | Açıklama |
|---|---|
| `/jira get <TICKET-KEY>` | 获取并分析 Jira 工单 |
| `/jira comment <TICKET-KEY>` | 添加进度评论 |
| `/jira transition <TICKET-KEY>` | 更改工单状态 |
| `/jira search <JQL>` | 使用 JQL 搜索问题 |

**`/jira get` 工作流**:
1. 从 Jira 获取工单（Vasıtasıyla MCP 或 REST API）
2. 提取所有字段：摘要、Tanım、Kabul Kriterleri、优先级、标签、关联问题
3. 可选获取评论以获取额外上下文
4. 生成结构化分析

**先决条件** - Jira 凭证配置（二选一）:

**选项 A — MCP Server（推荐）**:
将 `jira` 添加到 `mcpServers` 配置。

**选项 B — 环境变量**:
```bash
export JIRA_URL="https://yourorg.atlassian.net"
export JIRA_EMAIL="your.email@example.com"
export JIRA_API_TOKEN="your-api-token"
```

**与Diğer Komutlar集成**:
- 分析工单后使用 `/plan` 创建实施计划
- 使用 `tdd-workflow` skill 进行Test Güdümlü Geliştirme
- 使用 `/code-review` 审查Gerçekleştir
- 使用 `/jira comment` 或 `/jira transition` 更新工单状态

### /prune

**Kullanım**: 删除超过 30 天未提升的陈旧 pending instinct

**Tanım**: 清理超过 30 天的陈旧 instinct，这些 instinct 是自动生成但从未被审查或提升的。

**Kullanım**:
```
/prune                    # 删除超过 30 天的 instinct
/prune --max-age 60      # 自定义天数阈值
/prune --dry-run         # 预览而不删除
```

**Gerçekleştir**:
```bash
python3 "${CLAUDE_PLUGIN_ROOT}/skills/continuous-learning-v2/scripts/instinct-cli.py" prune
```

或手动安装时（`CLAUDE_PLUGIN_ROOT` 未设置）:
```bash
python3 ~/.claude/skills/continuous-learning-v2/scripts/instinct-cli.py prune
```

**参数**:

| 参数 | Açıklama |
|---|---|
| 无 | 删除超过 30 天的 instinct |
| `--max-age <days>` | 自定义时间阈值（天） |
| `--dry-run` | 预览将删除的内容，不实际删除 |

**En İyi Uygulamalar**:
- 使用 `--dry-run` 先预览要删除的内容
- 定期运行 prune 保持 instinct 库存整洁
- Not：只会删除 pending 状态的 instinct，不会删除已提升的
