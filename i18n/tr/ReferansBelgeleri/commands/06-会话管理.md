# Oturum Yönetimi命令

## Genel Bakış

Oturum Yönetimi命令Kullanılan保存、恢复和Yönet Claude Code 会话状态。这些命令对于在长会话中保持上下文、在会话之间交接工作、以及追踪进度至关重要。

## 命令列表

### /save-session

**Kullanım**: 保存会话状态 - 将当前会话的完整上下文写入日期文件

**Tanım**: 将当前会话的所有上下文保存到 `~/.claude/session-data/` Dizin下的日期文件中，İçerir：构建了什么、什么工作了、什么失败了、剩余工作、文件状态、决策记录和待解决 blockers。

**会话文件命名规范**:
- 格式: `YYYY-MM-DD-<short-id>-session.tmp`
- 有效字符: 字母 `a-z`/`A-Z`、数字 `0-9`、连字符 `-`、下划线 `_`
- 最小长度: 1 字符（但不建议过短）
- 推荐: 8+ 字符的小写字母/数字/连字符组合避免同日冲突

**Kullanım Zamanı**:
- 工作结束时（会话结束前）
- 接近上下文限制前（先保存，再开新会话）
- 复杂问题解决后（保存成功模式）
- Gerekli交接给未来会话时

**工作流**:
1. **收集上下文** - 读取所有修改过的文件，审查讨论内容
2. **创建Dizin** - `mkdir -p ~/.claude/session-data`
3. **写入文件** - 创建带时间戳的会话文件
4. **展示给用户** - 显示内容并请求确认

**会话文件格式**:

```markdown
# Session: YYYY-MM-DD

**Started:** [approximate time if known]
**Last Updated:** [current time]
**Project:** [project name or path]
**Topic:** [one-line summary of what this session was about]

---

## What We Are Building
[1-3 paragraphs describing the feature, bug fix, or task]

## What WORKED (with evidence)
- **[thing that works]** — confirmed by: [specific evidence]

## What Did NOT Work (and why)
- **[approach tried]** — failed because: [exact reason / error message]

## What Has NOT Been Tried Yet
- [approach / idea]

## Current State of Files
| File              | Status         | Notes                      |
| ----------------- | -------------- | -------------------------- |
| `path/to/file.ts` | PASS: Complete | [what it does]             |

## Decisions Made
- **[decision]** — reason: [why this was chosen over alternatives]

## Blockers & Open Questions
- [blocker / open question]

## Exact Next Step
[The single most important thing to do when resuming]
```

**关键原则**:
- "What Did NOT Work" 部分是最重要的 - 未来会话会盲目重试失败的方法
- 每个会话独立文件 - 绝不追加到之前会话
- 文件供下次会话Vasıtasıyla `/resume-session` 读取

**En İyi Uygulamalar**:
- 中途保存时（不是结束时），清楚标记进行中项目
- 包含具体错误信息和失败原因（"threw X error because Y"）
- 下一具体Adımlar要足够精确，无需思考就知道从哪Başla

### /resume-session

**Kullanım**: 恢复保存的会话

**Tanım**: 从 `~/.claude/session-data/` 加载并恢复之前保存的会话上下文。

**Kullanım Senaryosu**:
- Başla新会话时
- Gerekli继续之前的工作
- Gerekli回顾之前的问题解决过程

---

### /sessions

**Kullanım**: 浏览+搜索+Yönet会话历史

**Tanım**: 浏览、搜索和Yönet所有已保存的会话历史。

**功能**:
- 列出所有会话
- 按日期搜索
- Yönet会话别名
- 清理旧会话

---

### /checkpoint

**Kullanım**: 创建/Doğrulama工作流检查点

**Tanım**: 在工作流关键节点创建检查点，确保进度可追溯。

**Kullanım Senaryosu**:
- 重要AdımlarBitir时
- GerekliDoğrulama关键里程碑
- 多阶段Görev中

---

### /aside

**Kullanım**: 回答侧问而不丢失上下文

**Tanım**: 切换到侧向对话模式，处理次要问题后无缝Geri Dön主Görev。

**Kullanım Senaryosu**:
- Gerekli问一个快速问题
- Gerekli查阅文档
- Gerekli执行辅助Görev

---

## 相关命令

- `/save-session` - 保存当前会话
- `/resume-session` - 恢复会话
- `/sessions` - Yönet会话历史
