# Test İlişkili命令

本文档介绍 ECC 中KullanılanTest的专门命令，Destekle多种Programlama Dilleri和TestÇerçeveler。

---

## /go-test

**KullanımAçıklama**: Go 语言的 TDD 工作流。使用表驱动Test，先写Test再Gerçekleştir，Doğrulama 80%+ 覆盖率。

**Kullanım Yöntemi**:
```
/go-test [功能Tanım]
```

**Kullanım Senaryosu**:
- Gerçekleştir新的 Go 函数
- 为现有 Go 代码添加Test覆盖
- 修复 bug（先写失败的Test）
- 学习 Go TDD 方法论

**Örnek工作流**:
```go
// Step 1: 定义接口
func ValidateEmail(email string) error {
    panic("not implemented")
}

// Step 2: 写表驱动Test (RED)
tests := []struct {
    name    string
    email   string
    wantErr bool
}{
    {"simple email", "user@example.com", false},
    {"empty string", "", true},
    {"no at sign", "userexample.com", true},
}

// Step 3: 运行Test - Doğrulama FAIL

// Step 4: Gerçekleştir最小代码 (GREEN)
func ValidateEmail(email string) error {
    if email == "" {
        return ErrEmailEmpty
    }
    if !emailRegex.MatchString(email) {
        return ErrEmailInvalid
    }
    return nil
}

// Step 5: Doğrulama PASS

// Step 6: 检查覆盖率
go test -cover ./...
// coverage: 100.0%
```

**覆盖率命令**:
```bash
go test -cover ./...                    # 基础覆盖率
go test -coverprofile=coverage.out      # 生成覆盖率文件
go tool cover -html=coverage.out        # 浏览器查看
go test -race -cover ./...              # 竞态检测
```

## /kotlin-test

**KullanımAçıklama**: Kotlin 语言的 TDD 工作流。使用 Kotest Çerçeveler和 MockK，先写Test再Gerçekleştir，Doğrulama 80%+ 覆盖率（Kover）。

**Kullanım Yöntemi**:
```
/kotlin-test [功能Tanım]
```

**Kullanım Senaryosu**:
- Gerçekleştir新的 Kotlin 函数或类
- 为 Android/KMP 项目添加Test覆盖
- 修复 bug（先写失败的Test）
- 学习 Kotlin TDD 方法论

**Kotest Test风格**:
```kotlin
// StringSpec (最简单)
class CalculatorTest : StringSpec({
    "add two positive numbers" {
        Calculator.add(2, 3) shouldBe 5
    }
})

// FunSpec (标准单元Test)
class RegistrationValidatorTest : FunSpec({
    test("valid registration returns Valid") {
        val request = RegistrationRequest(
            name = "Alice",
            email = "alice@example.com",
            password = "SecureP@ss1"
        )
        val result = validateRegistration(request)
        result.shouldBeInstanceOf<ValidationResult.Valid>()
    }
})

// BehaviorSpec (BDD 风格)
class OrderServiceTest : BehaviorSpec({
    Given("a valid order") {
        When("placed") {
            Then("should be confirmed") { /* ... */ }
        }
    }
})
```

**协程Test**:
```kotlin
test("concurrent fetch completes") {
    runTest {
        val result = service.fetchAll()
        result.shouldNotBeEmpty()
    }
}
```

**覆盖率命令**:
```bash
./gradlew koverHtmlReport     # HTML 报告
./gradlew koverVerify         # Doğrulama覆盖率阈值
./gradlew koverXmlReport      # CI 用 XML 报告
./gradlew test                # 运行Test
```

## /rust-test

**KullanımAçıklama**: Rust 语言的 TDD 工作流。使用 `#[test]`、rstest、proptest 和 mockall，先写Test再Gerçekleştir，Doğrulama 80%+ 覆盖率（cargo-llvm-cov）。

**Kullanım Yöntemi**:
```
/rust-test [功能Tanım]
```

**Kullanım Senaryosu**:
- Gerçekleştir新的 Rust 函数、方法或 trait
- 为 Rust 项目添加Test覆盖
- 修复 bug（先写失败的Test）
- 学习 Rust TDD 方法论

**Test模式**:
```rust
// 单元Test
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn adds_two_numbers() {
        assert_eq!(add(2, 3), 5);
    }

    #[test]
    fn handles_error() -> Result<(), Box<dyn std::error::Error>> {
        let result = parse_config(r#"port = 8080"#)?;
        assert_eq!(result.port, 8080);
        Ok(())
    }
}

// 参数化Test (rstest)
#[rstest]
#[case("hello", 5)]
#[case("", 0)]
#[case("rust", 4)]
fn test_string_length(#[case] input: &str, #[case] expected: usize) {
    assert_eq!(input.len(), expected);
}

// 异步Test
#[tokio::test]
async fn fetches_data_successfully() {
    let client = TestClient::new().await;
    let result = client.get("/data").await;
    assert!(result.is_ok());
}

// 属性Test (proptest)
proptest! {
    #[test]
    fn encode_decode_roundtrip(input in ".*") {
        let encoded = encode(&input);
        let decoded = decode(&encoded).unwrap();
        assert_eq!(input, decoded);
    }
}
```

**覆盖率命令**:
```bash
cargo llvm-cov                    # 覆盖率摘要
cargo llvm-cov --html             # HTML 报告
cargo llvm-cov --fail-under-lines 80  # 低于阈值则失败
cargo test                        # 运行Test
```

## /cpp-test

**KullanımAçıklama**: C++ 语言的 TDD 工作流。使用 GoogleTest/GoogleMock 和 CMake/CTest，先写Test再Gerçekleştir，Doğrulama覆盖率（gcov/lcov）。

**Kullanım Yöntemi**:
```
/cpp-test [功能Tanım]
```

**Kullanım Senaryosu**:
- Gerçekleştir新的 C++ 类或函数
- 添加Test覆盖
- 修复 bug（先写失败的Test）
- 学习 C++ TDD 方法论

**GoogleTest 模式**:
```cpp
// 基本Test
TEST(SuiteName, TestName) {
    EXPECT_EQ(add(2, 3), 5);
    EXPECT_NE(result, nullptr);
    EXPECT_TRUE(is_valid);
    EXPECT_THROW(func(), std::invalid_argument);
}

// Fixture
class DatabaseTest : public ::testing::Test {
protected:
    void SetUp() override { db_ = create_test_db(); }
    void TearDown() override { db_.reset(); }
    std::unique_ptr<Database> db_;
};

TEST_F(DatabaseTest, InsertsRecord) {
    db_->insert("key", "value");
    EXPECT_EQ(db_->get("key"), "value");
}

// 参数化Test
class PrimeTest : public ::testing::TestWithParam<std::pair<int, bool>> {};

TEST_P(PrimeTest, ChecksPrimality) {
    auto [input, expected] = GetParam();
    EXPECT_EQ(is_prime(input), expected);
}

INSTANTIATE_TEST_SUITE_P(Primes, PrimeTest,
    ::testing::Values(
        std::make_pair(2, true),
        std::make_pair(4, false),
        std::make_pair(7, true)
    ));
```

**覆盖率命令**:
```bash
cmake -DCMAKE_CXX_FLAGS="--coverage" -B build
cmake --build build
ctest --test-dir build
lcov --capture --directory build --output-file coverage.info
genhtml coverage.info --output-directory coverage_html
```

## /flutter-test

**KullanımAçıklama**: Flutter/Dart Test。报告失败并逐步修复Test问题，涵盖单元、组件、金色和集成Test。

**Kullanım Yöntemi**:
```
/flutter-test                # 运行所有Test
/flutter-test --coverage     # 带覆盖率
/flutter-test test/file.dart # 运行特定Test文件
```

**Kullanım Senaryosu**:
- Doğrulama功能Gerçekleştir后没有破坏
- 检查新代码的Test覆盖率
- 修复特定Test文件
- 提交 PR 之前

**常见Test失败和修复**:

| 失败Tür | 典型修复 |
|----------|----------|
| `Expected: <X> Actual: <Y>` | 更新断言或修复Gerçekleştir |
| `Widget not found` | 修复 finder 选择器或更新 widget Ad |
| `Golden file not found` | 运行 `flutter test --update-goldens` 生成 |
| `MissingPluginException` | 在Test设置中 mock 平台通道 |
| `pumpAndSettle timed out` | 用显式 `pump(Duration)` 替换 |

## /test-coverage

**KullanımAçıklama**: 分析Test覆盖率，识别差距，并生成缺失Test以达到Hedef阈值（默认 80%）。

**Kullanım Yöntemi**:
```
/test-coverage
```

**工作流程**:
1. **检测TestÇerçeveler** - 根据项目Tür选择覆盖命令
2. **分析覆盖率报告** - 列出低于 80% 的文件
3. **生成缺失Test** - 按优先级生成Test
4. **Doğrulama** - 重新运行覆盖率确认改进

**TestÇerçeveler检测**:

| 指示器 | 覆盖命令 |
|--------|----------|
| `jest.config.*` | `npx jest --coverage` |
| `vitest.config.*` | `npx vitest run --coverage` |
| `pytest.ini` | `pytest --cov=src --cov-report=json` |
| `Cargo.toml` | `cargo llvm-cov --json` |
| `go.mod` | `go test -coverprofile=coverage.out` |

## /python-testing

**KullanımAçıklama**: Python Test工作流。使用 pytest，Destekle覆盖率分析。

**Kullanım Yöntemi**:
```
/python-testing [功能Tanım]
```

**Kullanım Senaryosu**:
- Gerçekleştir新的 Python 功能
- 添加Test覆盖
- 修复 bug
- FastAPI/Django/Flask 项目Test

**Pytest 特性**:
```python
# 参数化Test
@pytest.mark.parametrize("input,expected", [
    ("hello", 5),
    ("", 0),
    ("world", 5),
])
def test_string_length(input, expected):
    assert len(input) == expected

# Fixtures
@pytest.fixture
def client():
    with TestClient(app) as c:
        yield c

# 异步Test
@pytest.mark.asyncio
async def test_async_fetch():
    result = await fetch_data()
    assert result is not None
```
