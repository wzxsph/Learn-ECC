# Özel Geliştirme

## Genel Bakış

本指南介绍如何为ECC Hooks Sistemi开发自定义钩子，İçerir基础结构、En İyi Uygulamalar和exit 0要求。

## 基础结构

### 最小钩子Şablon

```javascript
// my-custom-hook.js
let data = '';
process.stdin.on('data', chunk => data += chunk);
process.stdin.on('end', () => {
  const input = JSON.parse(data);

  // 访问工具信息
  const toolName = input.tool_name;        // "Edit", "Bash", "Write"等
  const toolInput = input.tool_input;      // 工具特定参数

  // process.exit(2);

  // 始终将原始数据输出到stdout
  console.log(data);

```

## 退出码要求

### 关键规则：exit 0 on 非关键错误

**重要**: 所有钩子Zorunlu在非关键错误时exit 0，避免意外阻塞工具执行。

| 退出码 | 含义 | Kullanım Senaryosu |
|--------|------|----------|
| 0 | 成功 | 继续执行，或非关键Uyarı |
| 2 | 阻止 | 仅KullanılanPreToolUse的关键阻塞 |
| 其他非零 | 错误 | 仅记录日志，**不要使用** |

### 为什么非零退出码不好？

如果钩子以非零退出码退出（除2外），Claude Code会将整个工具调用标记为失败，这可能导致：
- 工具执行被意外中断
- 用户体验受损
- 难以调试的问题

### 正确处理错误

```javascript
// 错误Örnek - 不要这样做
process.stdin.on('end', () => {
  try {
    const input = JSON.parse(data);
    // 处理逻辑
  } catch (e) {
    console.error('[Hook] Error:', e.message);
    process.exit(1);  // 错误：会阻塞工具
  }
  console.log(data);
});

// 正确Örnek - 这样做
process.stdin.on('end', () => {
  try {
    const input = JSON.parse(data);
    // 处理逻辑
  } catch (e) {
    console.error('[Hook] Error:', e.message);
    // 非关键错误，exit 0继续执行
    console.log(data);
    return;
  }
  console.log(data);
});
```

## 钩子输入模式

### HookInput 接口

```typescript
interface HookInput {
  tool_name: string;          // 工具Ad
  tool_input: {               // 工具输入参数
    command?: string;         // Bash: 命令
    file_path?: string;        // Edit/Write/Read: 文件路径
    old_string?: string;       // Edit: 被替换的文本
    new_string?: string;       // Edit: 替换文本
    content?: string;          // Write: 文件内容
  };
  tool_output?: {             // PostToolUse only
    output?: string;          // 命令/工具输出
  };
}
```

### 访问工具信息

```javascript
process.stdin.on('end', () => {
  const input = JSON.parse(data);

  // 获取工具Ad
  console.log('Tool:', input.tool_name);

  // 根据工具Tür处理
  if (input.tool_name === 'Bash') {
    console.log('Command:', input.tool_input.command);
  }

  if (input.tool_name === 'Edit' || input.tool_name === 'Write') {
    console.log('File:', input.tool_input.file_path);
  }

  // PostToolUse可访问输出
  if (input.tool_output) {
    console.log('Output:', input.tool_output.output);
  }
});
```

## 异步钩子

对于不应阻塞主流程的钩子（如后台分析）：

```json
{
  "matcher": "Edit|Write|MultiEdit",
  "hooks": [{
    "type": "command",
    "command": "node my-slow-hook.js",
    "async": true,
    "timeout": 30
  }]
}
```

### 异步钩子Not事项

- 异步钩子运行在后台
- 无法阻塞工具执行
- Olmalı在30秒内Bitir
- 适Kullanılan：日志、分析、遥测

## 阻塞钩子

对于Zorunlu阻止工具执行的情况（PreToolUse only）：

```javascript
// 在PreToolUse中阻止
process.exit(2);  // 退出码2表示阻止
```

### 何时使用阻止

- Güvenlik检查失败（如检测到密钥）
- 违反硬性策略
- 可能导致数据丢失的操作

### 何时不要使用阻止

- Kod Stili问题（使用Uyarı代替）
- 非关键检查
- 建议类检查

## 性能En İyi Uygulamalar

### 保持快速

- PreToolUse钩子：<200ms
- PostToolUse同步钩子：<1秒
- 异步钩子：<30秒

### 避免阻塞操作

```javascript
// 不好：同步文件读取
const content = fs.readFileSync('large-file.txt', 'utf8');

// 好：异步或延迟加载
fs.readFile('large-file.txt', 'utf8', (err, content) => {
  // 处理
});
```

### 缓存结果

```javascript
// 缓存昂贵操作

let cacheTime = 0;
const CACHE_TTL = 60000; // 1分钟

function getCachedResult() {
  const now = Date.now();
  if (!cachedResult || now - cacheTime > CACHE_TTL) {
    cachedResult = expensiveOperation();

  }
```

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 工具被意外阻止 | 钩子exit非零（除2外） | 改用exit 0和stderrUyarı |
| 挂起 | 钩子没有结束 | 确保总是输出到stdout |
| 数据损坏 | 输出到stdout非JSON | 仅输出原始data |
| 性能问题 | 钩子太慢 | 优化或改为异步 |

## En İyi Uygulamalar总结

1. **始终exit 0 on 非关键错误** - 不要用非零退出码（除2外）
2. **使用console.error输出Uyarı** - 不要用console.log
3. **保持快速** - PreToolUse <200ms
4. **总是输出原始data到stdout** - 不要改变数据流
5. **使用异步处理长时间操作** - 设置async: true
6. **跨平台路径处理** - 使用path.join
7. **Test输出格式** - 确保stdout是原始JSON
8. **文档化你的钩子** - 添加清晰的description
