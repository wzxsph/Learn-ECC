# Güvenlik Kuralları

## 规则Genel Bakış

ECC Rules Güvenlik Kuralları是强制性的Güvenlik检查体系，旨在确保代码在提交前符合最高Güvenlik标准。该规则覆盖了从SecretsYönet到MCP服务器Güvenlik的各个方面，是所有代码变更Zorunlu遵守的底线。

## 核心要求

### 提交前强制检查清单

在提交任何代码前，ZorunluBitir以下Güvenlik检查：

| 检查项 | Açıklama |
|--------|------|
| 无硬编码Secrets | API密钥、密码、令牌等不得硬编码在源代码中 |
| 用户输入Doğrulama | 所有用户输入Zorunlu经过Doğrulama |
| SQLEnjeksiyon Koruması | Parametreli Sorgu Kullan防止SQL注入 |
| XSS防护 | 对用户输入进行HTML转义 |
| CSRF保护 | 确保CSRF保护已启用 |
| 认证授权Doğrulama | 确认身份Doğrulama和授权逻辑正确 |
| 限流机制 | 所有端点Zorunlu有速率限制 |
| 错误消息Güvenlik | 错误消息不得泄露敏感数据 |
