# Test Kuralları

## 规则Genel Bakış

ECC Rules Test Kuralları定义了确保Kod Kalitesi的强制性Test标准。该规则涵盖Test Güdümlü Geliştirme(TDD)、最低覆盖率要求和Test结构规范，确保所有代码都经过充分Doğrulama。

## 核心要求

### 最低Test覆盖率：80%

Zorunlu包含以下三种Test型：

| Test型 | Açıklama | 覆盖范围 |
|----------|------|----------|
| 单元Test | 独立函数、工具类、组件 | Individual units |
| 集成Test | API端点、数据库操作 | Integration points |
| Uçtan Uca Test | 关键用户流程 | Critical user flows |

### Test Güdümlü Geliştirme（TDD）

**强制工作流**：

```
1. 编写Test (RED)   - 先写一个会失败的Test
2. 运行Test          - DoğrulamaTest失败
3. 编写最小Gerçekleştir (GREEN) - 编写VasıtasıylaTest的最小代码
4. 运行Test          - DoğrulamaTestVasıtasıyla
5. 重构 (IMPROVE)    - 改进代码结构
6. Doğrulama覆盖率        - 确保达到80%+
```

## 实施细节

### AAA模式（Arrange-Act-Assert）

所有TestZorunlu遵循AAA结构：

```typescript
test('正确计算余弦相似度', () => {
  // Arrange - 准备Test数据
  const vector1 = [1, 0, 0];
  const vector2 = [0, 1, 0];

  // Act - 执行被Test的操作
  const similarity = calculateCosineSimilarity(vector1, vector2);

  // Assert - Doğrulama结果
  expect(similarity).toBe(0);
});
```

### Test命名规范

使用Tanım性AdAçıklama被Test的行为：

```typescript
// 推荐 - Tanım预期行为
test('当没有市场匹配查询时Geri Dön空数组', () => {});
test('当API密钥缺失时抛出错误', () => {});
test('当Redis不可用时回退到子字符串搜索', () => {});

// 避免 - 模糊命名
test('test1', () => {});
test('edge case', () => {});
```

### Test故障排除

当Test失败时：

| Adımlar | 操作 |
|------|------|
| 1 | 使用 **tdd-guide** agent |
| 2 | 检查Test隔离 |
| 3 | DoğrulamaMock正确性 |
| 4 | 修复Gerçekleştir，而非Test（除非Test本身有误） |

### AgentDestekle

| Ajan | Kullanım | Kullanım Zamanı |
|-------|------|----------|
| **tdd-guide** | Test Güdümlü Geliştirme指导 | 新功能开发、错误修复时强制使用 |
| **code-reviewer** | Kod İnceleme | 代码编写Bitir后 |

## 违规处理

### 覆盖率不足

- 覆盖率低于80%的代码不得合并
- CI/CD流水线应阻止低覆盖率代码提交
- 使用 `c8` 或类似工具生成覆盖率报告

### Test失败处理

```
1. 分析失败原因
2. 诊断是Gerçekleştir问题还是Test问题
3. 如果是Gerçekleştir问题 - 修复Gerçekleştir代码
4. 如果是Test问题 - 修复Test代码
5. 确保所有TestVasıtasıyla后再提交
```

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| FlakyTest | 异步操作、竞态条件 | 增加等待时间、使用mock |
| Test隔离失败 | 共享状态污染 | 每个Test前重置状态 |
| Mock不正确 | 期望值与实际不符 | Doğrulamamock设置是否正确 |
