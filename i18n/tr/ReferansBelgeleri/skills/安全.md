## Güvenlik审查

### KullanımAçıklama
Güvenlik审查技能确保所有代码遵循GüvenlikEn İyi Uygulamalar，识别潜在漏洞。

### Kullanım Zamanı
- Gerçekleştir身份Doğrulama或授权
- 处理用户输入或文件上传
- 创建新API端点
- 使用密钥或凭证
- Gerçekleştir支付功能
- 存储或传输敏感数据
- 集成第三方API

### Temel Kavramlar

**1. 密钥Yönet**
永远不在源代码中硬编码密钥。

**2. 输入Doğrulama**
所有用户输入ZorunluDoğrulama。

**3. SQLEnjeksiyon Koruması**
Parametreli Sorgu Kullan。

**4. XSS防护**
清理用户Sağla的HTML。

**5. CSRF保护**
在状态改变操作上使用CSRF令牌。

**6. 速率限制**
在所有API端点上启用速率限制。

### Güvenlik检查清单

| 检查项 | Tanım |
|--------|------|
| 密钥 | 无硬编码密钥，所有密钥在环境变量中 |
| 输入Doğrulama | 所有用户输入使用schemaDoğrulama |
| SQL注入 | 所有查询Parametreli Sorgu Kullan |
| XSS | 用户内容被清理 |
| CSRF | 启用保护 |
| 身份Doğrulama | 正确的令牌处理 |
| 授权 | Rol检查到位 |
| 速率限制 | 所有端点启用 |
| HTTPS | 生产环境强制使用 |
| Güvenlik头 | CSP、X-Frame-Options配置 |
| Hata Yönetimi | 错误消息不暴露敏感数据 |
| 日志 | 不记录敏感数据 |
| 依赖 | 更新，无漏洞 |

// SQLEnjeksiyon Koruması - 错误做法

## Güvenlik扫描

### KullanımAçıklama
使用AgentShield扫描Claude Code配置（`.claude/`Dizin）中的Güvenlik Açıkları、配置错误和注入风险。

### Kullanım Zamanı
- 设置新的Claude Code项目
- 修改`.claude/settings.json`、`CLAUDE.md`或MCP配置后
- 提交配置更改前
- 加入已有Claude Code配置的新仓库
- 定期Güvenlik检查

### 扫描内容

| 文件 | 检查项 |
|------|--------|
| `CLAUDE.md` | 硬编码密钥、自动运行指令、İpucu注入模式 |
| `settings.json` | 过度宽松的允许列表、缺少拒绝列表、危险绕过标志 |
| `mcp.json` | 风险MCP服务器、硬编码env密钥、npx供应链风险 |
| `hooks/` | Vasıtasıyla插值的命令注入、数据泄露、静默错误抑制 |
| `agents/*.md` | 不受限的工具访问、İpucu注入表面、缺少模型规格 |

### 使用Örnek

```bash
# 基本扫描
npx ecc-agentshield scan

# 指定路径扫描
npx ecc-agentshield scan --path /path/to/.claude

# 最低严重性过滤
npx ecc-agentshield scan --min-severity medium

# JSON格式（KullanılanCI/CD）
npx ecc-agentshield scan --format json

# 自动修复
npx ecc-agentshield scan --fix

# Opus深度分析（GerekliANTHROPIC_API_KEY）
export ANTHROPIC_API_KEY=your-key
npx ecc-agentshield scan --opus --stream
```

| 等级 | 分数 | 含义 |
|------|------|------|
| A | 90-100 | Güvenlik配置 |
| B | 75-89 | 小问题 |
| C | 60-74 | Gerekli关注 |
| D | 40-59 | 重大风险 |

## DjangoGüvenlik

### KullanımAçıklama
DjangoGüvenlikEn İyi Uygulamalar，保护应用程序免受常见漏洞攻击。

### Kullanım Zamanı
- 设置Django身份Doğrulama和授权
- Gerçekleştir用户权限和Rol
- 配置生产Güvenlik设置
- 审查Django应用程序Güvenlik问题
- 部署Django应用程序到生产环境

### 核心Güvenlik设置

```python
# settings/production.py
DEBUG = False  # 关键：生产环境绝不使用True

ALLOWED_HOSTS = os.environ.get('ALLOWED_HOSTS', '').split(',')

# Güvenlik头
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True
SECURE_HSTS_SECONDS = 31536000  # 1年
SECURE_HSTS_INCLUDE_SUBDOMAINS = True
SECURE_HSTS_PRELOAD = True
SECURE_CONTENT_TYPE_NOSNIFF = True
SECURE_BROWSER_XSS_FILTER = True
X_FRAME_OPTIONS = 'DENY'

# CookieGüvenlik
SESSION_COOKIE_HTTPONLY = True
CSRF_COOKIE_HTTPONLY = True
SESSION_COOKIE_SAMESITE = 'Lax'

# 密码Doğrulama
AUTH_PASSWORD_VALIDATORS = [
    {'NAME': 'django.contrib.auth.password_validation.MinimumLengthValidator', 'OPTIONS': {'min_length': 12}},
    {'NAME': 'django.contrib.auth.password_validation.CommonPasswordValidator'},
]
```

### SQLEnjeksiyon Koruması

```python
# Güvenlik：Django ORM自动转义参数
def get_user(username):
    return User.objects.get(username=username)  # Güvenlik

# Güvenlik：使用参数的raw()
def search_users(query):
    return User.objects.raw('SELECT * FROM users WHERE username = %s', [query])

# 危险：永远不要直接插值
def get_user_bad(username):
    return User.objects.raw(f'SELECT * FROM users WHERE username = {username}')  # 漏洞！
```

### XSS防护

```django
{# Django默认自动转义变量 - Güvenlik #}
{{ user_input }}  {# HTML会被转义 #}

{# 显式标记为Güvenlik仅Kullanılan可信内容 #}
{{ trusted_html|safe }}  {# 不转义 #}

{# 使用Şablon过滤器Güvenlik处理HTML #}
{{ user_input|striptags }}  {# 移除所有HTML标签 #}
```

### CSRF保护

```django
{# Şablon中使用 #}
<form method="post">
    {% csrf_token %}
    {{ form.as_p }}
    <button type="submit">提交</button>
</form>

{# AJAX请求 #}
function getCookie(name) {
    let cookieValue = null;
    if (document.cookie && document.cookie !== '') {
        const cookies = document.cookie.split(';');
        for (let i = 0; i < cookies.length; i++) {
            const cookie = cookies[i].trim();
            if (cookie.substring(0, name.length + 1) === (name + '=')) {
                cookieValue = decodeURIComponent(cookie.substring(name.length + 1));
                break;
            }
        }
    }
    return cookieValue;
}

fetch('/api/endpoint/', {
    method: 'POST',
    headers: {
        'X-CSRFToken': getCookie('csrftoken'),
        'Content-Type': 'application/json',
    },
    body: JSON.stringify(data)
});
```

### 快速Güvenlik检查清单

| 检查项 | Tanım |
|--------|------|
| `DEBUG = False` | 生产环境绝不使用DEBUG运行 |
| 仅HTTPS | 强制SSL，Güvenlikcookie |
| 强密钥 | 使用环境变量的SECRET_KEY |
| 密码Doğrulama | 启用所有密码Doğrulama器 |
| CSRF保护 | 默认启用，不要禁用 |
| XSS防护 | Django默认转义，不要对用户输入使用`\|safe` |
| SQL注入 | 使用ORM，永远不要在查询中拼接字符串 |
| 文件上传 | Doğrulama文件Tür和大小 |
| 速率限制 | API端点限流 |
| Güvenlik头 | CSP、X-Frame-Options、HSTS |
| 日志 | 记录Güvenlik事件 |
| 更新 | 保持Django和依赖更新 |
