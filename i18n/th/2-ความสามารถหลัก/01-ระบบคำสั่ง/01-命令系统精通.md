# 01 - 命令系统精通

深入学习 ECC 的命令系统，掌握所有命令类型和使用场景。

## 概述

ECC 提供 **75 个[命令](../../เอกสารอ้างอิง/commands/01-核心工作流.md)**，分为三大类：斜杠命令（Slash Commands）、脚本命令和 Agent 命令。这些命令通过标准化的工作流大幅提升开发效率。

## 命令文件格式

每个命令是 `.md` 文件，包含 YAML frontmatter + Markdown 正文：

```yaml
---
description: "命令的简短描述"
argument-hint: "[可选参数提示]"
name: command-name          # 可选，默认用文件名
command: true              # 可选，CLI 透传命令
allowed_tools: ["Bash"]    # 可选，限制可用工具
---
```

### Frontmatter 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| description | 是 | 在 / 命令列表中显示 |
| argument-hint | 否 | 参数提示 |
| name | 否 | 覆盖文件名作为命令名 |
| command | 否 | 标记为 CLI 透传命令 |
| allowed_tools | 否 | 限制可用工具 |

## 2. 命令链和组合

多个命令可以组合使用，形成复杂工作流：

```markdown
# 典型开发流程
/plan → /tdd → /code-review → /verify → /pr

# 问题修复流程
/build-fix → /code-review → /test-coverage

# 多模型协作
/multi-plan → /multi-execute → /pr
```

## 3. 自定义命令创建

### 命令存放位置

| 类型 | 路径 |
|------|------|
| 命令文件 | `commands/` |
| 注册机制 | `.claude-plugin/plugin.json` |

### 创建步骤

1. 在 `commands/` 目录创建 `.md` 文件
2. 添加 YAML frontmatter
3. 编写命令内容（Markdown 格式）
4. 命令自动注册到 ECC

## 4. 最佳实践

### 命令选择指南

| 场景 | 推荐命令 |
|------|----------|
| 新功能开发 | `/plan` → `/tdd` |
| 代码审查 | `/code-review` |
| Bug 修复 | `/build-fix` → `/verify` |
| 测试驱动 | `/tdd` → `/test-coverage` |
| 创建 PR | `/pr` |
| 学习模式 | `/learn` → `/evolve` |

### 命令执行顺序

遵循工作流原则：
1. 规划：`/plan` 或 `/prp-plan`
2. 实现：`/tdd` 或 feature 命令
3. 审查：`/code-review`
4. 验证：`/verify` 或 `/quality-gate`
5. 提交：`/pr`
