# กฎความปลอดภัย

## ภาพรวมของกฎ

ECC Rules กฎความปลอดภัยเป็นระบบการตรวจสอบความปลอดภัยที่บังคับ มีจุดมุ่งหมายเพื่อให้มั่นใจว่าโค้ด tuân thủ มาตรฐานความปลอดภัยสูงสุดก่อนการ commit กฎนี้ครอบคลุมทุกอย่างตั้งแต่การจัดการ Secrets ไปจนถึงความปลอดภัยของ MCP servers และเป็นเส้นฐานที่ต้อง tuân thủ สำหรับทุกการเปลี่ยนแปลงโค้ด

## ข้อกำหนดหลัก

### รายการตรวจสอบบังคับก่อนการ commit

ก่อน commit โค้ดใดๆ ต้องทำการตรวจสอบความปลอดภัยต่อไปนี้ให้เสร็จสิ้น:

| รายการตรวจสอบ | คำอธิบาย |
|--------|------|
| ไม่มี Secrets ที่ hardcode | API keys, รหัสผ่าน, tokens ฯลฯ ต้องไม่ hardcode ใน source code |
| การตรวจสอบข้อมูลนำเข้าของผู้ใช้ | ข้อมูลนำเข้าของผู้ใช้ทั้งหมดต้องผ่านการตรวจสอบ |
| การป้องกัน SQL injection | ใช้ parameterized queries เพื่อป้องกัน SQL injection |
| การป้องกัน XSS | ทำ HTML escaping ข้อมูลที่ผู้ใช้ป้อน |
| การป้องกัน CSRF | ตรวจสอบว่า CSRF protection เปิดใช้งานอยู่ |
| การยืนยันการตรวจสอบสิทธิ์ | ยืนยันว่า logic การตรวจสอบสิทธิ์และการยืนยันตัวตนถูกต้อง |
| Rate limiting | ทุก endpoints ต้องมี rate limiting |
| ความปลอดภัยของข้อความข้อผิดพลาด | ข้อความข้อผิดพลาดต้องไม่เปิดเผยข้อมูลที่ละเอียดอ่อน |
