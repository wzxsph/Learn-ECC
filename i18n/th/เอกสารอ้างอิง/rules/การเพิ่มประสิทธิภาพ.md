# การเพิ่มประสิทธิภาพ

## กลยุทธ์การเลือกโมเดล

**Haiku 4.5** (ความสามารถ 90% ของ Sonnet, ค่าใช้จ่าย 1/3):
- ตัวแทนที่เบาและถูกเรียกใช้บ่อย
- การเขียนโค้ดคู่และการสร้างโค้ด
- ตัวแทน worker ในระบบหลายตัวแทน

**Sonnet 4.6** (โมเดลการเขียนโค้ดที่ดีที่สุด):
- งานพัฒนาหลัก
- การประสานงาน workflows หลายตัวแทน
- งานการเขียนโค้ดที่ซับซ้อน

**Opus 4.5** (การให้เหตุผลเชิงลึกที่สุด):
- การตัดสินใจสถาปัตยกรรมที่ซับซ้อน
- ความต้องการในการให้เหตุผลสูงสุด
- งานวิจัยและวิเคราะห์

## การจัดการบริบทหน้าต่าง

หลีกเลี่ยงการใช้ 20% สุดท้ายของบริบทหน้าต่าง:
- การปรับโครงสร้างขนาดใหญ่
- การ implementation ฟีเจอร์ที่ครอบคลุมหลายไฟล์
- การ debug interactions ที่ซับซ้อน

งานที่ต้องการบริบทต่ำ:
- การแก้ไขไฟล์เดียว
- การสร้าง utilities ที่เป็นอิสระ
- การอัปเดตเอกสาร
- การแก้ไข bug ง่ายๆ

## การคิดขยาย + โหมดการวางแผน

การคิดขยายเปิดใช้งานตามค่าเริ่มต้น สำรองไว้สูงสุด 31,999 tokens สำหรับการให้เหตุผลภายใน

ควบคุมการคิดขยาย:
- **สลับ**: Option+T (macOS) / Alt+T (Windows/Linux)
- **กำหนดค่า**: ตั้งค่า `alwaysThinkingEnabled` ใน `~/.claude/settings.json`
- **เพดานงบประมาณ**: `export MAX_THINKING_TOKENS=10000`
- **โหมดรายละเอียด**: Ctrl+O เพื่อดูผลลัพธ์การคิด

สำหรับงานที่ซับซ้อนและต้องการการให้เหตุผลเชิงลึก:
1. ตรวจสอบว่าการคิดขยายเปิดใช้งาน (ค่าเริ่มต้น)
2. เปิดใช้งาน**โหมดการวางแผน**เพื่อรับวิธีการที่มีโครงสร้าง
3. ใช้หลายรอบของการวิจารณ์เพื่อการวิเคราะห์อย่างทะลุปรุโปร่ง
4. ใช้ sub-agents แบบแบ่งบทบาทเพื่อได้มุมมองที่หลากหลาย

## การแก้ไขปัญหาการสร้าง

ถ้าการสร้างล้มเหลว:
1. ใช้ตัวแทน **build-error-resolver**
2. วิเคราะห์ข้อความข้อผิดพลาด
3. แก้ไขทีละน้อย
4. ตรวจสอบหลังการแก้ไขแต่ละครั้ง
