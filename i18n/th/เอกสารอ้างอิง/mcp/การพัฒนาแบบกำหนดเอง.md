# การพัฒนาเซิร์ฟเวอร์ MCP แบบกำหนดเอง

## ภาพรวม

เซิร์ฟเวอร์ MCP สร้างขึ้นบน Model Context Protocol ใช้สำหรับขยายความสามารถของ Claude Code คู่มือนี้อธิบายวิธีพัฒนาและบูรณาการเซิร์ฟเวอร์ MCP แบบกำหนดเอง

## ประเภทเซิร์ฟเวอร์

### เซิร์ฟเวอร์โหมดกระบวนการ

เปิดตัวผ่าน local process เหมาะสำหรับบริการที่ต้องการดำเนินการในพื้นที่หรือเข้าถึงทรัพยากรในพื้นที่

```json
{
  "command": "node",
  "args": ["/path/to/your/server.js"],
  "env": {
    "YOUR_API_KEY": "value"
  }
}
```

### เซิร์ฟเวอร์โหมด HTTP

เข้าถึงเซิร์ฟเวอร์ระยะไกลผ่าน HTTP/HTTPS เหมาะสำหรับสถาปัตยกรรม microservices หรือบริการ cloud

```json
{
  "type": "http",
  "url": "https://your-mcp-server.example.com/mcp",
  "headers": {
    "Authorization": "Bearer YOUR_TOKEN"
  }
}
```

## ขั้นตอนการพัฒนา

### 1. ใช้ MCP SDK

SDK มีให้เลือกหลายภาษา:

```bash
# Node.js
npm install @modelcontextprotocol/sdk

# Python
pip install mcp
```

### 2. เปิดตัวเซิร์ฟเวอร์

**ตัวอย่าง Node.js:**

```javascript
const { Server } = require('@modelcontextprotocol/sdk');
const { StdioServerTransport } = require('@modelcontextprotocol/sdk');

const server = new Server(
  {
    name: "my-custom-server",
    version: "1.0.0"
  },
  {
    capabilities: {
      tools: {},
      resources: {}
    }
  }
);

// ลงทะเบียนเครื่องมือ
server.setRequestHandler('tools/list', async () => {
  return {
    tools: [
      {
        name: "my_tool",
        description: "ดำเนินการแบบกำหนดเอง",
        inputSchema: {
          type: "object",
          properties: {
            param: { type: "string" }
          }
        }
      }
    ]
  };
});

// จัดการการเรียกเครื่องมือ
server.setRequestHandler('tools/call', async (request) => {
  const { name, arguments: args } = request.params;

  if (name === "my_tool") {
    // ดำเนินการ
    return { content: [{ type: "text", text: "ผลลัพธ์" }] };
  }
});

// เปิดตัวเซิร์ฟเวอร์
async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
}

main();
```

**ตัวอย่าง Python:**

```python
from mcp.server import Server
from mcp.server.stdio import stdio_server

server = Server("my-custom-server")

@server.list_tools()
async def list_tools():
    return [
        Tool(
            name="my_tool",
            description="ดำเนินการแบบกำหนดเอง",
            inputSchema={
                "type": "object",
                "properties": {
                    "param": {"type": "string"}
                }
            }
        )
    ]

@server.call_tool()
async def call_tool(name, arguments):
    if name == "my_tool":
        # ดำเนินการ
        return [TextContent(type="text", text="ผลลัพธ์")]

async def main():
    async with stdio_server() as (read_stream, write_stream):
        await server.run(read_stream, write_stream, server.create_initialization_options())
```

## การตั้งค่าและการบูรณาการ

### 1. ทดสอบในพื้นที่

```json
{
  "mcpServers": {
    "my-custom": {
      "command": "node",
      "args": ["/absolute/path/to/server.js"],
      "env": {
        "DEBUG": "true"
      }
    }
  }
}
```

### 2. เพิ่มไปยัง ECC

เพิ่มการตั้งค่าไปยัง `mcp-configs/mcp-servers.json`:

```json
{
  "mcpServers": {
    "my-custom": {
      "command": "npx",
      "args": ["-y", "/path/to/your/server"],
      "env": {
        "YOUR_API_KEY": "YOUR_KEY_HERE"
      },
      "description": "คำอธิบายเซิร์ฟเวอร์แบบกำหนดเองของคุณ"
    }
  }
}
```

## แนวปฏิบัติที่ดีที่สุด

1. **การจัดการข้อผิดพลาด**: ส่งคืนการตอบสนองข้อผิดพลาดที่มีโครงสร้างเสมอ
2. **การตรวจสอบข้อมูลนำเข้า**: ตรวจสอบพารามิเตอร์ข้อมูลนำเข้าทั้งหมด ปฏิเสธคำขอที่ไม่ถูกต้อง
3. **การควบคุมเวลา**: ตั้งค่าเวลาที่เหมาะสม
4. **การบันทึก**: บันทึกการดำเนินการสำคัญเพื่อการ debug
5. **การทำความสะอาดทรัพยากร**: ตรวจสอบว่าการเชื่อมต่อและทรัพยากรถูกปล่อยอย่างถูกต้อง
6. **การจัดการเวอร์ชัน**: tuân thủ semantic versioning

## ประเภทเครื่องมือทั่วไป

| ประเภท | วัตถุประสงค์ |
|------|------|
| `tools/list` | แสดงรายการเครื่องมือที่ใช้ได้ |
| `tools/call` | เรียกเครื่องมือ |
| `resources/list` | แสดงรายการทรัพยากร |
| `resources/read` | อ่านทรัพยากร |
| `prompts/list` | แสดงรายการเทมเพลต prompt |
| `prompts/get` | รับ prompt |
