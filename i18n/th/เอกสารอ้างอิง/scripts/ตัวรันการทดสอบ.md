# ตัวรันการทดสอบ

เอกสารนี้แนะนำโครงสร้างและวิธีการรันการทดสอบของโครงการ ECC

## จุดเข้าการทดสอบ

**เส้นทาง**: `tests/run-all.js`

ตัวรันการทดสอบแบบรวมศูนย์, ค้นพบและรันไฟล์การทดสอบทั้งหมดโดยอัตโนมัติ

### วิธีใช้

```bash
# รันการทดสอบทั้งหมด
node tests/run-all.js

# รันไฟล์การทดสอบเดียว
node tests/lib/utils.test.js
node tests/hooks/hooks.test.js
```

### การค้นพบไฟล์การทดสอบ

ตัวรันการทดสอบค้นพบไฟล์การทดสอบผ่าน glob pattern `tests/**/*.test.js`

กฎ:
- ไฟล์ต้องอยู่ในไดเรกทอรี `tests/`
- ชื่อไฟล์ต้องลงท้ายด้วย `.test.js`
- โครงสร้างไดเรกทอรีถูกรักษาเพื่อแสดงผล

## โครงสร้างการทดสอบ

### โครงสร้างไดเรกทอรี

```
tests/
├── run-all.js              # ตัวรันการทดสอบหลัก
├── lib/                    # การทดสอบไลบรารีเครื่องมือ
│   ├── utils.test.js
│   ├── package-manager.test.js
│   └── ...
├── hooks/                  # การทดสอบการบูรณาการ Hook
│   └── hooks.test.js
├── integration/            # การทดสอบการบูรณาการ
├── scripts/                # การทดสอบสคริปต์
├── commands/               # การทดสอบคำสั่ง
├── docs/                   # การทดสอบเอกสาร
├── opencode-config.test.js
├── plugin-manifest.test.js
├── test_*.py               # การทดสอบ Python
└── conftest.py             # การตั้งค่า pytest
```

### ประเภทการทดสอบ

1. **การทดสอบหน่วย** (`*.test.js`)
   - ทดสอบฟังก์ชันและเครื่องมือที่เป็นอิสระ
   - ใช้ Node.js native assert หรือรูปแบบ Jest

2. **การทดสอบการบูรณาการ** (`tests/integration/`)
   - ทดสอบการทำงานร่วมกันของหลายคอมโพเนนต์
   - ทดสอบขั้นตอน hook ทั้งหมด

3. **การทดสอบ Python** (`test_*.py`)
   - การทดสอบ Python รูปแบบ pytest
   - ใช้ `conftest.py` สำหรับการตั้งค่า

## การตั้งค่าการทดสอบ

### การทดสอบ Node.js

ไฟล์การทดสอบใช้มาตรฐาน Node.js assert module:

```javascript
const assert = require('assert');

test('ตัวอย่างการทดสอบ', () => {
  assert.strictEqual(actual, expected, 'ข้อความเพิ่มเติม');
});
```

### การทดสอบ Python (pytest)

ตำแหน่ง: `tests/conftest.py`

ไฟล์การตั้งค่า pytest, ให้ fixtures และการตั้งค่าที่ใช้ร่วมกัน

```bash
# รันการทดสอบ Python
python -m pytest tests/

# รันการทดสอบเฉพาะ
python -m pytest tests/test_builder.py
```

## แนวปฏิบัติที่ดีที่สุดในการทดสอบ

### รูปแบบ AAA

```
test('คำอธิบายการทดสอบ', () => {
  // Arrange - เตรียมข้อมูลการทดสอบ
  const input = 'test data';

  // Act - ดำเนินการที่ถูกทดสอบ
  const result = myFunction(input);

  // Assert - ตรวจสอบผลลัพธ์
  assert.strictEqual(result, expected);
});
```

### การตั้งชื่อการทดสอบ

ใช้ชื่อที่สื่อความหมาย, อธิบายพฤติกรรมของการทดสอบอย่างชัดเจน:

```javascript
test('ส่งคืน array ว่างเมื่อไม่มีตลาดที่ตรงกับคำค้นหา', () => {});
test('โยนข้อผิดพลาดเมื่อ API key หาย', () => {});
test('Fallback ไปยังการค้นหาตัวอักษรย่อยเมื่อ Redis ไม่พร้อม', () => {});
```

### การแยกการทดสอบ

การทดสอบแต่ละครั้งควร:
- รันอย่างเป็นอิสระ, ไม่พึ่งพาการทดสอบอื่น
- ทำความสะอาดข้อมูลการทดสอบของตัวเอง
- ใช้ mock เพื่อแยกการพึ่งพาภายนอก

## การบูรณาการ CI

กำหนดค่าสคริปต์การทดสอบใน `package.json`:

```json
{
  "scripts": {
    "test": "validate-commands.js && tests/run-all.js"
  }
}
```

ตัวรันการทดสอบออกแบบมาเพื่อทำงานในสภาพแวดล้อม CI:
- รหัสออกที่ไม่ใช่ศูนย์แสดงถึงความล้มเหลว
- เอาต์พุต JSON สำหรับการ parse อัตโนมัติ
- ข้อความข้อผิดพลาดที่ชัดเจนสำหรับการ debug
