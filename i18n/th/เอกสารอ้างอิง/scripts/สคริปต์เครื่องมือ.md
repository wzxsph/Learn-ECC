# สคริปต์เครื่องมือ

เอกสารนี้อธิบายสคริปต์เครื่องมือหลักในโครงการ ECC (Everything Claude Code)

## สารบัญ

- [ecc.js](#eccjs) - จุดเข้า CLI หลัก
- [install-apply.js](#install-applyjs) - ตัวติดตั้ง
- [claw.js](#clawjs) - NanoClaw REPL
- [harness-audit.js](#harness-auditjs) - การตรวจสอบ Toolchain
- [catalog.js](#catalogjs) - เครื่องมือดูคatalogue ของคอมโพเนนต์
- [build-opencode.js](#build-opencodejs) - การสร้าง OpenCode

---

## ecc.js

**เส้นทาง**: `scripts/ecc.js`

จุดเข้า CLI หลักสำหรับการติดตั้งแบบเลือกของ ECC จัดส่งคำสั่งย่อยทั้งหมดอย่างเป็นหนึ่งเดียว

### รายการคำสั่ง

| คำสั่ง | สคริปต์ | คำอธิบาย |
|------|------|------|
| `install` | install-apply.js | ติดตั้งเนื้อหา ECC ไปยังเป้าหมาย |
| `plan` | install-plan.js | ตรวจสอบ manifest และแผนการติดตั้ง |
| `catalog` | catalog.js | ค้นพบไฟล์การกำหนดค่าการติดตั้งและ ID คอมโพเนนต์ |
| `consult` | consult.js | แนะนำคอมโพเนนต์ตามการค้นหาภาษาธรรมชาติ |
| `list-installed` | list-installed.js | ตรวจสอบสถานะการติดตั้งของบริบทปัจจุบัน |
| `doctor` | doctor.js | วินิจฉัยไฟล์การจัดการ ECC ที่หายไปหรือเบี่ยงเบน |
| `repair` | repair.js | กู้คืนไฟล์การจัดการ ECC ที่เบี่ยงเบนหรือหายไป |
| `auto-update` | auto-update.js | ดึงการเปลี่ยนแปลง ECC ล่าสุดและติดตั้งใหม่ |
| `status` | status.js | สอบถามสรุปการจัดเก็บสถานะ SQLite |
| `platform-audit` | platform-audit.js | ตรวจสอบคิว GitHub การอภิปราย Roadmap ฯลฯ |
| `security-ioc-scan` | ci/scan-supply-chain-iocs.js | สแกน Supply Chain IOC ในการพึ่งพาและการคงอยู่ของ AI |
| `sessions` | sessions-cli.js | แสดงรายการหรือตรวจสอบการจัดเก็บสถานะ SQLite |
| `work-items` | work-items.js | ติดตามรายการงาน Linear/GitHub ที่เชื่อมโยง |
| `session-inspect` | session-inspect.js | ส่งออก snapshot เซสชันปกติจาก dmux หรือเป้าหมายประวัติ Claude |
| `loop-status` | loop-status.js | ตรวจสอบการตื่นของ loop ที่ล้าสมัยและผลลัพธ์เครื่องมือที่ค้างอยู่ในสคริปต์ Claude |
| `uninstall` | uninstall.js | ลบไฟล์การจัดการ ECC ที่บันทึกใน install-state |

### ตัวอย่างการใช้งาน

```bash
# ติดตั้งภาษาโดยตรง
ecc typescript

# ติดตั้งพร้อมไฟล์การกำหนดค่า
ecc install --profile developer --target claude

# แสดงแผนการติดตั้ง
ecc plan --profile core --target cursor

# แสดงรายการคอมโพเนนต์ที่ใช้ได้
ecc catalog profiles
ecc catalog components --family language

# แสดงรายละเอียดคอมโพเนนต์
ecc catalog show framework:nextjs

# ขอคำแนะนำ
ecc consult "security reviews"

# ตรวจสอบการติดตั้งแล้ว
ecc list-installed --json

# วินิจฉัยปัญหา
ecc doctor --target cursor
ecc repair --dry-run

# อัพเดทอัตโนมัติ
ecc auto-update --dry-run

# สอบถามสถานะ
ecc status --json
ecc status --exit-code
ecc status --markdown --write status.md

# ตรวจสอบแพลตฟอร์ม
ecc platform-audit --json --allow-untracked docs/drafts/

# สแกนความปลอดภัย
ecc security-ioc-scan --home

# จัดการเซสชัน
ecc sessions
ecc sessions session-active --json

# ติดตามรายการงาน
ecc work-items upsert linear-ecc-20 --source linear --source-id ECC-20 --title "Review control-plane contract" --status blocked
ecc work-items sync-github --repo affaan-m/ECC

# ตรวจสอบสถานะ loop
ecc loop-status --json

# ถอนการติดตั้ง
ecc uninstall --target antigravity --dry-run
```

## install-apply.js

**เส้นทาง**: `scripts/install-apply.js`

รันไทม์การติดตั้ง ECC รักษาจุดเข้าภาษาแบบดั้งเดิมให้ครบถ้วนขณะที่ย้ายตรรกะการเปลี่ยนแปลงเฉพาะเป้าหมายไปยังโค้ด Node ที่ทดสอบได้

### เป้าหมายที่รองรับ

| เป้าหมาย | คำอธิบาย |
|------|--------------|
| `claude` | ติดตั้งใน ~/.claude/ กฎ/ทักษะอยู่ใน rules/ecc และ skills/ecc |
| `claude-project` | ติดตั้งใน ./.claude/ (ระดับโครงการ) |
| `cursor` | ติดตั้งกฎ, Hooks, การกำหนดค่า Cursor ใน ./.cursor/ |
| `antigravity` | ติดตั้งกฎ, Workflows, ทักษะ, Agents ใน ./.agent/ |
| `codex` | ติดตั้ง Agents/การกำหนดค่าที่ใช้ร่วมกันใน ~/.codex/ |
| `gemini` | ติดตั้งการกำหนดค่า Gemini ระดับโครงการใน ./.gemini/ |
| `opencode` | ติดตั้งคำสั่ง/Hooks/การกำหนดค่าที่ใช้ร่วมกันใน ~/.opencode/ |
| `codebuddy` | ติดตั้งคำสั่ง, Agents, ทักษะใน ./.codebuddy/ |
| `joycode` | ติดตั้งคำสั่ง, Agents, ทักษะใน ./.joycode/ |
| `qwen` | ติดตั้งใน ~/.qwen/ |
| `zed` | ติดตั้งใน ./.zed/ |

### โหมดการติดตั้ง

1. **โหมดภาษาดั้งเดิม**: `install.sh <language>` (เช่น `ecc-install typescript`)
2. **โหมด Profile**: `install.sh --profile <name> [--with <component>]... [--without <component>]...`
3. **โหมดโมดูล**: `install.sh --modules <id,id,...>`
4. **โหมดทักษะ**: `install.sh --skills <skill-id[,skill-id...]>`
5. **โหมดการแปลท้องถิ่น**: `install.sh --target claude|claude-project --locale <locale-code>`

### ตัวเลือก

- `--dry-run` - แสดงแผนการติดตั้งแต่ไม่คัดลอกไฟล์
- `--json` - ส่งออกรูปแบบ JSON ที่เครื่องอ่านได้
- `--config <path>` - โหลดเจตนาการติดตั้งจากไฟล์

## claw.js

**เส้นทาง**: `scripts/claw.js`

NanoClaw v2 - REPL ปราศจากการพึ่งพาภายนอก, ตระหนักถึงเซสชัน สำหรับ Everything Claude Code สร้างบน `claude -p`

### ฟังก์ชัน

- การคงอยู่ของประวัติเซสชันเก็บใน `~/.claude/claw/`
- รองรับการโหลดทักษะแบบไดนามิกเป็นบริบท
- การบีบอัดเซสชันเพื่อจัดการความยาวบริบท
- การแตกกิ่งและส่งออกเซสชัน
- การค้นหาข้ามเซสชัน

### คำสั่ง REPL

| คำสั่ง | คำอธิบาย |
|------|------|
| `/help` | แสดงวิธีใช้ |
| `/clear` | ล้างประวัติเซสชันปัจจุบัน |
| `/history` | พิมพ์ประวัติการสนทนาทั้งหมด |
| `/sessions` | แสดงรายการเซสชันที่บันทึกไว้ |
| `/model [name]` | แสดง/ตั้งค่าโมเดล |
| `/load <skill-name>` | โหลดทักษะในบริบทที่ใช้งานอยู่ |
| `/branch <session-name>` | แตกกิ่งเซสชันปัจจุบันไปยังเซสชันใหม่ |
| `/search <query>` | ค้นหาข้ามเซสชัน |
| `/compact` | เก็บ turns ล่าสุด บีบอัดบริบทเก่า |
| `/export <md\|json\|txt> [path]` | ส่งออกเซสชัน |
| `/metrics` | แสดงตัวชี้วัดเซสชัน |
| `exit` | ออกจาก REPL |

### ตัวแปรสภาพแวดล้อม

| ตัวแปร | ค่าเริ่มต้น | คำอธิบาย |
|------|--------|------|
| `CLAW_SESSION` | `default` | ชื่อเซสชันเริ่มต้น |
| `CLAW_MODEL` | `sonnet` | โมเดลเริ่มต้น |
| `CLAW_SKILLS` | ว่างเปล่า | รายการทักษะที่โหลด (คั่นด้วย comma) |

### ตัวอย่างการใช้งาน

```bash
# เริ่มต้นพร้อมเซสชันเริ่มต้น
node scripts/claw.js

# เริ่มต้นพร้อมเซสชันที่ระบุ
CLAW_SESSION=my-session node scripts/claw.js

# เริ่มต้นพร้อมทักษะที่โหลด
CLAW_SKILLS="javascript-patterns,python-testing" node scripts/claw.js
```

## harness-audit.js

**เส้นทาง**: `scripts/harness-audit.js`

เครื่องมือตรวจสอบ Toolchain แบบกำหนดได้ อิงจากการตรวจสอบไฟล์/กฎที่ชัดเจน ตรวจจับโหมด ECC repo vs โหมดโครงการผู้บริโภคโดยอัตโนมัติ

### หมวดการตรวจสอบ

| หมวด | คำอธิบาย |
|------|------|
| Tool Coverage | ความครบถ้วนของเครื่องมือ |
| Context Efficiency | ประสิทธิภาพบริบท |
| Quality Gates | ประตูคุณภาพ |
| Memory Persistence | การคงอยู่ของหน่วยความจำ |
| Eval Coverage | การครอบคลุมการประเมิน |
| Security Guardrails | ราวน์ความปลอดภัย |
| Cost Efficiency | ประสิทธิภาพต้นทุน |
| GitHub Integration | การรวม GitHub |
| Vercel/Netlify/Cloudflare/Fly Integration | การรวมแพลตฟอร์ม |

### ตัวอย่างการใช้งาน

```bash
# ตรวจสอบไดเรกทอรีปัจจุบัน
node scripts/harness-audit.js

# ระบุขอบเขต
node scripts/harness-audit.js --scope repo
node scripts/harness-audit.js --scope hooks
node scripts/harness-audit.js --scope skills

# ระบุไดเรกทอรีราก
node scripts/harness-audit.js --root /path/to/project

# ส่งออกรูปแบบ JSON
node scripts/harness-audit.js --format json

# รวมกัน
node scripts/harness-audit.js repo --format json --root .
```

## catalog.js

**เส้นทาง**: `scripts/catalog.js`

เครื่องมือค้นพบคอมโพเนนต์และไฟล์การกำหนดค่าการติดตั้ง ECC

### คำสั่ง

#### profiles

แสดงรายการโปรไฟล์การติดตั้งทั้งหมด

```bash
node scripts/catalog.js profiles
node scripts/catalog.js profiles --json
```

#### components

แสดงรายการคอมโพเนนต์การติดตั้ง กรองตามครอบครัวและเป้าหมายได้

```bash
node scripts/catalog.js components
node scripts/catalog.js components --family language
node scripts/catalog.js components --family framework
node scripts/catalog.js components --target claude
node scripts/catalog.js components --json
```

#### show

แสดงรายละเอียดของคอมโพเนนต์เฉพาะ

```bash
node scripts/catalog.js show <component-id>
node scripts/catalog.js show framework:nextjs --json
```

### ครอบครัวคอมโพเนนต์

- `baseline` - คอมโพเนนต์พื้นฐาน
- `language` - คอมโพเนนต์ภาษา (javascript, typescript, python, ฯลฯ)
- `framework` - คอมโพเนนต์เฟรมเวิร์ค (nextjs, react, vue, ฯลฯ)
- `capability` - คอมโพเนนต์ความสามารถ
- `agent` - คอมโพเนนต์ตัวแทน
- `skill` - คอมโพเนนต์ทักษะ

## build-opencode.js

**เส้นทาง**: `scripts/build-opencode.js`

สร้างโค้ด TypeScript ของปลั็กอิน OpenCode

### ฟังก์ชัน

1. ล้างไดเรกทอรี `dist`
2. คอมไพล์โค้ด TypeScript ในไดเรกทอรี `.opencode` ด้วย TypeScript compiler
3. ส่งออกไปยัง `.opencode/dist`

### ข้อกำหนดเบื้องต้น

ต้องติดตั้ง development dependencies ที่ราก TypeScript compiler ต้องพร้อมใช้งาน

### การใช้งาน

```bash
node scripts/build-opencode.js
```
