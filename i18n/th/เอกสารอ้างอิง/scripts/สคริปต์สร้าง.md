## release.sh

**เส้นทาง**: `scripts/release.sh`

สคริปต์อัพเกรดเวอร์ชันและเผยแพร่ปลั็กอิน

### ฟังก์ชัน

1. **การตรวจสอบเวอร์ชัน**
   - ยืนยันว่าระบุหมายเลขเวอร์ชันรูปแบบ semver แล้ว (X.Y.Z หรือ X.Y.Z-prerelease)
   - ยืนยันว่า branch ปัจจุบันคือ main
   - ยืนยันว่าพื้นที่ทำงานสะอาด

2. **อัพเดทไฟล์**
   - อัพเดทหมายเลขเวอร์ชันในไฟล์ manifest ของแพ็คเกจ/ปลั็กอินทั้งหมด
   - อัพเดทการอ้างอิงเวอร์ชันในเอกสาร
   - อัพเดทการกำหนดค่าตัวแทน

3. **การตรวจสอบการสร้าง**
   - รันการสร้าง OpenCode
   - รันการทดสอบการสร้าง
   - รันการทดสอบ manifest ของปลั็กอิน

4. **การดำเนินการ Git**
   - stage การเปลี่ยนแปลงทั้งหมด
   - สร้าง commit
   - สร้างและ push แท็ก

### การใช้งาน

```bash
# เผยแพร่เวอร์ชันใหม่
./scripts/release.sh 1.5.0

# เผยแพร่ล่วงหน้า
./scripts/release.sh 2.0.0-rc.1
```

## build-opencode.js

**เส้นทาง**: `scripts/build-opencode.js`

สร้างโค้ด TypeScript ของปลั็กอิน OpenCode

### กระบวนการสร้าง

1. ล้างไดเรกทอรี `dist`
2. ค้นหา TypeScript compiler (`typescript/bin/tsc`)
3. คอมไพล์ไดเรกทอรี `.opencode` ด้วย tsconfig.json ที่รากของโครงการ

### การใช้งาน

```bash
# สร้าง OpenCode
node scripts/build-opencode.js

# รันโดยอัตโนมัติเป็นส่วนหนึ่งของการเผยแพร่
./scripts/release.sh 1.5.0
```

### ข้อกำหนดเบื้องต้น

ต้องติดตั้ง development dependencies ที่ราก, TypeScript compiler ต้องพร้อมใช้งาน

## ขั้นตอนการเผยแพร่

```
1. ตรวจสอบว่าอยู่บน branch main และพื้นที่ทำงานสะอาด
2. รันสคริปต์เผยแพร่พร้อมหมายเลขเวอร์ชัน
3. สคริปต์อัตโนมัติ:
   ├── ตรวจสอบข้อกำหนดเบื้องต้น
   ├── อัพเดทการอ้างอิงเวอร์ชันทั้งหมด
   ├── สร้าง OpenCode
   ├── รันการทดสอบเพื่อตรวจสอบ
   ├── สร้าง Git commit
   └── push แท็ก
4. GitHub Actions ตรวจพบแท็กและเผยแพร่โดยอัตโนมัติ
```
