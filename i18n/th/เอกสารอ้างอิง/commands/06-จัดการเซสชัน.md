# คำสั่งจัดการเซสชัน

## ภาพรวม

คำสั่งจัดการเซสชันใช้สำหรับบันทึก กู้คืน และจัดการสถานะของ Claude Code sessions คำสั่งเหล่านี้มีความสำคัญสำหรับการรักษาบริบทในเซสชันยาว การส่งต่องานระหว่างเซสชัน และการติดตามความคืบหน้า

## รายการคำสั่ง

### /save-session

**วัตถุประสงค์**: บันทึกสถานะเซสชัน - เขียนบริบทเต็มของเซสชันปัจจุบันไปยังไฟล์วันที่

**คำอธิบาย**: บันทึกทุกบริบทของเซสชันปัจจุบันไปยังไฟล์ในไดเรกทอรี `~/.claude/session-data/` พร้อมวันที่ รวมถึง: สร้างอะไร, อะไรทำงานแล้ว, อะไรล้มเหลว, งานที่เหลือ, สถานะไฟล์, บันทึกการตัดสินใจ และ blockers ที่ยังค้างอยู่

**มาตรฐานการตั้งชื่อไฟล์เซสชัน**:
- รูปแบบ: `YYYY-MM-DD-<short-id>-session.tmp`
- ตัวอักษรที่ถูกต้อง: ตัวอักษร `a-z`/`A-Z`, ตัวเลข `0-9`, เครื่องหมายขีด `-`, ขีดล่าง `_`
- ความยาวขั้นต่ำ: 1 ตัวอักษร (แต่ไม่แนะนำให้สั้นเกินไป)
- แนะนำ: ชุดตัวอักษร/ตัวเลข/เครื่องหมายขีด 8+ ตัวแบบพิมพ์เล็กเพื่อหลีกเลี่ยงการชนกันในวันเดียวกัน

**เวลาที่ใช้**:
- เมื่อสิ้นสุดการทำงาน (ก่อนเซสชันจบ)
- ใกล้ถึงขีดจำกัดบริบท (บันทึกก่อน เปิดเซสชันใหม่)
- หลังแก้ไขปัญหาที่ซับซ้อน (บันทึกรูปแบบที่สำเร็จ)
- เมื่อต้องส่งต่องานให้เซสชันถัดไป

**ขั้นตอนการทำงาน**:
1. **รวบรวมบริบท** - อ่านไฟล์ที่แก้ไขทั้งหมด ตรวจสอบเนื้อหาการสนทนา
2. **สร้างไดเรกทอรี** - `mkdir -p ~/.claude/session-data`
3. **เขียนไฟล์** - สร้างไฟล์เซสชันพร้อม timestamp
4. **แสดงให้ผู้ใช้** - แสดงเนื้อหาและขอการยืนยัน

**รูปแบบไฟล์เซสชัน**:

```markdown
# Session: YYYY-MM-DD

**Started:** [เวลาโดยประมาณถ้าทราบ]
**Last Updated:** [เวลาปัจจุบัน]
**Project:** [ชื่อโปรเจกต์หรือ path]
**Topic:** [สรุปหนึ่งบรรทัดของสิ่งที่เซสชันนี้เกี่ยวกับ]

---

## What We Are Building
[1-3 ย่อหน้าอธิบายฟีเจอร์ การแก้ไขข้อบกพร่อง หรืองาน]

## What WORKED (with evidence)
- **[สิ่งที่ทำงานได้]** — ยืนยันโดย: [หลักฐานเฉพาะ]

## What Did NOT Work (and why)
- **[แนวทางที่ลอง]** — ล้มเหลวเพราะ: [เหตุผล/ข้อความแสดงข้อผิดพลาดที่แน่นอน]

## What Has NOT Been Tried Yet
- [แนวทาง / ไอเดีย]

## Current State of Files
| File              | Status         | Notes                      |
| ----------------- | -------------- | -------------------------- |
| `path/to/file.ts` | PASS: Complete | [สิ่งที่มันทำ]             |

## Decisions Made
- **[การตัดสินใจ]** — เหตุผล: [ทำไมเลือกสิ่งนี้แทนทางเลือกอื่น]

## Blockers & Open Questions
- [blocker / คำถามที่เปิดอยู่]

## Exact Next Step
[สิ่งสำคัญที่สุดที่ต้องทำเมื่อกลับมาทำต่อ]
```

**หลักการสำคัญ**:
- ส่วน "What Did NOT Work" สำคัญที่สุด - เซสชันถัดไปจะลองใช้วิธีที่ล้มเหลวอย่างไม่มีทางรู้
- ไฟล์เซสชันแต่ละเซสชันแยกกัน - ไม่เคยต่อท้ายไฟล์เซสชันก่อนหน้า
- ไฟล์สำหรับเซสชันถัดไปอ่านผ่าน `/resume-session`

**แนวปฏิบัติที่ดีที่สุด**:
- เมื่อบันทึกระหว่างทาง (ไม่ใช่ตอนจบ) ทำเครื่องหมายโปรเจกต์ที่กำลังดำเนินอยู่อย่างชัดเจน
- รวมข้อความแสดงข้อผิดพลาดและเหตุผลที่ล้มเหลวเฉพาะเจาะจง ("threw X error because Y")
- ขั้นตอนถัดไปที่เฉพาะเจาะจงพอที่จะรู้ว่าเริ่มจากไหนโดยไม่ต้องคิด

---

### /resume-session

**วัตถุประสงค์**: กู้คืนเซสชันที่บันทึกไว้

**คำอธิบาย**: โหลดและกู้คืนบริบทเซสชันที่บันทึกไว้ก่อนหน้าจาก `~/.claude/session-data/`

**สถานการณ์การใช้งาน**:
- เมื่อเริ่มเซสชันใหม่
- เมื่อต้องการดำเนินการต่อจากงานก่อนหน้า
- เมื่อต้องการทบทวนกระบวนการแก้ปัญหาก่อนหน้า

---

### /sessions

**วัตถุประสงค์**: เรียกดู+ค้นหา+จัดการประวัติเซสชัน

**คำอธิบาย**: เรียกดู ค้นหา และจัดการประวัติเซสชันทั้งหมดที่บันทึกไว้

**ฟังก์ชัน**:
- แสดงรายการเซสชันทั้งหมด
- ค้นหาตามวันที่
- จัดการ alias เซสชัน
- ล้างเซสชันเก่า

---

### /checkpoint

**วัตถุประสงค์**: สร้าง/ตรวจสอบจุดตรวจสอบขั้นตอนการทำงาน

**คำอธิบาย**: สร้างจุดตรวจสอบที่จุดสำคัญของขั้นตอนการทำงาน เพื่อให้ความคืบหน้าสามารถตรวจสอบย้อนกลับได้

**สถานการณ์การใช้งาน**:
- เมื่อขั้นตอนสำคัญเสร็จสิ้น
- เมื่อต้องการตรวจสอบ milestone สำคัญ
- ในงานหลายขั้นตอน

---

### /aside

**วัตถุประสงค์**: ตอบคำถามข้างเคียงโดยไม่สูญเสียบริบท

**คำอธิบาย**: สลับไปโหมดการสนทนาข้างเคียง จัดการคำถามรองแล้วกลับไปที่งานหลักอย่างราบรื่น

**สถานการณ์การใช้งาน**:
- เมื่อต้องถามคำถามด่วน
- เมื่อต้องการอ่านเอกสาร
- เมื่อต้องดำเนินการงานเสริม

---

## คำสั่งที่เกี่ยวข้อง

- `/save-session` - บันทึกเซสชันปัจจุบัน
- `/resume-session` - กู้คืนเซสชัน
- `/sessions` - จัดการประวัติเซสชัน
