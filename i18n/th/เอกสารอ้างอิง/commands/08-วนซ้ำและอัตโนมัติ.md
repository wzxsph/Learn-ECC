# คำสั่งวนซ้ำและอัตโนมัติ

## ภาพรวม

คำสั่งวนซ้ำและอัตโนมัติใช้สำหรับเริ่มและจัดการโหมด loop อัตโนมัติแบบปกครอง (autonomous loop mode)

## รายการคำสั่ง

### /loop-start

**วัตถุประสงค์**: เริ่มโหมด loop อัตโนมัติแบบปกครองพร้อมค่าเริ่มต้นที่ปลอดภัยและเงื่อนไขการหยุดที่ชัดเจน

**คำอธิบาย**: เริ่มโหมด loop อัตโนมัติแบบปกครองพร้อมค่าเริ่มต้นที่ปลอดภัยและเงื่อนไขการหยุดที่ชัดเจน

**วิธีใช้**: `/loop-start [pattern] [--mode safe|fast]`

**พารามิเตอร์**:

| พารามิเตอร์ | ค่า | คำอธิบาย |
|---|---|---|
| `pattern` | `sequential` | รันงานตามลำดับ ทีละอัน |
| | `continuous-pr` | loop ต่อเนื่องสำหรับสร้างและรวม PR |
| | `rfc-dag` | workflow DAG ที่อิงกับ RFC |
| | `infinite` | loop อนันต์ (ต้องมีเงื่อนไขการหยุดที่ชัดเจน) |
| `--mode` | `safe` (ค่าเริ่มต้น) | ประตูคุณภาพและ checkpoints ที่เข้มงวด |
| | `fast` | ลดประตูเพื่อความเร็ว |

**ขั้นตอนการทำงาน**:
1. ยืนยันสถานะ repo และกลยุทธ์ branch
2. เลือกโหมด loop และกลยุทธ์ระดับโมเดลที่เกี่ยวข้อง
3. เปิดใช้งาน hooks/profile ที่จำเป็นสำหรับโหมดที่เลือก
4. สร้างแผน loop และเขียน runbook ใน `.claude/plans/`
5. พิมพ์คำสั่งเริ่มต้นและ monitoring

**การตรวจสอบความปลอดภัยที่จำเป็น**:
- ยืนยันว่าการทดสอบผ่านก่อน loop แรก
- ตรวจสอบว่า `ECC_HOOK_PROFILE` ไม่ได้ถูกปิดใช้งานทั่วโลก
- ตรวจสอบว่า loop มีเงื่อนไขการหยุดที่ชัดเจน

**การส่งพารามิเตอร์**:
- `$ARGUMENTS`: `[pattern]` และ `[--mode safe|fast]` แบบเลือกได้
- pattern เป็นแบบเลือก (ค่าเริ่มต้นคือ sequential)
- mode flag เป็นแบบเลือก (ค่าเริ่มต้นคือ safe)

**แนวปฏิบัติที่ดีที่สุด**:
- ตรวจสอบว่ามีเงื่อนไขการหยุดที่ชัดเจนก่อนใช้โหมด infinite
- ใช้โหมด fast กับโค้ดที่ทดสอบแล้วเท่านั้น
- monitoring ความคืบหน้าของ loop ใช้ `/loop-status` เพื่อตรวจสอบสถานะ

**กับดักที่พบบ่อย**:
- เริ่ม loop โดยไม่ได้ตรวจสอบการทดสอบ
- ใช้โหมด infinite โดยไม่มีเงื่อนไขการหยุดที่ชัดเจน
- พยายามใช้โหมด safe เมื่อ hooks ถูกปิดใช้งานทั่วโลก

**การบูรณาการกับคำสั่งอื่น**:
- ใช้ `/loop-status` เพื่อ monitoring loop ที่กำลังรัน
- ใช้ `/santa-loop` สำหรับ loop สไตล์ Santa
- ใช้ `/checkpoint` เพื่อสร้าง checkpoints ที่จุดสำคัญ

- `/santa-loop` - สไตล์ Santa
