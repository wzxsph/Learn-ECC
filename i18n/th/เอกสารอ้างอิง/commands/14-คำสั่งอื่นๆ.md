# คำสั่งอื่นๆ

## ภาพรวม

คำสั่งอื่นๆ เบ็ดเตล็ด รวมถึงการบูรณาการ Jira, การดำเนินการ GAN, การสแกนความปลอดภัย และอื่นๆ

## รายการคำสั่ง

### /jira

**วัตถุประสงค์**: ปฏิสัมพันธ์กับ Jira tickets - รับ วิเคราะห์ แสดงความคิดเห็น เปลี่ยนสถานะ ค้นหา

**คำอธิบาย**: บูรณาการกับเครื่องมือจัดการโครงการ Jira รองรับการใช้งาน Jira tickets โดยตรงใน workflow

**วิธีใช้**:

| คำสั่ง | คำอธิบาย |
|---|---|
| `/jira get <TICKET-KEY>` | รับและวิเคราะห์ Jira ticket |
| `/jira comment <TICKET-KEY>` | เพิ่มความคิดเห็นความคืบหน้า |
| `/jira transition <TICKET-KEY>` | เปลี่ยนสถานะ ticket |
| `/jira search <JQL>` | ค้นหาปัญหาด้วย JQL |

**ขั้นตอนการทำงาน `/jira get`**:
1. รับ ticket จาก Jira (ผ่าน MCP หรือ REST API)
2. แยกทุกฟิลด์: สรุป คำอธิบาย เกณฑ์การยอมรับ ลำดับความสำคัญ ป้าย ปัญหาที่เกี่ยวข้อง
3. เลือกรับความคิดเห็นเพื่อรับบริบทเพิ่มเติม
4. สร้างการวิเคราะห์ที่มีโครงสร้าง

**เงื่อนไขเบื้องต้น** - การตั้งค่า Jira credentials (เลือกหนึ่ง):

**ตัวเลือก A — MCP Server (แนะนำ)**:
เพิ่ม `jira` ในการตั้งค่า `mcpServers`

**ตัวเลือก B — ตัวแปรสิ่งแวดล้อม**:
```bash
export JIRA_URL="https://yourorg.atlassian.net"
export JIRA_EMAIL="your.email@example.com"
export JIRA_API_TOKEN="your-api-token"
```

**การบูรณาการกับคำสั่งอื่น**:
- ใช้ `/plan` หลังวิเคราะห์ ticket เพื่อสร้างแผนการดำเนินการ
- ใช้ `tdd-workflow` skill สำหรับการพัฒนาที่ขับเคลื่อนด้วยการทดสอบ
- ใช้ `/code-review` เพื่อตรวจสอบการ implementation
- ใช้ `/jira comment` หรือ `/jira transition` เพื่ออัปเดตสถานะ ticket

### /prune

**วัตถุประสงค์**: ลบ instinct ที่ pending และล้าสมัยเกิน 30 วัน

**คำอธิบาย**: ทำความสะอาด instinct ที่ล้าสมัยเกิน 30 วัน instinct เหล่านี้ถูกสร้างอัตโนมัติแต่ไม่เคยถูกตรวจสอบหรือยกระดับ

**วิธีใช้**:
```
/prune                    # ลบ instinct ที่เก่ากว่า 30 วัน
/prune --max-age 60      # กำหนดจำนวนวันเอง
/prune --dry-run         # แสดงตัวอย่างโดยไม่ลบจริง
```

**การ implementation**:
```bash
python3 "${CLAUDE_PLUGIN_ROOT}/skills/continuous-learning-v2/scripts/instinct-cli.py" prune
```

หรือเมื่อติดตั้งด้วยตนเอง (เมื่อ `CLAUDE_PLUGIN_ROOT` ไม่ได้ตั้งค่า):
```bash
python3 ~/.claude/skills/continuous-learning-v2/scripts/instinct-cli.py prune
```

**พารามิเตอร์**:

| พารามิเตอร์ | คำอธิบาย |
|---|---|
| ไม่มี | ลบ instinct ที่เก่ากว่า 30 วัน |
| `--max-age <days>` | กำหนดเกณฑ์จำนวนวันเอง |
| `--dry-run` | แสดงตัวอย่างสิ่งที่จะลบ โดยไม่ลบจริง |

**แนวปฏิบัติที่ดีที่สุด**:
- ใช้ `--dry-run` ก่อนเพื่อดูเนื้อหาที่จะลบ
- รัน prune เป็นประจำเพื่อรักษาคลัง instinct ให้สะอาด
- หมายเหตุ: จะลบเฉพาะ instinct ที่มีสถานะ pending ไม่ลบ instinct ที่ยกระดับแล้ว
