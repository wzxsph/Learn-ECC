# คำสั่งเรียนรู้และปรับปรุง

## ภาพรวม

คำสั่งเรียนรู้และปรับปรุงใช้สำหรับแยกรูปแบบจากเซสชัน ติดตาม instinct และจัดการองค์ความรู้องค์กร

## รายการคำสั่ง

### /learn

**วัตถุประสงค์**: แยกรูปแบบที่นำกลับมาใช้ใหม่ได้จากเซสชันปัจจุบันและบันทึกเป็น skills ต้นแบบ

**คำอธิบาย**: วิเคราะห์เซสชันปัจจุบัน แยกรูปแบบใดๆ ที่ควรค่าแก่การบันทึกเป็น skills รัน `/learn` ได้ทุกเมื่อระหว่างเซสชัน

**ประเภทเนื้อหาที่แยกออกมา**:

| ประเภท | คำอธิบาย |
|---|---|
| **รูปแบบแก้ปัญหาข้อผิดพลาด** | ข้อผิดพลาดอะไร? สาเหตุรากเหง้าคืออะไร? การแก้ไขอะไรได้ผล? สามารถนำรูปแบบไปใช้กับข้อผิดพลาดที่คล้ายกันได้หรือไม่? |
| **เทคนิคการ debug** | ขั้นตอน debug ที่ไม่ชัดเจน, การผสมผสานเครื่องมือที่มีประสิทธิภาพ, รูปแบบการวินิจฉัย |
| **วิธีแก้ความลำบาก** | ข้อบกพร่องของไลบรารี, ข้อจำกัด API, การแก้ไขเฉพาะเวอร์ชัน |
| **รูปแบบเฉพาะโปรเจกต์** | ข้อตกลง codebase ที่ค้นพบ, การตัดสินใจสถาปัตยกรรม, รูปแบบการบูรณาการ |

**รูปแบบเอาต์พุต**: บันทึกไปยัง `~/.claude/skills/learned/[pattern-name].md`

```markdown
# [Descriptive Pattern Name]

**Extracted:** [Date]
**Context:** [Brief description of when this applies]

## Problem
[What problem this solves - be specific]

## Solution
[The pattern/technique/workaround]

## Example
[Code example if applicable]

## When to Use
[Trigger conditions - what should activate this skill]
```

**ขั้นตอนการทำงาน**:
1. ตรวจสอบรูปแบบที่แยกออกมาได้ในเซสชัน
2. ระบุข้อมูลเชิงลึกที่มีค่ามากที่สุด/นำกลับมาใช้ใหม่ได้มากที่สุด
3. ร่างไฟล์ skill
4. ขอการยืนยันจากผู้ใช้ก่อนบันทึก
5. บันทึกไปยัง `~/.claude/skills/learned/`

**แนวปฏิบัติที่ดีที่สุด**:
- ไม่ต้องแยกการแก้ไขที่ trivial (ข้อผิดพลาดการสะกด, ข้อผิดพลาดไวยากรณ์ง่ายๆ)
- ไม่ต้องแยกปัญหาครั้งเดียว (API หยุดทำงานเฉพาะเจาะจง ฯลฯ)
- โฟกัสที่รูปแบบที่จะประหยัดเวลาให้เซสชันถัดไป
- รักษา skills ให้ตรงจุด - หนึ่งรูปแบบต่อหนึ่ง skill

### /learn-eval

**วัตถุประสงค์**: แยกรูปแบบ + ประเมินคุณภาพตนเอง

**คำอธิบาย**: ประเมินคุณภาพพร้อมกับการแยกรูปแบบ

---

### /evolve

**วัตถุประสงค์**: วิเคราะห์ instinct + แนะนำโครงสร้างที่วิวัฒนาการ

**คำอธิบาย**: วิเคราะห์ instinct ที่เรียนรู้แล้ว ให้คำแนะนำโครงสร้างที่จะวิวัฒนาการ

---

### /promote

**วัตถุประสงค์**: ยกระดับ instinct ของโปรเจกต์ไปยังขอบเขตทั่วโลก

**คำอธิบาย**: ยกระดับ instinct เฉพาะโปรเจกต์ให้กลายเป็นความรู้ที่ใช้ได้ทั่วโลก

---

### /instinct-status

**วัตถุประสงค์**: แสดง instinct ทั้งหมดที่เรียนรู้

**คำอธิบาย**: แสดงสถานะและความมั่นใจของ instinct ทั้งหมดในปัจจุบัน

---

### /instinct-export

**วัตถุประสงค์**: ส่งออก instinct ไปยังไฟล์

**คำอธิบาย**: ส่งออก instinct เป็นรูปแบบไฟล์ที่แชร์ได้

---

### /instinct-import

**วัตถุประสงค์**: นำเข้า instinct จากไฟล์/URL

**คำอธิบาย**: นำเข้า instinct จากไฟล์หรือ URL ไปยังระบบ

---

### /skill-create

**วัตถุประสงค์**: วิเคราะห์ git history + สร้างไฟล์ skill

**คำอธิบาย**: วิเคราะห์ git history ของโปรเจกต์ แยกรูปแบบที่นำกลับมาใช้ใหม่ได้สร้างไฟล์ skill

**ขั้นตอนการทำงาน**:
1. วิเคราะห์ประวัติ git commit
2. ระบุรูปแบบที่ซ้ำกัน
3. สร้างไฟล์ skill
4. ตรวจสอบและบันทึก

---

### /skill-health

**วัตถุประสงค์**: แดชบอร์ดสุขภาพของชุด skills

**คำอธิบาย**: แสดงสถานะสุขภาพและสถิติการใช้งานของชุด skills

---

### /rules-distill

**วัตถุประสงค์**: สแกน skills + แยกหลักการข้ามโดเมน

**คำอธิบาย**: สแกน skills ทั้งหมด แยกหลักการทั่วไปข้ามโดเมน

---

## คำสั่งที่เกี่ยวข้อง

- `/learn` - แยกรูปแบบ
- `/skill-create` - สร้าง skill จาก git history
- `/instinct-status` - ดูสถานะ instinct
