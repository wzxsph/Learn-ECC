# คำสั่งจัดการโครงการ

## ภาพรวม

คำสั่งจัดการโครงการใช้สำหรับการตั้งค่าโครงการ โครงสร้างพื้นฐาน และการจัดการเครื่องมือ

## รายการคำสั่ง

### /projects

**วัตถุประสงค์**: แสดงรายการโครงการที่ทราบ + สถิติ instinct

**คำอธิบาย**: แสดงรายการโครงการทั้งหมดของผู้ใช้พร้อมข้อมูลสถิติ instinct ที่เกี่ยวข้อง

---

### /harness-audit

**วัตถุประสงค์**: ตรวจสอบการตั้งค่า agent harness

**คำอธิบาย**: ตรวจสอบและตรวจสอบการตั้งค่า Claude Code harness

**เนื้อหาการตรวจสอบ**:
- การตั้งค่า toolchain
- การตั้งค่าตัวแทน
-สถานะเซิร์ฟเวอร์ MCP
- การตั้งค่ากฎ

---

### /model-route

**วัตถุประสงค์**: กำหนดเส้นทางงานไปยังโมเดลที่เหมาะสม

**คำอธิบาย**: เลือกโมเดลที่เหมาะสมที่สุด (Haiku/Sonnet/Opus) ตามประเภทงาน

**คู่มือการเลือกโมเดล**:
- **Haiku**: งานเบา, การสร้างโค้ด, การเรียกใช้บ่อย
- **Sonnet**: งานพัฒนาหลัก, งานเขียนโค้ดที่ซับซ้อน
- **Opus**: การตัดสินใจสถาปัตยกรรม, การให้เหตุผลเชิงลึก, การวิจัยและวิเคราะห์

---

### /pm2

**วัตถุประสงค์**: เริ่มต้น PM2 process manager

**คำอธิบาย**: เริ่มต้นการตั้งค่า PM2 process manager

---

### /setup-pm

**วัตถุประสงค์**: ตั้งค่า package manager

**คำอธิบาย**: ตั้งค่า package manager ที่โครงการใช้

**Package managers ที่รองรับ**:
- npm
- pnpm
- yarn
- bun

**กำหนดค่าได้ผ่าน** ตัวแปรสิ่งแวดล้อม `CLAUDE_PACKAGE_MANAGER`

---

### /project-init

**วัตถุประสงค์**: เริ่มต้นโครงการใหม่

**คำอธิบาย**: เริ่มต้นโครงการใหม่พร้อมโครงสร้างและการตั้งค่ามาตรฐาน

---

## คำสั่งที่เกี่ยวข้อง

- `/projects` - แสดงรายการโครงการ
- `/harness-audit` - ตรวจสอบการตั้งค่า
- `/model-route` - กำหนดเส้นทางโมเดล
