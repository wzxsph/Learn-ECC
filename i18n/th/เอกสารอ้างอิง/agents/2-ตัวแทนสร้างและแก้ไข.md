# ตัวแทนสร้างและแก้ไข

ตัวแทนประเภทสร้างและแก้ไขมีหน้าที่วินิจฉัยและแก้ไขข้อผิดพลาดในการสร้าง ข้อผิดพลาดในการคอมไพล์ และปัญหาการขึ้นต่อกันสำหรับภาษาโปรแกรมต่างๆ

## รายการตัวแทน

| ชื่อตัวแทน | การใช้งาน | โมเดลที่ใช้ | เครื่องมือหลัก |
|------------|------|----------|----------|
| build-error-resolver | TypeScript/ข้อผิดพลาดการสร้างทั่วไป | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| go-build-resolver | การแก้ไขข้อผิดพลาดการสร้าง/คอมไพล์ Go | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| kotlin-build-resolver | การแก้ไขข้อผิดพลาดการสร้าง Kotlin/Gradle | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| rust-build-resolver | การแก้ไขข้อผิดพลาดการสร้าง/Borrow checker ของ Rust | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| cpp-build-resolver | การแก้ไขข้อผิดพลาดการสร้าง C++/CMake | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| java-build-resolver | การแก้ไขข้อผิดพลาดการสร้าง Java/Maven/Gradle | sonnet | Read, Write, Edit, Bash, Grep, Glob |

## build-error-resolver

### ชื่อและการใช้งาน
ผู้เชี่ยวชาญแก้ไขข้อผิดพลาด TypeScript และประเภท มุ่งเน้นให้การสร้างผ่าน ไม่ทำการเปลี่ยนแปลงสถาปัตยกรรม

### ความสามารถ
- การแก้ไขข้อผิดพลาด TypeScript
- การแก้ไขข้อผิดพลาดการสร้าง
- การแก้ไขปัญหาการขึ้นต่อกัน
- การแก้ไขข้อผิดพลาดการกำหนดค่า
- การแก้ไขข้อผิดพลาดด้วย diff ที่น้อยที่สุด

### สถานการณ์การใช้งาน
- เมื่อการสร้างล้มเหลว
- เมื่อเกิดข้อผิดพลาดประเภท TypeScript
- เมื่อมีปัญหาการแก้ไขโมดูล

### รายการเครื่องมือที่ใช้
- Read: อ่านเนื้อหาไฟล์
- Write: เขียนการแก้ไข
- Edit: แก้ไขไฟล์
- Bash: รัน tsc, npm build, eslint
- Grep: ค้นหารูปแบบข้อผิดพลาด
- Glob: ค้นหาไฟล์ที่เกี่ยวข้อง

### การทำงานร่วมกับตัวแทนอื่น
- ไม่ทำ refactoring → ใช้ refactor-cleaner
- ไม่ทำการเปลี่ยนแปลงสถาปัตยกรรม → ใช้ architect
- ไม่เพิ่มฟีเจอร์ใหม่ → ใช้ planner
- ไม่แก้ไขการทดสอบ → ใช้ tdd-guide
- ไม่จัดการปัญหาความปลอดภัย → ใช้ security-reviewer

### หลักการหลัก
- แก้ไขข้อผิดพลาดเท่านั้น ไม่ refactor
- เปลี่ยนแปลงน้อยที่สุด
- ตรวจสอบว่าการสร้างผ่านหลังการแก้ไขแต่ละครั้ง

## go-build-resolver

### ชื่อและการใช้งาน
ผู้เชี่ยวชาญแก้ไขข้อผิดพลาดการสร้าง Go, go vet และคอมไพล์ การแก้ไขข้อผิดพลาดการสร้าง Go, ปัญหา go vet และคำเตือน linter

### ความสามารถ
- การวินิจฉัยข้อผิดพลาดคอมไพล์ Go
- การแก้ไขคำเตือน go vet
- การแก้ไขปัญหา staticcheck/golangci-lint
- การแก้ไขปัญหาการขึ้นต่อกันของโมดูล
- การจัดการข้อผิดพลาดประเภทและ interface mismatch

### สถานการณ์การใช้งาน
- เมื่อการสร้าง Go ล้มเหลว
- เมื่อ go vet รายงานปัญหา
- เมื่อมีความขัดแย้งของการขึ้นต่อกันของโมดูล

### รายการเครื่องมือที่ใช้
- Read: อ่านเนื้อหาไฟล์
- Write: เขียนการแก้ไข
- Edit: แก้ไขไฟล์
- Bash: รัน go build, go vet, staticcheck, golangci-lint
- Grep: ค้นหารูปแบบข้อผิดพลาด
- Glob: ค้นหาไฟล์ที่เกี่ยวข้อง

### การทำงานร่วมกับตัวแทนอื่น
- go-reviewer ตรวจสอบคุณภาพโค้ด
- security-reviewer จัดการปัญหาที่เกี่ยวข้องกับความปลอดภัย

### รูปแบบการแก้ไขข้อผิดพลาดทั่วไป

| ประเภทข้อผิดพลาด | สาเหตุ | วิธีแก้ไข |
|----------|------|----------|
| undefined: X | ขาด import, สะกดผิด | เพิ่ม import หรือแก้ไขการสะกด |
| cannot use X as type Y | ประเภทไม่ตรงกัน | การแปลงประเภทหรือ dereference |
| X does not implement Y | ขาดเมธอด | ใช้ receiver ที่ถูกต้องเพื่อใช้งานเมธอด |
| import cycle not allowed | การพึ่งพาวนรอบ | แยกประเภทที่ใช้ร่วมกันไปยังแพ็คเกจใหม่ |
| cannot find package | ขาดการขึ้นต่อกัน | go get pkg@version หรือ go mod tidy |

## kotlin-build-resolver

### ชื่อและการใช้งาน
ผู้เชี่ยวชาญแก้ไขข้อผิดพลาดการสร้าง/คอมไพล์/การขึ้นต่อกัน Kotlin/Gradle การแก้ไขข้อผิดพลาดการสร้าง Kotlin, ข้อผิดพลาดคอมไพลเลอร์ Kotlin และปัญหา Gradle

### ความสามารถ
- การวินิจฉัยข้อผิดพลาดคอมไพล์ Kotlin
- การแก้ไขปัญหาการกำหนดค่า Gradle
- การแก้ไขความขัดแย้งและความไม่ตรงกันของเวอร์ชันการขึ้นต่อกัน
- การจัดการข้อผิดพลาดคอมไพลเลอร์ Kotlin
- การแก้ไขการละเมิด detekt และ ktlint

### สถานการณ์การใช้งาน
- เมื่อการสร้าง Kotlin ล้มเหลว
- เมื่อความขัดแย้งของ Gradle dependencies
- เมื่อคอมไพลเลอร์ Kotlin รายงานข้อผิดพลาด

### รายการเครื่องมือที่ใช้
- Read: อ่านเนื้อหาไฟล์
- Write: เขียนการแก้ไข
- Edit: แก้ไขไฟล์
- Bash: รัน ./gradlew build, detekt, ktlintCheck
- Grep: ค้นหารูปแบบข้อผิดพลาด
- Glob: ค้นหาไฟล์ที่เกี่ยวข้อง

### การทำงานร่วมกับตัวแทนอื่น
- kotlin-reviewer ตรวจสอบคุณภาพโค้ด
- security-reviewer จัดการปัญหาที่เกี่ยวข้องกับความปลอดภัย

### รูปแบบการแก้ไขข้อผิดพลาดทั่วไป

| ประเภทข้อผิดพลาด | สาเหตุ | วิธีแก้ไข |
|----------|------|----------|
| Unresolved reference: X | ขาด import, สะกดผิด | เพิ่ม import หรือการขึ้นต่อกัน |
| Type mismatch | ข้อผิดพลาดประเภท | เพิ่มการแปลงหรือแก้ไขประเภท |
| Smart cast impossible | คุณสมบัติที่เปลี่ยนแปลงได้หรือการเข้าถึงพร้อมกัน | ใช้ val ท้องถิ่นหรือ let |
| when expression must be exhaustive | sealed class when ขาด branch | เพิ่ม branch ที่ขาดหายหรือ else |
| Suspend function can only be called from coroutine | ขาด suspend modifier หรือ coroutine scope | เพิ่ม suspend modifier |

## rust-build-resolver

### ชื่อและการใช้งาน
ผู้เชี่ยวชาญแก้ไขข้อผิดพลาดการสร้าง/คอมไพล/การขึ้นต่อกัน Rust การแก้ไข cargo build errors, ปัญหา Borrow checker และ Cargo.toml

### ความสามารถ
- การวินิจฉัยข้อผิดพลาด cargo build/check
- การแก้ไขข้อผิดพลาด Borrow checker และ Lifetime
- การแก้ไข Feature implementation mismatch
- การจัดการปัญหา Cargo dependencies และ features
- การแก้ไขคำเตือน cargo clippy

### สถานการณ์การใช้งาน
- เมื่อการสร้าง Rust ล้มเหลว
- เมื่อ Borrow checker รายงานข้อผิดพลาด
- เมื่อ Cargo dependencies ขัดแย้ง

### รายการเครื่องมือที่ใช้
- Read: อ่านเนื้อหาไฟล์
- Write: เขียนการแก้ไข
- Edit: แก้ไขไฟล์
- Bash: รัน cargo check, clippy, fmt, tree
- Grep: ค้นหารูปแบบข้อผิดพลาด
- Glob: ค้นหาไฟล์ที่เกี่ยวข้อง

### การทำงานร่วมกับตัวแทนอื่น
- rust-reviewer ตรวจสอบคุณภาพโค้ด
- security-reviewer จัดการปัญหาที่เกี่ยวข้องกับความปลอดภัย

### รูปแบบการแก้ไขข้อผิดพลาดทั่วไป

| ประเภทข้อผิดพลาด | สาเหตุ | วิธีแก้ไข |
|----------|------|----------|
| cannot borrow as mutable | การยืมแบบ immutable ที่ใช้งานอยู่ | Refactor เพื่อจบการยืม immutable ก่อนการเข้าถึงแบบ mutable |
| does not live long enough | ค่าถูก drop ขณะยังถูกยืม | ขยายขอบเขต lifetime |
| cannot move out of | การย้ายหลังจากการอ้างอิง | ใช้ .clone() หรือ refactor |
| mismatched types | ข้อผิดพลาดประเภท | เพิ่ม .into(), as หรือการแปลงที่ชัดเจน |
| trait X is not implemented for Y | ขาด impl หรือ derive | เพิ่ม #[derive(Trait)] |

## cpp-build-resolver

### ชื่อและการใช้งาน
ผู้เชี่ยวชาญแก้ไขข้อผิดพลาดการสร้าง/CMake/คอมไพล์ C++ การแก้ไขข้อผิดพลาดการสร้าง ปัญหา Linker และข้อผิดพลาด Template

### ความสามารถ
- การวินิจฉัยข้อผิดพลาดคอมไพล์ C++
- การแก้ไขปัญหาการกำหนดค่า CMake
- การแก้ไขข้อผิดพลาด Linker (undefined references, ไฟล์ที่กำหนดซ้ำ)
- การจัดการข้อผิดพลาด Template instantiation
- การแก้ไขปัญหาการรวมและการขึ้นต่อกัน

### สถานการณ์การใช้งาน
- เมื่อการสร้าง C++ ล้มเหลว
- เมื่อการกำหนดค่า CMake ผิดพลาด
- เมื่อ Linker รายงานข้อผิดพลาด

### รายการเครื่องมือที่ใช้
- Read: อ่านเนื้อหาไฟล์
- Write: เขียนการแก้ไข
- Edit: แก้ไขไฟล์
- Bash: รัน cmake --build, clang-tidy, cppcheck
- Grep: ค้นหารูปแบบข้อผิดพลาด
- Glob: ค้นหาไฟล์ที่เกี่ยวข้อง

### การทำงานร่วมกับตัวแทนอื่น
- cpp-reviewer ตรวจสอบคุณภาพโค้ด
- security-reviewer จัดการปัญหาที่เกี่ยวข้องกับความปลอดภัย

### รูปแบบการแก้ไขข้อผิดพลาดทั่วไป

| ประเภทข้อผิดพลาด | สาเหตุ | วิธีแก้ไข |
|----------|------|----------|
| undefined reference to X | ขาดการใช้งานหรือไลบรารี | เพิ่มไฟล์ต้นทางหรือลิงก์ไลบรารี |
| no matching function for call | ประเภทพารามิเตอร์ผิด | แก้ไขประเภทหรือเพิ่ม overload |
| multiple definition of | สัญลักษณ์ซ้ำ | ใช้ inline หรือย้ายไป .cpp |
| cannot convert X to Y | ประเภทไม่ตรงกัน | เพิ่มการแปลงหรือแก้ไขประเภท |
| template argument deduction failed | พารามิเตอร์ template ผิด | แก้ไขพารามิเตอร์ template |

## java-build-resolver

### ชื่อและการใช้งาน
ผู้เชี่ยวชาญแก้ไขข้อผิดพลาดการสร้าง/คอมไพล/การขึ้นต่อกัน Java/Maven/Gradle ตรวจจับ Spring Boot หรือ Quarkus โดยอัตโนโนมติและใช้การแก้ไขเฉพาะ framework

### ความสามารถ
- การวินิจฉัยข้อผิดพลาดคอมไพล์ Java
- การแก้ไขปัญหาการกำหนดค่า Maven และ Gradle
- การแก้ไขความขัดแย้งและความไม่ตรงกันของเวอร์ชันการขึ้นต่อกัน
- การจัดการข้อผิดพลาด Annotation processor (Lombok, MapStruct, Spring, Quarkus)
- การแก้ไขการละเมิด Checkstyle และ SpotBugs

### สถานการณ์การใช้งาน
- เมื่อการสร้าง Java ล้มเหลว
- เมื่อ Maven/Gradle dependencies ขัดแย้ง
- เมื่อมีปัญหาการสร้างเฉพาะ framework

### รายการเครื่องมือที่ใช้
- Read: อ่านเนื้อหาไฟล์
- Write: เขียนการแก้ไข
- Edit: แก้ไขไฟล์
- Bash: รัน ./mvnw compile, ./gradlew build, dependency:tree
- Grep: ค้นหารูปแบบข้อผิดพลาด
- Glob: ค้นหาไฟล์ที่เกี่ยวข้อง

### การทำงานร่วมกับตัวแทนอื่น
- java-reviewer ตรวจสอบคุณภาพโค้ด
- security-reviewer จัดการปัญหาที่เกี่ยวข้องกับความปลอดภัย

### การตรวจจับ Framework
ตรวจจับ framework ของโปรเจกต์โดยอัตโนโนมติ:
- ถ้ามี `quarkus` → ใช้กฎ [QUARKUS]
- ถ้ามี `spring-boot` → ใช้กฎ [SPRING]
- ทั้งสองมี → ใช้ทั้งสองชุดกฎ

### รูปแบบการแก้ไขข้อผิดพลาดทั่วไป

| ประเภทข้อผิดพลาด | สาเหตุ | วิธีแก้ไข |
|----------|------|----------|
| cannot find symbol | ขาด import, สะกดผิด | เพิ่ม import หรือการขึ้นต่อกัน |
| incompatible types | ประเภทไม่ตรงกัน | เพิ่มการแปลงที่ชัดเจน |
| package X does not exist | ขาดการขึ้นต่อกัน | เพิ่มไปยัง pom.xml/build.gradle |
| No qualifying bean of type X | ขาด @Component หรือ component scan | เพิ่ม annotation หรือแก้ไข scan package |

## swift-build-resolver

### ชื่อและการใช้งาน
ผู้เชี่ยวชาญแก้ไขข้อผิดพลาดการสร้าง/คอมไพล/การขึ้นต่อกัน Swift/Xcode การแก้ไขข้อผิดพลาด swift build, การสร้าง Xcode ที่ล้มเหลว, ปัญหา SPM dependencies และปัญหาการ sign โค้ด

### ความสามารถ
- การวินิจฉัยข้อผิดพลาด swift build/xcodebuild
- การแก้ไข Type checker และ Protocol conformance errors
- การแก้ไขปัญหา Swift Concurrency และ Sendable
- การจัดการความล้มเหลวของการ resolve และเวอร์ชันของ SPM dependencies

- การแก้ไขความขัดแย้งและข้อจำกัดเวอร์ชันของ pubspec.yaml dependencies
- การแก้ไขความล้มเหลวของ build_runner code generation
- การจัดการข้อผิดพลาดการสร้างเฉพาะ Flutter (Android Gradle, iOS CocoaPods, web)

### สถานการณ์การใช้งาน
- เมื่อการสร้าง Dart/Flutter ล้มเหลว
- เมื่อ pub dependencies ขัดแย้ง
- เมื่อ build_runner generation ล้มเหลว

### รูปแบบการแก้ไขข้อผิดพลาดทั่วไป

| ประเภทข้อผิดพลาด | สาเหตุ | วิธีแก้ไข |
|----------|------|----------|
