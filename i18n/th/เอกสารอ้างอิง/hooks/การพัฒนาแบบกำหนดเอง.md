# การพัฒนาแบบกำหนดเอง

## ภาพรวม

คู่มือนี้แนะนำวิธีพัฒนา hook แบบกำหนดเองสำหรับระบบ ECC Hooks รวมถึงโครงสร้างพื้นฐาน แนวปฏิบัติที่ดีที่สุด และข้อกำหนด exit 0

## โครงสร้างพื้นฐาน

### แม่แบบ Hook ขั้นต่ำ

```javascript
// my-custom-hook.js
let data = '';
process.stdin.on('data', chunk => data += chunk);
process.stdin.on('end', () => {
  const input = JSON.parse(data);

  // เข้าถึงข้อมูลเครื่องมือ
  const toolName = input.tool_name;        // "Edit", "Bash", "Write" ฯลฯ
  const toolInput = input.tool_input;      // พารามิเตอร์เฉพาะของเครื่องมือ
  const toolOutput = input.tool_output;    // มีเฉพาะใน PostToolUse

  // เตือน (ไม่บล็อก): เขียนไปยัง stderr
  console.error('[Hook] ข้อมูลเตือน');

  // บล็อก (เฉพาะ PreToolUse): exit code 2
  // process.exit(2);

  // ส่งข้อมูลดิบไปยัง stdout เสมอ
  console.log(data);
});
```

## ข้อกำหนด Exit Code

### กฎสำคัญ: exit 0 เมื่อข้อผิดพลาดไม่สำคัญ

**สำคัญ**: ทุก hook ต้อง exit 0 เมื่อข้อผิดพลาดไม่สำคัญ เพื่อหลีกเลี่ยงการบล็อกการทำงานของเครื่องมือโดยไม่ตั้งใจ

| Exit Code | ความหมาย | สถานการณ์การใช้ |
|--------|------|------|
| 0 | สำเร็จ | ดำเนินการต่อ หรือเตือนที่ไม่สำคัญ |
| 2 | บล็อก | เฉพาะสำหรับการบล็อกที่สำคัญใน PreToolUse |
| อื่นๆ ที่ไม่ใช่ศูนย์ | ข้อผิดพลาด | บันทึก log เท่านั้น **ห้ามใช้** |

### ทำไม exit code ที่ไม่ใช่ศูนย์จึงไม่ดี

ถ้า hook exit ด้วย exit code ที่ไม่ใช่ศูนย์ (นอกจาก 2) Claude Code จะทำเครื่องหมายการเรียกเครื่องมือทั้งหมดว่าล้มเหลว ซึ่งอาจทำให้:
- การทำงานของเครื่องมือถูกขัดขวางโดยไม่ตั้งใจ
- ประสบการณ์ผู้ใช้ได้รับผลกระทบ
- ปัญหาที่ยากจะ debug

### การจัดการข้อผิดพลาดอย่างถูกต้อง

```javascript
// ตัวอย่างผิด - อย่าทำแบบนี้
process.stdin.on('end', () => {
  try {
    const input = JSON.parse(data);
    // ลอจิกการประมวลผล
  } catch (e) {
    console.error('[Hook] ข้อผิดพลาด:', e.message);
    process.exit(1);  // ผิด: จะบล็อกเครื่องมือ
  }
  console.log(data);
});

// ตัวอย่างถูก - ทำแบบนี้
process.stdin.on('end', () => {
  try {
    const input = JSON.parse(data);
    // ลอจิกการประมวลผล
  } catch (e) {
    console.error('[Hook] ข้อผิดพลาด:', e.message);
    // ข้อผิดพลาดไม่สำคัญ exit 0 เพื่อดำเนินการต่อ
    console.log(data);
    return;
  }
  console.log(data);
});
```

## รูปแบบ Input ของ Hook

### Interface HookInput

```typescript
interface HookInput {
  tool_name: string;          // ชื่อเครื่องมือ
  tool_input: {               // พารามิเตอร์ของเครื่องมือ
    command?: string;         // Bash: คำสั่ง
    file_path?: string;        // Edit/Write/Read: เส้นทางไฟล์
    old_string?: string;       // Edit: ข้อความที่ถูกแทนที่
    new_string?: string;       // Edit: ข้อความแทนที่
    content?: string;          // Write: เนื้อหาไฟล์
  };
  tool_output?: {             // เฉพาะ PostToolUse
    output?: string;          // ผลลัพธ์ของคำสั่ง/เครื่องมือ
  };
}
```

### การเข้าถึงข้อมูลเครื่องมือ

```javascript
process.stdin.on('end', () => {
  const input = JSON.parse(data);

  // รับชื่อเครื่องมือ
  console.log('เครื่องมือ:', input.tool_name);

  // ประมวลผลตามประเภทเครื่องมือ
  if (input.tool_name === 'Bash') {
    console.log('คำสั่ง:', input.tool_input.command);
  }

  if (input.tool_name === 'Edit' || input.tool_name === 'Write') {
    console.log('ไฟล์:', input.tool_input.file_path);
  }

  // PostToolUse สามารถเข้าถึงผลลัพธ์ได้
  if (input.tool_output) {
    console.log('ผลลัพธ์:', input.tool_output.output);
  }
});
```

## Hook แบบ Async

สำหรับ hook ที่ไม่ควรบล็อกการทำงานหลัก (เช่น การวิเคราะห์เบื้องหลัง):

```json
{
  "matcher": "Edit|Write|MultiEdit",
  "hooks": [{
    "type": "command",
    "command": "node my-slow-hook.js",
    "async": true,
    "timeout": 30
  }]
}
```

### หมายเหตุเกี่ยวกับ Async Hooks

- Hooks แบบ async รันในเบื้องหลัง
- ไม่สามารถบล็อกการทำงานของเครื่องมือ
- ควรเสร็จภายใน 30 วินาที
- เหมาะสำหรับ: logging, การวิเคราะห์, telemetry

---

## Hook แบบบล็อก

สำหรับกรณีที่ต้องบล็อกการทำงานของเครื่องมือ (เฉพาะ PreToolUse):

```javascript
// บล็อกใน PreToolUse
process.exit(2);  // exit code 2 หมายถึงบล็อก
```

### เมื่อใดควรใช้การบล็อก

- การตรวจสอบความปลอดภัยล้มเหลว (เช่น ตรวจพบคีย์)
- การละเมิดนโยบายที่เข้มงวด
- การดำเนินการที่อาจทำให้สูญเสียข้อมูล

### เมื่อใดไม่ควรใช้การบล็อก

- ปัญหารูปแบบโค้ด (ใช้การเตือนแทน)
- การตรวจสอบที่ไม่สำคัญ
- การตรวจสอบประเภทคำแนะนำ

## แนวปฏิบัติที่ดีที่สุดด้านประสิทธิภาพ

### รันให้เร็ว

- PreToolUse hooks: <200ms
- PostToolUse hooks แบบ sync: <1 วินาที
- Async hooks: <30 วินาที

### หลีกเลี่ยงการดำเนินการที่บล็อก

```javascript
// ไม่ดี: อ่านไฟล์แบบ sync
const content = fs.readFileSync('large-file.txt', 'utf8');

// ดี: อ่านแบบ async หรือ lazy load
fs.readFile('large-file.txt', 'utf8', (err, content) => {
  // ประมวลผล
});
```

### แคชผลลัพธ์

```javascript
// แคชการดำเนินการที่ expensive
let cachedResult = null;
let cacheTime = 0;
const CACHE_TTL = 60000; // 1 นาที

function getCachedResult() {
  const now = Date.now();
  if (!cachedResult || now - cacheTime > CACHE_TTL) {
    cachedResult = expensiveOperation();
    cacheTime = now;
  }
  return cachedResult;
}
```

## การทดสอบ Hooks

### ทดสอบด้วยตนเอง

```bash
# ทดสอบ PreToolUse hook
echo '{"tool_name":"Bash","tool_input":{"command":"echo test"}}' | node scripts/hooks/my-hook.js

# ทดสอบ PostToolUse hook
echo '{"tool_name":"Bash","tool_input":{"command":"echo test"},"tool_output":{"output":"test\n"}}' | node scripts/hooks/my-hook.js
```

### รูปแบบเอาต์พุตการทดสอบ

```javascript
// เอาต์พุต stdout ที่ถูกต้อง (JSON)
console.log(data);  // ข้อมูล input ดิบ

// เอาต์พุต stderr ที่ถูกต้อง (เตือน)
console.error('[Hook] ข้อความเตือน');

// ไม่ถูกต้อง - อย่าแสดงข้อความอื่นไปยัง stdout
console.log('ข้อความบางอย่าง');  // จะทำลาย data stream
```

## การ Debug Hooks

### เพิ่มเอาต์พุต debug

```javascript
const DEBUG = process.env.DEBUG_HOOKS === '1';

process.stdin.on('end', () => {
  if (DEBUG) {
    console.error('[DEBUG] ได้รับ input:', data);
  }
  // ลอจิกการประมวลผล
  if (DEBUG) {
    console.error('[DEBUG] ส่ง output');
  }
  console.log(data);
});
```

### ปัญหาทั่วไป

| ปัญหา | สาเหตุ | วิธีแก้ไข |
|------|------|----------|
| เครื่องมือถูกบล็อกโดยไม่ตั้งใจ | hook exit ด้วยค่าที่ไม่ใช่ศูนย์ (นอกจาก 2) | เปลี่ยนเป็น exit 0 และ stderr เตือน |
| ค้าง | hook ไม่จบ | ตรวจสอบว่าส่ง output ไปยัง stdout เสมอ |
| ข้อมูลเสียหาย | ส่งออกไปยัง stdout ไม่ใช่ JSON | ส่งออกเฉพาะ data ดิบ |
| ปัญหาประสิทธิภาพ | hook ทำงานช้าเกินไป | optimize หรือเปลี่ยนเป็น async |

## สรุปแนวปฏิบัติที่ดีที่สุด

1. **Exit 0 เสมอเมื่อข้อผิดพลาดไม่สำคัญ** - ห้ามใช้ exit code ที่ไม่ใช่ศูนย์ (นอกจาก 2)
2. **ใช้ console.error เพื่อแสดงเตือน** - ห้ามใช้ console.log
3. **รันให้เร็ว** - PreToolUse <200ms
4. **ส่งข้อมูล data ดิบไปยัง stdout เสมอ** - อย่าเปลี่ยน data stream
5. **ใช้ async สำหรับการดำเนินการที่ใช้เวลานาน** - ตั้งค่า async: true
6. **จัดการเส้นทางข้ามแพลตฟอร์ม** - ใช้ path.join
7. **ทดสอบรูปแบบเอาต์พุต** - ตรวจสอบว่า stdout เป็น JSON ดิบ
8. **เขียนเอกสาร hook** - เพิ่ม description ที่ชัดเจน
