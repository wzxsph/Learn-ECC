# รูปแบบการกำหนดค่า

## ภาพรวมโครงสร้าง hooks.json

ไฟล์การกำหนดค่าระบบ ECC Hooks ใช้รูปแบบ JSON พร้อมโครงสร้างหลักดังนี้:

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "hooks": {
    "PreToolUse": [...],
    "PostToolUse": [...],
    "Stop": [...],
    "SessionStart": [...],
    "SessionEnd": [...],
    "PreCompact": [...],
    "PostToolUseFailure": [...]
  }
}
```

## โครงสร้างระดับบนสุด

| ฟิลด์ | ประเภท | คำอธิบาย |
|------|------|------|
| `$schema` | string | URL ของ JSON Schema สำหรับการตรวจสอบความถูกต้องของการกำหนดค่า |
| `hooks` | object | ออบเจ็กต์ที่มีประเภท Hook ทั้งหมด |

---

## โครงสร้างอาร์เรย์ของ Hook

แต่ละประเภทเหตุการณ์ (PreToolUse, PostToolUse ฯลฯ) มีอาร์เรย์ของ Hook โดยแต่ละออบเจ็กต์ Hook มีโครงสร้างดังนี้:

```json
{
  "matcher": "Bash|Edit|Write",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/my-hook.js",
      "async": false,
      "timeout": 30
    }
  ],
  "description": "คำอธิบาย Hook",
  "id": "unique-hook-id"
}
```

### ฟิลด์ matcher

#### คำอธิบายประเภท
ฟิลด์ matcher ระบุว่า Hook ทำงานสำหรับประเภทเครื่องมือใด รองรับเครื่องมือเดียวหรือหลายเครื่องมือ (คั่นด้วย `|`)

#### ค่าที่ใช้ได้
| ค่า matcher | เครื่องมือที่ตรงกัน |
|-----------|-----------------|
| `*` | เครื่องมือทั้งหมด |
| `Bash` | เครื่องมือ Bash |
| `Edit` | เครื่องมือ Edit |
| `Write` | เครื่องมือ Write |
| `Read` | เครื่องมือ Read |
| `MultiEdit` | เครื่องมือ MultiEdit |
| `Bash\|Edit\|Write` | Bash, Edit หรือ Write |
| `Edit\|Write\|MultiEdit` | Edit, Write หรือ MultiEdit |

#### ตัวอย่างการใช้งาน
```json
// ตรงกับเครื่องมือทั้งหมด
"matcher": "*"

// ตรงกับเฉพาะเครื่องมือ Bash
"matcher": "Bash"

// ตรงกับเครื่องมือหลายตัว
"matcher": "Edit|Write|MultiEdit"
```

## โครงสร้างอาร์เรย์ hooks

อาร์เรย์ hooks มีออบเจ็กต์คำสั่ง Hook หนึ่งหรือหลายตัว:

### ฟิลด์ type

| ค่า type | คำอธิบาย |
|--------|------|
| `command` | รันคำสั่ง Node.js |

### ฟิลด์ command

คำสั่งที่จะรัน โดยทั่วไปคือเส้นทางสคริปต์ Node.js

### ฟิลด์ async (ตัวเลือก)

```json
"async": true
```

- `true`: รันแบบ async ไม่บล็อกการทำงานของเครื่องมือ
- `false` หรือละเว้น: รันแบบ sync บล็อกการทำงานของเครื่องมือ

### ฟิลด์ timeout (ตัวเลือก)

```json
"timeout": 30
```

เวลาสูงสุดในการรัน (วินาที) แนะนำ:
- Hooks แบบ sync: <200ms
- Hooks แบบ async: ≤30 วินาที

## ตัวอย่างการกำหนดค่าที่สมบูรณ์

### ตัวอย่าง PreToolUse Hook

```json
{
  "matcher": "Bash",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/pre-bash-dispatcher.js"
    }
  ],
  "description": "ตัวจัดส่ง pre-check ของ Bash สำหรับการตรวจสอบคุณภาพ, tmux, push และ GateGuard",
  "id": "pre:bash:dispatcher"
}
```

### ตัวอย่าง PostToolUse Hook

```json
{
  "matcher": "Edit|Write|MultiEdit",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/run-with-flags.js post:quality-gate scripts/hooks/quality-gate.js standard,strict",
      "async": true,
      "timeout": 30
    }
  ],
  "description": "รันการตรวจสอบ quality gate หลังแก้ไขไฟล์",
  "id": "post:quality-gate"
}
```

### ตัวอย่าง Stop Hook

```json
{
  "matcher": "*",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/run-with-flags.js stop:session-end scripts/hooks/session-end.js minimal,standard,strict",
      "async": true,
      "timeout": 10
    }
  ],
  "description": "บันทึกสถานะเซสชันหลังทุกคำตอบ",
  "id": "stop:session-end"
}
```

### ตัวอย่าง SessionStart Hook

```json
{
  "matcher": "*",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/session-start-bootstrap.js"
    }
  ],
  "description": "โหลดบริบทก่อนหน้าและตรวจจับตัวจัดการแพ็คเกจของเซสชันใหม่",
  "id": "session:start"
}
```

## รูปแบบการกำหนดค่าเหตุการณ์วงจรชีวิต

สำหรับเหตุการณ์วงจรชีวิตเช่น SessionStart, SessionEnd, PreCompact ให้ใช้โครงสร้างการกำหนดค่าที่แตกต่าง:

```json
{
  "description": "คำนิยาม lifecycle hook สำหรับการคงสภาพหน่วยความจำ",
  "events": [
    {
      "event": "SessionStart",
      "id": "session:start",
      "script": "scripts/hooks/session-start-bootstrap.js",
      "purpose": "โหลดบริบทก่อนหน้าที่จำกัดและตรวจจับสถานะโครงการเมื่อเริ่มเซสชัน",
      "blocking": false
    },
    {
      "event": "PreCompact",
      "id": "pre:compact",
      "script": "scripts/hooks/pre-compact.js",
      "purpose": "บันทึกสถานะเซสชันก่อนบีบอัดบริบท",
      "blocking": false
    }
  ]
}
```

### ฟิลด์เหตุการณ์วงจรชีวิต

| ฟิลด์ | คำอธิบาย |
|------|--------------|
| `event` | ประเภทเหตุการณ์ (SessionStart, PreCompact, SessionEnd ฯลฯ) |
| `id` | ตัวระบุที่ไม่ซ้ำกัน |
| `script` | เส้นทางสคริปต์ |
| `purpose` | คำอธิบายวัตถุประสงค์ |
| `blocking` | บล็อกได้หรือไม่ (ปกติเป็น false) |

## การปิดใช้งาน Hooks

### การปิดใช้งานโดยการแก้ไขการกำหนดค่า

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write",
        "hooks": [],
        "description": "แทนที่: อนุญาตการสร้างไฟล์ .md ทั้งหมด"
      }
    ]
  }
}
```

### การปิดใช้งานโดยตัวแปรสภาพแวดล้อม

```bash
# ปิดใช้งาน Hooks เฉพาะ (คั่นด้วย comma)
export ECC_DISABLED_HOOKS="pre:bash:tmux-reminder,post:edit:typecheck"

# ปิดใช้งาน GateGuard
export ECC_GATEGUARD=off
```

---

## ระเบียบการตั้งชื่อ Hook ID

ECC ใช้ระเบียบการตั้งชื่อที่คั่นด้วยโคลอน:

| คำนำหน้า | วัตถุประสงค์ | ตัวอย่าง |
|------|------|------|
| `pre:` | PreToolUse Hooks | `pre:bash:dispatcher` |
| `post:` | PostToolUse Hooks | `post:quality-gate` |
| `stop:` | Stop Hooks | `stop:session-end` |
| `session:` | Session lifecycle Hooks | `session:start` |

---

## ตัวห่อ run-with-flags.js

ECC ใช้ตัวห่อ `run-with-flags.js` เพื่อรัน Hooks พร้อมการกำหนดค่า runtime gating:

```
node scripts/hooks/run-with-flags.js <hook-id> <script-path> <profiles>
```

### คำอธิบายพารามิเตอร์
| พารามิเตอร์ | คำอธิบาย |
|------|------|
| `hook-id` | ตัวระบุ Hook ที่ไม่ซ้ำกัน |
| `script-path` | เส้นทางสคริปต์ที่จะรันจริง |
| `profiles` | รายการโปรไฟล์ที่คั่นด้วย comma (minimal, standard, strict) |

### วิธีการทำงาน
1. อ่านตัวแปรสภาพแวดล้อม `ECC_HOOK_PROFILE` (ค่าเริ่มต้น: standard)
2. ตรวจสอบว่า Hook ID อยู่ใน `ECC_DISABLED_HOOKS` หรือไม่
3. ถ้าอนุญาต รันสคริปต์

## การจัดการเส้นทางข้ามแพลตฟอร์ม

คำสั่ง Hook ของ ECC ใช้สคริปต์ Node.js เพื่อทำงานข้ามแพลตฟอร์ม:

```javascript
const path = require('path');
const homedir = require('os').homedir();

// การแก้ไขเส้นทางข้ามแพลตฟอร์ม
const configDir = path.join(homedir, '.claude');
const hookScript = path.join(configDir, 'scripts', 'hooks', 'my-hook.js');
```

