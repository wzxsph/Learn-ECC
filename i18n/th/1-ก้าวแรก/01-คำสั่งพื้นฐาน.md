# 03 - 基础命令

## 命令类型

ECC 提供三种命令类型：

### 1. 斜杠命令（Slash Commands）

直接输入使用：

| 命令 | 功能 | 示例 |
|------|------|------|
| `/ecc:plan` | 创建实施计划 | `/ecc:plan "添加用户认证"` |
| `/ecc:code-review` | 代码审查 | `/ecc:code-review` |
| `/ecc:build-fix` | 修复构建错误 | `/ecc:build-fix` |
| `/ecc:learn` | 从会话提取模式 | `/ecc:learn` |
| `/ecc:skill-create` | 从 Git 历史生成 Skill | `/ecc:skill-create` |
| `/ecc:sessions` | 会话历史管理 | `/ecc:sessions` |

注意：手动安装版本可能使用不带前缀的命令（如 `[/plan](../เอกสารอ้างอิง/commands/01-Workflowหลัก.md)`），插件安装使用 `/ecc:` 前缀。

### 2. Skills（主要工作流表面）

Skills 是 ECC 的主要工作流调用方式：

| Skill | 功能 |
|-------|------|
| `[tdd-workflow](../เอกสารอ้างอิง/skills/测试.md)` | 测试驱动开发 |
| `[e2e-testing](../เอกสารอ้างอิง/skills/测试.md)` | 端到端测试 |
| `security-review` | 安全审查 |
| `verification-loop` | 验证循环 |
| `search-first` | 研究优先工作流 |

调用方式：
```
使用 [tdd-workflow](../เอกสารอ้างอิง/skills/测试.md) skill
```

### 3. Agent 命令

通过 Agent 执行子任务：

| Agent | 用途 |
|-------|------|
| `planner` | 实施规划 |
| `code-reviewer` | 代码审查 |
| `build-error-resolver` | 构建错误修复 |
| `security-reviewer` | 安全审查 |
| `architect` | 架构设计 |

调用方式：
```
@planner
```

## 常用工作流

### 开发新功能

```
/ecc:plan "添加用户认证功能"
→ planner 创建实施蓝图
使用 [tdd-workflow](../เอกสารอ้างอิง/skills/测试.md) skill
→ tdd-guide 强制先写测试
/ecc:code-review
→ code-reviewer 检查代码
```

### 修复 Bug

```
使用 [tdd-workflow](../เอกสารอ้างอิง/skills/测试.md) skill
→ tdd-guide: 编写复现测试
→ 实施修复，验证测试通过
/ecc:code-review
→ code-reviewer: 检查回归
```

### 准备上线

```
/ecc:security-scan
→ security-reviewer: OWASP Top 10 审计
使用 [e2e-testing](../เอกสารอ้างอิง/skills/测试.md) skill
→ e2e-runner: 关键用户流测试
/ecc:test-coverage
→ 验证 80%+ 覆盖率
```

## MCP 命令

| 命令 | 功能 |
|------|------|
| `/mcp` | 管理 MCP 服务器 |
| `/ecc:status` | 检查状态 |
