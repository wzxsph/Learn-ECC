# แบบฝึกหัด

ทำแบบฝึกหัดต่อไปนี้เพื่อสร้างความเข้าใจพื้นฐาน ECC

## แบบฝึกหัด 1: การตรวจสอบสภาพแวดล้อม

**เป้าหมาย**: ยืนยันว่าติดตั้ง ECC สภาพแวดล้อมถูกต้อง

**ขั้นตอน**:
1. รันชุดทดสอบ: `node tests/run-all.js`
2. ตรวจสอบว่าทุกทดสอบผ่าน
3. แสดงโครงสร้างไดเรกทอรี `.claude/`

**มาตรฐานการยอมรับ**:
- ทดสอบทั้งหมดผ่าน
- สามารถเห็นไดเรกทอรี agents, skills, hooks, rules

---

## แบบฝึกหัด 2: การใช้คำสั่ง slash

**เป้าหมาย**: ทำความคุ้นเคยกับคำสั่ง slash ที่ใช้บ่อย

**ขั้นตอน**:
1. เปิด Claude Code
2. ทำตามคำสั่งต่อไปนี้ทีละขั้นตอน สังเกตผลลัพธ์:
   - `/help` หรือ `/ecc:plan`
   - `/ecc:code-review`
3. บันทึกรูปแบบผลลัพธ์ของแต่ละคำสั่ง

**มาตรฐานการยอมรับ**:
- เข้าใจฟังก์ชันและรูปแบบผลลัพธ์ของแต่ละคำสั่ง

---

## แบบฝึกหัด 3: สร้าง Agent ง่ายๆ

**เป้าหมาย**: เข้าใจรูปแบบไฟล์ Agent

**ขั้นตอน**:
1. ดูไฟล์ Agent ที่มีอยู่ในไดเรกทอรี `agents/`
2. ศึกษารูปแบบของ `planner.md`
3. สร้าง `hello-agent.md` ง่ายๆ ที่มีฟังก์ชันแสดงข้อความทักทาย

**มาตรฐานการยอมรับ**:
- ไฟล์ Agent ใหม่มีรูปแบบถูกต้อง
- มี YAML frontmatter (name, description, tools, model)

---

## แบบฝึกหัด 4: รัน Hook

**เป้าหมาย**: เข้าใจกลไก Hook

**ขั้นตอน**:
1. ดูการตั้งค่า Hook ในไดเรกทอรี `hooks/`
2. รัน `node scripts/hooks/run-with-flags.js --help` (ถ้ารองรับ)
3. เข้าใจจังหวะการ trigger และตรรกะการ execute ของ Hook

**มาตรฐานการยอมรับ**:
- สามารถอธิบายหลักการทำงานของ Hook
- เข้าใจความแตกต่างระหว่าง PreToolUse และ PostToolUse

---

## แบบฝึกหัด 5: ภารกิจบูรณาการ

**เป้าหมาย**: ทำโครงการเล็กๆ ให้เสร็จ

**ภารกิจ**: สร้างเครื่องมือจัดรูปแบบชื่อคน

**ข้อกำหนดฟังก์ชัน**:
- `formatName(name)` - จัดรูปแบบเป็น "นามสกุล, ชื่อ"
- `initials(name)` - ส่งกลับตัวอักษรย่อ

**ขั้นตอนการดำเนินการ**:
1. ใช้ `/ecc:plan` วางแผน
2. ใช้ `/tdd` พัฒนา
3. ใช้ `/ecc:code-review` review

**มาตรฐานการยอมรับ**:
- Implement ทั้งสองฟังก์ชันครบถ้วน
- มี test case
- ผ่าน code review
