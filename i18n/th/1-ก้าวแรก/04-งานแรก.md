# 04-งานแรก

## 学习目标

- 理解完整的开发流程
- 掌握 /ecc:plan → /ecc:code-review → /ecc:build-fix → 提交 的完整流程
- 能够独立完成一个简单功能的开发

---

## 完整的开发流程

ECC 推荐使用以下开发流程：

```
Step 1: /ecc:plan        → 规划任务
Step 2: /ecc:code-review → 审查代码
Step 3: /ecc:build-fix   → 修复构建
Step 4: 提交代码     → Git 提交
```

---

## Step 1: 使用 /ecc:plan 规划

### 输入

假设我们要实现一个简单的计算器功能：

```
/ecc:plan 实现一个计算器，支持加减乘除四种运算
```

### 预期输出

/ecc:plan 命令会：

1. **复述需求** - "实现一个计算器，支持加法、减法、乘法、除法四种基本运算..."
2. **风险评估** - "需要注意：除法除以零的情况..."
3. **分步骤计划** -
   - Phase 1: 创建计算器核心函数
   - Phase 2: 实现四种运算
   - Phase 3: 添加错误处理
   - Phase 4: 编写单元测试
4. **等待确认** - "请确认以上计划是否可以开始"

### 确认后

用户确认后，ECC 会等待用户输入开始每个阶段。

### 小技巧

- 使用 PRD 文件模式可以获得更详细的规划
- 复杂功能建议先写 PRD 再规划

---

## Step 2: 使用 /ecc:code-review 审查

### 输入

代码编写完成后，运行：

```
/ecc:code-review
```

### 预期输出

/ecc:code-review 会：

1. 收集变更文件
2. 进行 7 个类别的审查：
   - 正确性
   - 类型安全
   - 模式合规
   - 安全
   - 性能
   - 完整性
   - 可维护性
3. 生成审查报告，标注 CRITICAL/HIGH/MEDIUM/LOW 级别

### 审查报告示例

```
## 代码审查报告

### CRITICAL
- [ ] 硬编码凭证：src/auth.js 第 23 行

### HIGH
- [ ] 函数过长：src/calculator.js 第 45-78 行（34 行）
- [ ] 缺少错误处理：src/calculator.js 除法函数

### MEDIUM
- [ ] 命名不规范：变量 `tmp` 应改为 `temporary`

### 操作建议
1. 移除硬编码的 API Key
2. 重构 calculator.js 的除法函数，添加除零检查
3. 拆分过长的函数
```

### 小技巧

- 提交前务必运行 /ecc:code-review
- CRITICAL 和 HIGH 问题必须修复

---

## Step 3: 使用 /ecc:build-fix 修复

### 输入

如果构建失败，运行：

```
/ecc:build-fix
```

### 预期输出

/ecc:build-fix 会：

1. **检测构建系统** - "检测到 TypeScript 项目"
2. **运行构建** - "运行 tsc --noEmit..."
3. **解析错误** - 按文件分组
4. **逐步修复** - 一次修复一个错误
5. **验证** - 每次修复后重新构建

### 示例输出

```
[build-fix] 检测到 TypeScript 项目
[build-fix] 运行 tsc --noEmit...
[build-fix] 发现 3 个错误：

错误 1: src/calculator.ts:15 - 类型 'string' 不能赋给类型 'number'
错误 2: src/calculator.ts:28 - 属性 'result' 不存在
错误 3: src/calculator.ts:42 - 参数 'b' 从不使用

[build-fix] 开始修复错误 1...
[build-fix] 修复完成，验证中...
[build-fix] 构建成功！
```

### 小技巧

- 构建修复是增量的，一次修一个错误
- 可以随时 Ctrl+C 停止

---

## Step 4: 提交代码

### 提交流程

1. 确保所有测试通过
2. 确保构建成功
3. 确保代码审查问题已修复

### Git 提交规范

ECC 使用 conventional commits 格式：

```
<type>: <description>

[optional body]
```

**Type 类型**:
- `feat` - 新功能
- `fix` - 修复 bug
- `refactor` - 重构
- `docs` - 文档
- `test` - 测试
- `chore` - 日常维护
- `perf` - 性能优化
- `ci` - CI 配置

### 提交示例

```
feat: 实现计算器四种基本运算

- 添加加法函数
- 添加减法函数
- 添加乘法函数
- 添加除法函数（含除零检查）

解决：Issue #123
```

### 小技巧

- 使用 `/ecc:verify quick` 提交前快速验证
- 大功能分多次提交，小功能一次提交

## 完整示例

### 场景

实现一个简单的字符串工具函数：`truncate(str, maxLength)`，将字符串截断到指定长度。

### Step 1: 规划

```
/ecc:plan 实现 truncate 函数，将字符串截断到指定长度
```

输出：
```
## 需求重述
实现 truncate(str, maxLength) 函数，超过 maxLength 的部分用 ... 替代

## 风险评估
- maxLength 小于 3 时的处理
- str 为空或 null 的处理
- Unicode 字符的处理

## 分步骤计划
1. 创建 src/string-utils.ts
2. 实现 truncate 函数
3. 添加边界情况测试
4. 添加 JSDoc 文档

请确认是否可以开始。
```

### Step 2: 实现

（用户确认后，ECC 协助实现代码）

### Step 3: 审查

```
/ecc:code-review
```

输出：
```
## 代码审查报告
状态：APPROVED

### 审查结果
- 正确性：通过
- 类型安全：通过
- 模式合规：通过
- 安全：通过
- 性能：通过
- 完整性：建议添加更多边界测试
- 可维护性：通过

### 建议
- 添加空字符串测试
- 添加 null 值测试
```

### Step 4: 构建验证

```
/ecc:build-fix
```

```
[build-fix] 检测到 TypeScript 项目
[build-fix] 运行 tsc --noEmit...
[build-fix] 构建成功！
```

### Step 5: 提交

```
feat: 实现 truncate 字符串截断函数

- 支持指定最大长度
- 超出部分用 ... 替代
- 处理空字符串和 null 值
```

---


## 小结

1. **完整流程**: /ecc:plan → 实现 → /ecc:code-review → /ecc:build-fix → 提交
2. **/ecc:plan** 用于规划，确保方向正确
3. **/ecc:code-review** 用于审查，发现问题
4. **/ecc:build-fix** 用于修复构建错误
5. **提交** 使用 conventional commits 格式
