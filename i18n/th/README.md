## 背景故事

### 为什么会有这个项目？

我英语不太好，阅读 ECC 的英文文档比较吃力。所以我把 ECC 中的命令（Commands）、智能体（Agents）、技能（Skills）等的说明翻译成了中文，整理成这套学习课程。

เอกสารอ้างอิง就是这么来的 —— 就是把 ECC 原项目里的内容翻译了一下，告诉你每个命令、每个 Agent 是干什么用的。

### 课程是怎么来的？

坦白说，这套课程是 AI 生成的。AI 根据 ECC 的文件结构生成了基础内容，但缺少详细的思考和实际评估。如果你发现内容有问题或者不够清晰，欢迎提出修改建议。

## 核心概念

### ECC 是什么？

**ECC** (Everything Claude Code) 是一个 Claude Code 插件项目（[原始仓库](https://github.com/affaan-m/ECC)），提供了：
- **75 个命令** - 斜杠命令（如 `/plan`、`/code-review`）和脚本命令
- **60 个专业 Agent** - 代码审查、构建修复、架构设计等
- **232+ Skills** - 可复用的工作流模板
- **Hooks 系统** - 事件驱动的自动化机制
- **Rules 规则** - 强制性的开发规范

### Learn-ECC 是什么？

**Learn-ECC** 是我为了学习 ECC 而整理的一套系统化学习路径，不是 ECC 本身。

它采用「做中学」的设计理念，每个阶段都包含理论学习和实战练习。通过完成真实项目，你将掌握 Claude Code 的核心能力。

### เอกสารอ้างอิง是什么？

**เอกสารอ้างอิง**（./เอกสารอ้างอิง/）是对 ECC 原项目的中文翻译。我把 ECC 的英文文档翻译成中文，方便自己（和你）查阅。

内容包括：
- 完整的命令说明（每个命令是干什么的）
- 所有 Agent 的详细介绍
- Skills 目录结构和编写规范
- Hooks 系统的完整文档
- 最佳实践和设计模式

## 如何使用本课程

### 推荐学习顺序

1. **按阶段顺序学习** - 每个阶段建立在前一阶段的基础上
2. **理论 + 实践结合** - 每章都有配套练习，先理解再动手
3. **完成所有练习** - 练习是学习路径的核心部分

### 学习建议

- **入门阶段**：建议 2-3 小时完成，重点熟悉基本概念和操作
- **核心能力**：建议 8-12 小时，每个模块都要亲手实践
- **专家之路**：建议 6-8 小时，需要一定的项目经验
- **实战项目**：建议 10-15 小时，选择感兴趣的项目深入

### 学习检查点

完成每个阶段后，确认你已掌握：
- 能够独立完成该阶段的所有操作
- 能够向他人解释核心概念
- 能够举一反三应用于其他场景

## 成功技巧

1. **循序渐进**: 不要跳阶段学习，每个Stage都是下一个的基础
2. **动手实践**: 每学一个概念立即实践，不要只看不动手
3. **完成练习**: 每个Stage都有配套练习，认真完成才能巩固知识
4. **使用速查表**: 日常使用中善用速查表，不要死记硬背
5. **项目驱动**: Stage 4 用真实项目串联所有知识
6. **做笔记**: 记录学习心得和遇到的问题
7. **教他人**: 能教会别人才算掌握

## 常见问题 (FAQ)

### Q: 需要编程基础吗？

A: 需要基本的命令行和编程知识（JavaScript/Node.js 基础有助于理解部分功能），但不需要精通任何特定语言。入门阶段为零基础设计。

### Q: 可以跳过某些阶段吗？

A: Stage 1-3 建议按顺序完成，Stage 4 可以根据需要选择感兴趣的项目，Stage 5 随时可查。

### Q: 学完后能做什么？

A: 掌握生产级 Claude Code 工作流，能高效进行代码开发、审查、测试和自动化；能设计和实现自定义Agent系统；能定制Hooks和Skills。

### Q: 学习过程中遇到问题怎么办？

A: 1) 查阅 [เอกสารอ้างอิง](./เอกสารอ้างอิง/) 相关章节；2) 查看 [命令速查](./5-อ้างอิงเร็ว/cheatsheet-commands.md)；3) 提交 Issue 寻求帮助。

### Q: 如何验证学习效果？

A: 使用 [progress-tracker.md](./progress-tracker.md) 追踪进度，每个阶段都有明确的学习目标，完成所有练习即表示掌握该阶段内容。

### Q: 课程内容会更新吗？

A: 会持续更新，欢迎提交 PR 贡献内容。详细指南见 [contributing.md](./contributing.md)。
