# Оркестрация агентов

## 可用 Agent

位于 `~/.claude/agents/`:

| Agent | Назначение | Когда использовать |
|-------|------|----------|
| planner | Планирование реализации | 复杂功能、重构 |
| architect | Проектирование системы | 架构决策 |
| tdd-guide | Разработка через тестирование | 新功能、bug 修复 |
| code-reviewer | Код-ревью | 编写代码后 |
| security-reviewer | Анализ безопасности | 提交前 |
| build-error-resolver | Исправление ошибок сборки | 构建失败时 |
| e2e-runner | E2E тестирование | 关键用户流程 |
| refactor-cleaner | Очистка мёртвого кода | 代码维护 |
| doc-updater | Обновление документации | 更新文档时 |
| rust-reviewer | Rust Код-ревью | Rust 项目 |
| harmonyos-app-resolver | Разработка HarmonyOS приложений | HarmonyOS/ArkTS 项目 |

## 立即使用 Agent

无需用户Подсказка：
1. 复杂功能请求 → 使用 **planner** agent
2. 代码刚编写/修改 → 使用 **code-reviewer** agent
3. Bug 修复或新功能 → 使用 **tdd-guide** agent
4. 架构决策 → 使用 **architect** agent

## Параллельное выполнение задач

对于独立操作，始终使用并行 Task 执行：

```markdown
# 好：Параллельное выполнение
启动 3 个 agent 并行：
1. Agent 1: 认证模块Анализ безопасности
2. Агент 2: Проверка производительности кэш-системы
3. Agent 3: 工具Тип检查

# Плохо: Ненужное последовательное выполнение
Сначала agent 1, потом agent 2, потом agent 3
```

## Мультиперспективный анализ

对于复杂问题，使用分Роль sub-agents：
- 事实审查员
- 高级工程师
- Безопасность专家
- 一致性审查员
- 冗余检查员
