# Правила тестирования

## 规则Обзор

ECC Rules Правила тестирования定义了确保Качество кода的强制性Тестирование标准。该规则涵盖Разработка через тестирование(TDD)、最低覆盖率要求和Тестирование结构规范，确保所有代码都经过充分Проверка。

## 核心要求

### 最低Тестирование覆盖率：80%

Обязательно包含以下三种Тестирование型：

| Тестирование型 | Описание | 覆盖范围 |
|----------|------|----------|
| 单元Тестирование | 独立函数、工具类、组件 | Individual units |
| 集成Тестирование | API端点、数据库操作 | Integration points |
| E2E тестирование | 关键用户流程 | Critical user flows |

### Разработка через тестирование（TDD）

**强制工作流**：

```
1. 编写Тестирование (RED)   - 先写一个会失败的Тестирование
2. 运行Тестирование          - ПроверкаТестирование失败
3. 编写最小Реализовать (GREEN) - 编写ЧерезТестирование的最小代码
4. 运行Тестирование          - ПроверкаТестированиеЧерез
5. 重构 (IMPROVE)    - 改进代码结构
6. Проверка覆盖率        - 确保达到80%+
```

## 实施细节

### AAA模式（Arrange-Act-Assert）

所有ТестированиеОбязательно遵循AAA结构：

```typescript
test('正确计算余弦相似度', () => {
  // Arrange - 准备Тестирование数据
  const vector1 = [1, 0, 0];
  const vector2 = [0, 1, 0];

  // Act - 执行被Тестирование的操作
  const similarity = calculateCosineSimilarity(vector1, vector2);

  // Assert - Проверка结果
  expect(similarity).toBe(0);
});
```

### Тестирование命名规范

使用Описание性НазваниеОписание被Тестирование的行为：

```typescript
// 推荐 - Описание预期行为
test('当没有市场匹配查询时Назад空数组', () => {});
test('当API密钥缺失时抛出错误', () => {});
test('当Redis不可用时回退到子字符串搜索', () => {});

// 避免 - 模糊命名
test('test1', () => {});
test('edge case', () => {});
```

### Тестирование故障排除

当Тестирование失败时：

| Шаги | 操作 |
|------|------|
| 1 | 使用 **tdd-guide** agent |
| 2 | 检查Тестирование隔离 |
| 3 | ПроверкаMock正确性 |
| 4 | 修复Реализовать，而非Тестирование（除非Тестирование本身有误） |

### AgentПоддерживать

| Agent | Назначение | Когда использовать |
|-------|------|----------|
| **tdd-guide** | Разработка через тестирование指导 | 新功能开发、错误修复时强制使用 |
| **code-reviewer** | Код-ревью | 代码编写Завершить后 |

## 违规处理

### 覆盖率不足

- 覆盖率低于80%的代码不得合并
- CI/CD流水线应阻止低覆盖率代码提交
- 使用 `c8` 或类似工具生成覆盖率报告

### Тестирование失败处理

```
1. 分析失败原因
2. 诊断是Реализовать问题还是Тестирование问题
3. 如果是Реализовать问题 - 修复Реализовать代码
4. 如果是Тестирование问题 - 修复Тестирование代码
5. 确保所有ТестированиеЧерез后再提交
```

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| FlakyТестирование | 异步操作、竞态条件 | 增加等待时间、使用mock |
| Тестирование隔离失败 | 共享状态污染 | 每个Тестирование前重置状态 |
| Mock不正确 | 期望值与实际不符 | Проверкаmock设置是否正确 |
