## Python Patterns (python-patterns)

### НазначениеОписание
ПредоставлятьPythonic惯用模式、PEP 8标准、ТипПодсказка和Лучшие практики，Используется для构建健壮、高效和可维护的Python应用程序。

### Когда использовать
- 编写新的Python代码
- 审查Python代码
- 重构现有Python代码
- СпроектироватьPython包/模块

### Основные концепции
1. **可读性优先** - 代码Следует清晰明显
2. **显式优于隐式** - 避免魔法操作
3. **EAFP风格** - 更容易请求原谅而不是许可
4. **ТипПодсказка** - 使用Тип注解提高代码Безопасность性
5. **上下文Управлять器** - 使用`with`进行资源Управлять
6. **列表推导式** - 简单的数据转换
7. **生成器** - 惰性求值和大数据集

### 使用Пример

**ТипПодсказка：**
```python
from typing import Optional, List, Dict, Any

def process_user(
    user_id: str,
    data: Dict[str, Any],
    active: bool = True
) -> Optional[User]:
    if not active:
        return None
    return User(user_id, data)
```

**上下文Управлять器：**
```python
from contextlib import contextmanager

@contextmanager
def timer(name: str):
    start = time.perf_counter()
    yield
    elapsed = time.perf_counter() - start
    print(f"{name} took {elapsed:.4f} seconds")

with timer("data processing"):
    process_large_dataset()
```

**Обработка ошибок：**
```python
try:
    parsed = json.loads(data)
except json.JSONDecodeError as e:
    raise ValueError(f"Failed to parse data: {data}") from e
```

### НазначениеОписание

2. **零值有用** - СпроектироватьТип使其零值可直接使用
3. **接受接口，Назад结构体** - 函数接受接口参数，Назад具体Тип

6. **接口Спроектировать** - 小而专注的接口
7. **包组织** - 标准项目布局

### 使用Пример

**错误包装：**
```go
func LoadConfig(path string) (*Config, error) {
    data, err := os.ReadFile(path)
    if err != nil {
        return nil, fmt.Errorf("load config %s: %w", path, err)
    }
    // ...
}
```

**Worker Pool：**
```go
func WorkerPool(jobs <-chan Job, results chan<- Result, numWorkers int) {
    var wg sync.WaitGroup
    for i := 0; i < numWorkers; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            for job := range jobs {
                results <- process(job)
            }
        }()
    }
    wg.Wait()
}
```

**功能选项模式：**
```go
type Option func(*Server)

func WithTimeout(d time.Duration) Option {
    return func(s *Server) { s.timeout = d }
}

func NewServer(addr string, opts ...Option) *Server {
    s := &Server{addr: addr, timeout: 30 * time.Second}
    for _, opt := range opts {
        opt(s)
    }
    return s
}
```

## Rust Patterns (rust-patterns)

### НазначениеОписание
Предоставлять惯用的Rust模式、所有权系统、Обработка ошибок、trait、并发和Лучшие практики，Используется для构建Безопасность、高效的应用程序。

### Когда использовать
- 编写新的Rust代码
- 审查Rust代码
- 重构现有Rust代码
- Спроектироватьcrate结构和模块布局

### Основные концепции
1. **所有权和借用** - 编译时防止数据竞争和内存错误
2. **Result和?操作符** - 使用`?`进行错误传播
3. **枚举和穷尽匹配** - 使用枚举使不可能的状态无法表示
4. **Trait和泛型** - 零成本抽象
5. **Безопасность并发** - 使用`Arc<Mutex<T>>`、channels和async/await
6. **最小pub表面** - 按领域组织模块

### 使用Пример

**Обработка ошибок：**
```rust
use anyhow::{Context, Result};

fn load_config(path: &str) -> Result<Config> {
    let content = std::fs::read_to_string(path)
        .with_context(|| format!("failed to read config from {path}"))?;
    let config: Config = toml::from_str(&content)
        .with_context(|| format!("failed to parse config from {path}"))?;
    Ok(config)
}
```

**使用Cow灵活处理所有权：**
```rust
use std::borrow::Cow;

fn normalize(input: &str) -> Cow<'_, str> {
    if input.contains(' ') {
        Cow::Owned(input.replace(' ', "_"))
    } else {
        Cow::Borrowed(input) // 无需修改时零成本
    }
}
```

**Arc<Mutex<T>>共享可变状态：**
```rust
use std::sync::{Arc, Mutex};

let counter = Arc::new(Mutex::new(0));
let handles: Vec<_> = (0..10).map(|_| {
    let counter = Arc::clone(&counter);
    std::thread::spawn(move || {
        let mut num = counter.lock().expect("mutex poisoned");
        *num += 1;
    })
}).collect();
```

## C++ Coding Standards (cpp-coding-standards)

### НазначениеОписание
基于C++核心指南的现代C++（C++17/20/23）编码标准。强制ТипБезопасность、资源Безопасность、不可变性和清晰度。

### Когда использовать
- 编写新的C++代码
- 审查或重构C++代码
- 做出架构决策
- 强制一致的Стиль кода
- 选择语言特性

### Основные концепции
1. **RAII无处不在** - 绑定资源生命周期到对象生命周期
2. **默认不可变** - 从`const`/`constexpr`Начать
3. **ТипБезопасность** - 使用Тип系统防止编译时错误
4. **表达意图** - Название、Тип和概念应传达目的
5. **最小化复杂度** - 简单代码是正确的代码
6. **值语义优先于指针语义** - 优先Назад值和作用域对象

### 使用Пример

**智能指针使用：**
```cpp
// R.11 + R.20 + R.21: RAII和智能指针
auto widget = std::make_unique<Widget>("config");  // 唯一所有权
auto cache = std::make_shared<Cache>(1024);         // 共享所有权

// R.3: 原始指针是非拥有的观察者
void render(const Widget* w) {  // 不拥有w
    if (w) w->draw();
}
```

**Rule of Zero：**
```cpp

struct Employee {
    std::string name;
    std::string department;
    int id;
    // 无需定义析构函数、复制/移动构造函数或赋值运算符
};
```

## Swift 6.2 Approachable Concurrency (swift-concurrency-6-2)

### НазначениеОписание
Swift 6.2的"可接近并发"模式——默认单线程，使用`@concurrent`显式后台卸载，Через隔离一致性为主actorТипПредоставлять协议Поддерживать。

### Когда использовать
- 将Swift 5.x或6.0/6.1项目迁移到Swift 6.2
- 解决数据竞争Безопасность编译器错误
- СпроектироватьMainActor-based应用架构
- Реализовать协议一致性

### Основные концепции
1. **单线程默认** - 大多数代码是数据竞争Безопасность的；并发是显式 opt-in
2. **Async保持在调用actor** - 消除导致数据竞争错误的隐式卸载
3. **隔离一致性** - MainActorТипМожноБезопасность地Реализовать非隔离协议
4. **@concurrent显式opt-in** - 后台执行是有意的性能选择

### 使用Пример

**隔离一致性：**
```swift
extension StickerModel: @MainActor Exportable {
    func export() {
        photoProcessor.exportAsPNG()
    }
}
```

**@concurrentИспользуется для后台工作：**
```swift
nonisolated final class PhotoProcessor {
    @concurrent
    static func extractSubject(from data: Data) async -> Sticker {
        // 后台执行的昂贵工作
    }
}
```

## Dart/Flutter Patterns (dart-flutter-patterns)

### НазначениеОписание
生产就绪的Dart和Flutter模式，涵盖空Безопасность、不可变状态、异步组合、组件架构、流行状态УправлятьФреймворки（BLoC、Riverpod、Provider）、GoRouter导航、Dio网络请求、Freezed代码生成和清洁架构。

### Когда использовать
- 启动新的Flutter功能并Требуется状态Управлять、导航或数据访问的模式
- 审查或编写Dart代码并Требуется空Безопасность、密封Тип或异步组合的指导
- 设置新的Flutter项目并选择BLoC、Riverpod或Provider
- РеализоватьБезопасность的HTTP客户端、WebView集成或本地存储
- 编写Flutter组件、Cubit或Riverpod provider的Тестирование

### Основные концепции
1. **空Безопасность** - 避免`!`，优先使用`?.`/`??`/模式匹配
2. **不可变状态** - 密封类、`freezed`、`copyWith`
3. **异步组合** - 并发`Future.wait`、await后Безопасность使用BuildContext
4. **组件架构** - 提取为类（不是方法）、`const`传播、作用域重建
5. **状态Управлять** - BLoC/Cubit事件、Riverpod notifiers和派生providers

### 使用Пример

**密封状态类：**
```dart
sealed class AsyncState<T> {}
final class Loading<T> extends AsyncState<T> {}
final class Success<T> extends AsyncState<T> { final T data; const Success(this.data); }
final class Failure<T> extends AsyncState<T> { final Object error; const Failure(this.error); }

Widget buildFrom(AsyncState<User> state) => switch (state) {
    UserInitial() => const SizedBox.shrink(),
    UserLoading() => const CircularProgressIndicator(),
    UserLoaded(:final user) => UserCard(user: user),
    UserError(:final message) => ErrorText(message),
};
```

**Riverpod派生provider：**
```dart
@riverpod
double cartTotal(Ref ref) {
    final cart = ref.watch(cartNotifierProvider);
    final products = ref.watch(productsProvider).valueOrNull ?? [];
    return cart.fold(0.0, (total, item) {
        final product = products.firstWhereOrNull((p) => p.id == item.productId);
        return total + (product?.price ?? 0) * item.quantity;
    });
}
```
