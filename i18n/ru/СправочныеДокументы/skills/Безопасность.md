## Безопасность审查

### НазначениеОписание
Безопасность审查技能确保所有代码遵循БезопасностьЛучшие практики，识别潜在漏洞。

### Когда использовать
- Реализовать身份Проверка或授权
- 处理用户输入或文件上传
- 创建新API端点
- 使用密钥或凭证
- Реализовать支付功能
- 存储或传输敏感数据
- 集成第三方API

### Основные концепции

**1. 密钥Управлять**
永远不在源代码中硬编码密钥。

**2. 输入Проверка**
所有用户输入ОбязательноПроверка。

**3. SQLЗащита от инъекций**
Использовать параметризованные запросы。

**4. XSS防护**
清理用户Предоставлять的HTML。

**5. CSRF保护**
在状态改变操作上使用CSRF令牌。

**6. 速率限制**
在所有API端点上启用速率限制。

### Безопасность检查清单

| 检查项 | Описание |
|--------|------|
| 密钥 | 无硬编码密钥，所有密钥在环境变量中 |
| 输入Проверка | 所有用户输入使用schemaПроверка |
| SQL注入 | 所有查询Использовать параметризованные запросы |
| XSS | 用户内容被清理 |
| CSRF | 启用保护 |
| 身份Проверка | 正确的令牌处理 |
| 授权 | Роль检查到位 |
| 速率限制 | 所有端点启用 |
| HTTPS | 生产环境强制使用 |
| Безопасность头 | CSP、X-Frame-Options配置 |
| Обработка ошибок | 错误消息不暴露敏感数据 |
| 日志 | 不记录敏感数据 |
| 依赖 | 更新，无漏洞 |

## Безопасность扫描

### НазначениеОписание
使用AgentShield扫描Claude Code配置（`.claude/`Содержание）中的Уязвимости безопасности、配置错误和注入风险。

### Когда использовать
- 设置新的Claude Code项目
- 修改`.claude/settings.json`、`CLAUDE.md`或MCP配置后
- 提交配置更改前
- 加入已有Claude Code配置的新仓库
- 定期Безопасность检查

### 扫描内容

| 文件 | 检查项 |
|------|--------|
| `CLAUDE.md` | 硬编码密钥、自动运行指令、Подсказка注入模式 |
| `settings.json` | 过度宽松的允许列表、缺少拒绝列表、危险绕过标志 |
| `mcp.json` | 风险MCP服务器、硬编码env密钥、npx供应链风险 |
| `hooks/` | Через插值的命令注入、数据泄露、静默错误抑制 |
| `agents/*.md` | 不受限的工具访问、Подсказка注入表面、缺少模型规格 |

### 使用Пример

```bash
# 基本扫描
npx ecc-agentshield scan

# 指定路径扫描
npx ecc-agentshield scan --path /path/to/.claude

# 最低严重性过滤
npx ecc-agentshield scan --min-severity medium

# JSON格式（Используется дляCI/CD）
npx ecc-agentshield scan --format json

# 自动修复
npx ecc-agentshield scan --fix

# Opus深度分析（ТребуетсяANTHROPIC_API_KEY）
export ANTHROPIC_API_KEY=your-key
npx ecc-agentshield scan --opus --stream
```

### 严重性级别

| 等级 | 分数 | 含义 |
|------|------|------|
| A | 90-100 | Безопасность配置 |
| B | 75-89 | 小问题 |
| C | 60-74 | Требуется关注 |
| D | 40-59 | 重大风险 |
| F | 0-39 | 严重漏洞 |

## DjangoБезопасность

### НазначениеОписание
DjangoБезопасностьЛучшие практики，保护应用程序免受常见漏洞攻击。

### Когда использовать
- 设置Django身份Проверка和授权
- Реализовать用户权限和Роль
- 配置生产Безопасность设置
- 审查Django应用程序Безопасность问题
- 部署Django应用程序到生产环境

### 核心Безопасность设置

```python
# settings/production.py
DEBUG = False  # 关键：生产环境绝不使用True

ALLOWED_HOSTS = os.environ.get('ALLOWED_HOSTS', '').split(',')

# Безопасность头
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True
SECURE_HSTS_SECONDS = 31536000  # 1年
SECURE_HSTS_INCLUDE_SUBDOMAINS = True
SECURE_HSTS_PRELOAD = True
SECURE_CONTENT_TYPE_NOSNIFF = True
SECURE_BROWSER_XSS_FILTER = True
X_FRAME_OPTIONS = 'DENY'

# CookieБезопасность
SESSION_COOKIE_HTTPONLY = True
CSRF_COOKIE_HTTPONLY = True
SESSION_COOKIE_SAMESITE = 'Lax'

# 密码Проверка
AUTH_PASSWORD_VALIDATORS = [
    {'NAME': 'django.contrib.auth.password_validation.MinimumLengthValidator', 'OPTIONS': {'min_length': 12}},
    {'NAME': 'django.contrib.auth.password_validation.CommonPasswordValidator'},
]
```

### SQLЗащита от инъекций

```python
# Безопасность：Django ORM自动转义参数
def get_user(username):
    return User.objects.get(username=username)  # Безопасность

# Безопасность：使用参数的raw()
def search_users(query):
    return User.objects.raw('SELECT * FROM users WHERE username = %s', [query])

# 危险：永远不要直接插值
def get_user_bad(username):
    return User.objects.raw(f'SELECT * FROM users WHERE username = {username}')  # 漏洞！
```

### XSS防护

```django
{# Django默认自动转义变量 - Безопасность #}
{{ user_input }}  {# HTML会被转义 #}

{# 显式标记为Безопасность仅Используется для可信内容 #}
{{ trusted_html|safe }}  {# 不转义 #}

{# 使用Шаблон过滤器Безопасность处理HTML #}
{{ user_input|striptags }}  {# 移除所有HTML标签 #}
```

### CSRF保护

```django
{# Шаблон中使用 #}
<form method="post">
    {% csrf_token %}
    {{ form.as_p }}
    <button type="submit">提交</button>
</form>

{# AJAX请求 #}
function getCookie(name) {
    let cookieValue = null;
    if (document.cookie && document.cookie !== '') {
        const cookies = document.cookie.split(';');
        for (let i = 0; i < cookies.length; i++) {
            const cookie = cookies[i].trim();
            if (cookie.substring(0, name.length + 1) === (name + '=')) {
                cookieValue = decodeURIComponent(cookie.substring(name.length + 1));
                break;
            }
        }
    }
    return cookieValue;
}

fetch('/api/endpoint/', {
    method: 'POST',
    headers: {
        'X-CSRFToken': getCookie('csrftoken'),
        'Content-Type': 'application/json',
    },
    body: JSON.stringify(data)
});
```
