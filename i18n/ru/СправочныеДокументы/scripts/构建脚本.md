## release.sh

**路径**: `scripts/release.sh`

插件版本升级和发布脚本。

### 功能

1. **版本Проверка**
   - 确认Предоставлять了 semver 格式版本号 (X.Y.Z 或 X.Y.Z-prerelease)
   - 确认当前分支是 main
   - 确认工作区干净

2. **文件更新**
   - 更新所有包/插件清单中的版本号
   - 更新文档中的版本引用
   - 更新 agent 配置

3. **Проверка构建**
   - 运行 OpenCode 构建
   - Запустить сборкуТестирование
   - 运行插件清单Тестирование

4. **Git 操作**
   - 暂存所有更改
   - 创建提交
   - 创建并推送标签

### Способ использования

```bash
# 发布新版本
./scripts/release.sh 1.5.0

# 预发布版本
./scripts/release.sh 2.0.0-rc.1
```

## build-opencode.js

**路径**: `scripts/build-opencode.js`

构建 OpenCode 插件的 TypeScript 代码。

### 构建流程

1. 清理 `dist` Содержание
2. 查找 TypeScript 编译器 (`typescript/bin/tsc`)
3. 使用项目根Содержание的 tsconfig.json 编译 `.opencode` Содержание

### Способ использования

```bash
# 构建 OpenCode
node scripts/build-opencode.js

# 作为 release 的一部分自动运行
./scripts/release.sh 1.5.0
```

### Предварительные требования

Требуется已安装根Содержание的开发依赖,确保 TypeScript 编译器可用。

## 发布工作流程

```
1. 确保在 main 分支且工作区干净
2. 运行发布脚本并Предоставлять版本号
3. 脚本自动Завершить:
   ├── ПроверкаПредварительные требования
   ├── 更新所有版本引用
   ├── 构建 OpenCode
   ├── 运行ТестированиеПроверка
   ├── 创建 Git 提交
   └── 推送标签
4. GitHub Actions 检测到标签自动发布
```
