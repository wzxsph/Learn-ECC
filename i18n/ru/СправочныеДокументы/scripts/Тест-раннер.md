# Тестовые раннеры

本文档介绍 ECC 项目的Тестирование结构和运行方式。

## Тестирование入口

**路径**: `tests/run-all.js`

集中式Тестовые раннеры,自动发现并运行所有Тестирование文件。

### Способ использования

```bash
# 运行所有Тестирование
node tests/run-all.js

# 运行单个Тестирование文件
node tests/lib/utils.test.js
node tests/hooks/hooks.test.js
```

### Тестирование文件发现

Тестовые раннерыЧерез glob 模式 `tests/**/*.test.js` 自动发现Тестирование文件。

规则:
- 文件Обязательно位于 `tests/` Содержание下
- 文件名Обязательно以 `.test.js` 结尾
- Содержание结构会被保留Используется для显示

Тестовые раннеры会解析每个Тестирование文件的输出来汇总统计:
- `Passed: <数量>`
- `Failed: <数量>`

## Тестирование结构

### Содержание布局

```
tests/
├── run-all.js              # 主Тестовые раннеры
├── lib/                    # 库工具Тестирование
│   ├── utils.test.js
│   ├── package-manager.test.js
│   └── ...
├── hooks/                  # Hook 集成Тестирование
│   └── hooks.test.js
├── integration/            # 集成Тестирование
├── scripts/                # 脚本Тестирование
├── commands/               # 命令Тестирование
├── docs/                   # 文档Тестирование
├── opencode-config.test.js
├── plugin-manifest.test.js
├── test_*.py               # Python Тестирование
└── conftest.py             # pytest 配置
```

### Тестирование型

1. **单元Тестирование** (`*.test.js`)
   - Тестирование独立函数和工具
   - 使用 Node.js 原生 assert 或 Jest 风格

2. **集成Тестирование** (`tests/integration/`)
   - Тестирование多个组件协作
   - Тестирование钩子完整流程

3. **Python Тестирование** (`test_*.py`)
   - pytest 格式的 Python Тестирование
   - 使用 `conftest.py` 配置

## Тестирование配置

### Node.js Тестирование

Тестирование文件使用标准的 Node.js assert 模块:

```javascript
const assert = require('assert');

test('example test', () => {
  assert.strictEqual(actual, expected, 'Optional message');
});
```

### Python Тестирование (pytest)

位置: `tests/conftest.py`

pytest 配置文件,Предоставлять共享的 fixtures 和配置。

```bash
# 运行 Python Тестирование
python -m pytest tests/

# 运行特定Тестирование
python -m pytest tests/test_builder.py
```

## ТестированиеЛучшие практики

### AAA 模式

```
test('ТестированиеОписание', () => {
  // Arrange - 准备Тестирование数据
  const input = 'test data';

  // Act - 执行被测操作
  const result = myFunction(input);

  // Assert - Проверка结果
  assert.strictEqual(result, expected);
});
```

### Тестирование命名

使用Описание性Название,清晰ОписаниеТестирование的行为:

```javascript
test('returns empty array when no markets match query', () => {});
test('throws error when API key is missing', () => {});
test('falls back to substring search when Redis is unavailable', () => {});
```

### Тестирование隔离

每个ТестированиеСледует:
- 独立运行,不依赖其他Тестирование
- 清理自己的Тестирование数据
- 使用 mock 隔离外部依赖

## CI 集成

在 `package.json` 中配置Тестирование脚本:

```json
{
  "scripts": {
    "test": "validate-commands.js && tests/run-all.js"
  }
}
```

Тестовые раннерыСпроектировать为可在 CI 环境中工作:
- 非零退出码表示失败
- JSON 输出Используется для自动化解析
- 清晰的错误消息Используется для调试
