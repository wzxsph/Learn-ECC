# ECC документациятело系

ECC (Everything Claude Code) является一个 Claude Code плагин，提供 75 个命令、60 个 Agent、16 个 Skills、7 个правиладокументацияи完整 Hook/MCP конфигурациятело系。

---

## 1. ECC документациятело系概述

ECC 为 Claude Code 提供生产级разработка工作流支持，涵盖多个 AI Agent фреймворки（Claude Code、Codex、OpenCode、Cursor、Gemini）。

### 技术栈

| 层级 | 技术 | версия |
|------|------|------|
| 主要语言 | JavaScript/Node.js | >=18 |
| 次要语言 | Python | >=3.11 |
| пакет管理器 | Yarn | 4.9.2 |
| тестированиезапуск器 | пользовательский Node.js тестированиезапуск器 | - |
| покрытие | c8 | 11.x |
| кодпроверка | ESLint + markdownlint-cli | - |

### каталог结构

```
ECC/
├── agents/        # 60 个专业 Agent
├── commands/      # 75 个命令файл
├── skills/        # 16 个 Skills каталог
├── rules/         # 7 个правиладокументация（common + 语言特定）
├── hooks/         # 3 个 Hook документация
├── mcp/           # 6 个 MCP серверконфигурация
├── scripts/       # 54 个утилитыскрипты
├── README.md
```

## 10. 贡献гайд

### файл命名规范

- **命令、Skills、Agents、Hooks**: 小写 + 连символ符（如 `code-review.md`）
- **скрипты**: camelCase 或 kebab-case（如 `session-start.js`）
- **правила**: 按语言/темаорганизация

### 命令файлформат

```yaml
---
description: "简短описание"
argument-hint: "[опциональныйпараметр]"
name: command-name
command: true
allowed_tools: ["Bash"]
---
```

### Agent файлформат

```yaml
---
name: agent-name
description: Agent 用途
tools: ["Read", "Grep", "Glob", "Bash"]
model: sonnet
---
```

### Skill файлформат

```markdown
# Skill 名称

## When to Use
## How It Works
## Examples
```

### 提交流程

1. 创建新файл或изменение现有файл
2. обеспечение遵循命名规范
3. запусктестирование: `node tests/run-all.js`
4. запуск markdown lint: `npx markdownlint-cli '**/*.md' --ignore node_modules`
5. 创建 PR запроспроверка
