## /python-review

**НазначениеОписание**: Python 代码全面审查。检查 PEP 8 合规性、ТипПодсказка、Безопасность性和 Python 习惯Использование。

**Способ использования**:
```
/python-review
```

**Сценарий использования**:
- 编写或修改 Python 代码之后
- 提交 Python 变更之前
- 审查包含 Python 代码的 PR
- 学习 Python 习惯Использование和Лучшие практики

**审查Категория**:

### CRITICAL (Обязательно修复)
- SQL/命令注入漏洞
- 不Безопасность的 eval/exec 使用
- Pickle 不Безопасность反序列化
- Жёстко закодированные учётные данные
- YAML 不Безопасность加载
- 隐藏错误的裸 except 子句

### HIGH (Следует修复)
- 公共函数缺少ТипПодсказка
- 可变默认参数
- 静默吞没异常
- 资源Управлять未使用上下文Управлять器
- C 风格循环而非推导式
- 使用 type() 而非 isinstance()

### MEDIUM (建议考虑)
- PEP 8 格式违规
- 公共函数缺少文档字符串
- Print 语句而非日志
- 低效的字符串操作
- 魔术数字未使用命名常量
- 未使用 f-strings

**自动检查命令**:
```bash
mypy .                              # Тип检查
ruff check .                        # Linting
black --check .                     # 格式化检查
isort --check-only .                # 导入排序
bandit -r .                         # Безопасность扫描
pip-audit                           # 依赖审计
pytest --cov=app --cov-report=term-missing  # Тестирование覆盖
```

**常见问题修复**:

```python
# SQL инъекция修复
# 错误
query = f"SELECT * FROM users WHERE id = {user_id}"

# 正确 - Использовать параметризованные запросы
query = "SELECT * FROM users WHERE id = %s"
cursor.execute(query, (user_id,))

# 可变默认参数修复
# 错误
def process_items(items=[]):
    items.append("new")
    return items

# 正确
def process_items(items=None):
    if items is None:
        items = []
    items.append("new")
    return items

# 使用上下文Управлять器
# 错误
f = open("config.json")
data = f.read()
f.close()

# 正确
with open("config.json") as f:
    data = f.read()
```

## /go-review

**НазначениеОписание**: Go 代码全面审查。检查惯用模式、并发Безопасность性、Обработка ошибок和Безопасность性。

**Способ использования**:
```
/go-review
```

**Сценарий использования**:
- 编写或修改 Go 代码之后
- 提交 Go 变更之前
- 审查包含 Go 代码的 PR
- 学习 Go 惯用模式

**审查Категория**:

### CRITICAL (Обязательно修复)
- SQL/命令注入漏洞
- 无同步的竞态条件
- Goroutine 泄漏
- Жёстко закодированные учётные данные
- 不Безопасность的指针使用
- 关键路径忽略错误

### HIGH (Следует修复)
- 缺少错误的上下文包装
- Panic 而非错误Назад
- Context 未传播
- 未缓冲通道导致死锁
- 接口未Реализовать错误
- 缺少互斥锁保护

### MEDIUM (建议考虑)
- 非惯用代码模式
- 导出函数缺少 godoc 注释
- 低效的字符串拼接
- Slice 未预分配
- 未使用表驱动Тестирование

**自动检查命令**:
```bash

staticcheck ./...                    # 高级检查（如果安装）
golangci-lint run                   # Linting
go build -race ./...                # 竞态检测
govulncheck ./...                   # Уязвимости безопасности
```

**НазначениеОписание**: Kotlin 代码全面审查。检查惯用模式、空Безопасность、协程Безопасность性和Безопасность性。

**Способ использования**:
```
/kotlin-review
```

**Сценарий использования**:
- 编写或修改 Kotlin 代码之后
- 提交 Kotlin 变更之前
- 审查 Android/KMP 项目的 PR
- 学习 Kotlin 惯用模式

**审查Категория**:

### CRITICAL (Обязательно修复)
- SQL/命令注入漏洞
- 无充分Обоснование的强制解包 `!!`
- 平台Тип空Безопасность违规
- GlobalScope 使用（结构化并发违规）

- 非惯用 Kotlin（Java 风格模式）

- 大集合链缺少 Sequence

## /rust-review

**НазначениеОписание**: Rust 代码全面审查。检查所有权生命周期、Обработка ошибок、不Безопасность使用和惯用模式。

**Способ использования**:
```
/rust-review
```

**Сценарий использования**:
- 编写或修改 Rust 代码之后
- 提交 Rust 变更之前
- 审查 Rust 项目的 PR
- 学习 Rust 惯用模式

**审查Категория**:

### CRITICAL (Обязательно修复)
- 生产代码路径中未检查的 `unwrap()`/`expect()`
- 不带 `// SAFETY:` 注释的 `unsafe`
- 字符串插值的 SQL инъекция
- 未Проверка输入的命令注入
- Жёстко закодированные учётные данные
- 原始指针的释放后使用

### HIGH (Следует修复)
- 不必要的 `.clone()` 以满足借用检查器
- `String` 参数应用 `&str` 或 `impl AsRef<str>`
- 异步上下文中的阻塞（`std::thread::sleep`）
- 共享Тип缺少 `Send`/`Sync` 约束
- 关键枚举上的通配符 `_ =>` 匹配
- 大函数（>50行）

### MEDIUM (建议考虑)
- 热路径中的不必要分配
- 已知大小时缺少 `with_capacity`
- 无充分Обоснование抑制 clippy Предупреждение
- 公共 API 缺少 `///` 文档
- 考虑对可能忽略值的非 `must_use` НазадТип使用 `#[must_use]`

**自动检查命令**:
```bash
cargo check                           # 构建检查
cargo clippy -- -D warnings            # Lints
cargo fmt --check                      # 格式检查
cargo test                            # Тестирование
cargo audit                           # Безопасность审计（如果可用）
```

## /cpp-review

**НазначениеОписание**: C++ 代码全面审查。检查内存Безопасность、现代 C++ 习惯Использование、并发和Безопасность性。

**Способ использования**:
```
/cpp-review
```

**Сценарий использования**:
- 编写或修改 C++ 代码之后
- 提交 C++ 变更之前
- 审查 C++ 项目的 PR
- 检查内存Безопасность问题

**审查Категория**:

### CRITICAL (Обязательно修复)
- 无 RAII 的裸 `new`/`delete`
- 缓冲区溢出和释放后使用
- 无同步的数据竞态
- Через `system()` 的命令注入
- 未初始化变量读取
- 空指针解引用

### HIGH (Следует修复)
- 五法则违规
- 缺少 `std::lock_guard` / `std::scoped_lock`
- detach 线程的寿命Управлять不当
- C 风格强制转换而非 `static_cast`/`dynamic_cast`
- 缺少 `const` 正确性

### MEDIUM (建议考虑)
- 不必要的拷贝（传值而非 `const&`）
- 已知大小时缺少 `reserve()`
- 头文件中 `using namespace std;`
- 重要Назад值缺少 `[[nodiscard]]`
- 过于复杂的Шаблон元编程

**自动检查命令**:
```bash
clang-tidy --checks='*,-llvmlibc-*' src/*.cpp -- -std=c++17
cppcheck --enable=all --suppress=missingIncludeSystem src/
cmake --build build -- -Wall -Wextra -Wpedantic
```

**НазначениеОписание**: FastAPI 应用审查。检查架构、异步正确性、依赖注入、Pydantic 模式、Безопасность性、性能和可Тестирование性。

**Способ использования**:
```

```

**审查区域**:
- App factory、路由边界、中间件和异常处理器
- Pydantic 请求和响应模式分离
- 数据库会话、认证、分页和设置的依赖注入
- 异步数据库和外部 HTTP 模式
- CORS、认证、限流、日志和密钥处理
- OpenAPI 元数据和文档化响应模型
- Тестирование客户端设置和依赖覆盖
