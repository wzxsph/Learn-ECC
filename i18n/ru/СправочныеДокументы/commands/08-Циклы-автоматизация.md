# Циклы и автоматизация命令

## Обзор

Циклы и автоматизация命令Используется для启动和Управлять自主循环工作模式。

## 命令列表

### /loop-start

**Назначение**: 启动托管自主循环模式，带Безопасность默认设置和明确停止条件

**Описание**: 启动带Безопасность默认设置和明确停止条件的托管自主循环模式。

**Использование**: `/loop-start [pattern] [--mode safe|fast]`

**参数**:

| 参数 | 值 | Описание |
|---|---|---|
| `pattern` | `sequential` | 顺序执行Задача，一个接一个 |
| | `continuous-pr` | 持续 PR 创建和合并循环 |
| | `rfc-dag` | 基于 RFC 的有向无环图工作流 |
| | `infinite` | 无限循环（需明确停止条件） |
| `--mode` | `safe`（默认） | 严格质量门禁和检查点 |
| | `fast` | 为速度减少门禁 |

**工作流**:
1. 确认仓库状态和分支策略
2. 选择循环模式和相关模型层级策略
3. 为所选模式启用所需 hooks/profile
4. 创建循环计划并在 `.claude/plans/` 下写 runbook
5. 打印启动和监控命令

**必需Безопасность检查**:
- ПроверкаТестирование在首次循环前Через
- 确保 `ECC_HOOK_PROFILE` 未全局禁用
- 确保循环有明确停止条件

**参数传递**:
- `$ARGUMENTS`: `[pattern]` 和可选的 `[--mode safe|fast]`
- 模式是可选的（默认为 sequential）
- mode 标志是可选的（默认为 safe）

**Лучшие практики**:
- 在 infinite 模式之前确保设置明确的停止条件
- fast 模式仅在充分Тестирование的代码上使用
- 监控循环进度，使用 `/loop-status` 检查状态

**常见陷阱**:
- 在未ПроверкаТестирование的情况下启动循环
- 使用 infinite 模式而无明确停止条件
- 在全局禁用 hooks 时尝试使用Безопасность模式

**与Другие команды集成**:
- 使用 `/loop-status` 监控运行中循环
- 使用 `/santa-loop` 进行 Santa 风格循环
- 使用 `/checkpoint` 创建关键节点检查点
