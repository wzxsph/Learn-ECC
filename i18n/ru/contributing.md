# Learn-ECC Руководство по вкладу

Мы рады вашим вкладам в Learn-ECC! Это руководство объясняет, как добавлять новый контент, улучшать существующий или отправлять изменения.

## Типы вкладов

### 1. Добавление нового контента

Типы контента, которым можно содействовать:
- Новые главы или модули
- Новые упражнения
- Новые практические проекты
- Дополнения к шпаргалкам
- Улучшения перевода

### 2. Улучшение существующего контента

- Исправление ошибок
- Улучшение четкости объяснений
- Добавление дополнительных примеров
- Улучшение ответов на упражнения
- Обновление устаревшей информации

### 3. Сообщения о проблемах

- Опечатки или грамматические ошибки
- Неработающие примеры кода
- Неработающие ссылки
- Отсутствующий контент

## Стандарты формата контента

### Markdown формат

Весь контент должен быть в формате Markdown и содержать следующие разделы:

```markdown
# Заголовок

## Цели обучения

После завершения этого раздела вы сможете:
- Цель1
- Цель2

---

## Основное содержание

### Подзаголовок

Содержание...

---

## Упражнения

### Упражнение 1

Описание задачи...

### Метод проверки

1. Шаг проверки 1
2. Шаг проверки 2

---

## Следующие шаги

- Следующая глава: [Ссылка](./next.md)
- Назад: [Содержание](../README.md)
```

### Именование файлов

- Используйте китайские названия
- Используйте дефис `-` для разделения слов
- Именование файлов глав: `01-Имя-главы.md`, `02-Имя-главы.md`
- Именование файлов упражнений: `Упражнение-название.md`

### Стандарты ссылок

- Ссылки на учебник: `./Справочные-документы/имя-файла.md`
- Ссылки внутри курса: `./имя-главы.md`
- Относительные пути: вычисляются от текущего расположения файла

## Отправка изменений

### Шаги

1. Fork репозитория
2. Создайте ветку: `git checkout -b feature/описание-нового-контента`
3. Внесите изменения
4. Commit: `git commit -m "feat: добавить новый контент"`
5. Push на удаленный: `git push origin feature/описание-нового-контента`
6. Создайте Pull Request

### Стандарты сообщений коммитов

```
<type>: <description>

<optional body>
```

Типы type:
- `feat`: новая функция
- `fix`: исправление ошибки
- `docs`: обновление документации
- `test`: обновление тестов
- `refactor`: рефакторинг

## Контроль качества

Перед отправкой проверьте:

- [ ] Markdown формат корректен
- [ ] Все ссылки работают
- [ ] Примеры кода работают
- [ ] Упражнения имеют метод проверки
- [ ] Нет опечаток на китайском
- [ ] Соответствует стандартам формата

## Обратная связь

Если вы обнаружили проблему или у вас есть предложения, создайте Issue, включая:
- Описание проблемы
- Расположение файла
- Предлагаемый способ улучшения
