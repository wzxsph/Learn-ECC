# 项目二：Код-ревью

使用 ECC 构建自动化Код-ревью流程。

## 项目Цель

**预计时长**: 2-3 小时

创建一个自动化Код-ревью系统，自动检查Качество кода、Безопасность问题、规范遵守。

## 功能需求

### 1. 审查维度
- **Качество кода** - 复杂度、重复、命名
- **Безопасность审查** - 漏洞、敏感信息、注入
- **规范检查** - 编码规范、文档完整性
- **Тестирование覆盖** - 覆盖率、Тестирование质量

### 2. 审查流程
- 提交触发审查
- 并行多维度检查
- 汇总报告生成
- 问题分级标记

### 3. 集成能力
- Git Hook 集成
- CI/CD 集成
- GitHub PR 评论

## 技术方案

### Agent Спроектировать
- **code-reviewer** - 主审查 Agent
- **security-reviewer** - Безопасность专项
- **tdd-guide** - Тестирование质量评估
> 参考: [/code-review](../СправочныеДокументы/commands/01-Основной рабочий процесс.md) | [code-reviewer Agent](../СправочныеДокументы/agents/1.%20Код-ревью.md)

### 输出格式
```json
{
  "summary": { "critical": "0", "high": "2", "medium": "5" },
  "issues": [
    { "type": "security", "severity": "high", "file": "src/auth.js", "line": 42 }
  ]
}
```

## Критерии приёмки

- [ ] Поддерживать JavaScript/TypeScript 审查
- [ ] 检出率 > 80%
- [ ] 误报率 < 10%
- [ ] 报告格式友好
- [ ] 与 Git 工作流集成
