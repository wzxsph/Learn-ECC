# 03 - Основные команды

## Типы команд

ECC предоставляет три типа команд:

### 1. Слэш-команды (Slash Commands)

Используются напрямую:

| Команда | Функция | Пример |
|------|------|------|
| `/ecc:plan` | Создать план реализации | `/ecc:plan "Добавить аутентификацию пользователя"` |
| `/ecc:code-review` | Проверка кода | `/ecc:code-review` |
| `/ecc:build-fix` | Исправить ошибки сборки | `/ecc:build-fix` |
| `/ecc:learn` | Извлечь шаблоны из сессии | `/ecc:learn` |
| `/ecc:skill-create` | Создать Skill из истории Git | `/ecc:skill-create` |
| `/ecc:sessions` | Управление историей сессий | `/ecc:sessions` |

Примечание: Ручная установка может использовать команды без префикса (напр. `[/plan](../Справочные-документы/commands/01-Основные-рабочие-потоки.md)`), установка плагина использует префикс `/ecc:`.

### 2. Skills (Основная поверхность рабочих процессов)

Skills - это основной способ вызова рабочих процессов ECC:

| Skill | Функция |
|-------|------|
| `[tdd-workflow](../Справочные-документы/skills/Тестирование.md)` | Разработка через тестирование |
| `[e2e-testing](../Справочные-документы/skills/Тестирование.md)` | Сквозное тестирование |
| `security-review` | Проверка безопасности |
| `verification-loop` | Цикл проверки |
| `search-first` | Рабочий процесс с приоритетом исследования |

Способ вызова:
```
Используйте skill [tdd-workflow](../Справочные-документы/skills/Тестирование.md)
```

### 3. Agent команды

Выполнение подзадач через Agent:

| Agent | Назначение |
|-------|------|
| `planner` | Планирование реализации |
| `code-reviewer` | Проверка кода |
| `build-error-resolver` | Исправление ошибок сборки |
| `security-reviewer` | Проверка безопасности |
| `architect` | Архитектурное проектирование |

Способ вызова:
```
@planner
```

## Часто используемые рабочие процессы

### Разработка новой функции

```
/ecc:plan "Добавить функцию аутентификации пользователя"
→ planner создает план реализации
Используйте skill [tdd-workflow](../Справочные-документы/skills/Тестирование.md)
→ tdd-guide заставляет сначала написать тест
/ecc:code-review
→ code-reviewer проверяет код
```

### Исправление Bug

```
Используйте skill [tdd-workflow](../Справочные-документы/skills/Тестирование.md)
→ tdd-guide: пишет воспроизводящий тест
→ Реализует исправление, проверяет прохождение тестов
/ecc:code-review
→ code-reviewer: проверяет регрессию
```

### Подготовка к выпуску

```
/ecc:security-scan
→ security-reviewer: аудит OWASP Top 10
Используйте skill [e2e-testing](../Справочные-документы/skills/Тестирование.md)
→ e2e-runner: тестирует критические пользовательские потоки
/ecc:test-coverage
→ Проверка покрытия 80%+
```

## MCP команды

| Команда | Функция |
|------|------|
| `/mcp` | Управление MCP серверами |
| `/ecc:status` | Проверить статус |
