# 01 - [ECC](../Справочные-документы/README.md) Введение

## Что такое [ECC](../Справочные-документы/README.md)?

[ECC](../Справочные-документы/README.md) (Everything Claude Code) - это система плагинов Claude Code, предоставляющая производственные [Agent](../Справочные-документы/agents/1.%20Код-проверка.md), [Skills](../Справочные-документы/skills/Лучшие-практики.md), hooks, rules, [Commands](../Справочные-документы/commands/01-Основные-рабочие-потоки.md) и конфигурацию MCP. Создана на основе победившего проекта Anthropic Hackathon, прошла 10+ месяцев ежедневного использования.

## Что может [ECC](../Справочные-документы/README.md)?

### 1. Система [Agent](../Справочные-документы/agents/1.%20Код-проверка.md)
[ECC](../Справочные-документы/README.md) предоставляет 60+ специализированных [Agent](../Справочные-документы/agents/1.%20Код-проверка.md) для многоязычной проверки, исправления сборки, архитектурного проектирования и др.:

| [Agent](../Справочные-документы/agents/1.%20Код-проверка.md) | Назначение |
|-------|------|
| `planner` | Планирование реализации функций |
| `code-reviewer` | Проверка качества и безопасности кода |
| `security-reviewer` | Анализ уязвимостей безопасности |
| `build-error-resolver` | Исправление ошибок сборки |
| `architect` | Проектирование системной архитектуры |
| `tdd-guide` | Руководство по разработке через тестирование |

### 2. [Skills](../Справочные-документы/skills/Лучшие-практики.md) Рабочие процессы
[Skills](../Справочные-документы/skills/Лучшие-практики.md) - это основная поверхность рабочих процессов, [ECC](../Справочные-документы/README.md) предоставляет 232+ [Skills](../Справочные-документы/skills/Лучшие-практики.md):

- `tdd-workflow` - Разработка через тестирование
- `e2e-testing` - Сквозное тестирование
- `security-review` - Чек-лист проверки безопасности
- `continuous-learning-v2` - Непрерывное обучение на основе instinct
- `eval-harness` - Оценка цикла проверки

### 3. [Commands](../Справочные-документы/commands/01-Основные-рабочие-потоки.md) Команды
Сохранена совместимость с традиционными слэш-командами (всего 75+ команд):

| Команда | Функция |
|------|------|
| `/plan` | Создать план реализации |
| `/code-review` | Проверка кода |
| `/build-fix` | Исправить ошибки сборки |
| `/learn` | Извлечь шаблоны из сессии |
| `/skill-create` | Создать Skill из истории Git |
| `/sessions` | Управление историей сессий |

### 4. Hooks Автоматизация
Гибкая система хуков (8 типов событий):
- `SessionStart` - Загрузка контекста при запуске сессии
- `SessionEnd` - Сохранение состояния при завершении сессии
- `PreToolUse` - Валидация перед выполнением инструмента
- `PostToolUse` - Форматирование после выполнения инструмента

### 5. Rules Система правил
Комплексные стандарты разработки (общие + языкозависимые):
- **common/** - Общие принципы (безопасность, стиль кода, тестирование, производительность)
- **typescript/** - TypeScript/JavaScript
- **python/** - Python
- **golang/** - Go
- **swift/** - Swift
- **php/** - PHP
- **arkts/** - HarmonyOS/ArkTS

### 6. MCP Сервисы
Поддержка 14+ конфигураций MCP серверов: GitHub, Supabase, Vercel, Railway, Context7, Exa, Playwright и другие.

## Основные концепции

### [Agent](../Справочные-документы/agents/1.%20Код-проверка.md)
Повторно используемые исполнители подзадач, каждый [Agent](../Справочные-документы/agents/1.%20Код-проверка.md) имеет четкую ответственность. Путем комбинирования нескольких [Agent](../Справочные-документы/agents/1.%20Код-проверка.md) 构建复杂ные рабочие процессы.

### [Skill](../Справочные-документы/skills/Лучшие-практики.md)
Определяет рабочий процесс для выполнения конкретной задачи, является основной поверхностью рабочих процессов [ECC](../Справочные-документы/README.md).

### Hook
Скрипты, автоматически запускаемые в определенные моменты, поддерживают автоматизацию рабочих процессов.

### Rule
Обязательные правила, которые должен соблюдать Claude Code, гарантируют качество вывода.
