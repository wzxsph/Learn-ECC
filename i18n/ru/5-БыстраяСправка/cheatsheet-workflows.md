# Рабочие процессы速查

## Часто используемые шаблоны рабочих процессов

### 1. TDD процесс разработки

```
Требования → Написать тест (RED) → Реализовать (GREEN) → Рефакторинг (REFACTOR)
```

**Использование**: `/tdd`

---

### 2. Процесс проверки кода

```
Отправить код → Запустить проверку → Классифицировать проблемы → Исправить → Повторная проверка → Слить
```

**Использование**: `/code-review`

---

### 3. Процесс исправления ошибок

```
Обнаружить ошибку → Проанализировать причину → Локализовать код → Исправить → Проверка → Тестирование
```

**Использование**: `/build-fix`

---

### 4. Процесс создания Skill

```
Проанализировать требования → Изучить существующие паттерны → Написать шаблон → Тестирование и проверка
```

**Использование**: `/skill-create`

---

## Пользовательские рабочие процессы

Формат определения рабочего процесса:

```yaml
name: my-workflow
description: Мой рабочий процесс
steps:
  - name: step1
    command: do-something
  - name: step2
    command: do-other
    requires: [step1]
```
