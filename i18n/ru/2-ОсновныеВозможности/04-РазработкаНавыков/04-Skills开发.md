## 1. Skill 格式

### 1.1 基本结构

Skills 使用 Markdown 格式，包含清晰的章节结构：

```markdown
# Skill Название

## When to Use
## How It Works
## Examples
```

### 1.2 YAML Frontmatter

```yaml
---
name: skill-name
description: Skill НазначениеОписание
category: 分类
---
```

### 1.3 章节结构

| 章节 | Описание |
|------|------|
| **When to Use** | 何时使用此 Skill |
| **How It Works** | 工作原理和Шаги |
| **Examples** | 使用Пример |

## 3. 创建自定义 Skill

### 3.1 创建Шаги

1. **确定Назначение**：明确 Skill 要解决的问题
2. **编写结构**：按照 When to Use / How It Works / Examples 结构
3. **添加 YAML frontmatter**：Предоставлять元数据
4. **保存到 skills/ Содержание**：使用小写 + 连字符命名

### 3.2 Пример Skill

```markdown
---
name: my-custom-skill
description: 执行特定Задача的工作流
category: 自定义
---

# My Custom Skill

## When to Use

当Требуется进行特定Задача X 时使用此 Skill。

## How It Works

1. Шаги一：准备数据
2. Шаги二：执行核心操作
3. Шаги三：Проверка结果
4. Шаги四：清理资源

## Examples

### Пример 1：基本Использование

```
执行 My Custom Skill 来Завершить X
```

### Пример 2：高级Использование

```
使用自定义参数执行 My Custom Skill
```
```

### 3.3 命名规范

| Тип | 规范 | Пример |
|------|------|------|
| 文件名 | 小写 + 连字符 | `code-review.md`, `tdd-workflow.md` |
| Содержание | 按领域组织 | `skills/Языки программирования/` |
| 位置 | 项目 skills/ 或全局 ~/.claude/skills/ | |

## 4. 工作流Спроектировать

### 4.1 Шаги定义

清晰定义每个Шаги：

```markdown
## How It Works

1. **需求分析**：理解用户需求
2. **方案Спроектировать**：制定实施方案
3. **代码Реализовать**：编写代码
4. **ТестированиеПроверка**：确保质量
5. **Обновление документации**：记录变更
```

### 4.2 条件分支

处理不同Сценарий：

```markdown
## How It Works

1. 分析输入
2. 根据条件分支：
   - 如果是 A，执行Шаги X
   - 如果是 B，执行Шаги Y
   - 否则，执行Шаги Z
3. 汇总结果
```

### 4.3 Обработка ошибок

标准化的Обработка ошибок：

```markdown
## How It Works

1. 执行操作
2. 检查结果
   - 成功：继续Следующий шаг
   - 失败：记录错误并Предоставлять修复建议
3. Назад最终结果
```

## 5. 参数化Шаблон

### 5.1 参数定义

```markdown
## How It Works

1. 接收输入参数：
   - `name`: Название（必填）
   - `type`: Тип（可选，默认值：`standard`）
2. 处理参数
3. Назад结果
```

### 5.2 变量替换

```markdown
## Examples

### 基本Использование

使用默认参数：
```
执行 Skill，Название为 {{name}}
```
```

### 自定义参数

指定自定义Тип：
```
执行 Skill，Название为 {{name}}，Тип为 {{type}}
```
```

## 7. Skill 创建工作流

使用 `/skill-create` 命令分析 git 历史并生成 Skill 文件：

```bash
# 分析 git 历史并生成 Skill
/skill-create

# 指定分析范围
/skill-create --days 30

# 指定输出路径
/skill-create --output ./my-skills/
```

## 8. Лучшие практики

### 8.1 Спроектировать原则

| 原则 | Описание |
|------|------|
| **单一Ответственность** | 每个 Skill 只做一件事 |
| **可组合** | Skill 之间Можно组合使用 |
| **可复用** | Спроектировать通用而非特定 |
| **可Тестирование** | ПредоставлятьПроверкаШаги |

### 8.2 文档质量

| 要求 | Описание |
|------|------|
| 清晰的 When to Use | 让用户知道何时使用 |
| 详细的 How It Works | Шаги清晰可执行 |
| 实用的 Examples | Предоставлять真实用例 |
| Обработка ошибокОписание | 告诉用户如何处理错误 |

### 8.3 维护建议

- 定期更新以反映最新实践
- 根据用户反馈改进
- 保持简洁，避免过度复杂
- 添加版本号便于追踪
