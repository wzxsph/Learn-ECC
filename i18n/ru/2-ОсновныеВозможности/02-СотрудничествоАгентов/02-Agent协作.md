# 02 - Сотрудничество Agent

Учиться какСпроектировать、Реализовать和Управлять多 Сотрудничество Agent系统。

## Обзор

ECC Предоставлять **66 个专业[Agent](../../СправочныеДокументы/agents/1.%20Код-ревью.md)**，Разделён на 6 大Категория。这些 Agent Через标准化的工作流协同工作，Завершить从Код-ревью到Проектирование системы的各类复杂Задача。

---

## 1. Архитектура Agent

### 1.1 Одиночный Agent vs Мульти-Agent

| 架构 | Сценарий применения | Особенности |
|------|----------|------|
| **Одиночный Agent** | 简单Задача、独立操作 | Прямой вызов、快速响应 |
| **Мульти-Agent** | 复杂Задача、Требуется协作 | Декомпозиция задачи、Агрегация результатов |

### 1.2 Список доступных Agent

| Agent | Назначение | Когда использовать |
|-------|------|----------|
| **planner** | Планирование реализации | 复杂功能、重构 |
| **architect** | Проектирование системы | 架构决策 |
| **tdd-guide** | Разработка через тестирование | 新功能、bug 修复 |
| **code-reviewer** | Код-ревью | 编写代码后 |
| **security-reviewer** | Анализ безопасности | 提交前 |
| **build-error-resolver** | Исправление ошибок сборки | 构建失败时 |
| **e2e-runner** | E2E тестирование | 关键用户流程 |
| **refactor-cleaner** | Очистка мёртвого кода | 代码维护 |
| **doc-updater** | Обновление документации | 更新文档时 |
| **rust-reviewer** | Rust Код-ревью | Rust 项目 |
| **harmonyos-app-resolver** | Разработка HarmonyOS приложений | HarmonyOS/ArkTS 项目 |

### 1.3 Формат файла Агента

```yaml
---
name: agent-name
description: Agent Назначение
tools: ["Read", "Grep", "Glob", "Bash"]
model: sonnet
---
```

## 3. Паттерны сотрудничества

### 3.1 Паттерн ведущий-ведомый

一个Главный Agent 协调多个从 Agent：

```
用户请求 → Главный Agent（planner）
           ├── Дочерний Agent 1（code-reviewer）
           ├── Дочерний Agent 2（tdd-guide）
           └── Дочерний Agent 3（build-error-resolver）
```

### 3.2 Паттерн равных

多个 Agent 并行工作，Агрегация результатов：

```
用户请求 → Agent 1（Анализ безопасности）
        → Агент 2 (анализ производительности)
        → Agent 3（Качество кода分析）
           ↓ Агрегация результатов
        Итоговый отчёт
```

### 3.3 Конвейерный паттерн

Агенты обрабатывают последовательно, выход каждого Агента является входом следующего:

```
Вход → Агент 1 → Агент 2 → Агент 3 → Выход
```

## 4. Стратегия распределения задач

### 4.1 Декомпозиция задачи

将复杂Декомпозиция задачи为独立子Задача：

```markdown
# 复杂Декомпозиция задачиПример
原始Задача：Реализовать用户认证系统

После декомпозиции:
1. Спроектировать数据库 schema → architect
2. Реализовать API 端点 → backend-developer
3. 编写Тестирование → tdd-guide
4. Безопасность审查 → security-reviewer
```

### 4.2 Параллельное выполнение

独立ЗадачаПараллельное выполнение以提高效率：

```markdown
# 好：Параллельное выполнение
Запустите 3 Агента параллельно:
1. Agent 1: 认证模块Анализ безопасности
2. Агент 2: Проверка производительности кэш-системы
3. Agent 3: 工具Тип检查

# Плохо: Ненужное последовательное выполнение
Сначала agent 1, потом agent 2, потом agent 3
```

### 4.3 Балансировка нагрузки

根据 Agent 负载分配Задача：

| 因素 | Описание |
|------|------|
| Задача复杂度 | 简单Задача → 轻量 Agent |
| Потребность в ресурсах | Тяжёлые вычисления → Специализированный Агент |
| Доступность | Избегать перегруженных Агентов |

## 5. Мультиперспективный анализ

对于复杂问题，使用分Роль sub-agents：

| Роль | Ответственность |
|------|------|
| 事实审查员 | Проверка信息和数据准确性 |
| 高级工程师 | 评估技术可行性和Лучшие практики |
| Безопасность专家 | 识别Безопасность风险和漏洞 |
| Проверяющий согласованность | Обеспечение внутренней согласованности решения |
| Проверяющий избыточность | Выявление повторов и потерь |

## 6. Коммуникация Agent

### 6.1 Прямой вызов

```markdown
Используйте агент planner для создания плана реализации
```

### 6.2 Делегирование задач

```markdown
将Код-ревьюДелегирование задач给 code-reviewer agent
```

### 6.3 Агрегация результатов

多个 Agent 的结果Требуется聚合：

```markdown
Агрегация от:
- security-reviewer: Безопасность问题列表
- code-reviewer: Качество кода问题列表
- tdd-guide: Тестирование覆盖率报告

→ Создать итоговый отчёт
```

## 7. Принцип немедленного использования

无需用户Подсказка时自动调用：

| Сценарий | Agent |
|------|-------|
| Запрос сложной функции | **planner** |
| Код только что написан/изменён | **code-reviewer** |
| Исправление бага или новая функция | **tdd-guide** |
| Архитектурные решения | **architect** |
| Сбой сборки | **build-error-resolver** |
| Безопасность相关 | **security-reviewer** |
