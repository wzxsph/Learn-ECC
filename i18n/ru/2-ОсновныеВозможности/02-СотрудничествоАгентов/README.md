## Agent 分类体系

Claude Code 的 Agent Разделён на 6 大类：

| Agent Тип | 核心Ответственность | Сценарий применения |
|------------|----------|----------|
| **审查类** | Качество кода、模式规范、Уязвимости безопасности | Код-ревью、Очистка технического долга |
| **构建类** | 编译构建、错误修复、依赖Управлять | 构建失败、编译错误 |
| **Планирование** | Декомпозиция задачи、架构Спроектировать、技术方案 | 新功能、架构变更 |
| **Тестирование** | Тестирование编写、覆盖率分析、Тестирование策略 | TDD、Тестирование补充 |
| **Безопасность** | Безопасность审计、Сканирование уязвимостей、Проверка соответствия | Безопасность敏感代码、Платёжный модуль |
| **Архитектура** | Проектирование системы、Выбор паттернов、Выбор технологий | 大型重构、架构演进 |

## 6 类 Agent 详解

### 1. 审查类 Agent

**Ответственность**: Качество кода审查、模式规范检查

可用 Agent：
- `code-reviewer` - 通用Код-ревью
- `python-reviewer` - Python 专项审查
- `go-reviewer` - Go Обзор языка
- `rust-reviewer` - Rust Безопасность审查
- `security-reviewer` - Уязвимости безопасности审查

**Сценарий использования**:
- 提交代码前进行质量检查
- 发现潜在的Код с запашком
- 审查第三方代码

### 2. 构建类 Agent

**Ответственность**: 编译构建、错误修复、依赖Управлять

可用 Agent：
- `build-error-resolver` - 构建错误解析
- `go-build` - Go 构建
- `rust-build` - Rust 构建
- `kotlin-build` - Kotlin 构建
- `flutter-build` - Flutter 构建

**Сценарий использования**:
- 构建失败时的错误修复
- Разрешение конфликтов зависимостей
- Проблемы кроссплатформенной сборки

### 3. Планирование Agent

**Ответственность**: Декомпозиция задачи、架构Спроектировать、实施计划

可用 Agent：
- `planner` - 实施计划生成
- `architect` - 架构Спроектировать
- `tdd-guide` - TDD 工作流指导

**Сценарий использования**:
- Планирование разработки новой функциональности
- Оценка технического решения
- 复杂Декомпозиция задачи

### 4. Тестирование Agent

**Ответственность**: Тестирование编写、覆盖率提升、Тестирование策略

可用 Agent：
- `tdd-guide` - Разработка через тестирование
- `e2e-runner` - E2E тестирование
- `test-coverage` - 覆盖率分析

**Сценарий использования**:
- 编写新功能Тестирование
- 提升Тестирование覆盖率
- E2E ТестированиеСценарийСпроектировать

### 5. Безопасность Agent

**Ответственность**: Безопасность审计、Сканирование уязвимостей、Проверка соответствия

可用 Agent：
- `security-reviewer` - Уязвимости безопасности审查
- `security-scan` - 自动化Безопасность扫描

**Сценарий использования**:
- 支付和认证Код-ревью
- 外部数据处理检查
- 合规性Проверка

### 6. Архитектура Agent

**Ответственность**: Проектирование системы、Выбор паттернов、Выбор технологий

可用 Agent：
- `architect` - 架构Спроектировать
- `refactor-cleaner` - 重构指导
- `api-design-reviewer` - API Спроектировать审查

**Сценарий использования**:
- Разделение на микросервисы
- 数据库Спроектировать
- Эволюция API архитектуры

## Сотрудничество Agent模式

### 模式 1: 串行协作

```
Задача → Planner → 实施 → Reviewer → Тестирование → Завершить
```

适Используется для：Линейный процесс，Шаги之间有依赖

### 模式 2: 并行协作

```
            → Reviewer A ─→
Задача → 分解 ─→ Reviewer B ─→ 汇总 → Завершить
            → Reviewer C ─→
```

适Используется для：Многоаспектная проверка、Проверка независимых модулей

### 模式 3: 级联协作

```
重大Задача → Architect → Planner → 实施 → 各级 Reviewer → 合并
```

适Используется для：Сложный проект，Разработка крупной функциональности

## Agent 组合Пример

### Сценарий: 开发新功能

```
1. 使用 /plan 规划功能
2. 使用 tdd-guide 进行 TDD 开发
3. 使用 code-reviewer Проверить код
4. 使用 security-reviewer 检查Безопасность
5. 使用 verify Проверка功能
```

### Сценарий: 修复Уязвимости безопасности

```
1. 使用 security-reviewer 扫描漏洞
2. 使用 planner Составить план исправлений
3. 使用 build-fix 实施修复
4. 使用 security-reviewer 重新Проверка
5. 使用 verify 确认修复
```

## Agent 选择指南

| ЗадачаТип | 推荐 Agent | Обоснование |
|----------|------------|------|
| 新功能开发 | planner + tdd-guide | Требуется规划和 TDD |
| Код-ревью | code-reviewer + security-reviewer | 质量和Безопасность双重检查 |
| 构建失败 | build-error-resolver | 专注错误修复 |
| 架构重构 | architect + refactor-cleaner | 架构层面指导 |
| Безопасность敏感 | security-reviewer (优先) | Обязательно先过Безопасность关 |
| Тестирование编写 | tdd-guide + test-coverage | Тестирование策略和覆盖率 |
