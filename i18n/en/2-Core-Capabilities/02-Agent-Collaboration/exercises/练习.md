# Exercises: Agent Collaboration

## Exercise 1: Using Code Review Agents

### Objective
Learn to invoke code review Agents and understand their feedback mechanism.

### Scenario
Assume you wrote the following Python code (no actual execution needed, just in your mind):

```python
def get_user_data(user_id, include_password=False):
    user = db.query("SELECT * FROM users WHERE id = " + user_id)
    if include_password:
        return user
    return {"name": user.name, "email": user.email}
```

### Steps

1. **Analyze this code**
   Think about possible issues:
   - SQL injection vulnerability
   - Password leak risk
   - Missing error handling

2. **Select appropriate Agent**
   For this code, should use:
   - `security-reviewer` - Check security vulnerabilities
   - `python-reviewer` - Check Python best practices

3. **Simulate Agent feedback**
   Record what you think the Agent will provide:

   ```markdown
   ## Code Review Feedback

   ### Security Issues (CRITICAL)
   - SQL injection: Directly concatenating user_id into SQL statement
   - Password leak: include_password option may expose passwords

   ### Quality Issues (HIGH)
   - Missing error handling
   - Missing input validation

   ### Improvement Suggestions
   - Use parameterized queries
   - Never return password fields
   - Add exception handling
   ```

4. **Create fix plan**
   Using `/plan` approach, create fix plan:

   ```markdown
   # Fix Plan

   1. Fix SQL injection: Use parameterized queries
   2. Remove password return: Never return passwords
   3. Add input validation: Validate user_id format
   4. Add error handling: Handle user not existing case
   ```

### Acceptance Criteria
- [ ] Can identify security issues in code
- [ ] Understand review Agent feedback structure
- [ ] Can create fix plans

## Exercise 2: Using Build Fix Agents

### Objective
Learn how to use build Agents to diagnose and fix build errors.

### Scenario
Assume Go project has the following build errors:

```
# github.com/myproject/pkg
./main.go:15: undefined: " fmt"
./main.go:23: undefined: calculateTotal
./main.go:31: too many arguments to function call
```

### Steps

1. **Analyze error types**
   Categorize errors:

   | Error | Type | Severity |
   |-------|------|----------|
   | undefined: " fmt" | Import error | HIGH |
   | undefined: calculateTotal | Function undefined | HIGH |
   | too many arguments | Parameter error | MEDIUM |

2. **Design fix workflow**
   Using Agent collaboration:

   ```markdown
   # Build Fix Workflow

   1. /go-build - Run build to get complete errors
   2. /build-fix - Call build fix Agent
   3. Analyze error priority: Fix high priority first
   4. Fix each and verify
   5. /verify - Confirm build success
   ```

3. **Design fix solution for each error**

   **Error 1: undefined: " fmt"**
   ```go
   // Problem: Extra quotes
   import "fmt"  // Correct写法

   // Fix: Remove quotes
   ```

   **Error 2: undefined: calculateTotal**
   ```go
   // Problem: Function undefined or naming error
   // Fix: Check function definition, may need to add or rename
   ```

   **Error 3: too many arguments**
   ```go
   // Problem: Function call parameter count mismatch
   // Fix: Check function signature, correct call parameters
   ```

4. **Record fix experience**
   Create error pattern summary:

   ```markdown
   # Go Build Error Patterns

   ## Common Errors
   1. undefined: Import/function issues
   2. too many/few arguments: Parameter mismatch
   3. undefined: " xxx": Import syntax error

   ## Fix Strategies
   - First look at error count, then error locations
   - High priority errors may chain-cause low priority errors
   - Rebuild to verify after fixing
   ```

### Acceptance Criteria
- [ ] Can analyze build error types and priorities
- [ ] Designed complete fix workflow
- [ ] Recorded common error patterns

## Exercise 3: Agent Combination Workflow

### Objective
Design a complex task using multiple Agents collaborating to complete.

### Scenario
Develop a new user authentication module with:
- User registration (with password strength validation)
- User login (with brute force protection)
- JWT token generation and verification

### Steps

1. **Task decomposition**
   Decompose task into subtasks:

   ```markdown
   # Authentication Module Development Task

   ## Subtasks
   1. Database design (Schema)
   2. User registration API
   3. User login API
   4. JWT Token handling
   5. Security mechanisms (password strength, brute force protection)
   ```

2. **Agent assignment**
   Assign Agent for each subtask:

   | Subtask | Agent | Reason |
   |---------|-------|--------|
   | Database design | architect | Needs architecture perspective |
   | Registration API | tdd-guide + security-reviewer | TDD + Security |
   | Login API | tdd-guide + security-reviewer | TDD + Security |
   | JWT handling | security-reviewer | Security sensitive |
   | Security mechanisms | security-reviewer (priority) | Security first |

3. **Design workflow**

   ```
   [Start]
      ↓
   /plan (Plan overall)
      ↓
   ┌─────────────────────────────────────┐
   │  Parallel execution:                │
   │  - architect: Database design      │
   │  - tdd-guide: Design test strategy │
   └─────────────────────────────────────┘
      ↓
   [Implementation Phase]
      ↓
   ┌─────────────────────────────────────┐
   │  Serial + parallel hybrid:           │
   │  1. tdd-guide: Write registration   │
   │     tests                           │
   │  2. Implement registration         │
   │  3. security-reviewer: Security    │
   │     review                         │
   │  ────────────────────────────────── │
   │  1. tdd-guide: Write login tests    │
   │  2. Implement login                │
   │  3. security-reviewer: Security    │
   │     review                         │
   └─────────────────────────────────────┘
      ↓
   [Integration Phase]
      ↓
   ┌─────────────────────────────────────┐
   │  - architect: JWT architecture       │
   │  - security-reviewer: Overall      │
   │    security review                 │
   │  - code-reviewer: Code quality      │
   │    review                          │
   │  - /verify: Overall verification   │
   └─────────────────────────────────────┘
      ↓
   [Complete]
   ```

4. **Write workflow document**

   ```markdown
   # User Authentication Module Development Workflow

   ## Phase 1: Planning
   - Use /plan to create overall plan
   - Use architect to design database and JWT architecture

   ## Phase 2: TDD Development
   - Use tdd-guide to guide test writing
   - Write tests first, then implement features
   - Verify each subtask separately

   ## Phase 3: Security Review
   - Use security-reviewer to review each API
   - Special focus: Password handling, Token security, Injection prevention

   ## Phase 4: Integration Verification
   - Use code-reviewer for overall review
   - Use /verify to verify functional completeness

   ## Agent Collaboration Summary
   - architect: Architecture design (1x)
   - tdd-guide: Test guidance (2x)
   - security-reviewer: Security review (3x)
   - code-reviewer: Quality review (1x)
   ```

### Acceptance Criteria
- [ ] Task decomposition is reasonable, contains 5+ subtasks
- [ ] Agent assignment strategy is clear
- [ ] Workflow design is clear
- [ ] Workflow documented for reuse
