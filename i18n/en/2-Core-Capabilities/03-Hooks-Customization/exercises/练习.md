# Exercises: Hooks Customization

## Exercise 1: Create a PreToolUse Hook

### Objective
Create your first PreToolUse Hook to implement dangerous command warnings.

### Scenario
When user tries to execute dangerous commands (like `rm -rf`), show a warning prompt.

### Steps

1. **Design Hook Configuration**

   ```json
   {
     "name": "dangerous-command-warning",
     "matcher": {
       "tool": "Bash"
     },
     "condition": "command.includes('rm -rf') || command.includes('drop table')",
     "hooks": [
       {
         "type": "notification",
         "when": "before",
         "message": "Warning: Dangerous command detected! Please confirm operation is safe.",
         "continue": true
       }
     ]
   }
   ```

2. **Create Hook File**
   Create `hooks/dangerous-warning.json` in the project

3. **Test Hook**
   Design test scenarios:

   | Command | Expected Behavior |
   |---------|-------------------|
   | `rm -rf /tmp/test` | Show warning |
   | `rm -rf ./project` | Show warning |
   | `ls -la` | No warning |
   | `git commit` | No warning |

4. **Extend Hook Functionality**
   Try adding more dangerous command detection:

   ```json
   "condition": "command.includes('rm -rf') || 
                 command.includes('drop table') || 
                 command.includes('truncate') ||
                 command.includes('shutdown')"
   ```

5. **Add Whitelist**
   Some situations need to allow dangerous commands, add whitelist mechanism:

   ```json
   "condition": "dangerousCommand && !whitelisted"
   ```

### Acceptance Criteria
- [ ] Created PreToolUse Hook configuration file
- [ ] Defined dangerous command conditions
- [ ] Designed test scenarios
- [ ] Added at least 5 types of dangerous command detection

## Exercise 2: Create a PostToolUse Hook

### Objective
Create PostToolUse Hook for automatic notification after test completion.

### Scenario
When test execution completes, automatically check test results and send notifications.

### Steps

1. **Design Hook Configuration**

   ```json
   {
     "name": "test-result-notifier",
     "matcher": {
       "tool": "Bash"
     },
     "condition": "command.includes('npm test') || command.includes('pytest')",
     "hooks": [
       {
         "type": "notification",
         "when": "after",
         "message": "Test execution completed",
         "continue": true
       }
     ]
   }
   ```

2. **Enhance: Parse Test Results**
   Create a JavaScript processing script `scripts/parse-test-result.js`:

   ```javascript
   module.exports = function parseTestResult(output) {
     // Parse test output
     const passed = output.match(/(\d+) passed/);
     const failed = output.match(/(\d+) failed/);
     
     return {
       passed: passed ? parseInt(passed[1]) : 0,
       failed: failed ? parseInt(failed[1]) : 0,
       status: failed ? 'FAILED' : 'PASSED'
     };
   };
   ```

3. **Create Complete Hook Configuration**
   Combine script to create complete configuration:

   ```json
   {
     "name": "test-result-checker",
     "matcher": {
       "tool": "Bash"
     },
     "condition": "command.includes('npm test')",
     "hooks": [
       {
         "type": "command",
         "when": "after",
         "command": "node scripts/parse-test-result.js",
         "continue": true
       }
     ]
   }
   ```

4. **Design Multi Framework Adaptation**
   Support different test frameworks:

   | Framework | Output Keywords |
   |-----------|-----------------|
   | Jest | `Tests:` |
   | Pytest | `passed` |
   | Go test | `ok` |
   | RSpec | `examples` |

5. **Add CI Environment Scenario**
   Design behavior in CI environment:

   ```json
   {
     "condition": "isCI && testCommand",
     "hooks": [
       {
         "type": "command",
         "when": "after",
         "command": "node scripts/ci-reporter.js",
         "continue": true
       }
     ]
   }
   ```

### Acceptance Criteria
- [ ] Created PostToolUse Hook
- [ ] Implemented test result parsing
- [ ] Supported multiple test frameworks
- [ ] Designed CI scenario handling

## Exercise 3: Configure Quality Gate Hook

### Objective
Create a complete quality gate system for multi-dimensional checks before code commits.

### Scenario
Automatically run a series of quality checks before `git commit` to ensure code quality.

### Steps

1. **Define Quality Gate Check Items**

   | Check Item | Check Content | Failure Behavior |
   |------------|---------------|------------------|
   | Test coverage | >= 80% | Block commit |
   | Lint check | No errors | Block commit |
   | Security scan | No vulnerabilities | Warn + block |
   | Format check | Follows standards | Auto-fix |

2. **Design Hook Execution Flow**

   ```
   git commit
      ↓
   [PreToolUse Hook]
      ↓
   Run quality checks
      ├── Test coverage check
      ├── Lint check  
      ├── Security scan
      └── Format check
      ↓
   [Result Judgment]
      ↓
   ├── All passed → Allow commit
   ├── Warnings → Show warnings, continue commit
   └── Errors → Block commit
   ```

3. **Create Hook Configuration File**

   ```json
   {
     "name": "quality-gate",
     "matcher": {
       "tool": "Bash"
     },
     "condition": "command.includes('git commit')",
     "hooks": [
       {
         "type": "command",
         "when": "before",
         "command": "npm test && npm run lint && npm run security-scan",
         "continue": false
       }
     ]
   }
   ```

4. **Create Check Script**

   `scripts/quality-gate.js`:

   ```javascript
   const { execSync } = require('child_process');
   
   function runChecks() {
     const checks = [
       { name: 'Test', command: 'npm test', required: true },
       { name: 'Lint', command: 'npm run lint', required: true },
       { name: 'Coverage', command: 'npm run coverage', required: true },
       { name: 'Security', command: 'npm run security-scan', required: false }
     ];
   
     let allPassed = true;
     for (const check of checks) {
       try {
         execSync(check.command, { stdio: 'inherit' });
         console.log(`✓ ${check.name} passed`);
       } catch (error) {
         console.error(`✗ ${check.name} failed`);
         if (check.required) allPassed = false;
       }
     }
     return allPassed;
   }
   
   process.exit(runChecks() ? 0 : 1);
   ```

5. **Add Progressive Quality Gate**

   ```json
   {
     "name": "progressive-quality-gate",
     "stages": [
       { "name": "Fast check", "commands": ["npm run lint"], "blockOnFail": true },
       { "name": "Test", "commands": ["npm test"], "blockOnFail": true },
       { "name": "Security scan", "commands": ["npm run security-scan"], "blockOnFail": false },
       { "name": "Coverage", "commands": ["npm run coverage"], "blockOnFail": true }
     ]
   }
   ```

6. **Configure Skip Mechanism**

   ```json
   {
     "condition": "command.includes('git commit') && !command.includes('--no-verify')",
     "message": "Quality gate enabled. Use --no-verify to skip checks (not recommended)."
   }
   ```

### Acceptance Criteria
- [ ] Defined complete checklist
- [ ] Created Hook configuration file
- [ ] Implemented check script
- [ ] Supported skip mechanism
- [ ] Documented usage
