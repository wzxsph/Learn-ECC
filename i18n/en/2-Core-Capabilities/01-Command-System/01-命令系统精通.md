# 01 - Command System Mastery

Deep dive into ECC's command system, master all command types and usage scenarios.

## Overview

ECC provides **75 [commands](../../Reference-Docs/commands/01-Core-Workflow.md)**, divided into three categories: Slash Commands, script commands, and Agent commands. These commands greatly improve development efficiency through standardized workflows.

## Command File Format

Each command is a `.md` file containing YAML frontmatter + Markdown body:

```yaml
---
description: "Short command description"
argument-hint: "[Optional parameter hint]"
name: command-name          # Optional, defaults to filename
command: true              # Optional, CLI passthrough
allowed_tools: ["Bash"]    # Optional, restrict available tools
---
```

### Frontmatter Field Description

| Field | Required | Description |
|-------|----------|-------------|
| description | Yes | Shown in / command list |
| argument-hint | No | Parameter hint |
| name | No | Override filename as command name |
| command | No | Mark as CLI passthrough command |
| allowed_tools | No | Restrict available tools |

## 3. Custom Command Creation

### Command Storage Locations

| Type | Path |
|------|------|
| Command files | `commands/` |
| Registration mechanism | `.claude-plugin/plugin.json` |

### Creation Steps

1. Create `.md` file in `commands/` directory
2. Add YAML frontmatter
3. Write command content (Markdown format)
4. Command auto-registers to ECC

## 4. Best Practices

### Command Selection Guide

| Scenario | Recommended Commands |
|----------|----------------------|
| New feature development | `/plan` â†’ `/tdd` |
| Code review | `/code-review` |
| Bug fix | `/build-fix` â†’ `/verify` |
| Test-driven | `/tdd` â†’ `/test-coverage` |
| Create PR | `/pr` |
| Learning patterns | `/learn` â†’ `/evolve` |

### Command Execution Order

Follow workflow principles:
1. Plan: `/plan` or `/prp-plan`
2. Implement: `/tdd` or feature command
3. Review: `/code-review`
4. Verify: `/verify` or `/quality-gate`
5. Commit: `/pr`
