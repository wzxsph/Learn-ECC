# 프로젝트 2: 코드 검토

ECC를 사용하여 자동화 코드 검토 프로세스를 구축합니다.

## 프로젝트 목표

**예상 시간**: 2-3시간

코드 품질, 보안 문제, 규범 준수를 자동으로检查하는自动化代码审查系统을 만듭니다.

## 기능 요구사항

### 1. 검토 차원
- **코드 품질** - 복잡도, 중복, 명명
- **보안 검토** - 취약점, 민감 정보, 인젝션
- **규범检查** - 코딩 규범, 문서 완전성
- **테스트 커버리지** - 커버리지율, 테스트 품질

### 2. 검토 프로세스
- 커밋 시 검토トリガー
- 병렬 다차원 검사
- 요약 보고서 생성
- 문제 심각도 분류

### 3. 통합 능력
- Git Hook 통합
- CI/CD 통합
- GitHub PR 댓글

## 기술方案

### Agent 설계
- **code-reviewer** - 메인 검토 Agent
- **security-reviewer** - 보안 전문가
- **tdd-guide** - 테스트 품질 평가
> 참고: [/code-review](../참조문서/commands/01-核心工作流.md) | [code-reviewer Agent](../참조문서/agents/1.%20代码审查类.md)

### 출력 형식
```json
{
  "summary": { "critical": "0", "high": "2", "medium": "5" },
  "issues": [
    { "type": "security", "severity": "high", "file": "src/auth.js", "line": 42 }
  ]
}
```

## 수락 기준

- [ ] JavaScript/TypeScript 검토 지원
- [ ] 검출율 > 80%
- [ ] 오검출율 < 10%
- [ ] 보고서 형식 사용자 친화적
- [ ] Git 워크플로와 통합
