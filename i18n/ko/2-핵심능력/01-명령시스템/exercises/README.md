# 연습: 명령 시스템 마스터

## 연습 1: 모든 명령 분류 탐색

### 목표
Claude Code의 모든 명령 분류를 이해하고, 각 분류의 핵심 명령을 식별한다.

### 단계

1. **모든 사용 가능한 명령 나열**
   - Claude Code에서 `/`를 입력하여 모든 명령을 확인
   - 명령의 분류 구성 방식에 주의

2. **각 분류 탐색**
   - 각 분류에서 `/help` 또는 `?`를 사용하여 도움말을 가져옴
   - 각 분류의 핵심 명령과 용도를 기록

3. **명령清单 기록**
   자신의 명령 빠른 참조표를 생성：

   | 분류 | 명령 | 용도 | 사용 빈도 |
   |------|------|------|----------|
   | 개발 플로 | /plan | 실시 계획 작성 | ★★★ |
   | ... | ... | ... | ... |

### 완료 기준
- [ ] 최소 5개 분류의 명령을 나열할 수 있음
- [ ] 20+개의 자주 사용하는 명령을 기록
- [ ] 명령의 분류 로직을 이해

## 연습 2: 여러 명령 조합 사용

### 목표
여러 명령을 조합하여连贯한 개발 워크플로를 생성하는 방법을 학습.

### 시나리오
假设의 빌드 에러를 수리하고 코드 리뷰를 수행.

### 단계

1. **시나리오 시뮬레이션**
   Go 프로젝트 개발 중 빌드 에러 발생：

   ```
   # 시나리오 설정（실제 실행 불필요）
   수리가 필요한以下の 빌드 에러가 있다고 가정:
   - import cycle detected
   - undefined: someFunction
   ```

2. **명령 조합 연습**
   다음 명령의 조합을 순서대로思考하고 기록：

   a. 먼저 `/plan`을 사용하여 수리 절차 계획
   b. `/go-build`를 사용하여 빌드 시도
   c. `/build-fix`를 사용하여 에러 수리
   d. `/code-review`를 사용하여 리뷰
   e. `/verify`를 사용하여 수리 검증

3. **워크플로 기록**
   수리 워크플로 문서를 생성：

   ```markdown
   ## 빌드 수리 워크플로

   1. /plan - 문제 분석, 수리 계획 수립
   2. /build-fix - 수리 실행
   3. /code-review - 수리 품질 검사
   4. /verify - 기능 정상 검증
   ```

### 완료 기준
- [ ] 명령 조합의 로직을 이해
- [ ] 최소 3개의 커스텀 워크플로 생성
- [ ] 명령 조합의 베스트 프랙티스를 기록

## 연습 3: 커스텀 워크플로 생성

### 목표
프로젝트 요구사항 기반으로 자동화된 워크플로 명령 조합을 생성.

### 시나리오
假设의 Flutter 프로젝트용 개발 워크플로를 생성.

### 단계

1. **워크플로 목표 정의**
   Flutter 프로젝트에는以下の 워크플로가 필요：

   - 일상 개발: 코딩 → 테스트 → 빌드
   - 코드 리뷰: 자검 → 리뷰 → 수리
   - 릴리스 준비: 테스트 → 빌드 → 검증

2. **명령 조합 설계**

   **일상 개발 워크플로**:
   ```
   /flutter-build → /flutter-test → /verify
   ```

   **코드 리뷰 워크플로**:
   ```
   /flutter-review → /build-fix → /code-review
   ```

3. **워크플로 문서 생성**
   프로젝트에서 `CLAUDE_WORKFLOWS.md` 를 생성：

   ```markdown
   # Claude Code 워크플로

   ## Flutter 개발 워크플로

   ### 일상 개발
   1. `/plan`을 사용하여 임무 계획
   2. `/flutter-build`를 사용하여 빌드
   3. `/flutter-test`를 사용하여 테스트
   4. `/verify`를 사용하여 검증

   ### 코드 리뷰
   1. `/flutter-review`를 사용하여 자검
   2. `/build-fix`를 사용하여 문제 수리
   3. `/code-review`를 사용하여 리뷰 요청

   ### 릴리스 준비
   1. 완전한 테스트 스위트 실행
   2. 빌드 검증 실행
   3. `/verify`를 사용하여 최종 확인
   ```

4. **실천 검증**
   실제 프로젝트가 있으면 이 워크플로를 실행试着.

### 완료 기준
- [ ] 3개의 완전한 워크플로 설계
- [ ] 워크플로 문서 생성
- [ ] 실제 프로젝트에서 최소 1개의 워크플로 테스트

### 연습 3 워크플로 템플릿

```markdown
# [프로젝트명] Claude Code 워크플로

## 개발 플로
- 일상: /plan → [코딩] → /verify
- 수리: /build-fix → /code-review → /verify
- 릴리스: /test → /build → /verify

## 사용 설명
1. `/plan`을 사용하여任何 임무 시작
2. 코딩 완료 후 `/verify` 실행
3. 중요한 변경에는 `/code-review` 사용
```
