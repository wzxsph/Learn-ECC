# Learn-ECC 기여 가이드

Learn-ECC에 기여해 주셔서 감사합니다! 이 가이드는 새 콘텐츠 추가, 기존 콘텐츠 개선 또는 수정 제출 방법을 설명합니다.

## 기여 유형

### 1. 새 콘텐츠 추가

기여할 수 있는 콘텐츠 유형:
- 새 챕터 또는 모듈
- 새 연습 문제
- 새 실전 프로젝트
- 치트시트 보충
- 번역 개선

### 2. 기존 콘텐츠 개선

- 오류 수정
- 설명 명확성 개선
- 더 많은 예제 추가
- 연습 답안 보완
- 오래된 정보 업데이트

### 3. 문제 보고

- 오타 또는 문법 오류
- 작동하지 않는 코드 예제
- 깨진 링크
- 누락된 콘텐츠

## 콘텐츠 형식 표준

### Markdown 형식

모든 콘텐츠는 다음 섹션을 포함하는 Markdown 형식을 사용해야 합니다:

```markdown
# 제목

## 학습 목표

이 섹션을 완료하면 다음을 할 수 있습니다:
- 목표 1
- 목표 2

---

## 본문 콘텐츠

### 하위 제목

콘텐츠...

---

## 연습

### 연습 1

작업 설명...

### 검증 방법

1. 검증 단계 1
2. 검증 단계 2

---

## 다음 단계

- 다음 챕터: [링크](./next.md)
- 돌아가기: [목차](../README.md)
```

### 파일 명명

- 한국어 명명 사용
- 하이픈 `-`으로 단어 분리
- 챕터 파일 명명: `01-챕터이름.md`, `02-챕터이름.md`
- 연습 파일 명명: `연습-이름.md`

### 링크 표준

- 참고 문서 링크: `./참조문서/파일이름.md`
- 과정 내 링크: `./챕터이름.md`
- 상대 경로: 현재 파일 위치에서 계산

## 수정 제출

### 단계

1. 저장소 포크
2. 브랜치 생성: `git checkout -b feature/새콘텐츠설명`
3. 수정 수행
4. 커밋: `git commit -m "feat: 새 콘텐츠 추가"`
5. 원격에 푸시: `git push origin feature/새콘텐츠설명`
6. 풀 리퀘스트 생성

### 커밋 메시지 표준

```
<type>: <description>

<optional body>
```

Type 유형:
- `feat`: 새 기능
- `fix`: 버그 수정
- `docs`: 문서 업데이트
- `test`: 테스트 업데이트
- `refactor`: 리팩토링
