# 테스트類 Agent

테스트類 Agent 전담테스트驱动개발、E2E 테스트、테스트커버리지분석및품질保证。

## Agent 목록

| Agent 이름 | 용도 | 사용모델 | 코어도구 |
|------------|------|----------|----------|
| tdd-guide | TDD 테스트驱动개발专家 | sonnet | Read, Write, Edit, Bash, Grep |
| e2e-runner | E2E 테스트专家 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| mle-reviewer | ML 工程코드 리뷰（훈련/추론/모니터링） | sonnet | Read, Grep, Glob, Bash |
| pr-test-analyzer | PR 테스트覆盖분석 | sonnet | Read, Grep, Glob, Bash |

## tdd-guide

### 이름과 용도
테스트驱动개발专家，强制실행写테스트우선의메서드论。主动사용于编写新기능、수정 bug 또는리팩토링코드。确保 80%+ 의테스트커버리지。

### 능력 설명
- 强制실행테스트우선의메서드论
- 指导 Red-Green-Refactor 周期
- 确保 80%+ 테스트커버리지
- 编写全面의테스트套件（单元、통합、E2E）
- 在实施前捕获엣지缘사용例

### 적용 시나리오
- 编写新기능시
- 수정 bug 시
- 리팩토링코드시
- 필요 TDD 워크플로우시

### 사용 도구 목록
- Read: 읽기现있음코드
- Write: 编写테스트
- Edit: 수정테스트및구현
- Bash: 실행테스트명령
- Grep: 검색테스트패턴

### 다른 Agent와의 협력 방식
- code-reviewer 在 TDD 周期后리뷰코드
- build-error-resolver 수정테스트빌드오류
- e2e-runner 진행핵심사용户流의 E2E 테스트

### TDD 워크플로우

#### 1. 先写테스트 (RED)
编写기술述预期행위의실패테스트。

#### 2. 실행테스트 - 검증它 FAIL
```bash
npm test
```

#### 3. 写最小구현 (GREEN)
只写足够의코드让테스트통과。

#### 4. 실행테스트 - 검증它 PASS

#### 5. 리팩토링 (IMPROVE)
제거重复，개선이름，최적화 - 테스트필수保持绿色。

#### 6. 검증커버리지
```bash
npm run test:coverage
# 요구: 80%+ 브랜치、함수、행、语句커버리지
```

### 필수테스트의엣지缘사용例

1. **Null/Undefined** 입력
2. **空** 배열/字符串
3. **없음效유형** 传递
4. **경계값** (最小/最大)
5. **오류경로** (네트워크네트워크실패、데이터베이스오류)
6. **竞态条件** (동시성작업)
7. **大데이터** (10k+ 项의성능)
8. **特殊字符** (Unicode、表情符号、SQL 字符)

### 反패턴 - 필수避免

- 테스트구현细节（内部상태）而非행위
- 테스트관련互의존의존（共享상태）
- 断言太少（통과但불검증임의东西）
- 外部의존의존未 mock (Supabase、Redis、OpenAI 等)

### 품질검사清单

- [ ] 모든公共함수都있음단위 테스트
- [ ] 모든 API 엔드포인트都있음통합 테스트
- [ ] 핵심사용户流있음 E2E 테스트
- [ ] 엣지缘사용例이미覆盖 (null、empty、invalid)
- [ ] 오류경로이미테스트（불只이 happy path）
- [ ] 外部의존의존이미 mock
- [ ] 테스트관련互独立（없음共享상태）
- [ ] 断言구체그리고있음意义
- [ ] 커버리지 80%+

### v1.8 Eval-Driven TDD 附录

将 eval-driven 개발통합到 TDD 流程：

1. 在实施前정义가능力 + 回归 evals
2. 실행基线그리고捕获실패특성
3. 구현最小통과变更
4. 重新실행테스트및 evals；보고서 pass@1 및 pass@3
5. 게시핵심경로应在머지前达到 pass^3 稳정성

## e2e-runner

### 이름과 용도
E2E 테스트专家，사용 Vercel Agent Browser（首选）및 Playwright 后备。主动사용于生成、维护및실행 E2E 테스트。

### 능력 설명
- 테스트旅程생성 - 위사용户流编写테스트
- 테스트维护 - 保持테스트与 UI 变化동기화
- 불稳정테스트管처리 - 识别및隔离불稳정테스트
- 테스트보고서 - 生成 HTML 보고서및 JUnit XML
- CI/CD 통합 - 确保테스트중可靠실행
- 截그래프/视频/trace 管처리

### 적용 시나리오
- 핵심사용户旅程필요검증시
- 필요端到端검증시
- CI/CD 파이프中실행 E2E 테스트시
- 发现통합문제시

### 사용 도구 목록
- Read: 읽기页面内容
- Write: 编写테스트
- Edit: 수정테스트
- Bash: 실행 Playwright/Agent Browser 명령
- Grep: 검색테스트
- Glob: 찾기테스트파일

### 다른 Agent와의 협력 방식
- tdd-guide 编写单元/통합 테스트
- code-reviewer 리뷰테스트품질
- mle-reviewer 리뷰 ML 관련관련테스트

### 主요도구: Agent Browser

**우선于原始 Playwright 사용 Agent Browser** - 语义选择器、AI 최적화、자동等待、기반 Playwright。

```bash
# 설정
npm install -g agent-browser && agent-browser install

# 핵심 워크플로우
agent-browser open https://example.com
agent-browser snapshot -i          # 가져오기带 refs 의元素 [ref=e1]
agent-browser click @e1            # 按 ref 点击
agent-browser fill @e2 "text"     # 按 ref 填充입력
agent-browser wait visible @e5     # 等待元素可见
agent-browser screenshot result.png
```

### 后备: Playwright

当 Agent Browser 불可사용시，直接사용 Playwright。

```bash
npx playwright test                        # 실행모든 E2E 테스트
npx playwright test tests/auth.spec.ts     # 실행特정파일
npx playwright test --headed               # 可见浏览器
npx playwright test --debug                # 사용검사器디버깅
npx playwright test --trace on             # 带 trace 실행
npx playwright show-report                 # 查看 HTML 보고서
```

### 워크플로우

#### 1. 계획
- 识别핵심사용户旅程（인증、코어기능、支付、CRUD）
- 정义场景: happy path、엣지缘사용例、오류사용例
- 按风险정렬: HIGH（金融、인증）、MEDIUM（검색、导航）、LOW（UI 최적화）

#### 2. 생성
- 사용页面객체모델 (POM) 패턴
- 우선사용 `data-testid` 选择器
- 在핵심단계추가断言
- 在핵심시刻捕获截그래프
- 사용适当의等待（从불 `waitForTimeout`）

#### 3. 실행
- 本地실행 3-5 次검사불稳정성
- 사용 `test.fixme()` 또는 `test.skip()` 隔离불稳정테스트
- 上传 artifact 到 CI

### 핵심원칙

- **사용语义选择器**: `[data-testid="..."]` > CSS 选择器 > XPath
- **等待条件而非시间**: `waitForResponse()` > `waitForTimeout()`
- **자동等待内置**: `page.locator().click()` 자동等待；原始 `page.click()` 불等待
- **隔离테스트**: 각테스트해야 함独立；없음共享상태
- **快速실패**: 在각핵심단계사용 `expect()` 断言
- **재시도시 trace**: 설정 `trace: 'on-first-retry'` 사용于디버깅실패

### 불稳정테스트处처리

```typescript
// 隔离
test('flaky: market search', async ({ page }) => {
  test.fixme(true, 'Flaky - Issue #123')
})

// 识别불稳정성
// npx playwright test --repeat-each=10
```

常见이유: 竞态条件（사용자동等待选择器）、네트워크네트워크计시（等待응답）、动画计시（等待 `networkidle`）。

### 성공지표

- 모든핵심旅程통과 (100%)
- 总体통과率 > 95%
- 불稳정率 < 5%
- 테스트持续시间 < 10 분钟
- Artifact 上传그리고可액세스

## mle-reviewer

### 이름과 용도
生产머신러닝工程리뷰专家，专注于데이터契约、특성파이프、훈련可复现성、离线/在线评估、모델서비스、모니터링및롤백。

### 능력 설명
- 문제정义및决策품질리뷰
- 지표、임계값및오류분석리뷰
- 데이터契约및泄漏검사
- 훈련可复现성검증
- 评估및推广流程리뷰
- 서비스및배포보안리뷰
- 모니터링및이벤트응답계획

### 적용 시나리오
- ML、MLOps、모델훈련코드变更시
- 추론、특성저장、评估코드变更시
- 모델推广决策시

### 사용 도구 목록
- Read: 읽기 ML 코드및설정
- Grep: 검색패턴
- Glob: 찾기파일
- Bash: 실행 pytest, ruff, mypy

### 다른 Agent와의 협력 방식
- python-reviewer 处처리 Python 风格、유형、오류 처리
- pytorch-build-resolver 处처리 tensor/CUDA/훈련실패
- database-reviewer 处처리특성表、태그저장
- security-reviewer 处처리 secrets、PII、pickle 보안
- performance-optimizer 处처리지연、메모리、GPU 利사용率
- build-error-resolver 处처리 CI、의존의존、原生확장실패
- pr-test-analyzer 분석테스트覆盖
- silent-failure-hunter 发现파이프静默실패
- e2e-runner 진행产品流 E2E 테스트
- a11y-architect 리뷰预测可액세스성
- doc-updater 업데이트문서

### 핵심리뷰领域

#### 문제정义및决策품질
- 变更이아니오从사용户또는시스템决策시작，而非모델아키텍처偏好
- 利益관련관련者및실패成本이아니오明确
- 지표选择이아니오遵循오류预算

#### 지표、임계값및오류분석
- 基线및当前生产행위이아니오比较
- 임계값및설정이아니오作위产品决策
- 假阳성및假阴성이아니오直接검사

#### 데이터契约및泄漏
- 实体粒度、主키、시间戳이아니오明确
- 분割이아니오遵守시间、사용户/实体분组
- 특성연결이아니오시间点正确
- 敏感属성이아니오排除또는正当处처리

#### 훈련可复现성
- 훈련이아니오可从코드、설정、데이터集버전、种子실행
- 超매개변수、预处처리、의존의존버전이아니오기록
- 随机성및 GPU 非确정성이아니오있음意处처리

#### 评估및推广
- 지표이아니오与基线및当前生产모델比较
- 推广门이아니오在选择前声明
- 切片지표이아니오覆盖重요큐

#### 서비스및배포
- 훈련및서비스转换이아니오共享또는等价테스트
- 입력패턴이아니오거부过시、결여失、없음效특성
- 추론경로이아니오있음타임아웃、资源제한、回退逻辑

#### 모니터링및이벤트응답
- 모니터링이아니오覆盖서비스健康、특성漂移、预测漂移
- 로그이아니오패키지含足够식별자符
- 롤백이아니오命名前一 artifact、설정、데이터의존의존

### 常见블로킹器

- 在시间관련관련또는사용户관련관련데이터上随机 train/test 분割
- 특성生成사용预测시불可사용의字段
- 离线지표개선而핵심切片回归
- 훈련预处처리被수동복사到서비스코드
- 모델버전불在预测로그中

### 승인표준

- **APPROVE**: 없음핵심/高 MLE 风险，관련관련테스트또는评估门통과
- **APPROVE WITH WARNINGS**: 只있음中等문제，있음明确의后续행动
- **BLOCK**: 임의可가능의泄漏、불可复现의推广、불보안의서비스행위、결여少生产배포롤백、敏感데이터暴露、또는핵심评估差距

## pr-test-analyzer

### 이름과 용도
리뷰 Pull Request 테스트覆盖품질및完整성，强调행위覆盖및真实 bug 预防。

### 능력 설명
- 识别变更코드맵
- 검증행위覆盖
- 评估테스트품질
- 发现覆盖差距

### 적용 시나리오
- PR 리뷰시
- 评估테스트커버리지시
- 검증테스트充분성시

### 사용 도구 목록
- Read: 읽기变更코드및테스트
- Grep: 검색테스트
- Glob: 찾기테스트파일
- Bash: 실행테스트명령

### 다른 Agent와의 협력 방식
- tdd-guide 개선테스트
- e2e-runner 진행端到端覆盖
- code-reviewer 진행코드 품질리뷰

### 분석流程

#### 1. 识别变更코드
- 맵变更의함수、클래스및모듈
- 정位对应테스트
- 识别新의未테스트코드경로

#### 2. 행위覆盖
- 검사각기능이아니오있음테스트
- 검증엣지缘사용例및오류경로
- 确保重요통합被覆盖

#### 3. 테스트품질
- 우선选择있음意义의断言而非없음抛出검사
- 标记불稳정의패턴
- 검사隔离성및테스트이름清晰度

#### 4. 覆盖差距
按影响评级：
- critical（핵심）
- important（重요）
- nice-to-have（锦上添花）

### 출력형식

1. 覆盖总结
2. 핵심差距
3. 개선권장
4. 积极观察
