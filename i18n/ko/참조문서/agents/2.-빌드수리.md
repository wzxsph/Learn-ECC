## build-error-resolver

### 이름과 용도
TypeScript 빌드및유형오류해결专家。수정빌드실패또는유형오류，专注于让빌드통과，불做아키텍처수정。

### 능력 설명
- TypeScript 오류해결
- 빌드오류수정
- 의존의존문제수정
- 설정오류해결
- 最小化 diff 수정

### 적용 시나리오
- 빌드실패시
- TypeScript 유형오류发生시
- 모듈파싱문제시

### 사용 도구 목록
- Read: 읽기파일内容
- Write: 쓰기수정
- Edit: 편집파일
- Bash: 실행 tsc, npm build, eslint
- Grep: 검색오류패턴
- Glob: 찾기관련관련파일

### 다른 Agent와의 협력 방식
- 불做리팩토링 → 사용 refactor-cleaner
- 불做아키텍처变更 → 사용 architect
- 불추가新기능 → 사용 planner
- 불수정테스트 → 사용 tdd-guide
- 불处처리보안문제 → 사용 security-reviewer

### 코어원칙
- 只수정오류，불요리팩토링
- 最小化变更
- 검증每次수정后빌드통과

## go-build-resolver

### 능력 설명

- go vet 경고수정
- staticcheck/golangci-lint 문제수정
- 모듈의존의존문제해결
- 유형오류및인터페이스불매칭处처리

### 적용 시나리오

- go vet 报错시
- 모듈의존의존冲突시

### 사용 도구 목록
- Read: 읽기파일内容
- Write: 쓰기수정
- Edit: 편집파일
- Bash: 실행 go build, go vet, staticcheck, golangci-lint
- Grep: 검색오류패턴
- Glob: 찾기관련관련파일

### 다른 Agent와의 협력 방식

### 常见오류수정패턴

| 오류유형 | 이유 | 수정메서드 |
|----------|------|----------|
| undefined: X | 결여少가져오기、拼写오류 | 추가가져오기또는수정크기写 |
| cannot use X as type Y | 유형불매칭 | 유형转换또는解引사용 |
| X does not implement Y | 결여少메서드 | 사용正确 receiver 구현메서드 |
| import cycle not allowed | 루프의존의존 | 提取共享유형到新패키지 |
| cannot find package | 결여少의존의존 | go get pkg@version 또는 go mod tidy |

- 만약패키지含 `quarkus` → 应사용 [QUARKUS] 규칙
