# 빌드 수정클래스 Agent

빌드 수정클래스 Agent 전담진단和수정各种프로그래밍 언어的빌드오류、컴파일오류和依赖문제。

## Agent 목록

| Agent 이름 | 용도 | 사용모델 | 코어도구 |
|------------|------|----------|----------|
| build-error-resolver | TypeScript/通用빌드오류수정 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| go-build-resolver | Go 빌드/컴파일오류수정 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| kotlin-build-resolver | Kotlin/Gradle 빌드오류수정 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| rust-build-resolver | Rust 빌드/借用검사器오류수정 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| cpp-build-resolver | C++/CMake 빌드오류수정 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| java-build-resolver | Java/Maven/Gradle 빌드오류수정 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| swift-build-resolver | Swift/Xcode 빌드오류수정 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| dart-build-resolver | Dart/Flutter 빌드오류수정 | sonnet | Read, Write, Edit, Bash, Grep, Glob |

## build-error-resolver

### 이름과 용도
TypeScript 빌드和유형오류해결专家。수정빌드실패或유형오류，专注于让빌드变绿，不做아키텍처수정。

### 능력 설명
- TypeScript 오류해결
- 빌드오류수정
- 依赖문제수정
- 설정오류해결
- 最小化 diff 수정

### 적용 시나리오
- 빌드실패时
- TypeScript 유형오류发生时
- 모듈파싱문제时

### 사용 도구 목록
- Read: 읽기파일内容
- Write: 쓰기수정
- Edit: 편집파일
- Bash: 실행 tsc, npm build, eslint
- Grep: 搜索오류패턴
- Glob: 찾기相关파일

### 다른 Agent와의 협력 방식
- 不做리팩토링 → 사용 refactor-cleaner
- 不做아키텍처变更 → 사용 architect
- 不추가新기능 → 사용 planner
- 不수정테스트 → 사용 tdd-guide
- 不处理보안문제 → 사용 security-reviewer

### 코어원칙
- 只수정오류，不要리팩토링
- 最小化变更
- 검증每次수정后빌드통과

## go-build-resolver

### 이름과 용도
Go 빌드、vet 和컴파일오류해결专家。수정 Go 빌드오류、go vet 문제和 linter 경고。

### 능력 설명
- Go 컴파일오류진단
- go vet 경고수정
- staticcheck/golangci-lint 문제수정
- 모듈依赖문제해결
- 유형오류和인터페이스不匹配处理

### 적용 시나리오
- Go 빌드실패时
- go vet 报错时
- 모듈依赖冲突时

### 사용 도구 목록
- Read: 읽기파일内容
- Write: 쓰기수정
- Edit: 편집파일
- Bash: 실행 go build, go vet, staticcheck, golangci-lint
- Grep: 搜索오류패턴
- Glob: 찾기相关파일

### 다른 Agent와의 협력 방식
- go-reviewer 리뷰코드 품질
- security-reviewer 处理보안相关문제

### 常见오류수정패턴

| 오류유형 | 原因 | 수정메서드 |
|----------|------|----------|
| undefined: X | 缺少가져오기、拼写오류 | 추가가져오기或수정크기写 |
| cannot use X as type Y | 유형不匹配 | 유형转换或解引用 |
| X does not implement Y | 缺少메서드 | 사용正确 receiver 구현메서드 |
| import cycle not allowed | 루프依赖 | 提取共享유형到新패키지 |
| cannot find package | 缺少依赖 | go get pkg@version 或 go mod tidy |

## rust-build-resolver

### 이름과 용도
Rust 빌드、컴파일和依赖오류해결专家。수정 cargo 빌드오류、借用검사器문제和 Cargo.toml 문제。

### 능력 설명

- 借用검사器和라이프사이클오류수정
- 特征구현不匹配해결
- Cargo 依赖和特征문제处理
- cargo clippy 경고수정

### 적용 시나리오
- Rust 빌드실패时
- 借用검사器报错时
- Cargo 依赖冲突时

### 사용 도구 목록
- Read: 읽기파일内容
- Write: 쓰기수정
- Edit: 편집파일
- Bash: 실행 cargo check, clippy, fmt, tree
- Grep: 搜索오류패턴
- Glob: 찾기相关파일

### 다른 Agent와의 협력 방식
- rust-reviewer 리뷰코드 품질
- security-reviewer 处理보안相关문제

### 常见오류수정패턴

| 오류유형 | 原因 | 수정메서드 |
|----------|------|----------|
| cannot borrow as mutable | 不可变借用활성화 | 리팩토링以先종료不可变借用 |
| does not live long enough | 值在仍被借用时丢弃 | 확장라이프사이클作用域 |
| cannot move out of | 从引用后이동 | 사용 .clone() 或리팩토링 |
| mismatched types | 유형오류 | 추가 .into()、as 或显式转换 |
| trait X is not implemented for Y | 缺少 impl 或 derive | 추가 #[derive(Trait)] |

- Checkstyle 和 SpotBugs 违规수정

### 적용 시나리오
- Java 빌드실패时
- Maven/Gradle 依赖冲突时
- 프레임워크特定빌드문제时

### 사용 도구 목록
- Read: 읽기파일内容

- Grep: 搜索오류패턴
- Glob: 찾기相关파일

### 다른 Agent와의 협력 방식
- java-reviewer 리뷰코드 품질
- security-reviewer 处理보안相关문제

### 프레임워크감지
자동감지프로젝트프레임워크：
- 如果패키지含 `quarkus` → 应用 [QUARKUS] 규칙
- 如果패키지含 `spring-boot` → 应用 [SPRING] 규칙
- 两者都있음 → 应用两个규칙集

### 常见오류수정패턴

| 오류유형 | 原因 | 수정메서드 |
|----------|------|----------|
| cannot find symbol | 缺少가져오기、拼写오류 | 추가가져오기或依赖 |
| incompatible types | 유형不匹配 | 추가显式转换 |
| package X does not exist | 缺少依赖 | 추가到 pom.xml/build.gradle |
| No qualifying bean of type X | 缺少 @Component 或컴포넌트扫描 | 추가注解或수정扫描패키지 |

- security-reviewer 处理보안相关문제

### 常见오류수정패턴

| 오류유형 | 原因 | 수정메서드 |
|----------|------|----------|
| The name 'X' isn't defined | 缺少가져오기或拼写오류 | 추가正确的 import |

| Because X depends on Y >=A and Z depends on Y <B | Pub 버전冲突 | 调整버전约束或추가 dependency_overrides |
