# 빌드 수정명령

本문서介绍 ECC 中用于수정各언어빌드오류的专门명령。

---

## /go-build

**용도설명**: 逐步수정 Go 빌드오류、go vet 경고和 linter 문제。

**사용메서드**:
```
/go-build
```

**사용场景**:
- `go build ./...` 실패
- `go vet ./...` 보고서문제
- `golangci-lint run` 표시경고
- 모듈依赖损坏
- 풀更改后빌드실패

**워크플로우**:
1. **실행진단** - 실행 `go build`、`go vet`、`staticcheck`
2. **파싱오류** - 按파일分组并按심각程度排序
3. **逐步수정** - 一次一个오류
4. **검증수정** - 每次更改后重新실행빌드
5. **보고서总结** - 표시已수정和剩余문제

**常见오류수정**:

| 오류 | 典型수정 |
|------|----------|
| `undefined: X` | 추가가져오기或수정拼写 |
| `cannot use X as Y` | 유형转换或수정赋值 |
| `missing return` | 추가 return 语句 |
| `X does not implement Y` | 추가缺失메서드 |
| `import cycle` | 리팩토링패키지구조 |
| `declared but not used` | 제거或사용변수 |
| `cannot find package` | `go get` 或 `go mod tidy` |

**진단명령**:
```bash
go build ./...                # 主빌드검사
go vet ./...                  # 静态분석
staticcheck ./...             # 高级 lint（如果可用）
golangci-lint run             # Linting
go mod verify                # 모듈검증
go mod tidy -v               # 整理依赖
```

## /kotlin-build

**용도설명**: 逐步수정 Kotlin/Gradle 빌드오류、컴파일器경고和依赖문제。

**사용메서드**:
```
/kotlin-build
```

**사용场景**:
- `./gradlew build` 실패
- Kotlin 컴파일器보고서오류
- `./gradlew detekt` 보고서违规
- Gradle 依赖파싱실패
- 풀更改后빌드실패

**常见오류수정**:

| 오류 | 典型수정 |
|------|----------|
| `Unresolved reference: X` | 추가가져오기或依赖 |
| `Type mismatch` | 수정유형转换或赋值 |
| `'when' must be exhaustive` | 추가缺失的密封클래스브랜치 |
| `Suspend function can only be called from coroutine` | 추가 `suspend` 修饰符 |
| `Smart cast impossible` | 사용局部 `val` 或 `let` |
| `Could not resolve dependency` | 수정버전或추가仓库 |

**진단명령**:
```bash
./gradlew build 2>&1                      # 主빌드검사
./gradlew detekt 2>&1                      # 静态분석（如果설정）
./gradlew ktlintCheck 2>&1                # 형식검사（如果설정）
./gradlew dependencies --configuration runtimeClasspath | head -100  # 依赖문제
./gradlew build --refresh-dependencies     # 깊이새로고침（캐시或依赖元데이터可疑时）
```

## /rust-build

**용도설명**: 逐步수정 Rust 빌드오류、借用검사器문제和依赖문제。

**사용메서드**:
```
/rust-build
```

**사용场景**:
- `cargo build` 或 `cargo check` 실패
- `cargo clippy` 보고서경고
- 借用검사器或라이프사이클오류阻止컴파일
- Cargo 依赖파싱실패
- 풀更改后빌드실패

**常见오류수정**:

| 오류 | 典型수정 |
|------|----------|
| `cannot borrow as mutable` | 리팩토링以在사용可变访问之前종료不可变借用 |
| `does not live long enough` | 사용拥있음유형或추가라이프사이클标注 |
| `cannot move out of` | 리팩토링以가져오기소유권；仅作为最后手段克隆 |
| `mismatched types` | 추가 `.into()`、`as` 或显式转换 |
| `trait X not implemented` | 추가 `#[derive(Trait)]` 或手动구현 |
| `unresolved import` | 추가到 Cargo.toml 或수정 `use` 路径 |
| `cannot find value` | 추가가져오기或수정路径 |

**진단명령**:
```bash
cargo check 2>&1                               # 主빌드검사
cargo clippy -- -D warnings 2>&1               # Lints
cargo fmt --check 2>&1                         # 형식검사
cargo tree --duplicates                        # 重复依赖
cargo audit                                   # 보안审计（如果可用）
```

## /cpp-build

**용도설명**: 逐步수정 C++ 빌드오류、CMake 문제和链接器문제。

**사용메서드**:
```
/cpp-build
```

**사용场景**:
- `cmake --build build` 실패
- 链接器오류（未定义引用、多次定义）
- 模板인스턴스化실패
- 패키지含/依赖문제
- 풀更改后빌드실패

**常见오류수정**:

| 오류 | 典型수정 |
|------|----------|
| `undeclared identifier` | 추가 `#include` 或수정拼写 |
| `no matching function` | 수정参数유형或추가重载 |
| `undefined reference` | 链接库或추가구현 |
| `multiple definition` | 사용 `inline` 或移到 .cpp |
| `incomplete type` | 用 `#include` 교체前向声明 |
| `no member named X` | 수정成员名称或추가 include |
| `cannot convert X to Y` | 추가适当转换 |
| `CMake Error` | 수정 CMakeLists.txt 설정 |

**진단명령**:
```bash
cmake -B build -S .                        # CMake 설정
cmake --build build 2>&1 | head -100       # 빌드
clang-tidy src/*.cpp -- -std=c++17         # 静态분석（如果可用）
cppcheck --enable=all src/                 # 额外분석（如果可用）
```

## /gradle-build

**용도설명**: 수정 Android 和 Kotlin Multiplatform (KMP) 프로젝트的 Gradle 빌드오류。

**사용메서드**:
```
/gradle-build
```

**사용场景**:
- Android 프로젝트빌드실패
- KMP 프로젝트컴파일오류
- Gradle 동기화실패
- 依赖冲突

**프로젝트유형检测**:

| 指示器 | 빌드명령 |
|--------|----------|
| `build.gradle.kts` + `composeApp/` (KMP) | `./gradlew composeApp:compileKotlinMetadata` |
| `build.gradle.kts` + `app/` (Android) | `./gradlew app:compileDebugKotlin` |
| `settings.gradle.kts` 带모듈 | `./gradlew assemble` |
| 설정了 detekt | `./gradlew detekt` |

**常见오류수정**:

| 오류 | 수정 |
|------|------|
| `commonMain` 中未파싱的引用 | 검사依赖是否在 `commonMain.dependencies {}` 中 |
| Expect 声明없음 actual | 在각平台源集中추가 `actual` 구현 |
| Compose 컴파일器버전不匹配 | 在 `libs.versions.toml` 中对齐 Kotlin 和 Compose 컴파일器버전 |
| 重复클래스 | 用 `./gradlew dependencies` 검사冲突依赖 |
| KSP 오류 | 실행 `./gradlew kspCommonMainKotlinMetadata` 重新生成 |
| 설정캐시문제 | 검사非可직렬화的태스크输入 |

## /flutter-build

**용도설명**: 逐步수정 Dart 분석器오류和 Flutter 빌드실패。

**사용메서드**:
```
/flutter-build
```

**사용场景**:
- `flutter analyze` 보고서오류
- `flutter build` 임의平台실패
- `flutter pub get` 버전冲突
- `build_runner` 生成코드실패
- 풀更改后빌드실패

**常见오류수정**:

| 오류 | 典型수정 |
|------|----------|
| `A value of type 'X?' can't be assigned to 'X'` | 추가 `?? default` 或空值保护 |
| `The name 'X' isn't defined` | 추가가져오기或수정拼写 |
| `Non-nullable instance field must be initialized` | 추가초기화器或 `late` |
| `Version solving failed` | 调整 pubspec.yaml 中的버전约束 |
| `Missing concrete implementation of 'X'` | 구현缺失的인터페이스메서드 |
| `build_runner: Part of X expected` | 삭제过时的 `.g.dart` 并重新빌드 |

**진단명령**:
```bash
flutter analyze 2>&1                  # 분석
flutter pub get 2>&1                  # 依赖
dart run build_runner build --delete-conflicting-outputs 2>&1  # 코드生成（如果사용 build_runner）
flutter build apk 2>&1                # 平台빌드
flutter build web 2>&1                # Web 빌드
```

## 빌드 수정명령对比表

| 명령 | 언어/平台 | 主要도구 | 常见문제 |
|------|----------|----------|----------|
| `/go-build` | Go | go build, go vet | 유형오류、가져오기루프 |
| `/kotlin-build` | Kotlin | Gradle, detekt | 유형不匹配、when 非穷举 |
| `/rust-build` | Rust | cargo check, clippy | 借用오류、라이프사이클 |
| `/cpp-build` | C++ | CMake, clang-tidy | 链接器오류、模板문제 |
| `/gradle-build` | Android/KMP | Gradle | 依赖冲突、설정오류 |
| `/flutter-build` | Flutter/Dart | Flutter analyze | 空보안、분석오류 |

## 通用수정전략

모든빌드 수정명령遵循相同的基本전략：

1. **先수정빌드오류** - 코드필수컴파일
2. **其次수정 lint 경고** - 수정可疑构造
3. **然后수정형식경고** - 样式和모범 사례
4. **一次수정一个** - 검증每次更改
5. **最小更改** - 不要리팩토링，只是수정

**중지条件**:
如果发生以下情况，프록시将중지并보고서：
- 相同오류在 3 次尝试后仍然存在
- 수정引入了更多오류
- 필요아키텍처更改
- 缺少外部依赖
