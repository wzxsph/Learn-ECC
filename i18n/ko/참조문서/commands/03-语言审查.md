# 언어 리뷰명령

本문서介绍 ECC 中用于各프로그래밍 언어코드 리뷰的专门명령。

---

## /python-review

**용도설명**: Python 코드全面리뷰。검사 PEP 8 合规性、타입 힌트、보안和 Python 习惯用法。

**사용메서드**:
```
/python-review
```

**사용场景**:
- 编写或修改 Python 코드之后
- 커밋 Python 变更之前
- 리뷰패키지含 Python 코드的 PR
- 학습 Python 习惯用法和모범 사례

**리뷰클래스别**:

### CRITICAL (필수수정)
- SQL/명령注入漏洞
- 不보안的 eval/exec 사용
- Pickle 不보안역직렬화
- 평문 자격 증명
- YAML 不보안加载
- 숨김오류的裸 except 子句

### HIGH (해야 함수정)
- 公共함수缺少타입 힌트
- 可变기본参数
- 静默吞没异常
- 资源管理未사용上下文管理器
- C 风格루프而非推导式
- 사용 type() 而非 isinstance()

### MEDIUM (권장고려)
- PEP 8 형식违规
- 公共함수缺少문서字符串
- Print 语句而非로그
- 低效的字符串操作
- 魔术数字未사용命名상수
- 未사용 f-strings

**자동검사명령**:
```bash
mypy .                              # 유형검사
ruff check .                        # Linting
black --check .                     # 형식化검사
isort --check-only .                # 가져오기排序
bandit -r .                         # 보안扫描
pip-audit                           # 依赖审计
pytest --cov=app --cov-report=term-missing  # 테스트覆盖
```

**常见문제수정**:

```python
# SQL 인젝션수정
# 오류
query = f"SELECT * FROM users WHERE id = {user_id}"

# 正确 - 사용参数化查询
query = "SELECT * FROM users WHERE id = %s"
cursor.execute(query, (user_id,))

# 可变기본参数수정
# 오류
def process_items(items=[]):
    items.append("new")
    return items

# 正确
def process_items(items=None):
    if items is None:
        items = []
    items.append("new")
    return items

# 사용上下文管理器
# 오류
f = open("config.json")
data = f.read()
f.close()

# 正确
with open("config.json") as f:
    data = f.read()
```

## /go-review

**용도설명**: Go 코드全面리뷰。검사惯用패턴、동시성보안、오류 처리和보안。

**사용메서드**:
```
/go-review
```

**사용场景**:
- 编写或修改 Go 코드之后
- 커밋 Go 变更之前
- 리뷰패키지含 Go 코드的 PR
- 학습 Go 惯用패턴

**리뷰클래스别**:

### CRITICAL (필수수정)
- SQL/명령注入漏洞
- 없음동기화的竞态条件
- Goroutine 泄漏
- 평문 자격 증명
- 不보안的指针사용
- 핵심路径무시오류

### HIGH (해야 함수정)
- 缺少오류的上下文패키지装
- Panic 而非오류돌아가기
- Context 未传播
- 未缓冲通道导致死锁
- 인터페이스未구현오류
- 缺少互斥锁保护

### MEDIUM (권장고려)
- 非惯用코드패턴
- 내보내기함수缺少 godoc 주석
- 低效的字符串拼接
- Slice 未预分配
- 未사용表驱动테스트

**자동검사명령**:
```bash
go vet ./...                         # 静态분석
staticcheck ./...                    # 高级검사（如果설치）
golangci-lint run                   # Linting
go build -race ./...                # 竞态检测
govulncheck ./...                   # 보안漏洞
```

## /kotlin-review

**용도설명**: Kotlin 코드全面리뷰。검사惯用패턴、空보안、코루틴보안和보안。

**사용메서드**:
```
/kotlin-review
```

**사용场景**:
- 编写或修改 Kotlin 코드之后
- 커밋 Kotlin 变更之前
- 리뷰 Android/KMP 프로젝트的 PR
- 학습 Kotlin 惯用패턴

**리뷰클래스别**:

### CRITICAL (필수수정)
- SQL/명령注入漏洞
- 없음充分理由的强制解패키지 `!!`
- 平台유형空보안违规
- GlobalScope 사용（구조化동시성违规）
- 평문 자격 증명
- 不보안역직렬화

### HIGH (해야 함수정)
- 可变状态（可用不可变时）
- 코루틴上下文内的阻塞调用
- 长루프中缺少取消검사
- 密封클래스 `when` 非穷举
- 大함수（>50行）
- 深嵌套（>4层）

### MEDIUM (권장고려)
- 非惯用 Kotlin（Java 风格패턴）
- 缺少尾随逗号
- Scope 함수误用或嵌套
- 大集合链缺少 Sequence
- 冗余显式유형

**자동검사명령**:
```bash
./gradlew build                      # 빌드검사
./gradlew detekt                     # 静态분석
./gradlew ktlintCheck                # 형식검사
./gradlew test                       # 테스트
```

## /rust-review

**용도설명**: Rust 코드全面리뷰。검사소유권라이프사이클、오류 처리、不보안사용和惯用패턴。

**사용메서드**:
```
/rust-review
```

**사용场景**:
- 编写或修改 Rust 코드之后
- 커밋 Rust 变更之前
- 리뷰 Rust 프로젝트的 PR
- 학습 Rust 惯用패턴

**리뷰클래스别**:

### CRITICAL (필수수정)
- 生产코드路径中未검사的 `unwrap()`/`expect()`
- 不带 `// SAFETY:` 주석的 `unsafe`
- 字符串插值的 SQL 인젝션
- 未검증输入的명령注入
- 평문 자격 증명
- 原始指针的释放后사용

### HIGH (해야 함수정)
- 필요 없음要的 `.clone()` 以满足借用검사器
- `String` 参数应用 `&str` 或 `impl AsRef<str>`
- 비동기上下文中的阻塞（`std::thread::sleep`）
- 共享유형缺少 `Send`/`Sync` 约束
- 핵심枚举上的通配符 `_ =>` 匹配
- 大함수（>50行）

### MEDIUM (권장고려)
- 热路径中的필요 없음要分配
- 已知크기时缺少 `with_capacity`
- 없음充分理由抑制 clippy 경고
- 公共 API 缺少 `///` 문서
- 고려对可能무시值的非 `must_use` 돌아가기유형사용 `#[must_use]`

**자동검사명령**:
```bash
cargo check                           # 빌드검사
cargo clippy -- -D warnings            # Lints
cargo fmt --check                      # 형식검사
cargo test                            # 테스트
cargo audit                           # 보안审计（如果可用）
```

## /cpp-review

**용도설명**: C++ 코드全面리뷰。검사메모리 안전성、现代 C++ 习惯用法、동시성和보안。

**사용메서드**:
```
/cpp-review
```

**사용场景**:
- 编写或修改 C++ 코드之后
- 커밋 C++ 变更之前
- 리뷰 C++ 프로젝트的 PR
- 검사메모리 안전성문제

**리뷰클래스别**:

### CRITICAL (필수수정)
- 없음 RAII 的裸 `new`/`delete`
- 缓冲区溢出和释放后사용
- 없음동기화的데이터竞态
- 통과 `system()` 的명령注入
- 未초기화변수읽기
- 空指针解引用

### HIGH (해야 함수정)
- 五法则违规
- 缺少 `std::lock_guard` / `std::scoped_lock`
- detach 스레드的寿命管理不当
- C 风格强制转换而非 `static_cast`/`dynamic_cast`
- 缺少 `const` 正确性

### MEDIUM (권장고려)
- 필요 없음要的拷贝（传值而非 `const&`）
- 已知크기时缺少 `reserve()`
- 头파일中 `using namespace std;`
- 重要돌아가기值缺少 `[[nodiscard]]`
- 过于复杂的模板元编程

**자동검사명령**:
```bash
clang-tidy --checks='*,-llvmlibc-*' src/*.cpp -- -std=c++17
cppcheck --enable=all --suppress=missingIncludeSystem src/
cmake --build build -- -Wall -Wextra -Wpedantic
```

## /flutter-review

**용도설명**: Flutter/Dart 코드 리뷰。검사惯用패턴、컴포넌트모범 사례、상태 관리、성능、可访问性和보안。

**사용메서드**:
```
/flutter-review
```

**사용场景**:
- 커밋패키지含 Flutter/Dart 变更的 PR 之前（빌드和테스트통과后）
- 구현新기능后早期发现문제
- 리뷰他人的 Flutter 코드
- 审核컴포넌트、상태 관리컴포넌트或서비스클래스
- 生产게시之前

**리뷰区域**:

| 区域 | 심각级别 |
|------|----------|
| 硬인코딩密钥、明文 HTTP | CRITICAL |
| 아키텍처违规、상태 관리反패턴 | CRITICAL |
| 컴포넌트重建문제、资源泄漏 | HIGH |
| 缺少 `dispose()`、`await` 后的 BuildContext | HIGH |
| Dart 空보안、缺少오류/加载状态 | HIGH |
| Const 传播、컴포넌트组合 | HIGH |
| `build()` 中的昂贵작업 | HIGH |
| 可访问性、语义태그 | MEDIUM |
| 状态转换缺少테스트 | HIGH |
| 硬인코딩字符串（l10n） | MEDIUM |

## /fastapi-review

**용도설명**: FastAPI 应用리뷰。검사아키텍처、비동기正确性、依赖注入、Pydantic 패턴、보안、성능和可테스트性。

**사용메서드**:
```
/fastapi-review [파일或디렉토리]
```

**리뷰区域**:
- App factory、라우팅边界、미들웨어和异常处理器
- Pydantic 요청和응답패턴分离
- 데이터베이스세션、인증、分页和설정的依赖注入
- 비동기데이터베이스和外部 HTTP 패턴
- CORS、인증、限流、로그和密钥处理
- OpenAPI 元데이터和문서화응답모델
- 테스트클라이언트설정和依赖覆盖
