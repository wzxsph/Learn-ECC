# 빌드 수정 명령

본 문서는 ECC에서 각 언어 빌드 오류를 수정하기 위한专门 명령을 소개합니다.

---

## /go-build

**용도 설명**: 점진적 Go 빌드 오류, go vet 경고 및 linter 문제 수정.

**사용 메서드**:
```
/go-build
```

**사용 시나리오**:
- `go build ./...` 실패
- `go vet ./...` 보고서 문제
- `golangci-lint run` 표시 경고
- 모듈 의존성 손상
- 풀 변경 후 빌드 실패

**워크플로우**:
1. **진단 실행** - `go build`, `go vet`, `staticcheck` 실행
2. **오류 파싱** - 파일별로 그룹화 및 심각도 순으로 정렬
3. **점진적 수정** - 한 번에 하나씩
4. **수정 검증** - 매 변경 후 빌드 다시 실행
5. **보고서 요약** - 수정된 항목 및 남은 문제 표시

**일반적인 오류 수정**:

| 오류 | 일반 수정 |
|------|----------|
| `undefined: X` | 가져오기 추가 또는 맞춤법 수정 |
| `cannot use X as Y` | 유형 변환 또는 할당 수정 |
| `missing return` | return 문 추가 |
| `X does not implement Y` |缺失 메서드 추가 |
| `import cycle` | 패키지 구조 리팩토링 |
| `declared but not used` | 변수 제거 또는 사용 |
| `cannot find package` | `go get` 또는 `go mod tidy` |

**진단 명령**:
```bash
go build ./...                # 주 빌드 검사
go vet ./...                  # 정적 분석
staticcheck ./...             # 고급 lint (사용 가능한 경우)
golangci-lint run             # Linting
go mod verify                # 모듈 검증
go mod tidy -v               # 의존성 정리
```

## /kotlin-build

**용도 설명**: 점진적 Kotlin/Gradle 빌드 오류, 컴파일러 경고 및 의존성 문제 수정.

**사용 메서드**:
```
/kotlin-build
```

**사용 시나리오**:
- `./gradlew build` 실패
- Kotlin 컴파일러 보고서 오류
- `./gradlew detekt` 보고서 위반
- Gradle 의존성 파싱 실패
- 풀 변경 후 빌드 실패

**일반적인 오류 수정**:

| 오류 | 일반 수정 |
|------|----------|
| `Unresolved reference: X` | 가져오기 또는 의존성 추가 |
| `Type mismatch` | 유형 변환 또는 할당 수정 |
| `'when' must be exhaustive` |缺失의密封 클래스 브랜치 추가 |
| `Suspend function can only be called from coroutine` | `suspend`修飾符 추가 |
| `Smart cast impossible` | 지역 `val` 또는 `let` 사용 |
| `Could not resolve dependency` | 버전 수정 또는 저장소 추가 |

**진단 명령**:
```bash
./gradlew build 2>&1                      # 주 빌드 검사
./gradlew detekt 2>&1                      # 정적 분석 (설정된 경우)
./gradlew ktlintCheck 2>&1                # 포맷팅 검사 (설정된 경우)
./gradlew dependencies --configuration runtimeClasspath | head -100  # 의존성 문제
./gradlew build --refresh-dependencies     # 깊이 새로고침 (캐시 또는 의존성 메타데이터 의심 시)
```

## /rust-build

**용도 설명**: 점진적 Rust 빌드 오류, 빌로우 검사기 문제 및 의존성 문제 수정.

**사용 메서드**:
```
/rust-build
```

**사용 시나리오**:
- `cargo build` 또는 `cargo check` 실패
- `cargo clippy` 보고서 경고
- 빌로우 검사기 또는 라이프사이클 오류가 컴파일 차단
- Cargo 의존성 파싱 실패
- 풀 변경 후 빌드 실패

**일반적인 오류 수정**:

| 오류 | 일반 수정 |
|------|----------|
| `cannot borrow as mutable` | 가변 접근 사용 전 불변 빌로우 종료하도록 리팩토링 |
| `does not live long enough` | 소유 유형 사용 또는 라이프사이클 라벨 추가 |
| `cannot move out of` | 소유권 가져오기 위해 리팩토링; 마지막 수단으로만 복제 |
| `mismatched types` | `.into()`, `as` 또는 명시적 변환 추가 |
| `trait X not implemented` | `#[derive(Trait)]` 추가 또는 수동 구현 |
| `unresolved import` | Cargo.toml에 추가 또는 `use` 경로 수정 |
| `cannot find value` | 가져오기 추가 또는 경로 수정 |

**진단 명령**:
```bash
cargo check 2>&1                               # 주 빌드 검사
cargo clippy -- -D warnings 2>&1               # Lints
cargo fmt --check 2>&1                         # 포맷팅 검사
cargo tree --duplicates                        # 중복 의존성
cargo audit                                   # 보안 감사 (사용 가능한 경우)
```

## /cpp-build

**용도 설명**: 점진적 C++ 빌드 오류, CMake 문제 및 링커 문제 수정.

**사용 메서드**:
```
/cpp-build
```

**사용 시나리오**:
- `cmake --build build` 실패
- 링커 오류 (정의되지 않은 참조, 여러 정의)
- 템플릿 인스턴스화 실패
- 패키지 포함/의존성 문제
- 풀 변경 후 빌드 실패

**일반적인 오류 수정**:

| 오류 | 일반 수정 |
|------|----------|
| `undeclared identifier` | `#include` 추가 또는 맞춤법 수정 |
| `no matching function` | 매개변수 유형 수정 또는重载 추가 |
| `undefined reference` | 라이브러리 링크 또는 구현 추가 |
| `multiple definition` | `inline` 사용 또는 .cpp로 이동 |
| `incomplete type` | 전방 선언 교체 전 `#include` |
| `no member named X` | 멤버 이름 수정 또는 include 추가 |
| `cannot convert X to Y` | 적절한 변환 추가 |
| `CMake Error` | CMakeLists.txt 설정 수정 |

**진단 명령**:
```bash
cmake -B build -S .                        # CMake 설정
cmake --build build 2>&1 | head -100       # 빌드
clang-tidy src/*.cpp -- -std=c++17         # 정적 분석 (사용 가능한 경우)
cppcheck --enable=all src/                 # 추가 분석 (사용 가능한 경우)
```

## /gradle-build

**용도 설명**: Android 및 Kotlin Multiplatform (KMP) 프로젝트의 Gradle 빌드 오류 수정.

**사용 메서드**:
```
/gradle-build
```

**사용 시나리오**:
- Android 프로젝트 빌드 실패
- KMP 프로젝트 컴파일 오류
- Gradle 동기화 실패
- 의존성 충돌

**프로젝트 유형 감지**:

| 지시자 | 빌드 명령 |
|--------|----------|
| `build.gradle.kts` + `composeApp/` (KMP) | `./gradlew composeApp:compileKotlinMetadata` |
| `build.gradle.kts` + `app/` (Android) | `./gradlew app:compileDebugKotlin` |
| `settings.gradle.kts`带모듈 | `./gradlew assemble` |
| detekt 설정됨 | `./gradlew detekt` |

**일반적인 오류 수정**:

| 오류 | 수정 |
|------|------|
| `commonMain`에서 미파싱된 참조 | 의존성이 `commonMain.dependencies {}`에 있는지 확인 |
| Expect 선언 없음 actual | 각 플랫폼 소스 집합에 `actual` 구현 추가 |
| Compose 컴파일러 버전 불일치 | `libs.versions.toml`에서 Kotlin 및 Compose 컴파일러 버전对齐 |
| 중복 클래스 | `./gradlew dependencies`로 충돌 의존성 확인 |
| KSP 오류 | `./gradlew kspCommonMainKotlinMetadata` 실행하여 다시 생성 |
| 설정 캐시 문제 | 직렬화할 수 없는 태스크 입력 확인 |

## /flutter-build

**용도 설명**: 점진적 Dart 분석기 오류 및 Flutter 빌드 실패 수정.

**사용 메서드**:
```
/flutter-build
```

**사용 시나리오**:
- `flutter analyze` 보고서 오류
- `flutter build` 임의 플랫폼 실패
- `flutter pub get` 버전 충돌
- `build_runner` 생성 코드 실패
- 풀 변경 후 빌드 실패

**일반적인 오류 수정**:

| 오류 | 일반 수정 |
|------|----------|
| `A value of type 'X?' can't be assigned to 'X'` | `?? default` 또는 null 보호 추가 |
| `The name 'X' isn't defined` | 가져오기 추가 또는 맞춤법 수정 |
| `Non-nullable instance field must be initialized` | 초기화器 또는 `late` 추가 |
| `Version solving failed` | pubspec.yaml에서 버전 제약 조건 조정 |
| `Missing concrete implementation of 'X'` |缺失의 인터페이스 메서드 구현 |
| `build_runner: Part of X expected` | 오래된 `.g.dart` 삭제하고 다시 빌드 |

**진단 명령**:
```bash
flutter analyze 2>&1                  # 분석
flutter pub get 2>&1                  # 의존성
dart run build_runner build --delete-conflicting-outputs 2>&1  # 코드 생성 (build_runner 사용 시)
flutter build apk 2>&1                # 플랫폼 빌드
flutter build web 2>&1                # Web 빌드
```

## 범용 수정 전략

모든 빌드 수정 명령은 동일한 기본 전략을 따릅니다:

1. **먼저 빌드 오류 수정** - 코드가 반드시 컴파일되어야 함
2. **다음 lint 경고 수정** - 의심스러운 구조 수정
3. **그런 다음 형식 경고 수정** - 스타일 및 모범 사례
4. **한 번에 하나씩** - 매 변경 검증
5. **최소 변경** - 리팩토링하지 말고 수정만

**중지 조건**:
다음 경우에 프록시가 중지하고 보고합니다:
- 동일한 오류가 3회 시도 후에도 계속 존재
- 수정이 더 많은 오류를 유발
- 아키텍처 변경 필요
- 외부 의존성 누락
