# 기타 명령

## 개요

기타杂项명령，패키지括 Jira 통합、GAN 操作、보안扫描等기능。

## 명령列表

### /jira

**용도**: 与 Jira 工单交互 - 가져오기、분석、评论、转换状态、搜索

**描述**: 与 Jira 프로젝트 관리도구통합，지원直接워크플로우中사용 Jira tickets。

**用法**:

| 명령 | 설명 |
|---|---|
| `/jira get <TICKET-KEY>` | 가져오기并분석 Jira 工单 |
| `/jira comment <TICKET-KEY>` | 추가进度评论 |
| `/jira transition <TICKET-KEY>` | 更改工单状态 |
| `/jira search <JQL>` | 사용 JQL 搜索문제 |

**`/jira get` 워크플로우**:
1. 从 Jira 가져오기工单（통과 MCP 或 REST API）
2. 提取모든字段：摘要、描述、验收표준、优先级、태그、关联문제
3. 선택가져오기评论以가져오기额外上下文
4. 生成구조化분석

**先决条件** - Jira 凭证설정（二选一）:

**选项 A — MCP Server（권장）**:
将 `jira` 추가到 `mcpServers` 설정。

**选项 B — 环境변수**:
```bash
export JIRA_URL="https://yourorg.atlassian.net"
export JIRA_EMAIL="your.email@example.com"
export JIRA_API_TOKEN="your-api-token"
```

**与기타 명령통합**:
- 분석工单后사용 `/plan` 생성实施계획
- 사용 `tdd-workflow` skill 进行테스트驱动개발
- 사용 `/code-review` 리뷰구현
- 사용 `/jira comment` 或 `/jira transition` 업데이트工单状态

### /prune

**용도**: 삭제超过 30 天未提升的陈旧 pending instinct

**描述**: 정리超过 30 天的陈旧 instinct，这些 instinct 是자동生成但从未被리뷰或提升的。

**用法**:
```
/prune                    # 삭제超过 30 天的 instinct
/prune --max-age 60      # 自定义天数임계값
/prune --dry-run         # 预览而不삭제
```

**구현**:
```bash
python3 "${CLAUDE_PLUGIN_ROOT}/skills/continuous-learning-v2/scripts/instinct-cli.py" prune
```

或手动설치时（`CLAUDE_PLUGIN_ROOT` 未설정）:
```bash
python3 ~/.claude/skills/continuous-learning-v2/scripts/instinct-cli.py prune
```

**参数**:

| 参数 | 설명 |
|---|---|
| 없음 | 삭제超过 30 天的 instinct |
| `--max-age <days>` | 自定义时间임계값（天） |
| `--dry-run` | 预览将삭제的内容，不实际삭제 |

**모범 사례**:
- 사용 `--dry-run` 先预览要삭제的内容
- 定期실행 prune 保持 instinct 库存整洁
- 注意：只会삭제 pending 状态的 instinct，不会삭제已提升的
