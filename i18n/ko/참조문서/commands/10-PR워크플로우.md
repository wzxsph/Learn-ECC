# Pull Request 워크플로우명령

## 개요

PR 워크플로우명령用于생성、管理和리뷰 GitHub Pull Requests。

## 명령列表

### /pr

**용도**: 从当前브랜치생성 GitHub PR，带 unpushed 커밋

**描述**: 发现模板、분석变更、푸시브랜치并생성 PR。분석完整커밋历史，사용 `git diff [base-branch]...HEAD` 查看모든变更。

**输入**: `$ARGUMENTS` - 선택，可패키지含基础브랜치名和/或标志（如 `--draft`）

**파싱규칙**:
- 提取임의识别的标志（如 `--draft`）
- 将剩余的非标志文本作为基础브랜치名
- 如果未指定，기본为 `main`

**워크플로우**:

| 단계 | 설명 |
|---|---|
| **Phase 1: VALIDATE** | 검사前置条件（不在base브랜치、없음未커밋变更、있음ahead커밋、없음已있음PR） |
| **Phase 2: DISCOVER** | 搜索PR模板、분석커밋、분석파일变更、검사相关계획产物 |
| **Phase 3: PUSH** | 푸시브랜치（如果필요则先rebase） |
| **Phase 4: CREATE** | 사용模板或기본형식생성PR |
| **Phase 5: VERIFY** | 검증PR생성성공 |
| **Phase 6: OUTPUT** | 보고서PR URL和下一步 |

**PR 생성검증검사**:

| 검사 | 条件 | 실패时动作 |
|---|---|---|
| 不在base브랜치 | 当前브랜치 ≠ base | 중지："先切换到기능브랜치" |
| 작업区干净 | 없음未커밋变更 | 경고："先커밋或stash" |
| 있음ahead커밋 | `git log origin/<base>..HEAD` 非空 | 중지："없음ahead커밋" |
| 없음已있음PR | `gh pr list` 为空 | 중지："PR已存在" |

**模板搜索순서**:
1. `.github/PULL_REQUEST_TEMPLATE/` 디렉토리（如存在）
2. `.github/PULL_REQUEST_TEMPLATE.md`
3. `.github/pull_request_template.md`
4. `docs/pull_request_template.md`

**기본 PR 형식**（없음模板时）:
```markdown
## Summary
<1-2 sentence description>

## Changes
<bulleted list grouped by area>

## Files Changed
<table of changed files>

## Testing
<how changes were tested, or "Needs testing">

## Related Issues
<linked issues or "None">
```

**边缘情况处理**:
- **없음 `gh` CLI**: 중지并提示설치
- **未인증**: 提示실행 `gh auth login`
- **브랜치分叉**: 사용 `git fetch && git rebase` 然后푸시
- **大型 PR (>20 파일)**: 경고并권장拆分

**与기타 명령통합**:
- 사용 `/code-review <number>` 리뷰 PR
- 사용 `gh pr merge <number>` 머지준비 완료的 PR
- 사용 `/plan-prd` 或 `/plan` 생성相关계획产物

### /review-pr

**용도**: 리뷰 GitHub PR 或本地未커밋变更

**描述**: 리뷰 GitHub 上的 Pull Request 或本地未커밋变更。PR 리뷰패턴适配自 PRPs-agentic-eng。

**输入**: `$ARGUMENTS` - 可为 PR 编号、PR URL 或空白（本地리뷰）

---

#### 패턴选择

| 输入 | 动作 |
|---|---|
| PR 编号（如 `42`） | 作为 PR 编号사용 |
| URL（如 `github.com/.../pull/42`） | 提取 PR 编号 |
| 브랜치名 | 통과 `gh pr list --head <branch>` 찾기 PR |
| 空白 | 사용本地리뷰패턴 |

---

#### 本地리뷰패턴

**워크플로우**:
1. **GATHER** - 收集变更: `git diff --name-only HEAD`
2. **REVIEW** - 리뷰각파일（보안문제、품질문제、모범 사례）
3. **REPORT** - 生成带심각级别和수정권장的보고서

**리뷰清单**:

| 클래스别 | 체크 항목 |
|---|---|
| **보안문제 (CRITICAL)** | 평문 자격 증명、SQL注入、XSS、缺少输入검증、不보안依赖、路径遍历 |
| **코드 품질 (HIGH)** | 함수>50行、파일>800行、嵌套>4层、缺少오류 처리、console.log、TODO/FIXME |
| **모범 사례 (MEDIUM)** | 变更패턴、emoji사용、新코드缺少테스트、可访问性문제 |

**决策**:

| 条件 | 决策 |
|---|---|
| 存在 CRITICAL 或 HIGH 문제 | **BLOCK** - 阻止커밋 |
| 仅 MEDIUM/LOW 문제 | 통과但带评论 |
| 없음문제且검증통과 | **APPROVE** |

---

#### PR 리뷰패턴

**워크플로우**:
1. **FETCH** - 가져오기 PR 元데이터和 diff
2. **CONTEXT** - 빌드리뷰上下文（프로젝트규칙、계획产物、变更파일）
3. **REVIEW** - 읽기完整파일，应用 7 클래스리뷰清单
4. **VALIDATE** - 실행프로젝트유형的검증명령
5. **DECIDE** - 根据发现形成권장
6. **REPORT** - 생성리뷰产物
7. **PUBLISH** - 게시리뷰到 GitHub
8. **OUTPUT** - 보고서给用户

**7 클래스리뷰清单**:

| 클래스别 | 체크 항목 |
|---|---|
| **Correctness** | 逻辑오류、边界情况、空处理、竞态条件 |
| **Type Safety** | 유형不匹配、不보안類型转换、`any` 사용、缺少泛型 |
| **Pattern Compliance** | 符合프로젝트约定（命名、파일구조、오류 처리、가져오기） |
| **Security** | 注入、인증缺口、secret暴露、SSRF、路径遍历、XSS |
| **Performance** | N+1查询、缺少인덱스、없음界루프、메모리泄漏、大payload |
| **Completeness** | 缺少테스트、缺少오류 처리、不完整마이그레이션、缺少문서 |
| **Maintainability** | 死코드、魔法数字、深嵌套、命名不清、缺少유형 |

**검증명령检测**:

| 프로젝트유형 | 检测파일 | 검증명령 |
|---|---|---|
| Node.js/TypeScript | `package.json` | `npm run typecheck`, `npm run lint`, `npm test`, `npm run build` |
| Rust | `Cargo.toml` | `cargo clippy`, `cargo test`, `cargo build` |
| Go | `go.mod` | `go vet ./...`, `go test ./...`, `go build ./...` |
| Python | `pyproject.toml` | `pytest` |

**决策级别**:

| 级别 | 含义 | 动作 |
|---|---|---|
| CRITICAL | 보안漏洞或데이터丢失风险 | 필수수정才能머지 |
| HIGH | 可能导致문제的 bug 或逻辑오류 | 머지前应수정 |
| MEDIUM | 코드 품질或모범 사례缺失 | 권장수정 |
| LOW | 样式문제或小권장 | 선택 |

**게시리뷰到 GitHub**:
```bash
# APPROVE
gh pr review <NUMBER> --approve --body "<summary>"

# REQUEST_CHANGES
gh pr review <NUMBER> --request-changes --body "<summary with required fixes>"

# COMMENT only (draft PR 或정보性)
gh pr review <NUMBER> --comment --body "<summary>"
```
