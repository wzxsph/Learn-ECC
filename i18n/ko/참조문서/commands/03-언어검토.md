# 언어 리뷰 명령

본 문서는 ECC에서 각 프로그래밍 언어 코드 리뷰용专门 명령을 소개합니다.

---

## /python-review

**용도 설명**: Python 코드 포괄적 리뷰. PEP 8 준수, 타입 힌트, 보안 및 Python 관용구를 검사합니다.

**사용 메서드**:
```
/python-review
```

**사용 시나리오**:
- Python 코드 작성 또는 수정 후
- Python 변경 커밋 전
- Python 코드를 포함한 PR 리뷰
- Python 관용구 및 모범 사례 학습

**리뷰 범주**:

### CRITICAL (필수 수정)
- SQL/명령 인젝션 취약점
-不安全한 eval/exec 사용
- Pickle不安全 역직렬화
- 평문 자격 증명
- YAML不安全 로드
- 숨은 오류의裸 except 절

### HIGH (해야 함 수정)
- 공용 함수에 타입 힌트 누락
- 가변 기본 매개변수
- 자동吞噬 예외
- 컨텍스트 관리자를 사용하지 않는 리소스 관리
- C 스타일 루프而非 리스트 내포
- type() 사용而非 isinstance()

### MEDIUM (권장 고려)
- PEP 8 형식 위반
- 공용 함수에 문서 문자열 누락
- print 문而非 로그
- 비효율적인 문자열 조작
- 매직 넘버가 명명된 상수 사용 안 함
- f-strings 미사용

**자동 검사 명령**:
```bash
mypy .                              # 유형 검사
ruff check .                        # Linting
black --check .                     # 포맷팅 검사
isort --check-only .                # 가져오기 정렬
bandit -r .                         # 보안 스캔
pip-audit                           # 의존성 감사
pytest --cov=app --cov-report=term-missing  # 테스트 커버리지
```

**일반적인 문제 수정**:

```python
# SQL 인젝션 수정
# 오류
query = f"SELECT * FROM users WHERE id = {user_id}"

# 올바른 것 - 파라미터화된 쿼리 사용
query = "SELECT * FROM users WHERE id = %s"
cursor.execute(query, (user_id,))

# 가변 기본 매개변수 수정
# 오류
def process_items(items=[]):
    items.append("new")
    return items

# 올바른 것
def process_items(items=None):
    if items is None:
        items = []
    items.append("new")
    return items

# 컨텍스트 관리자 사용
# 오류
f = open("config.json")
data = f.read()
f.close()

# 올바른 것
with open("config.json") as f:
    data = f.read()
```

## /go-review

**용도 설명**: Go 코드 포괄적 리뷰. 관용적 패턴, 동시성 보안, 오류 처리 및 보안을 검사합니다.

**사용 메서드**:
```
/go-review
```

**사용 시나리오**:
- Go 코드 작성 또는 수정 후
- Go 변경 커밋 전
- Go 코드를 포함한 PR 리뷰
- Go 관용적 패턴 학습

**리뷰 범주**:

### CRITICAL (필수 수정)
- SQL/명령 인젝션 취약점
- 동기화 없는 경합 조건
- Goroutine泄漏
- 평문 자격 증명
-不安全 포인터 사용
- 핵심 경로에 오류 처리 없음

### HIGH (해야 함 수정)
- 오류에 컨텍스트 패키지 누락
- panic而非 오류 반환
- Context 전파 안 함
- 미버퍼 채널로 인한 데드락
- 인터페이스 미구현 오류
- 뮤텍스 보호 누락

### MEDIUM (권장 고려)
- 非관용 코드 패턴
- 내보내기 함수에 godoc 주석 누락
- 비효율적인 문자열 연결
- Slice 사전 할당 안 함
- 표驱动 테스트 미사용

**자동 검사 명령**:
```bash
go vet ./...                         # 정적 분석
staticcheck ./...                    # 고급 검사 (설치 시)
golangci-lint run                   # Linting
go build -race ./...                # 경합 감지
govulncheck ./...                   # 보안 취약점
```

## /kotlin-review

**용도 설명**: Kotlin 코드 포괄적 리뷰. 관용적 패턴, null 보안, 코루틴 보안 및 보안을 검사합니다.

**사용 메서드**:
```
/kotlin-review
```

**사용 시나리오**:
- Kotlin 코드 작성 또는 수정 후
- Kotlin 변경 커밋 전
- Android/KMP 프로젝트 PR 리뷰
- Kotlin 관용적 패턴 학습

**리뷰 범주**:

### CRITICAL (필수 수정)
- SQL/명령 인젝션 취약점
- 충분한 이유 없는 강제 해제 `!!`
- 플랫폼 유형 null 보안 위반
- GlobalScope 사용 (구조화된 동시성 위반)
- 평문 자격 증명
-不安全 역직렬화

### HIGH (해야 함 수정)
- 가변 상태 (불변으로 충분할 때)
- 코루틴 컨텍스트 내阻塞 호출
- 긴 루프에서 취소 검사 누락
- 봉인 클래스 `when` 非穷举
- 큰 함수 (>50행)
- 깊은 중첩 (>4층)

### MEDIUM (권장 고려)
- 非관용 Kotlin (Java 스타일 패턴)
- 후행 쉼표 누락
- Scope 함수 오용 또는 중첩
- 큰 컬렉션 체인에 Sequence 누락
-冗余 명시적 유형

**자동 검사 명령**:
```bash
./gradlew build                      # 빌드 검사
./gradlew detekt                     # 정적 분석
./gradlew ktlintCheck                # 포맷팅 검사
./gradlew test                       # 테스트
```

## /rust-review

**용도 설명**: Rust 코드 포괄적 리뷰. 소유권 라이프사이클, 오류 처리, unsafe 사용 및 관용적 패턴을 검사합니다.

**사용 메서드**:
```
/rust-review
```

**사용 시나리오**:
- Rust 코드 작성 또는 수정 후
- Rust 변경 커밋 전
- Rust 프로젝트 PR 리뷰
- Rust 관용적 패턴 학습

**리뷰 범주**:

### CRITICAL (필수 수정)
- 프로덕션 코드 경로에서 검사되지 않은 `unwrap()`/`expect()`
- `// SAFETY:` 주석 없는 `unsafe`
- 문자열Interpolation의 SQL 인젝션
- 미검증 입력의 명령 인젝션
- 평문 자격 증명
- 원시 포인터의 사용 후 해제

### HIGH (해야 함 수정)
- 빌로우 검사기를 만족시키기 위해 불필요한 `.clone()`
- `String` 매개변수에 `&str` 또는 `impl AsRef<str>` 적용 필요
- 비동기 컨텍스트에서의阻塞 (`std::thread::sleep`)
- 공유 유형에 `Send`/`Sync` 제약 조건 누락
- 핵심 열거형上の通配符 `_ =>` 매칭
- 큰 함수 (>50행)

### MEDIUM (권장 고려)
- 热路径에서 불필요한 할당
- 크기 알 때 `with_capacity` 누락
- 불충분한 이유로 clippy 경고 억제
- 공용 API에 `///` 문서 누락
- 무시할 수 있는 값에 대해 non-`must_use` 반환 유형에 `#[must_use]` 사용 고려

**자동 검사 명령**:
```bash
cargo check                           # 빌드 검사
cargo clippy -- -D warnings            # Lints
cargo fmt --check                      # 포맷팅 검사
cargo test                            # 테스트
cargo audit                           # 보안 감사 (사용 가능한 경우)
```

## /cpp-review

**용도 설명**: C++ 코드 포괄적 리뷰. 메모리 안전성, 현대 C++ 관용구, 동시성 및 보안을 검사합니다.

**사용 메서드**:
```
/cpp-review
```

**사용 시나리오**:
- C++ 코드 작성 또는 수정 후
- C++ 변경 커밋 전
- C++ 프로젝트 PR 리뷰
- 메모리 안전성 문제 검사

**리뷰 범주**:

### CRITICAL (필수 수정)
- RAII 없는裸 `new`/`delete`
- 버퍼 오버플로 및 사용 후 해제
- 동기화 없는 데이터 경합
- `system()` 통한 명령 인젝션
- 미초기화 변수 읽기
-空指针 역참조

### HIGH (해야 함 수정)
- 五法则 위반
- `std::lock_guard`/`std::scoped_lock` 누락
- detach 스레드의寿命 관리 부적절
- C 스타일 캐스트而非 `static_cast`/`dynamic_cast`
- `const` 정확성 누락

### MEDIUM (권장 고려)
- 불필요한 복사 (값 전달而非 `const&`)
- 크기 알 때 `reserve()` 누락
- 헤더 파일에서 `using namespace std;`
- 중요 반환 값에 `[[nodiscard]]` 누락
- 너무 복잡한 템플릿 메타프로그래밍

**자동 검사 명령**:
```bash
clang-tidy --checks='*,-llvmlibc-*' src/*.cpp -- -std=c++17
cppcheck --enable=all --suppress=missingIncludeSystem src/
cmake --build build -- -Wall -Wextra -Wpedantic
```

## /flutter-review

**용도 설명**: Flutter/Dart 코드 리뷰. 관용적 패턴, 컴포넌트 모범 사례, 상태 관리, 성능, 접근성 및 보안을 검사합니다.

**사용 메서드**:
```
/flutter-review
```

**사용 시나리오**:
- Flutter/Dart 변경을 포함한 PR 커밋 전 (빌드 및 테스트 통과 후)
- 새 기능 구현 후 문제 조기 발견
- 다른 사람의 Flutter 코드 리뷰
- 컴포넌트, 상태 관리 컴포넌트 또는 서비스 클래스审计
- 프로덕션 게시 전

**리뷰 영역**:

| 영역 | 심각도 수준 |
|------|----------|
| 하드코딩된 키, 평문 HTTP | CRITICAL |
| 아키텍처 위반, 상태 관리 안티패턴 | CRITICAL |
| 컴포넌트 재구성 문제, 리소스泄漏 | HIGH |
| `dispose()` 누락, `await` 후 BuildContext | HIGH |
| Dart null 보안, 오류/로딩 상태 누락 | HIGH |
| Const 전파, 컴포넌트 조합 | HIGH |
| `build()` 中的昂贵 작업 | HIGH |
| 접근성, 시맨틱 태그 | MEDIUM |
| 상태 전환 테스트 누락 | HIGH |
| 하드코딩된 문자열 (l10n) | MEDIUM |

## /fastapi-review

**용도 설명**: FastAPI 앱 리뷰. 아키텍처, 비동기 정확성, 의존성 주입, Pydantic 패턴, 보안, 성능 및 테스트 가능성을 검사합니다.

**사용 메서드**:
```
/fastapi-review [파일 또는 디렉토리]
```

**리뷰 영역**:
- 앱 팩토리, 라우팅 경계, 미들웨어 및 예외 처리기
- Pydantic 요청 및 응답 패턴 분리
- 데이터베이스 세션, 인증, 페이징 및 설정의 의존성 주입
- 비동기 데이터베이스 및 외부 HTTP 패턴
- CORS, 인증,限流, 로그 및 키 처리
- OpenAPI 메타데이터 및 문서화 응답 모델
- 테스트 클라이언트 설정 및 의존성 오버라이드
