# 세션 관리명령

## 개요

세션 관리명령用于저장、복구和管理 Claude Code 세션状态。这些명령对于在长세션中保持上下文、在세션之间交接작업、以及추적进度至关重要。

## 명령列表

### /save-session

**용도**: 저장세션状态 - 将当前세션的完整上下文쓰기日期파일

**描述**: 将当前세션的모든上下文저장到 `~/.claude/session-data/` 디렉토리下的日期파일中，패키지括：빌드了什么、什么작업了、什么실패了、剩余작업、파일状态、决策기록和待해결 blockers。

**세션파일命名사양**:
- 형식: `YYYY-MM-DD-<short-id>-session.tmp`
- 있음效字符: 字母 `a-z`/`A-Z`、数字 `0-9`、连字符 `-`、下划线 `_`
- 最小길이: 1 字符（但不권장过短）
- 권장: 8+ 字符的小写字母/数字/连字符组合避免同日冲突

**사용 타이밍**:
- 작업종료时（세션종료前）
- 接近上下文限制前（先저장，再开新세션）
- 复杂문제해결后（저장성공패턴）
- 필요交接给未来세션时

**워크플로우**:
1. **收集上下文** - 읽기모든修改过的파일，리뷰讨论内容
2. **생성디렉토리** - `mkdir -p ~/.claude/session-data`
3. **쓰기파일** - 생성带时间戳的세션파일
4. **展示给用户** - 표시内容并요청확인

**세션파일형식**:

```markdown
# Session: YYYY-MM-DD

**Started:** [approximate time if known]
**Last Updated:** [current time]
**Project:** [project name or path]
**Topic:** [one-line summary of what this session was about]

---

## What We Are Building
[1-3 paragraphs describing the feature, bug fix, or task]

## What WORKED (with evidence)
- **[thing that works]** — confirmed by: [specific evidence]

## What Did NOT Work (and why)
- **[approach tried]** — failed because: [exact reason / error message]

## What Has NOT Been Tried Yet
- [approach / idea]

## Current State of Files
| File              | Status         | Notes                      |
| ----------------- | -------------- | -------------------------- |
| `path/to/file.ts` | PASS: Complete | [what it does]             |

## Decisions Made
- **[decision]** — reason: [why this was chosen over alternatives]

## Blockers & Open Questions
- [blocker / open question]

## Exact Next Step
[The single most important thing to do when resuming]
```

**핵심원칙**:
- "What Did NOT Work" 일부是最重要的 - 未来세션会盲目재시도실패的메서드
- 각세션独立파일 - 绝不追加到之前세션
- 파일供下次세션통과 `/resume-session` 읽기

**모범 사례**:
- 中途저장时（아님종료时），清楚标记진행 중프로젝트
- 패키지含具体오류정보和실패原因（"threw X error because Y"）
- 下一具体단계要足够精确，없음需思考就知道从哪시작

### /resume-session

**용도**: 복구저장的세션

**描述**: 从 `~/.claude/session-data/` 加载并복구之前저장的세션上下文。

**사용场景**:
- 시작新세션时
- 필요계속之前的작업
- 필요回顾之前的문제해결过程

---

### /sessions

**용도**: 浏览+搜索+管理세션历史

**描述**: 浏览、搜索和管理모든已저장的세션历史。

**기능**:
- 列出모든세션
- 按日期搜索
- 管理세션别名
- 정리旧세션

---

### /checkpoint

**용도**: 생성/검증워크플로우검사点

**描述**: 在워크플로우핵심노드생성검사点，确保进度可追溯。

**사용场景**:
- 重要단계완료时
- 필요검증핵심里程碑
- 多단계태스크中

---

### /aside

**용도**: 回答侧问而不丢失上下文

**描述**: 切换到侧向对话패턴，处理次要문제后없음缝돌아가기主태스크。

**사용场景**:
- 필요问一个快速문제
- 필요查阅문서
- 필요실행辅助태스크

---

## 相关명령

- `/save-session` - 저장当前세션
- `/resume-session` - 복구세션
- `/sessions` - 管理세션历史
