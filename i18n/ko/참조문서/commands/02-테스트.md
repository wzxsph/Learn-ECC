本문서介绍 ECC 中用于테스트的专门명령，지원多种프로그래밍 언어和테스트프레임워크。

## /go-test

**용도설명**: Go 언어的 TDD 워크플로우。사용表驱动테스트，先写테스트再구현，검증 80%+ 커버리지。

**사용메서드**:
```
/go-test [기능描述]
```

**사용场景**:
- 구현新的 Go 함수
- 为现있음 Go 코드추가테스트覆盖
- 수정 bug（先写실패的테스트）
- 학습 Go TDD 메서드论

**예제워크플로우**:
```go
// Step 1: 定义인터페이스
func ValidateEmail(email string) error {
    panic("not implemented")
}

// Step 2: 写表驱动테스트 (RED)
tests := []struct {
    name    string
    email   string
    wantErr bool
}{
    {"simple email", "user@example.com", false},
    {"empty string", "", true},
    {"no at sign", "userexample.com", true},
}

// Step 3: 실행테스트 - 검증 FAIL

// Step 4: 구현最小코드 (GREEN)
func ValidateEmail(email string) error {
    if email == "" {
        return ErrEmailEmpty
    }
    if !emailRegex.MatchString(email) {
        return ErrEmailInvalid
    }
    return nil
}

// Step 5: 검증 PASS

// Step 6: 검사커버리지
go test -cover ./...
// coverage: 100.0%
```

**커버리지명령**:
```bash
go test -cover ./...                    # 基础커버리지
go test -coverprofile=coverage.out      # 生成커버리지파일
go tool cover -html=coverage.out        # 浏览器查看
go test -race -cover ./...              # 竞态检测
```

## /kotlin-test

**용도설명**: Kotlin 언어的 TDD 워크플로우。사용 Kotest 프레임워크和 MockK，先写테스트再구현，검증 80%+ 커버리지（Kover）。

**사용메서드**:
```
/kotlin-test [기능描述]
```

**사용场景**:
- 구현新的 Kotlin 함수或클래스
- 为 Android/KMP 프로젝트추가테스트覆盖
- 수정 bug（先写실패的테스트）
- 학습 Kotlin TDD 메서드论

**Kotest 테스트风格**:
```kotlin
// StringSpec (最简单)
class CalculatorTest : StringSpec({
    "add two positive numbers" {
        Calculator.add(2, 3) shouldBe 5
    }
})

// FunSpec (표준단위 테스트)
class RegistrationValidatorTest : FunSpec({
    test("valid registration returns Valid") {
        val request = RegistrationRequest(
            name = "Alice",
            email = "alice@example.com",
            password = "SecureP@ss1"
        )
        val result = validateRegistration(request)
        result.shouldBeInstanceOf<ValidationResult.Valid>()
    }
})

// BehaviorSpec (BDD 风格)
class OrderServiceTest : BehaviorSpec({
    Given("a valid order") {
        When("placed") {
            Then("should be confirmed") { /* ... */ }
        }
    }
})
```

**코루틴테스트**:
```kotlin
test("concurrent fetch completes") {
    runTest {
        val result = service.fetchAll()
        result.shouldNotBeEmpty()
    }
}
```

**커버리지명령**:
```bash
./gradlew koverHtmlReport     # HTML 보고서
./gradlew koverVerify         # 검증커버리지임계값
./gradlew koverXmlReport      # CI 用 XML 보고서
./gradlew test                # 실행테스트
```

## /rust-test

**용도설명**: Rust 언어的 TDD 워크플로우。사용 `#[test]`、rstest、proptest 和 mockall，先写테스트再구현，검증 80%+ 커버리지（cargo-llvm-cov）。

**사용메서드**:
```
/rust-test [기능描述]
```

**사용场景**:
- 구현新的 Rust 함수、메서드或 trait
- 为 Rust 프로젝트추가테스트覆盖
- 수정 bug（先写실패的테스트）
- 학습 Rust TDD 메서드论

**테스트패턴**:
```rust
// 단위 테스트
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn adds_two_numbers() {
        assert_eq!(add(2, 3), 5);
    }

    #[test]
    fn handles_error() -> Result<(), Box<dyn std::error::Error>> {
        let result = parse_config(r#"port = 8080"#)?;
        assert_eq!(result.port, 8080);
        Ok(())
    }
}

// 参数化테스트 (rstest)
#[rstest]
#[case("hello", 5)]
#[case("", 0)]
#[case("rust", 4)]
fn test_string_length(#[case] input: &str, #[case] expected: usize) {
    assert_eq!(input.len(), expected);
}

// 비동기테스트
#[tokio::test]
async fn fetches_data_successfully() {
    let client = TestClient::new().await;
    let result = client.get("/data").await;
    assert!(result.is_ok());
}

// 属性테스트 (proptest)
proptest! {
    #[test]
    fn encode_decode_roundtrip(input in ".*") {
        let encoded = encode(&input);
        let decoded = decode(&encoded).unwrap();
        assert_eq!(input, decoded);
    }
}
```

**커버리지명령**:
```bash
cargo llvm-cov                    # 커버리지摘要
cargo llvm-cov --html             # HTML 보고서
cargo llvm-cov --fail-under-lines 80  # 低于임계값则실패
cargo test                        # 실행테스트
```

## /cpp-test

**용도설명**: C++ 언어的 TDD 워크플로우。사용 GoogleTest/GoogleMock 和 CMake/CTest，先写테스트再구현，검증커버리지（gcov/lcov）。

**사용메서드**:
```
/cpp-test [기능描述]
```

**사용场景**:
- 구현新的 C++ 클래스或함수
- 추가테스트覆盖
- 수정 bug（先写실패的테스트）
- 학습 C++ TDD 메서드论

**GoogleTest 패턴**:
```cpp
// 基本테스트
TEST(SuiteName, TestName) {
    EXPECT_EQ(add(2, 3), 5);
    EXPECT_NE(result, nullptr);
    EXPECT_TRUE(is_valid);
    EXPECT_THROW(func(), std::invalid_argument);
}

// Fixture
class DatabaseTest : public ::testing::Test {
protected:
    void SetUp() override { db_ = create_test_db(); }
    void TearDown() override { db_.reset(); }
    std::unique_ptr<Database> db_;
};

TEST_F(DatabaseTest, InsertsRecord) {
    db_->insert("key", "value");
    EXPECT_EQ(db_->get("key"), "value");
}

// 参数化테스트
class PrimeTest : public ::testing::TestWithParam<std::pair<int, bool>> {};

TEST_P(PrimeTest, ChecksPrimality) {
    auto [input, expected] = GetParam();
    EXPECT_EQ(is_prime(input), expected);
}

INSTANTIATE_TEST_SUITE_P(Primes, PrimeTest,
    ::testing::Values(
        std::make_pair(2, true),
        std::make_pair(4, false),
        std::make_pair(7, true)
    ));
```

**커버리지명령**:
```bash
cmake -DCMAKE_CXX_FLAGS="--coverage" -B build
cmake --build build
ctest --test-dir build
lcov --capture --directory build --output-file coverage.info
genhtml coverage.info --output-directory coverage_html
```

## /flutter-test

**용도설명**: Flutter/Dart 테스트。보고서실패并逐步수정테스트문제，涵盖单元、컴포넌트、金色和통합 테스트。

**사용메서드**:
```
/flutter-test                # 실행모든테스트
/flutter-test --coverage     # 带커버리지
/flutter-test test/file.dart # 실행特定테스트파일
```

**사용场景**:
- 검증기능구현后없음破坏
- 검사新코드的테스트커버리지
- 수정特定테스트파일
- 커밋 PR 之前

**常见테스트실패和수정**:

| 실패유형 | 典型수정 |
|----------|----------|
| `Expected: <X> Actual: <Y>` | 업데이트断言或수정구현 |
| `Widget not found` | 수정 finder 选择器或업데이트 widget 名称 |
| `Golden file not found` | 실행 `flutter test --update-goldens` 生成 |
| `MissingPluginException` | 在테스트설정中 mock 平台通道 |
| `pumpAndSettle timed out` | 用显式 `pump(Duration)` 교체 |

## /test-coverage

**용도설명**: 분석테스트커버리지，识别差距，并生成缺失테스트以达到目标임계값（기본 80%）。

**사용메서드**:
```
/test-coverage
```

**워크플로우**:
1. **检测테스트프레임워크** - 根据프로젝트유형选择覆盖명령
2. **분석커버리지보고서** - 列出低于 80% 的파일
3. **生成缺失테스트** - 按优先级生成테스트
4. **검증** - 重新실행커버리지확인개선

**테스트프레임워크检测**:

| 指示器 | 覆盖명령 |
|--------|----------|
| `jest.config.*` | `npx jest --coverage` |
| `vitest.config.*` | `npx vitest run --coverage` |
| `pytest.ini` | `pytest --cov=src --cov-report=json` |
| `Cargo.toml` | `cargo llvm-cov --json` |
| `go.mod` | `go test -coverprofile=coverage.out` |

## /python-testing

**용도설명**: Python 테스트워크플로우。사용 pytest，지원커버리지분석。

**사용메서드**:
```
/python-testing [기능描述]
```

**사용场景**:
- 구현新的 Python 기능
- 추가테스트覆盖
- 수정 bug
- FastAPI/Django/Flask 프로젝트테스트

**Pytest 特性**:
```python
# 参数化테스트
@pytest.mark.parametrize("input,expected", [
    ("hello", 5),
    ("", 0),
    ("world", 5),
])
def test_string_length(input, expected):
    assert len(input) == expected

# Fixtures
@pytest.fixture
def client():
    with TestClient(app) as c:
        yield c

# 비동기테스트
@pytest.mark.asyncio
async def test_async_fetch():
    result = await fetch_data()
    assert result is not None
```
