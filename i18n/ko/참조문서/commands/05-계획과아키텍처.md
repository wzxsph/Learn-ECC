# 계획과 아키텍처명령

## 개요

계획과 아키텍처명령用于产品계획、기능설계和시스템아키텍처决策。这些명령确保在写코드之前있음清晰的계획，减少返工和提高코드 품질。

## 명령列表

### /plan

**용도**: 通用实施계획 - 复述需求、评估风险、생성分단계实施계획

**描述**: 在触碰임의코드之前，等待用户확인后생성全面的实施계획。수용自由形式需求或 PRD markdown 파일。

**输入패턴**:

| 输入 | 패턴 | 行为 |
|---|---|---|
| `path/to/name.prd.md` | PRD artifact 패턴 | 읽기 PRD，选择下一个待交付里程碑，生成 `.claude/plans/{name}.plan.md` |
| 기타 markdown 路径 | 引用패턴 | 읽기파일作为上下文，产生内联계획 |
| 自由形式文本 | 对话패턴 | 产生内联계획 |
| 空输入 | 澄清패턴 | 询问要계획什么 |

**핵심参数**:
- `$ARGUMENTS`: `[feature description | path/to/*.prd.md]` - 기능描述或 PRD 파일路径

**워크플로우**:
1. **复述需求** - 澄清필요빌드什么
2. **识别风险** - 列出潜在문제和阻碍
3. **생성단계계획** - 分解为단계性实施
4. **等待확인** - 필수收到用户승인后才能계속

**输出형식** (PRD artifact 패턴):
```markdown
# Plan: {Feature Name}

**Source PRD**: {path}
**Selected Milestone**: {milestone or phase name}
**Complexity**: {Small | Medium | Large}

## Summary
{2-3 sentences}

## Patterns to Mirror
| Category | Source | Pattern |
|---|---|---|
| Naming | `path:line` | {short description} |
| Errors | `path:line` | {short description} |
| Tests | `path:line` | {short description} |

## Files to Change
| File | Action | Why |
|---|---|---|
| `path` | CREATE / UPDATE / DELETE | {reason} |

## Tasks
### Task 1: {name}
- **Action**: {what to do}
- **Mirror**: {pattern to follow}
- **Validate**: {command that proves correctness}

## Validation
```bash
{project-specific validation commands}
```

## Risks
| Risk | Likelihood | Mitigation |
|---|---|---|
```

**모범 사례**:
- 사용前先研究코드库中的现있음패턴
- 在 PRD artifact 패턴中생성 `.claude/plans/` 디렉토리（如不存在）
- 如果 PRD 패키지含 `Delivery Milestones` 表格，只업데이트所选行状态

**常见陷阱**:
- 空输入时不询问澄清就尝试계획
- 不等待用户확인就시작写코드
- 跳过 Pattern Grounding 단계

**与기타 명령통합**:
- 계획后사용 `tdd-workflow` skill 进行테스트驱动개발
- 사용 `/build-fix` 수정빌드오류
- 사용 `/code-review` 리뷰완료구현
- 사용 `/pr` 或 `/prp-pr` 열기 Pull Request

### /plan-prd

**용도**: 交互式 PRD 生成器

**描述**: 生成以문제为中心的产品需求문서，패키지含문제定义、用户画像、성공지표和 MVP 范围。

**사용场景**: 
- 确定要빌드的기능
- 필요明确产品需求
- 从零시작新기능계획

**워크플로우**:
1. FRAME - 了解用户和문제
2. GROUND - 收集证据
3. DECIDE - 确定范围和假设
4. GENERATE - 生成 PRD 파일

---

### /prp-plan

**용도**: 全面的기능계획

**描述**: 完整的프로젝트계획，涵盖코드库분석、风险评估和分단계实施계획。

**사용场景**:
- 复杂기능的详细계획
- 필요코드库上下文분석
- 多단계实施프로젝트

---

### /prp-prd

**용도**: PRP 워크플로우 PRD 生成器

**描述**: 사용 PRP（Plan-Record-Produce）워크플로우生成产品需求문서。

---

### /prp-implement

**용도**: 실행 PRP 계획+검증루프

**描述**: 按照 PRP 워크플로우실행계획，패키지含持续的검증和검사。

---

### /prp-pr

**용도**: 从 PRP 워크플로우생성 PR

**描述**: 从 PRP 워크플로우的结果생성 GitHub Pull Request。

---

### /prp-commit

**용도**: PRP 검증커밋

**描述**: 在 PRP 워크플로우中실행검증并커밋코드。

---

### /multi-plan

**용도**: 多모델协作계획

**描述**: 结合 Codex 和 Gemini 的双모델분석，生成综合实施계획。

**사용场景**:
- 필요多角度분석
- 复杂跨领域프로젝트
- 필요平衡프론트엔드和백엔드考量

---
