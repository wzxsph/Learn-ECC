# 루프와 자동화명령

## 개요

루프와 자동화명령用于시작和管理自主루프작업패턴。

## 명령列表

### /loop-start

**용도**: 시작托管自主루프패턴，带보안기본설정和明确중지条件

**描述**: 시작带보안기본설정和明确중지条件的托管自主루프패턴。

**用法**: `/loop-start [pattern] [--mode safe|fast]`

**参数**:

| 参数 | 值 | 설명 |
|---|---|---|
| `pattern` | `sequential` | 순서실행태스크，一个接一个 |
| | `continuous-pr` | 持续 PR 생성和머지루프 |
| | `rfc-dag` | 基于 RFC 的있음向없음环图워크플로우 |
| | `infinite` | 없음限루프（需明确중지条件） |
| `--mode` | `safe`（기본） | 严格품질门禁和검사点 |
| | `fast` | 为速度减少门禁 |

**워크플로우**:
1. 확인仓库状态和브랜치전략
2. 选择루프패턴和相关모델레이어전략
3. 为所选패턴활성화所需 hooks/profile
4. 생성루프계획并在 `.claude/plans/` 下写 runbook
5. 인쇄시작和모니터링명령

**必需보안검사**:
- 검증테스트在首次루프前통과
- 确保 `ECC_HOOK_PROFILE` 未全局비활성화
- 确保루프있음明确중지条件

**参数传递**:
- `$ARGUMENTS`: `[pattern]` 和선택的 `[--mode safe|fast]`
- 패턴是선택的（기본为 sequential）
- mode 标志是선택的（기본为 safe）

**모범 사례**:
- 在 infinite 패턴之前确保설정明确的중지条件
- fast 패턴仅在充分테스트的코드上사용
- 모니터링루프进度，사용 `/loop-status` 검사状态

**常见陷阱**:
- 在未검증테스트的情况下시작루프
- 사용 infinite 패턴而없음明确중지条件
- 在全局비활성화 hooks 时尝试사용보안패턴

**与기타 명령통합**:
- 사용 `/loop-status` 모니터링실행中루프
- 사용 `/santa-loop` 进行 Santa 风格루프
- 사용 `/checkpoint` 생성핵심노드검사点

---

### /loop-status

**용도**: 검사실행中루프的状态

**描述**: 查看当前실행中루프的状态和进度。

---

### /santa-loop

**용도**: Santa 风格自主루프

**描述**: 一种特殊风格的自主루프패턴。

---

## 相关명령

- `/loop-start` - 시작루프
- `/loop-status` - 검사状态
- `/santa-loop` - Santa 风格
