# 학습과 개선명령

## 개요

학습과 개선명령用于从세션中提取패턴、추적 instinct 和管理组织知识。

## 명령列表

### /learn

**용도**: 从当前세션中提取可复用패턴并저장为候选 skills

**描述**: 분석当前세션，提取임의值得저장为 skills 的패턴。실행 `/learn` 可在세션期间임의时候사용。

**提取内容유형**:

| 유형 | 描述 |
|---|---|
| **오류해결方案패턴** | 什么오류？根本原因是什么？什么수정了？클래스似오류可复用吗？ |
| **디버깅기술** | 非显而易见的디버깅단계、있음效的도구组合、진단패턴 |
| **变通메서드** | 库怪癖、API 限制、特定버전수정 |
| **프로젝트特定패턴** | 发现的코드库约定、아키텍처决策、통합패턴 |

**输出형식**: 저장到 `~/.claude/skills/learned/[pattern-name].md`

```markdown
# [Descriptive Pattern Name]

**Extracted:** [Date]
**Context:** [Brief description of when this applies]

## Problem
[What problem this solves - be specific]

## Solution
[The pattern/technique/workaround]

## Example
[Code example if applicable]

## When to Use
[Trigger conditions - what should activate this skill]
```

**워크플로우**:
1. 리뷰세션中可提取的패턴
2. 识别最있음价值/可复用的见解
3. 起草 skill 파일
4. 요청用户확인后再저장
5. 저장到 `~/.claude/skills/learned/`

**모범 사례**:
- 不要提取trivial수정（拼写오류、简单语法오류）
- 不要提取一次性문제（特定 API 中断等）
- 关注未来세션可节省时间的패턴
- 保持 skills 专注 - 一个패턴一个 skill

### /learn-eval

**용도**: 提取패턴 + 自我评估품질

**描述**: 在提取패턴的同时进行품질评估。

---

### /evolve

**용도**: 분석 instinct + 권장进化구조

**描述**: 분석已학습的 instinct，提供구조演进권장。

---

### /promote

**용도**: 将프로젝트 instinct 提升到全局范围

**描述**: 将프로젝트特定的 instinct 提升为全局可用的知识。

---

### /instinct-status

**용도**: 표시모든학습的 instinct

**描述**: 표시当前모든 instinct 的状态和置信度。

---

### /instinct-export

**용도**: 내보내기 instinct 到파일

**描述**: 将 instinct 내보내기为可分享的파일형식。

---

### /instinct-import

**용도**: 从파일/URL 가져오기 instinct

**描述**: 从파일或 URL 가져오기 instinct 到시스템中。

---

### /skill-create

**용도**: 분석 git 历史+生成 skill 파일

**描述**: 분석프로젝트 git 历史，提取可复用的패턴生成 skill 파일。

**워크플로우**:
1. 분석 git 커밋历史
2. 识别重复패턴
3. 生成 skill 파일
4. 검증并저장

---

### /skill-health

**용도**: Skill 组合健康仪表板

**描述**: 표시 skill 组合的健康状态和사용통계。

---

### /rules-distill

**용도**: 扫描 skills + 提取跨领域원칙

**描述**: 扫描모든 skills，提取跨领域的通用원칙。

---
