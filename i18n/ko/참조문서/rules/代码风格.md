# 코딩 스타일규칙

## 규칙개요

ECC Rules 코딩 스타일규칙定义了코드组织的코어원칙和모범 사례。这些규칙确保코드保持可读性、유지보수성和一致性，涵盖从不可变性到命名约定的모든方面。

## 코어요구

### 不可变性（핵심원칙）

**필수**：始终생성新对象，绝不修改现있음对象。

```typescript
// 오류예제 - 修改原对象
function modify(user, name) {
  user.name = name;  // 改变了原对象
  return user;
}

// 正确예제 - 돌아가기新对象
function update(user, name) {
  return { ...user, name };  // 생성新副本
}
```

**理由**：不可变데이터防止숨김副作用，使디버깅更容易，并지원보안동시성。

### 코어설계원칙

| 원칙 | 설명 | 实践 |
|------|------|------|
| **KISS** | Keep It Simple | 首选最简单的可行方案，避免过早최적화，优先고려清晰而非巧妙 |
| **DRY** | Don't Repeat Yourself | 将重复逻辑提取到共享함수或도구클래스，避免복사粘贴导致的구현漂移 |
| **YAGNI** | You Aren't Gonna Need It | 不빌드未来可能필요的特性或抽象，简单시작，在真正필요时리팩토링 |
### 파일组织

| 요구 | 설명 |
|------|------|
| 多小파일 > 少大파일 | 高内聚、低耦合 |
| 典型行数 | 200-400行，最多800行 |
| 大모듈拆分 | 从大모듈中提取도구함수 |
| 组织方式 | 按기능/领域组织，而非按유형 |

### 오류 처리

| 요구 | 설명 |
|------|------|
| 显式处理 | 각레이어都要显式处理오류 |
| UI코드 | 提供用户友好的오류 메시지 |
| 서비스端 | 기록详细的오류上下文 |
| 금지行为 | 不得静默吞掉오류 |

### 输入검증

| 요구 | 설명 |
|------|------|
| 边界검증 | 在시스템边界검증모든输入 |
| Schema검증 | 尽可能사용基于Schema的검증 |
| 快速실패 | 用清晰的오류 메시지快速실패 |
| 外部데이터 | 永远不信任外部데이터（API응답、用户输入、파일内容） |
## 实施细节

### 命名约定

| 유형 | 约定 | 예제 |
|------|------|------|
| 변수和함수 | camelCase + 描述性名称 | `calculateTotal`, `userName` |
| 布尔值 | is/has/should/can 前缀 | `isActive`, `hasPermission` |
| 인터페이스、유형、컴포넌트 | PascalCase | `UserProfile`, `ButtonComponent` |
| 상수 | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT`, `API_TIMEOUT` |
| 自定义Hooks | camelCase + use前缀 | `useUserData`, `useFetch` |

### 코드异味

#### 深层嵌套

超过4层嵌套应사용早期돌아가기리팩토링：

```typescript
// 避免
function processOrder(order) {
  if (order) {
    if (order.items) {
      if (order.items.length > 0) {
        // 处理逻辑
      }
    }
  }
}

// 권장
function processOrder(order) {
  if (!order || !order.items || order.items.length === 0) {
    return;
  }
  // 处理逻辑
}
```

#### 魔法数字

사용命名상수代替魔法数字：

```typescript
// 避免
setTimeout(callback, 300000);

// 권장
const SESSION_TIMEOUT_MS = 5 * 60 * 1000; // 5分钟
setTimeout(callback, SESSION_TIMEOUT_MS);
```

#### 长함수

拆分大型함수为보유清晰职责的小型함수：

| 함수길이 | 권장 |
|----------|------|
| <50行 | 理想 |
| 50-100行 | 可수용，고려拆分 |
| >100行 | 필수拆分 |

## 违规处理

### 코드 품질 검사清单

완료작업前필수검사：

- [ ] 코드可读且命名良好
- [ ] 함수短小（<50行）
- [ ] 파일专注（<800行）
- [ ] 없음深层嵌套（>4层）
- [ ] 适当的오류 처리
- [ ] 없음硬인코딩值（사용상수或설정）
- [ ] 사용不可变패턴（없음mutation）

### 심각程度分级

| 级别 | 含义 | 操作 |
|------|------|------|
| CRITICAL | 보안漏洞或데이터丢失风险 | **阻止** - 머지前필수수정 |
| HIGH | Bug或重大품질문제 | **경고** - 머지前应수정 |
| MEDIUM | 유지보수성문제 | **提示** - 권장수정 |
| LOW | 样式或小권장 | **备注** - 선택 |
