# 성능 최적화

## 모델选择전략

**Haiku 4.5**（Sonnet 90% 能力，1/3 成本）：
- 频繁调用的轻量级 agent
- 结对编程和코드生成
- 多 agent 시스템中的 worker agent

**Sonnet 4.6**（最佳인코딩모델）：
- 主要개발작업
- 编排多 agent 워크플로우
- 复杂인코딩태스크

**Opus 4.5**（最깊이추론）：
- 复杂아키텍처决策
- 最大추론需求
- 研究和분석태스크

## 上下文窗口管理

避免사용上下文窗口的最后 20%：
- 大规模리팩토링
- 跨多파일的기능구현
- 디버깅复杂交互

低上下文敏感度태스크：
- 单파일편집
- 独立도구생성
- 문서업데이트
- 简单 bug 수정

## 확장思考 + 계획패턴

확장思考기본활성화，为内部추론保留最多 31,999 tokens。

控制확장思考：
- **切换**: Option+T (macOS) / Alt+T (Windows/Linux)
- **설정**: 在 `~/.claude/settings.json` 中설정 `alwaysThinkingEnabled`
- **预算上限**: `export MAX_THINKING_TOKENS=10000`
- **详细패턴**: Ctrl+O 查看思考输出

필요깊이추론的复杂태스크：
1. 确保확장思考已활성화（기본）
2. 활성화**계획패턴**以获得구조化메서드
3. 사용多轮批评进行彻底분석
4. 사용分角色 sub-agents 获得不同视角

## 빌드故障排除

如果빌드실패：
1. 사용 **build-error-resolver** agent
2. 분석오류 메시지
3. 增量수정
4. 每次수정后검증
