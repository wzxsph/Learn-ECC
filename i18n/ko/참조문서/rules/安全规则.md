# 보안 규칙

## 규칙개요

ECC Rules 보안 규칙是强制性的보안검사体系，旨在确保코드在커밋前符合最高보안표준。该규칙覆盖了从Secrets管理到MCP서버보안的各个方面，是모든코드变更필수遵守的底线。

## 코어요구

### 커밋前强制검사清单

在커밋임의코드前，필수완료以下보안검사：

| 체크 항목 | 설명 |
|--------|------|
| 없음硬인코딩Secrets | API密钥、密码、令牌等不得硬인코딩在源코드中 |
| 用户输入검증 | 모든用户输入필수经过검증 |
| SQL注入防护 | 사용参数化查询防止SQL注入 |
| XSS防护 | 对用户输入进行HTML转义 |
| CSRF保护 | 确保CSRF保护已활성화 |
| 인증권한 부여검증 | 확인身份검증和권한 부여逻辑正确 |
| 限流机制 | 모든端点필수있음速率限制 |
| 오류 메시지보안 | 오류 메시지不得泄露敏感데이터 |
