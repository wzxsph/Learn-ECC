# Agent 오케스트레이션

## 可用 Agent

位于 `~/.claude/agents/`:

| Agent | 용도 | 사용 타이밍 |
|-------|------|----------|
| planner | 实施계획 | 复杂기능、리팩토링 |
| architect | 시스템설계 | 아키텍처决策 |
| tdd-guide | 테스트驱动개발 | 新기능、bug 수정 |
| code-reviewer | 코드 리뷰 | 编写코드后 |
| security-reviewer | 보안분석 | 커밋前 |
| build-error-resolver | 수정빌드오류 | 빌드실패时 |
| e2e-runner | E2E 테스트 | 핵심사용자 플로우 |
| refactor-cleaner | 死코드정리 | 코드维护 |
| doc-updater | 문서업데이트 | 업데이트문서时 |
| rust-reviewer | Rust 코드 리뷰 | Rust 프로젝트 |
| harmonyos-app-resolver | HarmonyOS 应用개발 | HarmonyOS/ArkTS 프로젝트 |

## 立即사용 Agent

없음需用户提示：
1. 复杂기능요청 → 사용 **planner** agent
2. 코드刚编写/修改 → 사용 **code-reviewer** agent
3. Bug 수정或新기능 → 사용 **tdd-guide** agent
4. 아키텍처决策 → 사용 **architect** agent

## 병렬태스크실행

对于独立操作，始终사용병렬 Task 실행：

```markdown
# 好：병렬실행
시작 3 个 agent 병렬：
1. Agent 1: 인증모듈보안분석
2. Agent 2: 캐시시스템성능리뷰
3. Agent 3: 도구유형검사

# 坏：필요 없음要的순서실행
先 agent 1，然后 agent 2，然后 agent 3
```

## 多视角분석

对于复杂문제，사용分角色 sub-agents：
- 事实리뷰员
- 高级工程师
- 보안专家
- 一致性리뷰员
- 冗余검사员
