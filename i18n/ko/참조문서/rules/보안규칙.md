# 보안 규칙

## 규칙 개요

ECC Rules 보안 규칙은强制性 보안 검사 체계로, 코드가 커밋 전에 최고의 보안 표준을 준수하도록 보장합니다. 이 규칙은 Secrets 관리부터 MCP 서버 보안에 이르기까지 모든 측면을 다루며, 모든 코드 변경이 반드시 준수해야 할底线을 제공합니다.

## 코드 요구사항

### 커밋 전强制性 검사 목록

임의의 코드를 커밋하기 전에, 다음 보안 검사를 반드시 완료해야 합니다:

| 체크 항목 | 설명 |
|--------|------|
| 하드코딩된 Secrets 없음 | API 키, 비밀번호, 토큰 등은 소스 코드에 하드코딩하면 안 됨 |
| 사용자 입력 검증 | 모든 사용자 입력은 반드시 검증되어야 함 |
| SQL 인젝션 방어 | 파라미터화된 쿼리 사용해서 SQL 인젝션 방지 |
| XSS 방어 | 사용자 입력에 대한 HTML 이스케이프 처리 |
| CSRF 보호 | CSRF 보호가 활성화되어 있는지 확인 |
| 인증/권한 부여 검증 | 인증 및 권한 부여 로직이 올바른지 확인 |
| 속도 제한 메커니즘 | 모든 엔드포인트에 속도 제한이 있어야 함 |
| 오류 메시지 보안 | 오류 메시지가 민감한 데이터를 노출하면 안 됨 |
