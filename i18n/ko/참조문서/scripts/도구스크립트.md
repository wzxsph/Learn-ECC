## ecc.js

**경로**: `scripts/ecc.js`

ECC 선택적 설치의 CLI 주요 진입점으로, 모든 하위 명령을 통합调度합니다.

### 명령 목록

| 명령 | 스크립트 | 설명 |
|------|------|------|
| `install` | install-apply.js | ECC 내용을 목표에 설치 |
| `plan` | install-plan.js | 설치清单 및 파싱 계획 검사 |
| `catalog` | catalog.js | 설치 설정 파일 및 컴포넌트 ID 발견 |
| `consult` | consult.js | 자연어로 컴포넌트 권장 查询 |
| `list-installed` | list-installed.js | 현재 컨텍스트의 설치 상태 검사 |
| `doctor` | doctor.js | 누락되거나漂移한 ECC 관리 파일 진단 |
| `repair` | repair.js |漂移되거나 누락된 ECC 관리 파일 복구 |
| `auto-update` | auto-update.js | 최신 ECC 변경 사항 풀 및 재설치 |
| `status` | status.js | SQLite 상태 저장소 요약 Query |
| `platform-audit` | platform-audit.js | GitHub 대기열, 토론, 로드맵 감사 |
| `security-ioc-scan` | ci/scan-supply-chain-iocs.js | 공급망 IOC 스캔 |
| `sessions` | sessions-cli.js | SQLite 상태 저장소에서 세션 나열 또는 검사 |
| `work-items` | work-items.js | 연결된 Linear, GitHub 작업 항목 추적 |
| `session-inspect` | session-inspect.js | dmux 또는 Claude 히스토리에서 세션 스냅샷 발생 |
| `loop-status` | loop-status.js | Claude 스크립트의过期 루프 깨우기 및 보류 도구 결과 검사 |
| `uninstall` | uninstall.js | install-state에서 ECC 관리 파일 삭제 |

### 사용 예제

```bash
# 직접 언어 설치
ecc typescript

# 설정 파일로 설치
ecc install --profile developer --target claude

# 설치 계획 보기
ecc plan --profile core --target cursor

# 사용 가능한 컴포넌트 나열
ecc catalog profiles
ecc catalog components --family language

# 컴포넌트 상세 보기
ecc catalog show framework:nextjs

# 권장 사항 Query
ecc consult "security reviews"

# 설치된 것 검사
ecc list-installed --json

# 문제 진단
ecc doctor --target cursor
ecc repair --dry-run

# 자동 업데이트
ecc auto-update --dry-run

# 상태 Query
ecc status --json
ecc status --exit-code
ecc status --markdown --write status.md

# 플랫폼 감사
ecc platform-audit --json --allow-untracked docs/drafts/

# 보안 스캔
ecc security-ioc-scan --home

# 세션 관리
ecc sessions
ecc sessions session-active --json

# 작업 항목 추적
ecc work-items upsert linear-ecc-20 --source linear --source-id ECC-20 --title "Review control-plane contract" --status blocked
ecc work-items sync-github --repo affaan-m/ECC

# 루프 상태 검사
ecc loop-status --json

# 제거
ecc uninstall --target antigravity --dry-run
```

## install-apply.js

**경로**: `scripts/install-apply.js`

ECC 설치 실행 시, 기존 언어 설치 진입점을 완전하게 유지하면서, 목표별 변경 로직을 테스트 가능한 Node 코드로 이동합니다.

### 지원 목표

| 목표 | 설명 |
|------|------|
| `claude` | ~/.claude/에 설치, 규칙/스킬은 rules/ecc 및 skills/ecc에 배치 |
| `claude-project` | ./.claude/에 설치 (프로젝트급) |
| `cursor` | 규칙, 훅, 커서 설정을 ./.cursor/에 설치 |
| `antigravity` | 규칙, 워크플로우, 스킬, 프록시를 ./.agent/에 설치 |
| `codex` | 공유 프록시/설정을 ~/.codex/에 설치 |
| `gemini` | 프로젝트급 Gemini 설정을 ./.gemini/에 설치 |
| `opencode` | 공유 명령/훅/설정을 ~/.opencode/에 설치 |
| `codebuddy` | 명령, 프록시, 스킬을 ./.codebuddy/에 설치 |
| `joycode` | 명령, 프록시, 스킬을 ./.joycode/에 설치 |
| `qwen` | ~/.qwen/에 설치 |
| `zed` | ./.zed/에 설치 |

### 설치 방식

1. **기존 언어 패턴**: `install.sh <language>` (예: `ecc-install typescript`)
2. **Profile 패턴**: `install.sh --profile <name> [--with <component>]... [--without <component>]...`
3. **모듈 패턴**: `install.sh --modules <id,id,...>`
4. **스킬 패턴**: `install.sh --skills <skill-id[,skill-id...]>`
5. **지역화 패턴**: `install.sh --target claude|claude-project --locale <locale-code>`

### 옵션

- `--dry-run` - 설치 계획만 표시, 파일 복사 안 함
- `--json` - 기계가 읽을 수 있는 JSON 형식 출력
- `--config <path>` - 파일에서 설치 의도 로드

## claw.js

**경로**: `scripts/claw.js`

NanoClaw v2 - Everything Claude Code를 위한제로 외부 의존, 세션 인식 REPL로, `claude -p` 기반으로 빌드됩니다.

### 기능

- 세션 히스토리가 `~/.claude/claw/`에 영속적으로 저장
- 동적 스킬 로드를 컨텍스트로 지원
- 컨텍스트 길이 관리를 위한 세션 압축
- 세션 분기 및 내보내기
- 크로스 세션 검색

### REPL 명령

| 명령 | 설명 |
|------|------|
| `/help` | 도움말 표시 |
| `/clear` | 현재 세션 히스토리 지움 |
| `/history` | 전체 대화 히스토리 인쇄 |
| `/sessions` | 저장된 세션 나열 |
| `/model [name]` | 모델 표시/설정 |
| `/load <skill-name>` | 스킬을 활성 컨텍스트로 로드 |
| `/branch <session-name>` | 현재 세션을 새 세션으로 분기 |
| `/search <query>` | 크로스 세션 검색 |
| `/compact` | 가장 최근 항목 유지, 오래된 컨텍스트 압축 |
| `/export <md\|json\|txt> [path]` | 세션 내보내기 |
| `/metrics` | 세션 지표 표시 |
| `exit` | REPL 종료 |

### 환경 변수

| 변수 | 기본값 | 설명 |
|------|--------|------|
| `CLAW_SESSION` | `default` | 초기 세션 이름 |
| `CLAW_MODEL` | `sonnet` | 기본 모델 |
| `CLAW_SKILLS` | 빈 값 | 로드할 스킬 목록 (쉼표分隔) |

### 사용 예제

```bash
# 기본 세션으로 시작
node scripts/claw.js

# 지정된 세션으로 시작
CLAW_SESSION=my-session node scripts/claw.js

# 스킬을 로드하고 시작
CLAW_SKILLS="javascript-patterns,python-testing" node scripts/claw.js
```

## harness-audit.js

**경로**: `scripts/harness-audit.js`

명시적 파일/규칙 검사에 기반한确定性 도구 체인 감사 도구입니다. 자동 감지 ECC 저장소 패턴 vs 소비자 프로젝트 패턴.

### 감사 범주

| 범주 | 설명 |
|------|------|
| Tool Coverage | 도구 커버리지 완전성 |
| Context Efficiency | 컨텍스트 효율성 |
| Quality Gates | 품질 Gate |
| Memory Persistence | 기억 영속화 |
| Eval Coverage | 평가 커버리지 |
| Security Guardrails | 보안 레일 |
| Cost Efficiency | 비용 효율성 |
| GitHub Integration | GitHub 통합 |
| Vercel/Netlify/Cloudflare/Fly Integration | 각 플랫폼 통합 |

### 사용 예제

```bash
# 현재 디렉토리 감사
node scripts/harness-audit.js

# 범위 지정
node scripts/harness-audit.js --scope repo
node scripts/harness-audit.js --scope hooks
node scripts/harness-audit.js --scope skills

# 루트 디렉토리 지정
node scripts/harness-audit.js --root /path/to/project

# JSON 형식 출력
node scripts/harness-audit.js --format json

# 조합 사용
node scripts/harness-audit.js repo --format json --root .
```

## catalog.js

**경로**: `scripts/catalog.js`

ECC 설치 컴포넌트 및 설정 파일을 발견하는 도구입니다.

### 명령

#### profiles

모든 설치 설정 파일을 나열합니다.

```bash
node scripts/catalog.js profiles
node scripts/catalog.js profiles --json
```

#### components

패밀리 및 목표별 필터링이 가능한 설치 컴포넌트를 나열합니다.

```bash
node scripts/catalog.js components
node scripts/catalog.js components --family language
node scripts/catalog.js components --family framework
node scripts/catalog.js components --target claude
node scripts/catalog.js components --json
```

#### show

특정 컴포넌트의 상세 정보를 표시합니다.

```bash
node scripts/catalog.js show <component-id>
node scripts/catalog.js show framework:nextjs --json
```

### 컴포넌트 패밀리

- `baseline` - 기본 컴포넌트
- `language` - 언어 컴포넌트 (javascript, typescript, python 등)
- `framework` - 프레임워크 컴포넌트 (nextjs, react, vue 등)
- `capability` - 역량 컴포넌트
- `agent` - 에이전트 컴포넌트
- `skill` - 스킬 컴포넌트

## build-opencode.js

**경로**: `scripts/build-opencode.js`

OpenCode 플러그인의 TypeScript 코드를 빌드합니다.

### 기능

1. `dist` 디렉토리 정리
2. TypeScript 컴파일러로 `.opencode` 디렉토리 아래 코드 컴파일
3. `.opencode/dist`에 출력

### 전제 조건

루트 디렉토리의 개발 종속성이 설치되어야 하며, TypeScript 컴파일러를 사용할 수 있어야 합니다.

### 사용

```bash
node scripts/build-opencode.js
```
