# 테스트 실행기

본 문서는 ECC 프로젝트의 테스트 구조 및 실행 방식을 소개합니다.

## 테스트 진입점

**경로**: `tests/run-all.js`

중앙 집중식 테스트 실행기로, 테스트 파일을 자동 발견 및 실행합니다.

### 사용 메서드

```bash
# 모든 테스트 실행
node tests/run-all.js

# 단일 테스트 파일 실행
node tests/lib/utils.test.js
node tests/hooks/hooks.test.js
```

### 테스트 파일 발견

테스트 실행기는 glob 패턴 `tests/**/*.test.js`로 테스트 파일을 자동 발견합니다.

규칙:
- 파일이 `tests/` 디렉토리 아래에 있어야 함
- 파일명이 `.test.js`로 끝나야 함
- 디렉토리 구조가 표시에 유지됨

### 출력 형식

```
╔══════════════════════════════════════════════════════════╗
║           Everything Claude Code - Test Suite             ║
╚══════════════════════════════════════════════════════════╝

━━━ Running lib/utils.test.js ━━━
(테스트 출력...)
━━━ Running hooks/hooks.test.js ━━━
(테스트 출력...)

╔══════════════════════════════════════════════════════════╗
║                     Final Results                        ║
╠══════════════════════════════════════════════════════════╣
║  Total Tests:    42                                      ║
║  Passed:         42  ✓                                   ║
║  Failed:         0                                        ║
╚══════════════════════════════════════════════════════════╝
```

### 테스트 통계

테스트 실행기가 각 테스트 파일의 출력을 파싱하여 통계를 집계합니다:
- `Passed: <수량>`
- `Failed: <수량>`

## 테스트 구조

### 디렉토리 레이아웃

```
tests/
├── run-all.js              # 주 테스트 실행기
├── lib/                    # 라이브러리 도구 테스트
│   ├── utils.test.js
│   ├── package-manager.test.js
│   └── ...
├── hooks/                  # Hook 통합 테스트
│   └── hooks.test.js
├── integration/            # 통합 테스트
├── scripts/                # 스크립트 테스트
├── commands/               # 명령 테스트
├── docs/                   # 문서 테스트
├── opencode-config.test.js
├── plugin-manifest.test.js
├── test_*.py               # Python 테스트
└── conftest.py             # pytest 설정
```

### 테스트 유형

1. **단위 테스트** (`*.test.js`)
   - 독립 함수 및 도구 테스트
   - Node.js原生 assert 또는 Jest 스타일 사용

2. **통합 테스트** (`tests/integration/`)
   - 여러 컴포넌트协作 테스트
   - 훅 완전한 흐름 테스트

3. **Python 테스트** (`test_*.py`)
   - pytest 형식의 Python 테스트
   - `conftest.py` 설정 사용

## 테스트 설정

### Node.js 테스트

테스트 파일은 표준 Node.js assert 모듈을 사용합니다:

```javascript
const assert = require('assert');

test('example test', () => {
  assert.strictEqual(actual, expected, 'Optional message');
});
```

### Python 테스트 (pytest)

위치: `tests/conftest.py`

pytest 설정 파일로, 공유 fixtures 및 설정을 제공합니다.

```bash
# Python 테스트 실행
python -m pytest tests/

# 특정 테스트 실행
python -m pytest tests/test_builder.py
```

## 테스트 모범 사례

### AAA 패턴

```
test('테스트 설명', () => {
  // Arrange - 테스트 데이터 준비
  const input = 'test data';

  // Act - 테스트할 작업 실행
  const result = myFunction(input);

  // Assert - 결과 검증
  assert.strictEqual(result, expected);
});
```

### 테스트 명명

설명적 이름을 사용하여 테스트의 동작을 명확히 설명합니다:

```javascript
test('returns empty array when no markets match query', () => {});
test('throws error when API key is missing', () => {});
test('falls back to substring search when Redis is unavailable', () => {});
```

### 테스트 격리

각 테스트가:
- 다른 테스트에 의존하지 않고 독립 실행
- 자체 테스트 데이터를 정리
- 외부 의존성을 격리하기 위해 mock 사용

## CI 통합

`package.json`에서 테스트 스크립트 설정:

```json
{
  "scripts": {
    "test": "validate-commands.js && tests/run-all.js"
  }
}
```

테스트 실행기는 CI 환경에서 작업하도록 설계되었습니다:
- 非零 종료 코드가 실패 표시
- 자동화 파싱을 위한 JSON 출력
- 디버깅을 위한 명확한 오류 메시지
