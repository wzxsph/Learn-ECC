# 테스트 실행기

本문서介绍 ECC 프로젝트的테스트구조和실행方式。

## 테스트入口

**路径**: `tests/run-all.js`

集中式테스트 실행기,자동发现并실행모든테스트파일。

### 사용메서드

```bash
# 실행모든테스트
node tests/run-all.js

# 실행单个테스트파일
node tests/lib/utils.test.js
node tests/hooks/hooks.test.js
```

### 테스트파일发现

테스트 실행기통과 glob 패턴 `tests/**/*.test.js` 자동发现테스트파일。

규칙:
- 파일필수位于 `tests/` 디렉토리下
- 파일名필수以 `.test.js` 结尾
- 디렉토리 구조会被保留用于표시

### 输出형식

```

║           Everything Claude Code - Test Suite             ║
╚══════════════════════════════════════════════════════════╝

(테스트输出...)
━━━ Running hooks/hooks.test.js ━━━
(테스트输出...)

╔══════════════════════════════════════════════════════════╗
║                     Final Results                        ║
╠══════════════════════════════════════════════════════════╣
║  Total Tests:    42                                      ║
║  Passed:         42  ✓                                   ║
║  Failed:         0                                        ║
╚══════════════════════════════════════════════════════════╝
```

### 테스트통계

테스트 실행기会파싱각테스트파일的输出来汇总통계:
- `Passed: <수량>`
- `Failed: <수량>`

## 테스트구조

### 디렉토리布局

```
tests/
├── run-all.js              # 主테스트 실행기
├── lib/                    # 库도구테스트
│   ├── utils.test.js
│   ├── package-manager.test.js
│   └── ...
├── hooks/                  # Hook 통합 테스트
│   └── hooks.test.js
├── integration/            # 통합 테스트
├── scripts/                # 스크립트테스트
├── commands/               # 명령테스트
├── docs/                   # 문서테스트
├── opencode-config.test.js
├── plugin-manifest.test.js
├── test_*.py               # Python 테스트
└── conftest.py             # pytest 설정
```

### 테스트類型

1. **단위 테스트** (`*.test.js`)
   - 테스트独立함수和도구
   - 사용 Node.js 原生 assert 或 Jest 风格

2. **통합 테스트** (`tests/integration/`)
   - 테스트多个컴포넌트协作
   - 테스트钩子完整流程

3. **Python 테스트** (`test_*.py`)
   - pytest 형식的 Python 테스트
   - 사용 `conftest.py` 설정

## 테스트설정

### Node.js 테스트

테스트파일사용표준的 Node.js assert 모듈:

```javascript
const assert = require('assert');

test('example test', () => {
  assert.strictEqual(actual, expected, 'Optional message');
});
```

### Python 테스트 (pytest)

位置: `tests/conftest.py`

pytest 설정파일,提供共享的 fixtures 和설정。

```bash
# 실행 Python 테스트
python -m pytest tests/

# 실행特定테스트
python -m pytest tests/test_builder.py
```

## 테스트모범 사례

### AAA 패턴

```
test('테스트描述', () => {
  // Arrange - 准备테스트데이터
  const input = 'test data';

  // Act - 실행被测操作
  const result = myFunction(input);

  // Assert - 검증结果
  assert.strictEqual(result, expected);
});
```

### 테스트命名

사용描述性名称,清晰설명테스트的行为:

```javascript
test('returns empty array when no markets match query', () => {});
test('throws error when API key is missing', () => {});
test('falls back to substring search when Redis is unavailable', () => {});
```

### 테스트隔离

각테스트해야 함:
- 独立실행,不依赖기타테스트
- 정리自己的테스트데이터
- 사용 mock 隔离外部依赖

## CI 통합

在 `package.json` 中설정테스트스크립트:

```json
{
  "scripts": {
    "test": "validate-commands.js && tests/run-all.js"
  }
}
```

테스트 실행기설계为可在 CI 环境中작업:
- 非零退出码表示실패
- JSON 输出用于자동化파싱
- 清晰的오류 메시지用于디버깅
