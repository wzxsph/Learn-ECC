# 빌드 스크립트

본 문서는 ECC 프로젝트의 빌드 및 게시 스크립트를 소개합니다.

---

## release.sh

**경로**: `scripts/release.sh`

플러그인 버전 업그레이드 및 게시 스크립트입니다.

### 기능

1. **버전 검증**
   - semver 형식 버전 번호 제공 확인 (X.Y.Z 또는 X.Y.Z-prerelease)
   - 현재 브랜치가 main인지 확인
   - 작업 영역이 깨끗한지 확인

2. **파일 업데이트**
   - 모든 패키지/플러그인 목록의 버전 번호 업데이트
   - 문서의 버전 참조 업데이트
   - 에이전트 설정 업데이트

3. **빌드 검증**
   - OpenCode 빌드 실행
   - 빌드 테스트 실행
   - 플러그인 목록 테스트 실행

4. **Git 작업**
   - 모든 변경 사항 staging
   - 커밋 생성
   - 태그 생성 및 푸시

### 사용 메서드

```bash
# 새 버전 게시
./scripts/release.sh 1.5.0

# 사전 게시 버전
./scripts/release.sh 2.0.0-rc.1
```

### 전제 조건

- Git 작업 영역이 반드시 깨끗해야 함
- main 브랜치에 반드시 있어야 함
- 모든 목록 파일이 반드시 존재해야 함

## build-opencode.js

**경로**: `scripts/build-opencode.js`

OpenCode 플러그인의 TypeScript 코드를 빌드합니다.

### 빌드 흐름

1. `dist` 디렉토리 정리
2. TypeScript 컴파일러 찾기 (`typescript/bin/tsc`)
3. 프로젝트 루트 디렉토리의 tsconfig.json으로 `.opencode` 디렉토리 컴파일

### 사용 메서드

```bash
# OpenCode 빌드
node scripts/build-opencode.js

# 게시의 일부로 자동 실행
./scripts/release.sh 1.5.0
```

### 전제 조건

루트 디렉토리의 개발 종속성이 설치되어야 하며, TypeScript 컴파일러를 사용할 수 있어야 합니다.

## 게시 워크플로우

```
1. main 브랜치에 있고 작업 영역이 깨끗한지 확인
2. 게시 스크립트 실행 및 버전 번호 제공
3. 스크립트가 자동 완료:
   ├── 전제 조건 검증
   ├── 모든 버전 참조 업데이트
   ├── OpenCode 빌드
   ├── 테스트 검증 실행
   ├── Git 커밋 생성
   └── 태그 푸시
4. GitHub Actions가 태그를 감지하여 자동 게시
```
