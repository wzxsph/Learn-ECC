# 도구 함수 라이브러리

本문서介绍 `scripts/lib/` 디렉토리下的코어도구 함수 라이브러리。

## utils.js

**路径**: `scripts/lib/utils.js`

跨平台的도구 함수 라이브러리,适用于 Windows、macOS 和 Linux。

### 平台检测

```javascript
const { isWindows, isMacOS, isLinux } = require('./utils');

// isWindows: true on Windows
// isMacOS: true on macOS
// isLinux: true on Linux
```

### 디렉토리操作

```javascript
const {
  getHomeDir,      // 가져오기用户主디렉토리
  getClaudeDir,    // 가져오기 ~/.claude 디렉토리
  getSessionsDir,  // 가져오기 ~/.claude/session-data 디렉토리
  getTempDir,      // 가져오기临时디렉토리
  ensureDir,       // 确保디렉토리存在,不存在则생성
  findFiles        // 찾기匹配패턴的파일
} = require('./utils');

// 생성디렉토리(如果不存在)
ensureDir('/path/to/dir');

// 찾기파일
const files = findFiles('/path', '*.js', { recursive: true });
```

### 日期时间

```javascript
const {
  getDateString,      // 돌아가기 YYYY-MM-DD 형식
  getTimeString,      // 돌아가기 HH:MM 형식
  getDateTimeString   // 돌아가기 YYYY-MM-DD HH:MM:SS 형식
} = require('./utils');
```

### 세션和프로젝트

```javascript
const {
  sanitizeSessionId,    // 정리세션 ID,去除없음效字符
  getSessionIdShort,    // 가져오기短세션 ID
  getGitRepoName,       // 가져오기 git 仓库名称
  getProjectName        // 가져오기프로젝트名称
} = require('./utils');

// 정리세션 ID
const cleanId = sanitizeSessionId('my-session@123'); // 'my-session-123'

// 가져오기프로젝트名
const project = getProjectName();
```

### 파일操作

```javascript
const {
  readFile,        // 보안읽기파일,실패돌아가기 null
  writeFile,       // 쓰기파일
  appendFile,      // 追加到파일
  replaceInFile,   // 교체파일内容
  countInFile,     // 통계패턴出现次数
  grepFile         // 搜索파일中的패턴
} = require('./utils');

// 읽기파일
const content = readFile('/path/to/file.txt');

// 교체파일内容(首次匹配)
replaceInFile('/path', 'old', 'new');

// 교체모든匹配
replaceInFile('/path', 'old', 'new', { all: true });

// 통계出现次数
const count = countInFile('/path', /pattern/g);

// 搜索并돌아가기匹配行
const matches = grepFile('/path', /regex/);
// 돌아가기: [{ lineNumber: 1, content: '...' }]
```

### 字符串处理

```javascript
const { stripAnsi } = require('./utils');

// 제거 ANSI 转义序列(颜色等)
const clean = stripAnsi('\x1b[32mHello\x1b[0m'); // 'Hello'
```

### Hook I/O

```javascript
const { readStdinJson, log, output } = require('./utils');

// 从 stdin 읽기 JSON(用于 hooks)
const input = await readStdinJson({ timeoutMs: 5000 });

// 输出到 stderr(对用户可见)
log('Debug message');

// 输出到 stdout(돌아가기给 Claude)
output({ result: 'data' });
```

### 시스템명령

```javascript
const { commandExists, runCommand, isGitRepo, getGitModifiedFiles } = require('./utils');

// 검사명령是否存在
if (commandExists('git')) { /* ... */ }

// 실행명령(仅限白名单명령)
const result = runCommand('git rev-parse --show-toplevel');
if (result.success) {
  console.log(result.output);
}

// 검사是否是 git 仓库
if (isGitRepo()) { /* ... */ }

// 가져오기修改的파일
const files = getGitModifiedFiles(['\\.js$', '\\.md$']);
```

## package-manager.js

**路径**: `scripts/lib/package-manager.js`

패키지 관리자检测和选择도구。

### 지원的패키지 관리자

- npm
- pnpm
- yarn
- bun

### 检测优先级

1. 环境변수 `CLAUDE_PACKAGE_MANAGER`
2. 프로젝트설정 `.claude/package-manager.json`
3. `package.json` 的 `packageManager` 字段
4. lock 파일检测
5. 全局用户偏好 `~/.claude/package-manager.json`
6. 기본值 npm

### 코어 API

```javascript
const {
  getPackageManager,           // 가져오기当前프로젝트的패키지 관리자
  setPreferredPackageManager,  // 설정全局偏好
  setProjectPackageManager,    // 설정프로젝트偏好
  getAvailablePackageManagers, // 가져오기시스템上可用的패키지 관리자
  detectFromLockFile,          // 从 lock 파일检测
  detectFromPackageJson,       // 从 package.json 检测
  getRunCommand,               // 가져오기실행스크립트的명령
  getExecCommand,              // 가져오기실행二进制파일的명령
  getSelectionPrompt,          // 가져오기交互式选择提示
  getCommandPattern            // 生成명령匹配正则
} = require('./package-manager');

// 가져오기当前패키지 관리자
const { name, config, source } = getPackageManager();
// name: 'pnpm', 'npm', 'yarn', 或 'bun'
// config: { installCmd, runCmd, execCmd, testCmd, ... }
// source: 'environment', 'project-config', 'lock-file', ...

// 설정全局偏好
setPreferredPackageManager('pnpm');

// 설정프로젝트偏好
setProjectPackageManager('yarn', '/path/to/project');

// 가져오기실행명령
const testCmd = getRunCommand('test'); // 'pnpm test' 或 'npm test' 等
const buildCmd = getRunCommand('build');

// 가져오기실행명령
const eslintCmd = getExecCommand('eslint', '--fix .');
```

### 보안검사

`getRunCommand` 和 `getExecCommand` 会검증输入:
- 스크립트名只허용字母、数字、dash、underscore、dot、slash、@
- 参数只허용字母、数字、空白、dash、dots、slashes、equals、colons、commas、quotes、@

## session-manager.js

**路径**: `scripts/lib/session-manager.js`

세션 관리库,提供세션的 CRUD 操作。

### 세션存储

- 新형식: `~/.claude/session-data/YYYY-MM-DD-[session-id]-session.tmp`
- 旧형식호환성: `~/.claude/sessions/YYYY-MM-DD-session.tmp`

### 코어 API

```javascript
const {
  parseSessionFilename,      // 파싱세션파일名
  getSessionPath,            // 가져오기세션파일完整路径
  getSessionContent,         // 읽기세션内容
  parseSessionMetadata,      // 파싱세션元데이터
  getSessionStats,           // 가져오기세션통계
  getSessionTitle,           // 가져오기세션标题
  getSessionSize,            // 가져오기세션크기(人클래스可读형식)
  getAllSessions,            // 가져오기모든세션(지원分页和筛选)
  getSessionById,            // 按 ID 가져오기单个세션
  writeSessionContent,       // 쓰기세션内容
  appendSessionContent,      // 追加세션内容
  deleteSession,             // 삭제세션
  sessionExists              // 검사세션是否存在
} = require('./session-manager');

// 가져오기모든세션
const { sessions, total, offset, limit, hasMore } = getAllSessions({
  limit: 50,
  offset: 0,
  date: '2026-05-26',
  search: 'abc123'
});

// 가져오기单个세션
const session = getSessionById('abc123', true);
// session: { filename, shortId, date, sessionPath, content, metadata, stats, ... }

// 파싱세션元데이터
const metadata = parseSessionMetadata(content);
// metadata: { title, date, started, lastUpdated, project, branch, worktree, completed, inProgress, notes, context }

// 가져오기세션통계
const stats = getSessionStats(sessionPathOrContent);
// stats: { totalItems, completedItems, inProgressItems, lineCount, hasNotes, hasContext }

// 쓰기세션
writeSessionContent(sessionPath, '# New Session\n\nContent...');

// 追加内容
appendSessionContent(sessionPath, '\n\nMore content...');

// 삭제세션
deleteSession(sessionPath);
```

### 세션元데이터형식

세션内容中的元데이터:

```markdown
# Session Title

**Date:** 2026-05-26
**Started:** 14:30
**Last Updated:** 15:45
**Project:** my-project
**Branch:** main
**Worktree:** frontend

### Completed
- [x] Task 1
- [x] Task 2

### In Progress
- [ ] Task 3

### Notes for Next Session
Remember to check the config file.

### Context to Load
```
previous-context-content
```
```

## install-executor.js

**路径**: `scripts/lib/install-executor.js`

ECC 설치실행器,负责实际실행설치操作。

### 主要기능

- 생성설치계획
- 실행파일복사操作
- 处理설치状态기록

### 코어 API

```javascript
const {
  applyInstallPlan,       // 应用설치계획
  listAvailableLanguages  // 列出可用언어
} = require('./install-executor');

// 列出可用언어
const languages = listAvailableLanguages();
// ['typescript', 'javascript', 'python', ...]
```
