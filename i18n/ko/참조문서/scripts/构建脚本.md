# 빌드 스크립트

本문서介绍 ECC 프로젝트的빌드和게시스크립트。

---

## release.sh

**路径**: `scripts/release.sh`

플러그인버전업그레이드和게시스크립트。

### 기능

1. **버전검증**
   - 확인提供了 semver 형식버전号 (X.Y.Z 或 X.Y.Z-prerelease)
   - 확인当前브랜치是 main
   - 확인작업区干净

2. **파일업데이트**
   - 업데이트모든패키지/플러그인清单中的버전号
   - 업데이트문서中的버전引用
   - 업데이트 agent 설정

3. **검증빌드**
   - 실행 OpenCode 빌드
   - 실행빌드테스트
   - 실행플러그인清单테스트

4. **Git 操作**
   - 暂存모든更改
   - 생성커밋
   - 생성并푸시태그

### 사용메서드

```bash
# 게시新버전
./scripts/release.sh 1.5.0

# 预게시버전
./scripts/release.sh 2.0.0-rc.1
```

### 前提条件

- Git 작업区필수干净
- 필수位于 main 브랜치
- 모든清单파일필수存在

## build-opencode.js

**路径**: `scripts/build-opencode.js`

빌드 OpenCode 플러그인的 TypeScript 코드。

### 빌드流程

1. 정리 `dist` 디렉토리
2. 찾기 TypeScript 컴파일器 (`typescript/bin/tsc`)
3. 사용프로젝트根디렉토리的 tsconfig.json 컴파일 `.opencode` 디렉토리

### 사용메서드

```bash
# 빌드 OpenCode
node scripts/build-opencode.js

# 作为 release 的一일부자동실행
./scripts/release.sh 1.5.0
```

### 前提条件

필요已설치根디렉토리的개발依赖,确保 TypeScript 컴파일器可用。

## 게시워크플로우

```
1. 确保在 main 브랜치且작업区干净
2. 실행게시스크립트并提供버전号
3. 스크립트자동완료:
   ├── 검증前提条件
   ├── 업데이트모든버전引用
   ├── 빌드 OpenCode
   ├── 실행테스트검증
   ├── 생성 Git 커밋
   └── 푸시태그
4. GitHub Actions 检测到태그자동게시
```
