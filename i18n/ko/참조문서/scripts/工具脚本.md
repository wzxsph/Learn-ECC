## 디렉토리

- [ecc.js](#eccjs) - 主명령行入口

## ecc.js

**路径**: `scripts/ecc.js`

ECC 选择性설치的 CLI 主入口点,统一调度모든子명령。

### 명령列表

| 명령 | 스크립트 | 설명 |
|------|------|------|
| `install` | install-apply.js | 설치 ECC 内容到目标 |
| `plan` | install-plan.js | 검사설치清单和파싱계획 |
| `catalog` | catalog.js | 发现설치설정파일和컴포넌트 ID |
| `consult` | consult.js | 根据自然언어查询권장컴포넌트 |
| `list-installed` | list-installed.js | 검사当前上下文的설치状态 |
| `doctor` | doctor.js | 진단缺失或漂移的 ECC 管理파일 |
| `repair` | repair.js | 복구漂移或缺失的 ECC 管理파일 |
| `auto-update` | auto-update.js | 풀最新 ECC 更改并重新설치 |
| `status` | status.js | 查询 SQLite 状态存储摘要 |
| `platform-audit` | platform-audit.js | 审计 GitHub 큐、讨论、路线图等 |
| `security-ioc-scan` | ci/scan-supply-chain-iocs.js | 扫描依赖和 AI 도구持久化面的供应链 IOC |
| `sessions` | sessions-cli.js | 列出或검사 SQLite 状态存储中的세션 |
| `work-items` | work-items.js | 추적链接的 Linear、GitHub 작업项 |
| `session-inspect` | session-inspect.js | 从 dmux 或 Claude 历史目标发出사양세션快照 |
| `loop-status` | loop-status.js | 검사 Claude 스크립트中的过时루프唤醒和보류도구结果 |
| `uninstall` | uninstall.js | 삭제기록在 install-state 中的 ECC 管理파일 |

### 사용예제

```bash
# 直接설치언어
ecc typescript

# 带설정파일的설치
ecc install --profile developer --target claude

# 查看설치계획
ecc plan --profile core --target cursor

# 列出可用컴포넌트
ecc catalog profiles
ecc catalog components --family language

# 표시컴포넌트详情
ecc catalog show framework:nextjs

# 咨询권장
ecc consult "security reviews"

# 검사已설치
ecc list-installed --json

# 진단문제
ecc doctor --target cursor
ecc repair --dry-run

# 자동업데이트
ecc auto-update --dry-run

# 状态查询
ecc status --json
ecc status --exit-code
ecc status --markdown --write status.md

# 平台审计
ecc platform-audit --json --allow-untracked docs/drafts/

# 보안扫描
ecc security-ioc-scan --home

# 세션 관리
ecc sessions
ecc sessions session-active --json

# 작업项추적
ecc work-items upsert linear-ecc-20 --source linear --source-id ECC-20 --title "Review control-plane contract" --status blocked
ecc work-items sync-github --repo affaan-m/ECC

# 검사루프状态
ecc loop-status --json

# 제거
ecc uninstall --target antigravity --dry-run
```

## install-apply.js

**路径**: `scripts/install-apply.js`

ECC 설치실행时,保持传统언어설치入口点完整,同时将目标特定的变更逻辑移入可테스트的 Node 코드。

### 지원的目标

| 目标 | 설명 |
|------|------|
| `claude` | 설치到 ~/.claude/,규칙/技能放在 rules/ecc 和 skills/ecc |
| `claude-project` | 설치到 ./.claude/ (프로젝트级) |
| `cursor` | 설치규칙、钩子、光标설정到 ./.cursor/ |
| `antigravity` | 설치규칙、워크플로우、技能、프록시到 ./.agent/ |
| `codex` | 설치共享프록시/설정到 ~/.codex/ |
| `gemini` | 설치프로젝트级 Gemini 설정到 ./.gemini/ |
| `opencode` | 설치共享명령/钩子/설정到 ~/.opencode/ |
| `codebuddy` | 설치명령、프록시、技能到 ./.codebuddy/ |
| `joycode` | 설치명령、프록시、技能到 ./.joycode/ |
| `qwen` | 설치到 ~/.qwen/ |
| `zed` | 설치到 ./.zed/ |

### 설치方式

1. **传统언어패턴**: `install.sh <language>` (如 `ecc-install typescript`)
2. **Profile 패턴**: `install.sh --profile <name> [--with <component>]... [--without <component>]...`
3. **모듈패턴**: `install.sh --modules <id,id,...>`
4. **技能패턴**: `install.sh --skills <skill-id[,skill-id...]>`
5. **本地化패턴**: `install.sh --target claude|claude-project --locale <locale-code>`

### 选项

- `--dry-run` - 표시설치계획但不복사파일
- `--json` - 发出机器可读的 JSON 형식
- `--config <path>` - 从파일加载설치意图

## claw.js

**路径**: `scripts/claw.js`

NanoClaw v2 - 面向 Everything Claude Code 的零外部依赖、세션感知的 REPL,基于 `claude -p` 빌드。

### 기능

- 세션历史持久化存储在 `~/.claude/claw/`
- 지원动态加载技能作为上下文
- 세션압축以管理上下文길이
- 세션브랜치和내보내기
- 跨세션搜索

### REPL 명령

| 명령 | 설명 |
|------|------|
| `/help` | 표시帮助 |
| `/clear` | 清除当前세션历史 |
| `/history` | 인쇄完整对话历史 |
| `/sessions` | 列出已저장的세션 |
| `/model [name]` | 표시/설정모델 |
| `/load <skill-name>` | 加载技能到活动上下文 |
| `/branch <session-name>` | 将当前세션브랜치到新세션 |
| `/search <query>` | 跨세션搜索 |
| `/compact` | 保留最近的轮次,압축旧上下文 |
| `/export <md\|json\|txt> [path]` | 내보내기세션 |
| `/metrics` | 표시세션지표 |
| `exit` | 退出 REPL |

### 环境변수

| 변수 | 기본值 | 설명 |
|------|--------|------|
| `CLAW_SESSION` | `default` | 初始세션名称 |
| `CLAW_MODEL` | `sonnet` | 기본모델 |
| `CLAW_SKILLS` | 空 | 加载的技能列表 (逗号分隔) |

### 사용예제

```bash
# 사용기본세션시작
node scripts/claw.js

# 사용指定세션시작
CLAW_SESSION=my-session node scripts/claw.js

# 加载技能시작
CLAW_SKILLS="javascript-patterns,python-testing" node scripts/claw.js
```

## harness-audit.js

**路径**: `scripts/harness-audit.js`

确定性도구链审计도구,基于显式파일/규칙검사。자동检测 ECC 仓库패턴 vs 消费者프로젝트패턴。

### 审计클래스别

| 클래스别 | 설명 |
|------|------|
| Tool Coverage | 도구覆盖完整性 |
| Context Efficiency | 上下文效率 |
| Quality Gates | 품질门禁 |
| Memory Persistence | 记忆持久化 |
| Eval Coverage | 评估覆盖 |
| Security Guardrails | 보안护栏 |
| Cost Efficiency | 成本效率 |
| GitHub Integration | GitHub 통합 |
| Vercel/Netlify/Cloudflare/Fly Integration | 各平台통합 |

### 사용예제

```bash
# 审计当前디렉토리
node scripts/harness-audit.js

# 指定作用域
node scripts/harness-audit.js --scope repo
node scripts/harness-audit.js --scope hooks
node scripts/harness-audit.js --scope skills

# 指定根디렉토리
node scripts/harness-audit.js --root /path/to/project

# JSON 형식输出
node scripts/harness-audit.js --format json

# 组合사용
node scripts/harness-audit.js repo --format json --root .
```

## catalog.js

**路径**: `scripts/catalog.js`

发现 ECC 설치컴포넌트和설정파일的도구。

### 명령

#### profiles

列出모든설치설정파일。

```bash
node scripts/catalog.js profiles
node scripts/catalog.js profiles --json
```

#### components

列出설치컴포넌트,可按家族和目标筛选。

```bash
node scripts/catalog.js components
node scripts/catalog.js components --family language
node scripts/catalog.js components --family framework
node scripts/catalog.js components --target claude
node scripts/catalog.js components --json
```

#### show

표시特定컴포넌트的详细정보。

```bash
node scripts/catalog.js show <component-id>
node scripts/catalog.js show framework:nextjs --json
```

### 컴포넌트家族

- `baseline` - 基础컴포넌트
- `language` - 언어컴포넌트 (javascript, typescript, python, etc.)
- `framework` - 프레임워크컴포넌트 (nextjs, react, vue, etc.)
- `capability` - 能力컴포넌트
- `agent` - 프록시컴포넌트
- `skill` - 技能컴포넌트

## build-opencode.js

**路径**: `scripts/build-opencode.js`

빌드 OpenCode 플러그인的 TypeScript 코드。

### 기능

1. 정리 `dist` 디렉토리
2. 사용 TypeScript 컴파일器컴파일 `.opencode` 디렉토리下的코드
3. 输出到 `.opencode/dist`

### 前提条件

필요已설치根디렉토리的개발依赖,确保 TypeScript 컴파일器可用。

### 사용

```bash
node scripts/build-opencode.js
```
