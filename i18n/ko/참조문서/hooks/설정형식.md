# 설정 형식

## hooks.json 구조개요

ECC Hooks 시스템的설정파일采用JSON형식，主要구조如下：

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "hooks": {
    "PreToolUse": [...],
    "PostToolUse": [...],
    "Stop": [...],
    "SessionStart": [...],
    "SessionEnd": [...],
    "PreCompact": [...],
    "PostToolUseFailure": [...]
  }
}
```

## 顶层구조

| 字段 | 유형 | 설명 |
|------|------|------|
| `$schema` | string | JSON Schema URL，用于검증설정 |
| `hooks` | object | 패키지含모든钩子유형的对象 |

---

## 钩子数组구조

각事件유형（PreToolUse、PostToolUse等）패키지含一个钩子数组，각钩子对象보유以下구조：

```json
{
  "matcher": "Bash|Edit|Write",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/my-hook.js",
      "async": false,
      "timeout": 30
    }
  ],
  "description": "钩子描述",
  "id": "unique-hook-id"
}
```

### matcher 字段

#### 유형설명
matcher字段用于指定触发钩子的도구유형，지원单个도구或多个도구（用`|`分隔）。

#### 可用值
| matcher值 | 匹配的도구있음哪些 |
|-----------|-----------------|
| `*` | 모든도구 |
| `Bash` | Bash도구 |
| `Edit` | Edit도구 |
| `Write` | Write도구 |
| `Read` | Read도구 |
| `MultiEdit` | MultiEdit도구 |
| `Bash\|Edit\|Write` | Bash、Edit或Write |
| `Edit\|Write\|MultiEdit` | Edit、Write或MultiEdit |

#### 사용예제
```json
// 匹配모든도구
"matcher": "*"

// 仅匹配Bash도구
"matcher": "Bash"

// 匹配多个도구
"matcher": "Edit|Write|MultiEdit"
```

## hooks 数组구조

hooks数组패키지含一个或多个钩子명령对象：

### type 字段

| type值 | 설명 |
|--------|------|
| `command` | 실행Node.js명령 |

### command 字段

要실행的명령，通常是Node.js스크립트路径。

### async 字段（선택）

```json
"async": true
```

- `true`: 비동기실행，不阻塞도구실행
- `false`或省略: 동기화실행，阻塞도구실행

### timeout 字段（선택）

```json
"timeout": 30
```

最大실행时间（秒）。권장：
- 동기화钩子: <200ms
- 비동기钩子: ≤30秒

## 完整설정예제

### PreToolUse 钩子예제

```json
{
  "matcher": "Bash",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/pre-bash-dispatcher.js"
    }
  ],
  "description": "Bash预检分发器，用于품질、tmux、푸시和GateGuard검사",
  "id": "pre:bash:dispatcher"
}
```

### PostToolUse 钩子예제

```json
{
  "matcher": "Edit|Write|MultiEdit",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/run-with-flags.js post:quality-gate scripts/hooks/quality-gate.js standard,strict",
      "async": true,
      "timeout": 30
    }
  ],
  "description": "파일편집后실행품질门검사",
  "id": "post:quality-gate"
}
```

### Stop 钩子예제

```json
{
  "matcher": "*",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/run-with-flags.js stop:session-end scripts/hooks/session-end.js minimal,standard,strict",
      "async": true,
      "timeout": 10
    }
  ],
  "description": "每次응답后저장세션状态",
  "id": "stop:session-end"
}
```

### SessionStart 钩子예제

```json
{
  "matcher": "*",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/session-start-bootstrap.js"
    }
  ],
  "description": "加载先前上下文并감지新세션的패키지 관리자",
  "id": "session:start"
}
```

## 라이프사이클事件설정 형식

对于SessionStart、SessionEnd、PreCompact等라이프사이클事件，사용不同的설정구조：

```json
{
  "description": "메모리持久化的라이프사이클钩子定义",
  "events": [
    {
      "event": "SessionStart",
      "id": "session:start",
      "script": "scripts/hooks/session-start-bootstrap.js",
      "purpose": "在세션시작时加载있음界限的先前上下文并감지프로젝트状态",
      "blocking": false
    },
    {
      "event": "PreCompact",
      "id": "pre:compact",
      "script": "scripts/hooks/pre-compact.js",
      "purpose": "在上下文압축前持久化세션状态",
      "blocking": false
    }
  ]
}
```

### 라이프사이클事件字段

| 字段 | 설명 |
|------|------|
| `event` | 事件유형（SessionStart、PreCompact、SessionEnd等） |
| `id` | 唯一标识符 |
| `script` | 스크립트路径 |
| `purpose` | 용도描述 |
| `blocking` | 是否阻塞（通常为false） |

## 비활성화钩子

### 통과편집설정비활성화

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write",
        "hooks": [],
        "description": "覆盖：허용모든.md파일생성"
      }
    ]
  }
}
```

### 통과环境변수비활성화

```bash
# 비활성화特定钩子（逗号分隔）
export ECC_DISABLED_HOOKS="pre:bash:tmux-reminder,post:edit:typecheck"

# 비활성화GateGuard
export ECC_GATEGUARD=off
```

## 钩子ID命名사양

ECC사용冒号分隔的命名约定：

| 前缀 | 용도 | 예제 |
|------|------|------|
| `pre:` | PreToolUse钩子 | `pre:bash:dispatcher` |
| `post:` | PostToolUse钩子 | `post:quality-gate` |
| `stop:` | Stop钩子 | `stop:session-end` |
| `session:` | 세션라이프사이클钩子 | `session:start` |

## run-with-flags.js 封装器

ECC사용`run-with-flags.js`封装器来실행钩子，지원설정파일的실행时门控：

```
node scripts/hooks/run-with-flags.js <hook-id> <script-path> <profiles>
```

### 参数설명
| 参数 | 설명 |
|------|------|
| `hook-id` | 钩子的唯一标识符 |
| `script-path` | 实际要실행的스크립트路径 |
| `profiles` | 逗号分隔的설정파일列表（minimal, standard, strict） |

### 작업原理
1. 읽기`ECC_HOOK_PROFILE`环境변수（기본: standard）
2. 검사钩子ID是否在`ECC_DISABLED_HOOKS`中
3. 如果허용，则실행스크립트

## 跨平台路径处理

ECC的hook명령사용Node.js스크립트구현跨平台行为：

```javascript
const path = require('path');
const homedir = require('os').homedir();

// 跨平台路径파싱
const configDir = path.join(homedir, '.claude');
const hookScript = path.join(configDir, 'scripts', 'hooks', 'my-hook.js');
```
