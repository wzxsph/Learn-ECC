# 사용자 정의 개발

## 개요

本가이드介绍如何为ECC Hooks 시스템개발自定义钩子，패키지括基础구조、모범 사례和exit 0요구。

## 基础구조

### 最小钩子模板

```javascript
// my-custom-hook.js
let data = '';
process.stdin.on('data', chunk => data += chunk);
process.stdin.on('end', () => {
  const input = JSON.parse(data);

  // 访问도구정보
  const toolName = input.tool_name;        // "Edit", "Bash", "Write"等
  const toolInput = input.tool_input;      // 도구特定参数
  const toolOutput = input.tool_output;    // 仅在PostToolUse可用

  // 경고（非阻塞）：쓰기stderr
  console.error('[Hook] 경고정보');

  // 阻止（仅PreToolUse）：exit code 2
  // process.exit(2);

  // 始终将原始데이터输出到stdout
  console.log(data);
});
```

## 退出码요구

### 핵심규칙：exit 0 on 非핵심오류

**重要**: 모든钩子필수在非핵심오류时exit 0，避免意外阻塞도구실행。

| 退出码 | 含义 | 사용场景 |
|--------|------|----------|
| 0 | 성공 | 계속실행，或非핵심경고 |
| 2 | 阻止 | 仅用于PreToolUse的핵심阻塞 |
| 기타非零 | 오류 | 仅기록로그，**不要사용** |

### 为什么非零退出码不好？

如果钩子以非零退出码退出（除2外），Claude Code会将整个도구调用标记为실패，这可能导致：
- 도구실행被意外中断
- 用户体验受损
- 难以디버깅的문제

### 正确处理오류

```javascript
// 오류예제 - 不要这样做
process.stdin.on('end', () => {
  try {
    const input = JSON.parse(data);
    // 处理逻辑
  } catch (e) {
    console.error('[Hook] Error:', e.message);
    process.exit(1);  // 오류：会阻塞도구
  }
  console.log(data);
});

// 正确예제 - 这样做
process.stdin.on('end', () => {
  try {
    const input = JSON.parse(data);
    // 处理逻辑
  } catch (e) {
    console.error('[Hook] Error:', e.message);
    // 非핵심오류，exit 0계속실행
    console.log(data);
    return;
  }
  console.log(data);
});
```

## 钩子输入패턴

### HookInput 인터페이스

```typescript
interface HookInput {
  tool_name: string;          // 도구名称
  tool_input: {               // 도구输入参数
    command?: string;         // Bash: 명령
    file_path?: string;        // Edit/Write/Read: 파일路径
    old_string?: string;       // Edit: 被교체的文本
    new_string?: string;       // Edit: 교체文本
    content?: string;          // Write: 파일内容
  };
  tool_output?: {             // PostToolUse only
    output?: string;          // 명령/도구输出
  };
}
```

### 访问도구정보

```javascript
process.stdin.on('end', () => {
  const input = JSON.parse(data);

  // 가져오기도구名称
  console.log('Tool:', input.tool_name);

  // 根据도구유형处理
  if (input.tool_name === 'Bash') {
    console.log('Command:', input.tool_input.command);
  }

  if (input.tool_name === 'Edit' || input.tool_name === 'Write') {
    console.log('File:', input.tool_input.file_path);
  }

  // PostToolUse可访问输出
  if (input.tool_output) {
    console.log('Output:', input.tool_output.output);
  }
});
```

## 비동기钩子

对于不应阻塞主流程的钩子（如后台분석）：

```json
{
  "matcher": "Edit|Write|MultiEdit",
  "hooks": [{
    "type": "command",
    "command": "node my-slow-hook.js",
    "async": true,
    "timeout": 30
  }]
}
```

### 비동기钩子注意事项

- 비동기钩子실행在后台
- 불가능阻塞도구실행
- 해야 함在30秒内완료
- 适用于：로그、분석、遥测

## 阻塞钩子

对于필수阻止도구실행的情况（PreToolUse only）：

```javascript
// 在PreToolUse中阻止
process.exit(2);  // 退出码2表示阻止
```

### 何时사용阻止

- 보안검사실패（如检测到密钥）
- 违反硬性전략
- 可能导致데이터丢失的操作

### 何时不要사용阻止

- 코딩 스타일문제（사용경고代替）
- 非핵심검사
- 권장클래스검사

## 성능모범 사례

### 保持快速

- PreToolUse钩子：<200ms
- PostToolUse동기화钩子：<1秒
- 비동기钩子：<30秒

### 避免阻塞操作

```javascript
// 不好：동기화파일읽기
const content = fs.readFileSync('large-file.txt', 'utf8');

// 好：비동기或지연加载
fs.readFile('large-file.txt', 'utf8', (err, content) => {
  // 处理
});
```

### 캐시结果

```javascript
// 캐시昂贵操作
let cachedResult = null;
let cacheTime = 0;
const CACHE_TTL = 60000; // 1分钟

function getCachedResult() {
  const now = Date.now();
  if (!cachedResult || now - cacheTime > CACHE_TTL) {
    cachedResult = expensiveOperation();
    cacheTime = now;
  }
  return cachedResult;
}
```

## 테스트钩子

### 手动테스트

```bash
# 테스트PreToolUse钩子
echo '{"tool_name":"Bash","tool_input":{"command":"echo test"}}' | node scripts/hooks/my-hook.js

# 테스트PostToolUse钩子
echo '{"tool_name":"Bash","tool_input":{"command":"echo test"},"tool_output":{"output":"test\n"}}' | node scripts/hooks/my-hook.js
```

### 테스트输出형식

```javascript
// 正确的stdout输出（JSON）
console.log(data);  // 原始输入데이터

// 正确的stderr输出（경고）
console.error('[Hook] Warning message');

// 不正确的做法 - 不要输出기타内容到stdout
console.log('Some message');  // 这会破坏데이터流
```

### 常见문제

| 문제 | 原因 | 해결方案 |
|------|------|----------|
| 도구被意外阻止 | 钩子exit非零（除2外） | 改用exit 0和stderr경고 |
| 挂起 | 钩子없음종료 | 确保总是输出到stdout |
| 데이터损坏 | 输出到stdout非JSON | 仅输出原始data |
| 성능문제 | 钩子太慢 | 최적화或改为비동기 |

## 모범 사례总结

1. **始终exit 0 on 非핵심오류** - 不要用非零退出码（除2外）
2. **사용console.error输出경고** - 不要用console.log
3. **保持快速** - PreToolUse <200ms
4. **总是输出原始data到stdout** - 不要改变데이터流
5. **사용비동기处理长时间操作** - 설정async: true
6. **跨平台路径处理** - 사용path.join
7. **테스트输出형식** - 确保stdout是原始JSON
8. **문서화你的钩子** - 추가清晰的description
