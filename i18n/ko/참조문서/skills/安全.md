# 보안

本일부涵盖보안리뷰、漏洞扫描和各프레임워크的보안모범 사례。

---

## 보안리뷰

### 용도설명
보안리뷰技能确保모든코드遵循보안모범 사례，识别潜在漏洞。

### 사용 타이밍
- 구현身份검증或권한 부여
- 处理用户输入或파일上传
- 생성新API端点
- 사용密钥或凭证
- 구현支付기능
- 存储或传输敏感데이터
- 통합第三方API

### 코어개념

**1. 密钥管理**
永远不在源코드中硬인코딩密钥。

**2. 输入검증**
모든用户输入필수검증。

**3. SQL注入防护**
사용参数化查询。

**4. XSS防护**
정리用户提供的HTML。

**5. CSRF保护**
在状态改变操作上사용CSRF令牌。

**6. 速率限制**
在모든API端点上활성화速率限制。

### 보안검사清单

| 체크 항목 | 描述 |
|--------|------|
| 密钥 | 없음硬인코딩密钥，모든密钥在环境변수中 |
| 输入검증 | 모든用户输入사용schema검증 |
| SQL注入 | 모든查询사용参数化查询 |
| XSS | 用户内容被정리 |
| CSRF | 활성화保护 |
| 身份검증 | 正确的令牌处理 |
| 권한 부여 | 角色검사到位 |
| 速率限制 | 모든端点활성화 |
| HTTPS | 生产环境强制사용 |
| 보안头 | CSP、X-Frame-Options설정 |
| 오류 처리 | 오류 메시지不暴露敏感데이터 |
| 로그 | 不기록敏感데이터 |
| 依赖 | 업데이트，없음漏洞 |

// 密钥管理 - 오류做法

## 보안扫描

### 용도설명
사용AgentShield扫描Claude Code설정（`.claude/`디렉토리）中的보안漏洞、설정오류和注入风险。

### 사용 타이밍
- 설정新的Claude Code프로젝트
- 수정`.claude/settings.json`、`CLAUDE.md`或MCP설정后
- 커밋설정更改前
- 加入已있음Claude Code설정的新仓库
- 定期보안검사

### 扫描内容

| 파일 | 체크 항목 |
|------|--------|
| `CLAUDE.md` | 硬인코딩密钥、자동실행指令、提示注入패턴 |
| `settings.json` | 过度宽松的허용列表、缺少거부列表、危险绕过标志 |
| `mcp.json` | 风险MCP서버、硬인코딩env密钥、npx供应链风险 |
| `hooks/` | 통과插值的명령注入、데이터泄露、静默오류抑制 |
| `agents/*.md` | 不受限的도구访问、提示注入表面、缺少모델规格 |

### 사용예제

```bash
# 基本扫描
npx ecc-agentshield scan

# 指定路径扫描
npx ecc-agentshield scan --path /path/to/.claude

# 最低심각性过滤
npx ecc-agentshield scan --min-severity medium

# JSON형식（用于CI/CD）
npx ecc-agentshield scan --format json

# 자동수정
npx ecc-agentshield scan --fix

# Opus깊이분석（필요ANTHROPIC_API_KEY）
export ANTHROPIC_API_KEY=your-key
npx ecc-agentshield scan --opus --stream
```

### 심각性级别

| 等级 | 分数 | 含义 |
|------|------|------|
| A | 90-100 | 보안설정 |
| B | 75-89 | 小문제 |
| C | 60-74 | 필요关注 |
| D | 40-59 | 重大风险 |
| F | 0-39 | 심각漏洞 |

## Django보안

### 용도설명
Django보안모범 사례，保护应用程序免受常见漏洞攻击。

### 사용 타이밍
- 설정Django身份검증和권한 부여
- 구현用户权限和角色
- 설정生产보안설정
- 리뷰Django应用程序보안문제
- 배포Django应用程序到生产环境

### 코어보안설정

```python
# settings/production.py
DEBUG = False  # 핵심：生产环境绝不사용True

ALLOWED_HOSTS = os.environ.get('ALLOWED_HOSTS', '').split(',')

# 보안头
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True
SECURE_HSTS_SECONDS = 31536000  # 1年
SECURE_HSTS_INCLUDE_SUBDOMAINS = True
SECURE_HSTS_PRELOAD = True
SECURE_CONTENT_TYPE_NOSNIFF = True
SECURE_BROWSER_XSS_FILTER = True
X_FRAME_OPTIONS = 'DENY'

# Cookie보안
SESSION_COOKIE_HTTPONLY = True
CSRF_COOKIE_HTTPONLY = True
SESSION_COOKIE_SAMESITE = 'Lax'

# 密码검증
AUTH_PASSWORD_VALIDATORS = [
    {'NAME': 'django.contrib.auth.password_validation.MinimumLengthValidator', 'OPTIONS': {'min_length': 12}},
    {'NAME': 'django.contrib.auth.password_validation.CommonPasswordValidator'},
]
```

### SQL注入防护

```python
# 보안：Django ORM자동转义参数
def get_user(username):
    return User.objects.get(username=username)  # 보안

# 보안：사용参数的raw()
def search_users(query):
    return User.objects.raw('SELECT * FROM users WHERE username = %s', [query])

# 危险：永远不要直接插值
def get_user_bad(username):
    return User.objects.raw(f'SELECT * FROM users WHERE username = {username}')  # 漏洞！
```

### XSS防护

```django
{# Django기본자동转义변수 - 보안 #}
{{ user_input }}  {# HTML会被转义 #}

{# 显式标记为보안仅用于可信内容 #}
{{ trusted_html|safe }}  {# 不转义 #}

{# 사용模板过滤器보안处理HTML #}
{{ user_input|striptags }}  {# 제거모든HTML태그 #}
```

### CSRF保护

```django
{# 模板中사용 #}
<form method="post">
    {% csrf_token %}
    {{ form.as_p }}
    <button type="submit">커밋</button>
</form>

{# AJAX요청 #}
function getCookie(name) {
    let cookieValue = null;
    if (document.cookie && document.cookie !== '') {
        const cookies = document.cookie.split(';');
        for (let i = 0; i < cookies.length; i++) {
            const cookie = cookies[i].trim();
            if (cookie.substring(0, name.length + 1) === (name + '=')) {
                cookieValue = decodeURIComponent(cookie.substring(name.length + 1));
                break;
            }
        }
    }
    return cookieValue;
}

fetch('/api/endpoint/', {
    method: 'POST',
    headers: {
        'X-CSRFToken': getCookie('csrftoken'),
        'Content-Type': 'application/json',
    },
    body: JSON.stringify(data)
});
```

### 快速보안검사清单

| 체크 항목 | 描述 |
|--------|------|
| `DEBUG = False` | 生产环境绝不사용DEBUG실행 |
| 仅HTTPS | 强制SSL，보안cookie |
| 强密钥 | 사용环境변수的SECRET_KEY |
| 密码검증 | 활성화모든密码검증器 |
| CSRF保护 | 기본활성화，不要비활성화 |
| XSS防护 | Django기본转义，不要对用户输入사용`\|safe` |
| SQL注入 | 사용ORM，永远不要在查询中拼接字符串 |
| 파일上传 | 검증파일유형和크기 |
| 速率限制 | API端点限流 |
| 보안头 | CSP、X-Frame-Options、HSTS |
| 로그 | 기록보안事件 |
| 업데이트 | 保持Django和依赖업데이트 |
