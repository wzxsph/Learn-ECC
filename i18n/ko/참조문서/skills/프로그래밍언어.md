## Python Patterns (python-patterns)

### 용도 설명
Pythonic 관용구, PEP 8 표준, 타입 힌트 및 모범 사례를 제공하여 강력하고 효율적이며 유지보수 가능한 Python 애플리케이션을 구축합니다.

### 사용 타이밍
- 새로운 Python 코드 작성 시
- Python 코드 리뷰 시
- 기존 Python 코드 리팩토링 시
- Python 패키지/모듈 설계 시

### 코어 개념
1. **가독성 우선** - 코드가 명확하고 оче직해야 함
2. **명시적이优于隐식** - 매직 작업 피하기
3. **EAFP 스타일** -許可より許しを请う easier than asking permission
4. **타입 힌트** - 유형 주석을 사용하여 코드 보안 향상
5. **컨텍스트 관리자** - `with`로 리소스 관리
6. **리스트 내포** - 간단한 데이터 변환
7. **生成器** -惰성 평가 및大数据集

### 사용 예제

**타입 힌트:**
```python
from typing import Optional, List, Dict, Any

def process_user(
    user_id: str,
    data: Dict[str, Any],
    active: bool = True
) -> Optional[User]:
    if not active:
        return None
    return User(user_id, data)
```

**컨텍스트 관리자:**
```python
from contextlib import contextmanager

@contextmanager
def timer(name: str):
    start = time.perf_counter()
    yield
    elapsed = time.perf_counter() - start
    print(f"{name} took {elapsed:.4f} seconds")

with timer("data processing"):
    process_large_dataset()
```

**오류 처리:**
```python
try:
    parsed = json.loads(data)
except json.JSONDecodeError as e:
    raise ValueError(f"Failed to parse data: {data}") from e
```

## Go Patterns (golang-patterns)

### 용도 설명
관용적 Go 패턴, 모범 사례 및 규칙을 제공하여 강력하고 효율적이며 유지보수 가능한 Go 애플리케이션을 구축합니다.

### 사용 타이밍
- 새로운 Go 코드 작성 시
- Go 코드 리뷰 시
- 기존 Go 코드 리팩토링 시
- Go 패키지/모듈 설계 시

### 코어 개념
1. **简洁和清晰** - Go는 기교보다简洁偏好
2. **零值 유용** --zero값이 직접 사용 가능한 유형 설계
3. **인터페이스を受け取り、構造体を返す** - 함수가 인터페이스 매개변수を受け取り, 구체적 유형 반환
4. **오류 처리** - 오류는 값이며, `error` 반환 값 패턴 따르기
5. **동시성 패턴** - goroutine 및 channel 사용
6. **인터페이스 설계** - 작고 집중된 인터페이스
7. **패키지 구성** - 표준 프로젝트 레이아웃

### 사용 예제

**오류 패키지 처리:**
```go
func LoadConfig(path string) (*Config, error) {
    data, err := os.ReadFile(path)
    if err != nil {
        return nil, fmt.Errorf("load config %s: %w", path, err)
    }
    // ...
}
```

**워커 풀:**
```go
func WorkerPool(jobs <-chan Job, results chan<- Result, numWorkers int) {
    var wg sync.WaitGroup
    for i := 0; i < numWorkers; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            for job := range jobs {
                results <- process(job)
            }
        }()
    }
    wg.Wait()
}
```

**기능 옵션 패턴:**
```go
type Option func(*Server)

func WithTimeout(d time.Duration) Option {
    return func(s *Server) { s.timeout = d }
}

func NewServer(addr string, opts ...Option) *Server {
    s := &Server{addr: addr, timeout: 30 * time.Second}
    for _, opt := range opts {
        opt(s)
    }
    return s
}
```

## Rust Patterns (rust-patterns)

### 용도 설명
관용적 Rust 패턴, 소유권 시스템, 오류 처리, trait, 동시성 및 모범 사례를 제공하여 안전하고 효율적인 애플리케이션을 구축합니다.

### 사용 타이밍
- 새로운 Rust 코드 작성 시
- Rust 코드 리뷰 시
- 기존 Rust 코드 리팩토링 시
- crate 구조 및 모듈 레이아웃 설계 시

### 코어 개념
1. **소유권 및 빌로우** - 컴파일 시 데이터 경합 및 메모리 오류防止
2. **Result 및 ? 연산자** - `?`로 오류 전파
3. **열거형 및穷举 매칭** - 열거형으로 불가능한 상태를 불가능으로 표현
4. **Trait 및 제네릭** -零비용 추상화
5. **안전한 동시성** - `Arc<Mutex<T>>`, channels 및 async/await 사용
6. **최소 pub 표면** - 영역별로 모듈 구성

### 사용 예제

**오류 처리:**
```rust
use anyhow::{Context, Result};

fn load_config(path: &str) -> Result<Config> {
    let content = std::fs::read_to_string(path)
        .with_context(|| format!("failed to read config from {path}"))?;
    let config: Config = toml::from_str(&content)
        .with_context(|| format!("failed to parse config from {path}"))?;
    Ok(config)
}
```

**Cow로 유연하게 소유권 처리:**
```rust
use std::borrow::Cow;

fn normalize(input: &str) -> Cow<'_, str> {
    if input.contains(' ') {
        Cow::Owned(input.replace(' ', "_"))
    } else {
        Cow::Borrowed(input) // 수정 필요 없을 때 zero-cost
    }
}
```

**Arc<Mutex<T>>로 공유可变 상태:**
```rust
use std::sync::{Arc, Mutex};

let counter = Arc::new(Mutex::new(0));
let handles: Vec<_> = (0..10).map(|_| {
    let counter = Arc::clone(&counter);
    std::thread::spawn(move || {
        let mut num = counter.lock().expect("mutex poisoned");
        *num += 1;
    })
}).collect();
```

1. **Null 보안** - 유형 시스템 및 안전 호출 연산자 사용
2. **기본 불변** - `val` 우선 사용, `var` 피하기
3. **데이터 클래스** -主要持有 데이터의 유형용
4. **봉인 클래스** -穷举 유형 계층 구조
5. **코루틴** - 구조화된 동시성

### 사용 예제

```kotlin

suspend fun fetchUserWithPosts(userId: String): UserProfile =

        val user = async { userService.getUser(userId) }

    }
```

## C++ Coding Standards (cpp-coding-standards)

### 용도 설명
C++ 코어 가이드 기반의 현대 C++ (C++17/20/23) 인코딩 표준입니다. 유형 안전성, 리소스 보안, 불변성 및明晰성을 강제합니다.

### 사용 타이밍
- 새로운 C++ 코드 작성 시
- C++ 코드 리뷰 또는 리팩토링 시
- 아키텍처 결정 시
- 일관된 코딩 스타일 강제 시
- 언어 기능 선택 시

### 코어 개념
1. **RAII 어디서나** - 리소스 라이프사이클을 객체 라이프사이클에 바인딩
2. **기본 불변** - `const`/`constexpr`로 시작
3. **유형 안전성** - 유형 시스템으로 컴파일 시 오류防止
4. **의도 표현** - 이름, 유형 및 개념이 목적을 전달해야 함
5. **최소화 복잡도** - 간단한 코드가 올바른 코드
6. **값 의미론 우선 포인터 의미론** - 값 및作用域 对象 우선 반환

### 사용 예제

**스마트 포인터 사용:**
```cpp
// R.11 + R.20 + R.21: RAII 및 스마트 포인터
auto widget = std::make_unique<Widget>("config");  // 유일 소유권
auto cache = std::make_shared<Cache>(1024);         // 공유 소유권

// R.3: 원시 포인터는 非拥有 的 관찰자
void render(const Widget* w) {  // w를 소유하지 않음
    if (w) w->draw();
}
```

**Zero 규칙:**
```cpp
// C.20: 컴파일러가 특수成员 생성让其
struct Employee {
    std::string name;
    std::string department;
    int id;
    // 소멸자, 복사/이동 생성자 또는 할당 연산자 정의 불필요
};
```

## Java Coding Standards (java-coding-standards)

### 용도 설명
Spring Boot 및 Quarkus 서비스용 Java 인코딩 표준으로, 명명, 불변성, Optional 사용, 스트림 처리, 예외, 제네릭 및 프로젝트 레이아웃을 포함합니다.

### 사용 타이밍
- Spring Boot 또는 Quarkus 프로젝트에서 Java 코드 작성 또는 리뷰 시
- 명명, 불변성 또는 예외 처리 규칙 강제 시
- records, 봉인 클래스 또는 패턴 매칭 (Java 17+) 사용 시
- Optional, 스트림 또는 제네릭 사용 리뷰 시

### 코어 개념
1. **프레임워크 감지** - 빌드 파일로 Spring Boot 또는 Quarkus 감지

4. **Optional 사용** - find* 메서드에서 Optional 반환

6. **응답형 패턴 (Quarkus)** - Uni/Multi 반환

```java
public class MarketService {
    private final MarketRepository marketRepository;

    public MarketService(MarketRepository marketRepository) {
        this.marketRepository = marketRepository;
    }
}
```

```java
return market
```

## Swift 6.2 Approachable Concurrency (swift-concurrency-6-2)

### 용도 설명
Swift 6.2의 "접근 가능한 동시성" 패턴 - 기본单스레드, `@concurrent`로 명시적后台제거,隔离 일관성으로 주 actor 유형에 프로토콜 지원 제공.

### 사용 타이밍
- Swift 5.x 또는 6.0/6.1 프로젝트를 Swift 6.2로 마이그레이션 시
- 데이터 경합 보안 컴파일러 오류 해결 시
- MainActor 기반 앱 아키텍처 설계 시
- 프로토콜 일관성 구현 시

### 코어 개념
1. **单스레드 기본** - 대부분의 코드는 데이터 경합 보안임; 동시성은 명시적 opt-in
2. **Async는 호출 actor에 유지** - 데이터 경합 오류를 유발하는隐式 제거消除
3. **隔离 일관성** - MainActor 유형이 안전하게非隔离 프로토콜 구현
4. **@concurrent 명시적 opt-in** -后台 실행은 의도적인 성능 선택

### 사용 예제

**隔离 일관성:**
```swift
extension StickerModel: @MainActor Exportable {
    func export() {
        photoProcessor.exportAsPNG()
    }
}
```

**后台 작업용 @concurrent:**
```swift
nonisolated final class PhotoProcessor {
    @concurrent
    static func extractSubject(from data: Data) async -> Sticker {
        //后台执行的昂贵 작업
    }
}
```

## Dart/Flutter Patterns (dart-flutter-patterns)

### 용도 설명
null 보안, 불변 상태, 비동기 조합, 컴포넌트 아키텍처, 인기 상태 관리 프레임워크 (BLoC, Riverpod, Provider), GoRouter 탐색, Dio 네트워크 요청, Freezed 코드 생성 및 클린 아키텍처를 다루는.production 준비된 Dart 및 Flutter 패턴.

### 사용 타이밍
- 새 Flutter 기능을 시작하고 상태 관리, 탐색 또는 데이터 액세스 패턴이 필요한 경우
- Dart 코드를 리뷰하거나 작성하고 null 보안, 봉인 유형 또는 비동기 조합에 대한 지침이 필요한 경우
- 새 Flutter 프로젝트를 설정하고 BLoC, Riverpod 또는 Provider 선택 시
- 안전한 HTTP 클라이언트, WebView 통합 또는 로컬 저장소 구현 시
- Flutter 컴포넌트, Cubit 또는 Riverpod provider 테스트 작성 시

### 코어 개념
1. **Null 보안** - `!` 피하기, `?.`/`??`/패턴 매칭 우선
2. **불변 상태** - 봉인 클래스, `freezed`, `copyWith`
3. **비동기 조합** - 동시성 `Future.wait`, await 후 안전한 BuildContext 사용
4. **컴포넌트 아키텍처** - 클래스로 추출 (메서드 아닌), `const` 전파,作用域 재구성
5. **상태 관리** - BLoC/Cubit 이벤트, Riverpod notifiers 및 파생 providers

### 사용 예제

**봉인 상태 클래스:**
```dart
sealed class AsyncState<T> {}
final class Loading<T> extends AsyncState<T> {}
final class Success<T> extends AsyncState<T> { final T data; const Success(this.data); }
final class Failure<T> extends AsyncState<T> { final Object error; const Failure(this.error); }

Widget buildFrom(AsyncState<User> state) => switch (state) {
    UserInitial() => const SizedBox.shrink(),
    UserLoading() => const CircularProgressIndicator(),
    UserLoaded(:final user) => UserCard(user: user),
    UserError(:final message) => ErrorText(message),
};
```

**Riverpod 파생 provider:**
```dart
@riverpod
double cartTotal(Ref ref) {
    final cart = ref.watch(cartNotifierProvider);
    final products = ref.watch(productsProvider).valueOrNull ?? [];
    return cart.fold(0.0, (total, item) {
        final product = products.firstWhereOrNull((p) => p.id == item.productId);
        return total + (product?.price ?? 0) * item.quantity;
    });
}
```

### 코어 개념
1. **xUnit** - 표준 .NET 생태계 테스트 프레임워크
2. **FsUnit.xUnit** - xUnit의 F# 친화적assertion 구문

4. **FsCheck.xUnit** - xUnit에 통합된 속성 테스트

### 사용 예제

**Unquote assertions:**
```fsharp
[<Fact>]
let ``order total sums item prices`` () =
    let items = [ { Sku = "A"; Quantity = 2; Price = 10m }
                  { Sku = "B"; Quantity = 1; Price = 5m } ]
    let total = Order.calculateTotal items
    test <@ total = 25m @>
```

**FsCheck 속성 테스트:**
```fsharp
[<Property>]
let ``order total is always non-negative`` (items: NonEmptyList<PositiveInt * decimal>) =
    let orderItems = items.Get |> List.map (fun (qty, price) ->
        { Sku = "SKU"; Quantity = qty.Get; Price = abs price })
    let total = Order.calculateTotal orderItems
    total >= 0m
```
