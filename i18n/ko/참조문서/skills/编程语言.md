## Python Patterns (python-patterns)

### 용도설명
提供Pythonic惯用패턴、PEP 8표준、타입 힌트和모범 사례，用于빌드健壮、高效和可维护的Python应用程序。

### 사용 타이밍
- 编写新的Python코드
- 리뷰Python코드
- 리팩토링现있음Python코드
- 설계Python패키지/모듈

### 코어개념
1. **可读性优先** - 코드해야 함清晰明显
2. **显式优于隐式** - 避免魔法操作
3. **EAFP风格** - 更容易요청原谅而아님许可
4. **타입 힌트** - 사용유형注解提高코드보안
5. **上下文管理器** - 사용`with`진행资源管理
6. **列表推导式** - 简单的데이터转换
7. **生成器** - 惰性求值和大데이터集

### 사용예제

**타입 힌트：**
```python
from typing import Optional, List, Dict, Any

def process_user(
    user_id: str,
    data: Dict[str, Any],
    active: bool = True
) -> Optional[User]:
    if not active:
        return None
    return User(user_id, data)
```

**上下文管理器：**
```python
from contextlib import contextmanager

@contextmanager
def timer(name: str):
    start = time.perf_counter()
    yield
    elapsed = time.perf_counter() - start
    print(f"{name} took {elapsed:.4f} seconds")

with timer("data processing"):
    process_large_dataset()
```

**오류 처리：**
```python
try:
    parsed = json.loads(data)
except json.JSONDecodeError as e:
    raise ValueError(f"Failed to parse data: {data}") from e
```

## Go Patterns (golang-patterns)

### 용도설명
提供惯用的Go패턴、모범 사례和约定，用于빌드健壮、高效和可维护的Go应用程序。

### 사용 타이밍
- 编写新的Go코드
- 리뷰Go코드
- 리팩토링现있음Go코드
- 설계Go패키지/모듈

### 코어개념
1. **简洁和清晰** - Go偏好简洁而아님技巧
2. **零值있음用** - 설계유형使其零值可直接사용
3. **수용인터페이스，돌아가기구조体** - 함수수용인터페이스参数，돌아가기具体유형
4. **오류 처리** - 오류是值，遵循`error`돌아가기值패턴
5. **동시성패턴** - 사용goroutine和channel진행동시성
6. **인터페이스설계** - 小而专注的인터페이스
7. **패키지组织** - 표준프로젝트布局

### 사용예제

**오류패키지装：**
```go
func LoadConfig(path string) (*Config, error) {
    data, err := os.ReadFile(path)
    if err != nil {
        return nil, fmt.Errorf("load config %s: %w", path, err)
    }
    // ...
}
```

**Worker Pool：**
```go
func WorkerPool(jobs <-chan Job, results chan<- Result, numWorkers int) {
    var wg sync.WaitGroup
    for i := 0; i < numWorkers; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            for job := range jobs {
                results <- process(job)
            }
        }()
    }
    wg.Wait()
}
```

**기능选项패턴：**
```go
type Option func(*Server)

func WithTimeout(d time.Duration) Option {
    return func(s *Server) { s.timeout = d }
}

func NewServer(addr string, opts ...Option) *Server {
    s := &Server{addr: addr, timeout: 30 * time.Second}
    for _, opt := range opts {
        opt(s)
    }
    return s
}
```

## Rust Patterns (rust-patterns)

### 용도설명
提供惯用的Rust패턴、소유권시스템、오류 처리、trait、동시성和모범 사례，用于빌드보안、高效的应用程序。

### 사용 타이밍
- 编写新的Rust코드
- 리뷰Rust코드
- 리팩토링现있음Rust코드
- 설계crate구조和모듈布局

### 코어개념
1. **소유권和借用** - 컴파일时防止데이터竞争和메모리오류
2. **Result和?操作符** - 사용`?`진행오류传播
3. **枚举和穷尽匹配** - 사용枚举使不可能的状态불가능表示
4. **Trait和泛型** - 零成本抽象
5. **보안동시성** - 사용`Arc<Mutex<T>>`、channels和async/await
6. **最小pub表面** - 按领域组织모듈

### 사용예제

**오류 처리：**
```rust
use anyhow::{Context, Result};

fn load_config(path: &str) -> Result<Config> {
    let content = std::fs::read_to_string(path)
        .with_context(|| format!("failed to read config from {path}"))?;
    let config: Config = toml::from_str(&content)
        .with_context(|| format!("failed to parse config from {path}"))?;
    Ok(config)
}
```

**사용Cow灵活处理소유권：**
```rust
use std::borrow::Cow;

fn normalize(input: &str) -> Cow<'_, str> {
    if input.contains(' ') {
        Cow::Owned(input.replace(' ', "_"))
    } else {
        Cow::Borrowed(input) // 없음需수정时零成本
    }
}
```

**Arc<Mutex<T>>共享可变状态：**
```rust
use std::sync::{Arc, Mutex};

let counter = Arc::new(Mutex::new(0));
let handles: Vec<_> = (0..10).map(|_| {
    let counter = Arc::clone(&counter);
    std::thread::spawn(move || {
        let mut num = counter.lock().expect("mutex poisoned");
        *num += 1;
    })
}).collect();
```

## Kotlin Patterns (kotlin-patterns)

### 용도설명
提供惯用的Kotlin패턴、모범 사례和约定，用于빌드健壮、高效和可维护的Kotlin应用程序，패키지括코루틴、空보안和DSL빌드器。

### 사용 타이밍
- 编写新的Kotlin코드
- 리뷰Kotlin코드
- 리팩토링现있음Kotlin코드
- 설계Kotlin모듈或库
- 설정Gradle Kotlin DSL빌드

### 코어개념
1. **空보안** - 사용유형시스템和보안调用操作符
2. **기본不可变** - 优先사용`val`而아님`var`
3. **데이터클래스** - 用于主要持있음데이터的유형
4. **密封클래스** - 穷尽유형层次구조
5. **코루틴** - 구조化동시성
6. **확장함수** - 없음需继承추가기능
7. **DSL빌드器** - 사용`@DslMarker`和lambda接收器

### 사용예제

**空보안：**
```kotlin
fun getUserEmail(userId: String): String {
    val user = userRepository.findById(userId)
    return user?.email ?: "unknown@example.com"
}
```

**密封클래스用于穷尽结果：**
```kotlin
sealed class Result<out T> {
    data class Success<T>(val data: T) : Result<T>()
    data class Failure(val error: AppError) : Result<Nothing>()
    data object Loading : Result<Nothing>()
}
```

**구조化동시성：**
```kotlin
suspend fun fetchUserWithPosts(userId: String): UserProfile =
    coroutineScope {
        val user = async { userService.getUser(userId) }
        val posts = async { postService.getUserPosts(userId) }
        UserProfile(user = user.await(), posts = posts.await())
    }
```

## C++ Coding Standards (cpp-coding-standards)

### 용도설명
基于C++코어가이드的现代C++（C++17/20/23）인코딩표준。强制타입 안전성、资源보안、不可变性和清晰度。

### 사용 타이밍
- 编写新的C++코드
- 리뷰或리팩토링C++코드
- 做出아키텍처决策
- 强制一致的코딩 스타일
- 选择언어特性

### 코어개념
1. **RAII없음处不在** - 绑定资源라이프사이클到对象라이프사이클
2. **기본不可变** - 从`const`/`constexpr`시작
3. **타입 안전성** - 사용유형시스템防止컴파일时오류
4. **表达意图** - 名称、유형和개념应传达目的
5. **最小化复杂度** - 简单코드是正确的코드
6. **值语义优先于指针语义** - 优先돌아가기值和作用域对象

### 사용예제

**智能指针사용：**
```cpp
// R.11 + R.20 + R.21: RAII和智能指针
auto widget = std::make_unique<Widget>("config");  // 唯一소유권
auto cache = std::make_shared<Cache>(1024);         // 共享소유권

// R.3: 原始指针是非拥있음的观察者
void render(const Widget* w) {  // 不拥있음w
    if (w) w->draw();
}
```

**Rule of Zero：**
```cpp
// C.20: 让컴파일器生成特殊成员
struct Employee {
    std::string name;
    std::string department;
    int id;
    // 없음需定义析构함수、복사/이동构造함수或赋值运算符
};
```

## Java Coding Standards (java-coding-standards)

### 용도설명
Spring Boot和Quarkus서비스的Java인코딩표준，패키지括命名、不可变性、Optional사용、流处理、异常、泛型和프로젝트布局。

### 사용 타이밍
- 在Spring Boot或Quarkus프로젝트中编写或리뷰Java코드
- 强制命名、不可变性或异常处理约定
- 사용records、密封클래스或패턴 매칭（Java 17+）
- 리뷰Optional、流或泛型的사용

### 코어개념
1. **프레임워크감지** - 根据빌드파일감지Spring Boot或Quarkus
2. **命名约定** - 클래스/Records用PascalCase，메서드/字段用camelCase
3. **不可变性** - 优先사용records和final字段
4. **Optional사용** - 从find*메서드돌아가기Optional
5. **依赖注入** - 构造함수注入优于字段注入
6. **응답式패턴（Quarkus）** - 돌아가기Uni/Multi

### 사용예제

**Spring Boot构造함수注入：**
```java
@Service
public class MarketService {
    private final MarketRepository marketRepository;

    public MarketService(MarketRepository marketRepository) {
        this.marketRepository = marketRepository;
    }
}
```

**Optional사용：**
```java
return market
```

## Swift 6.2 Approachable Concurrency (swift-concurrency-6-2)

### 용도설명
Swift 6.2的"可接近동시성"패턴——기본单스레드，사용`@concurrent`显式后台제거，통과隔离一致性为主actor유형提供프로토콜지원。

### 사용 타이밍
- 将Swift 5.x或6.0/6.1프로젝트마이그레이션到Swift 6.2
- 해결데이터竞争보안컴파일器오류
- 설계MainActor-based应用아키텍처
- 구현프로토콜一致性

### 코어개념
1. **单스레드기본** - 大多数코드是데이터竞争보안的；동시성是显式 opt-in
2. **Async保持在调用actor** - 消除导致데이터竞争오류的隐式제거
3. **隔离一致性** - MainActor유형가능보안地구현非隔离프로토콜
4. **@concurrent显式opt-in** - 后台실행是있음意的성능选择

### 사용예제

**隔离一致性：**
```swift
extension StickerModel: @MainActor Exportable {
    func export() {
        photoProcessor.exportAsPNG()
    }
}
```

**@concurrent用于后台작업：**
```swift
nonisolated final class PhotoProcessor {
    @concurrent
    static func extractSubject(from data: Data) async -> Sticker {
        // 后台실행的昂贵작업
    }
}
```

## Dart/Flutter Patterns (dart-flutter-patterns)

### 용도설명
生产준비 완료的Dart和Flutter패턴，涵盖空보안、不可变状态、비동기组合、컴포넌트아키텍처、流行상태 관리프레임워크（BLoC、Riverpod、Provider）、GoRouter导航、Dio网络요청、Freezed코드生成和清洁아키텍처。

### 사용 타이밍
- 시작新的Flutter기능并필요상태 관리、导航或데이터访问的패턴
- 리뷰或编写Dart코드并필요空보안、密封유형或비동기组合的指导
- 설정新的Flutter프로젝트并选择BLoC、Riverpod或Provider
- 구현보안的HTTP클라이언트、WebView통합或本地存储
- 编写Flutter컴포넌트、Cubit或Riverpod provider的테스트

### 코어개념
1. **空보안** - 避免`!`，优先사용`?.`/`??`/패턴 매칭
2. **不可变状态** - 密封클래스、`freezed`、`copyWith`
3. **비동기组合** - 동시성`Future.wait`、await后보안사용BuildContext
4. **컴포넌트아키텍처** - 提取为클래스（아님메서드）、`const`传播、作用域重建
5. **상태 관리** - BLoC/Cubit事件、Riverpod notifiers和派生providers

### 사용예제

**密封状态클래스：**
```dart
sealed class AsyncState<T> {}
final class Loading<T> extends AsyncState<T> {}
final class Success<T> extends AsyncState<T> { final T data; const Success(this.data); }
final class Failure<T> extends AsyncState<T> { final Object error; const Failure(this.error); }

Widget buildFrom(AsyncState<User> state) => switch (state) {
    UserInitial() => const SizedBox.shrink(),
    UserLoading() => const CircularProgressIndicator(),
    UserLoaded(:final user) => UserCard(user: user),
    UserError(:final message) => ErrorText(message),
};
```

**Riverpod派生provider：**
```dart
@riverpod
double cartTotal(Ref ref) {
    final cart = ref.watch(cartNotifierProvider);
    final products = ref.watch(productsProvider).valueOrNull ?? [];
    return cart.fold(0.0, (total, item) {
        final product = products.firstWhereOrNull((p) => p.id == item.productId);
        return total + (product?.price ?? 0) * item.quantity;
    });
}
```

4. **FsCheck.xUnit** - 통합到xUnit的属性테스트

### 사용예제

**Unquote断言：**
```fsharp

let ``order total sums item prices`` () =
    let items = [ { Sku = "A"; Quantity = 2; Price = 10m }
                  { Sku = "B"; Quantity = 1; Price = 5m } ]
    let total = Order.calculateTotal items
    test <@ total = 25m @>
```

**FsCheck属性테스트：**
