# 보안

본 섹션은 보안 리뷰, 취약점 스캔 및 각 프레임워크의 보안 모범 사례를 다룹니다.

---

## 보안 리뷰

### 용도 설명
보안 리뷰 스킬은 모든 코드가 보안 모범 사례를 준수하고 잠재적 취약점을 식별하도록 보장합니다.

### 사용 타이밍
- 인증 또는 권한 부여 구현 시
- 사용자 입력 또는 파일 업로드 처리 시
- 새 API 엔드포인트 생성 시
- 키 또는 자격 증명 사용 시
- 결제 기능 구현 시
- 민감한 데이터 저장 또는 전송 시
- 서드파티 API 통합 시

### 코어 개념

**1. 키 관리**
소스 코드에 키를 절대 하드코딩하지 마세요.

**2. 입력 검증**
모든 사용자 입력은 반드시 검증해야 합니다.

**3. SQL 인젝션 방어**
파라미터화된 쿼리를 사용하세요.

**4. XSS 방어**
사용자가 제공한 HTML을 정리하세요.

**5. CSRF 보호**
상태 변경 작업에 CSRF 토큰을 사용하세요.

**6. 속도 제한**
모든 API 엔드포인트에서 속도 제한을 활성화하세요.

### 보안 검사 목록

| 체크 항목 | 설명 |
|--------|------|
| 키 | 하드코딩된 키 없음, 모든 키가 환경 변수에 있음 |
| 입력 검증 | 모든 사용자 입력에 스키마 검증 사용 |
| SQL 인젝션 | 모든 쿼리에 파라미터화된 쿼리 사용 |
| XSS | 사용자 콘텐츠가 정리됨 |
| CSRF | 보호 활성화됨 |
| 인증 | 올바른 토큰 처리 |
| 권한 부여 | 역할 검사到位 |
| 속도 제한 | 모든 엔드포인트에 활성화됨 |
| HTTPS | 프로덕션 환경에서 강제 사용 |
| 보안 헤더 | CSP, X-Frame-Options 설정 |
| 오류 처리 | 오류 메시지가 민감한 데이터 노출 안 함 |
| 로그 | 민감한 데이터 기록 안 함 |
| 의존성 | 업데이트, 취약점 없음 |

## 보안 스캔

### 용도 설명
AgentShield를 사용해서 Claude Code 설정 (`.claude/` 디렉토리)의 보안 취약점, 설정 오류 및 인젝션 위험을 스캔합니다.

### 사용 타이밍
- 새 Claude Code 프로젝트 설정 시
- `.claude/settings.json`, `CLAUDE.md` 또는 MCP 설정 수정 후
- 설정 변경을 커밋하기 전
- 기존 Claude Code 설정이 있는 새 저장소 가입 시
- 정기적인 보안 검사 시

### 스캔 내용

| 파일 | 체크 항목 |
|------|--------|
| `CLAUDE.md` | 하드코딩된 키, 자동 실행 명령, 프롬프트 인젝션 패턴 |
| `settings.json` | 너무 관대한 허용 목록, 거절 목록 부족, 위험한 우회 플래그 |
| `mcp.json` | 위험 MCP 서버, 하드코딩된 env 키, npx 공급망 위험 |
| `hooks/` | 삽입을 통한 명령 인젝션, 데이터 유출, 자동 오류 억제 |
| `agents/*.md` | 무제한 도구 액세스, 프롬프트 인젝션 표면, 모델 사양 부족 |

### 사용 예제

```bash
# 기본 스캔
npx ecc-agentshield scan

# 경로 지정 스캔
npx ecc-agentshield scan --path /path/to/.claude

# 최소 심각성 필터링
npx ecc-agentshield scan --min-severity medium

# JSON 형식 (CI/CD용)
npx ecc-agentshield scan --format json

# 자동 수정
npx ecc-agentshield scan --fix

# Opus 깊이 분석 (ANTHROPIC_API_KEY 필요)
export ANTHROPIC_API_KEY=your-key
npx ecc-agentshield scan --opus --stream
```

### 심각성 수준

| 등급 | 점수 | 의미 |
|------|------|------|
| A | 90-100 | 보안 설정 |
| B | 75-89 | 작은 문제 |
| C | 60-74 | 주의 필요 |
| D | 40-59 | 주요 위험 |
| F | 0-39 | 심각한 취약점 |

## Django 보안

### 용도 설명
일반적인 취약점 공격으로부터 애플리케이션을 보호하는 Django 보안 모범 사례입니다.

### 사용 타이밍
- Django 인증 및 권한 부여 설정 시
- 사용자 권한 및 역할 구현 시
- 프로덕션 보안 설정 시
- Django 애플리케이션 보안 문제 리뷰 시
- Django 애플리케이션을 프로덕션 환경에 배포 시

### 코어 보안 설정

```python
# settings/production.py
DEBUG = False  # 핵심: 프로덕션 환경에서는 절대 True 사용 안 함

ALLOWED_HOSTS = os.environ.get('ALLOWED_HOSTS', '').split(',')

# 보안 헤더
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True
SECURE_HSTS_SECONDS = 31536000  # 1년
SECURE_HSTS_INCLUDE_SUBDOMAINS = True
SECURE_HSTS_PRELOAD = True
SECURE_CONTENT_TYPE_NOSNIFF = True
SECURE_BROWSER_XSS_FILTER = True
X_FRAME_OPTIONS = 'DENY'

# Cookie 보안
SESSION_COOKIE_HTTPONLY = True
CSRF_COOKIE_HTTPONLY = True
SESSION_COOKIE_SAMESITE = 'Lax'

# 비밀번호 검증
AUTH_PASSWORD_VALIDATORS = [
    {'NAME': 'django.contrib.auth.password_validation.MinimumLengthValidator', 'OPTIONS': {'min_length': 12}},
    {'NAME': 'django.contrib.auth.password_validation.CommonPasswordValidator'},
]
```

### SQL 인젝션 방어

```python
# 보안: Django ORM이 자동으로 파라미터 이스케이프
def get_user(username):
    return User.objects.get(username=username)  # 보안

# 보안: 파라미터 사용
def search_users(query):
    return User.objects.raw('SELECT * FROM users WHERE username = %s', [query])

# 위험: 절대 문자열Interpolation 직접 사용하지 마세요
def get_user_bad(username):
    return User.objects.raw(f'SELECT * FROM users WHERE username = {username}')  # 취약점!
```

### XSS 방어

```django
{# Django 기본 자동 이스케이프 변수 - 보안 #}
{{ user_input }}  {# HTML이 이스케이프됩니다 #}

{# 안전하게 명시적으로 표시 - 신뢰할 수 있는 콘텐츠만 #}
{{ trusted_html|safe }}  {# 이스케이프 안 함 #}

{# 템플릿 필터로 HTML 안전하게 처리 #}
{{ user_input|striptags }}  {# 모든 HTML 태그 제거 #}
```

### CSRF 보호

```django
{# 템플릿에서 사용 #}
<form method="post">
    {% csrf_token %}
    {{ form.as_p }}
    <button type="submit">커밋</button>
</form>

{# AJAX 요청 #}
function getCookie(name) {
    let cookieValue = null;
    if (document.cookie && document.cookie !== '') {
        const cookies = document.cookie.split(';');
        for (let i = 0; i < cookies.length; i++) {
            const cookie = cookies[i].trim();
            if (cookie.substring(0, name.length + 1) === (name + '=')) {
                cookieValue = decodeURIComponent(cookie.substring(name.length + 1));
                break;
            }
        }
    }
    return cookieValue;
}

fetch('/api/endpoint/', {
    method: 'POST',
    headers: {
        'X-CSRFToken': getCookie('csrftoken'),
        'Content-Type': 'application/json',
    },
    body: JSON.stringify(data)
});
```
