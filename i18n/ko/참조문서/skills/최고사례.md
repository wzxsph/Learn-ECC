## 인코딩표준

### 용도설명
프로젝트 간 적용의베이스라인인코딩 규칙，패키지괄이름、가독성、불변성및코드 품질리뷰。

### 사용 타이밍
- 시작새프로젝트또는모듈
- 리뷰코드 품질및유지보수성
- 리팩토링현재있음코드준수규칙
- 강제이름、형식또는구조일관성
- 설정linting、형식화또는유형검사규칙
- 통합새기여자인코딩 규칙

### 코어원칙

**1. 가독성우선**
- 코드읽기횟수보다작성횟수
- 명확의변수및함수이름
- 우선사용자체문서화코드그러나비주석
- 일관의형식화

**2. KSS원칙**
- 유지단순，불과도할설계
- 피함조기최적화
- 容易理解 > 똑똑함의코드

**3. DRY원칙**
- 提取公共논리까지함수
- 생성可复用컴포넌트
- 크로스모듈共享도구
- 피함복사붙여넣기 프로그래밍

**4. YAGN원칙**
- 불빌드尚未필요의특성
- 피함 speculative generality
- 仅에필요시추가复杂性
- 先단순，必要시리팩토링

  const [debouncedValue, setDebouncedValue] = useState<T>(value)

    const handler = setTimeout(() => {
      setDebouncedValue(value)
    }, delay)
    return () => clearTimeout(handler)
  }, [value, delay])

## 오류 처리

### 용도설명
크로스TypeScript、Python및Go의健壮오류 처리패턴，패키지괄유형화오류、오류경계、재시도、서킷기및사용자 친화좋오류 메시지。

### 사용 타이밍
- 위새모듈또는서비스설계오류유형또는예외계층구조
- 위불可靠의外部의존추가재시도논리또는서킷기
- 리뷰AP엔드점 이否欠오류 처리
- 구현사용자 친화좋오류 메시지및피드백
- 디버깅캐스케이드실패또는자동오류吞그리고

### 코어원칙

1. **快速실패그리고大声** - 에경계处暴露오류
2. **유형화오류优于字符串消息** - 오류는등公民，있음구조
3. **사용자 메시지 ≠ 개발자 메시지** - 向사용자표시友좋文本，서버엔드기록완전컨텍스트
4. **영원히불자동 무시오류** - 각`catch`블록필수처리、재새던지거나기록
5. **오류이AP合约의一일부** - 기록클라이언트가능收까지의각오류코드

### 指数退避재시도

```typescript
interface RetryOptions {
  maxAttempts?: number
  baseDelayMs?: number
  maxDelayMs?: number
  retryf?: (error: unknown) => boolean
}

async function withRetry<T>(fn: () => Promise<T>, options: RetryOptions = {}): Promise<T> {
  const { maxAttempts = 3, baseDelayMs = 500, maxDelayMs = 10_000, retryf = () => true } = options

  let lastError: unknown

  for (let attempt = 1; attempt <= maxAttempts; attempt++) {
    try {
      return await fn()
    } catch (error) {
      lastError = error
      if (attempt === maxAttempts || !retryf(error)) throw error

      const jitter = Math.random() * baseDelayMs
      const delay = Math.min(baseDelayMs * 2 ** (attempt - 1) + jitter, maxDelayMs)
      await new Promise(resolve => setTimeout(resolve, delay))
    }
  }

  throw lastError
}
```

### autonomous-loops

**용도**: 자체루프패턴및아키텍처 - 从단순순서파이프라인까지 RFC 주도의다중프록시 DAG 시스템

**루프패턴계열**:
| 패턴 | 复杂度 | 적용 시나리오 |
|------|--------|----------|
| Sequential Pipeline | 낮 | 日常개발단계、스크립트화워크플로우 |
| NanoClaw REPL | 낮 | 交互式영속세션 |
| nfinite Agentic Loop | 중 | 병렬内容생성、사양주도작업 |
| Continuous Claude PR Loop | 중 | 다중일 반복프로젝트，C 접근 제어 |
| De-Sloppify Pattern | 추가 | 임의 mplementer 단계후의품질정리 |
| Ralphinho / RFC-Driven DAG | 높 | 大型기능、다중单元병렬작업및머지큐 |

**사용 타이밍**:
- 설정자체개발워크플로우
- 적절한 선택의루프아키텍처
- 빌드 C/CD 스타일의持续개발파이프라인
- 실행포함머지协调의병렬프록시

**핵심패턴**:

1. **Sequential Pipeline** - 단순의 `claude -p` 파이프라인
2. **NanoClaw REPL** - 내장영속화루프，session-aware
3. **nfinite Agentic Loop** - 사양주도의병렬생성
4. **Continuous Claude PR Loop** - 프로덕션급 shell 스크립트，자동생성 PR、대기 C、자동머지
5. **De-Sloppify Pattern** - 추가专门의정리/리팩토링단계
6. **Ralphinho** - RFC 주도의다중프록시 DAG 오케스트레이션
