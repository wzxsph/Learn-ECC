# 03 - 기본 명령

## 명령 유형

ECC는 3가지 유형의 명령을 제공：

### 1. 슬래시 명령（Slash Commands）

직접 입력하여 사용：

| 명령 | 기능 | 예 |
|------|------|------|
| `/ecc:plan` | 실시 계획 작성 | `/ecc:plan "사용자 인증 추가"` |
| `/ecc:code-review` | 코드 리뷰 | `/ecc:code-review` |
| `/ecc:build-fix` | 빌드 에러 수리 | `/ecc:build-fix` |
| `/ecc:learn` | 세션에서 패턴 추출 | `/ecc:learn` |
| `/ecc:skill-create` | Git 히스토리에서 Skill 생성 | `/ecc:skill-create` |
| `/ecc:sessions` | 세션 히스토리 관리 | `/ecc:sessions` |

주의: 수동 설치 버전은 접두사 없는 명령을 사용（如`[/plan](../참조문서/commands/01-核心工作流.md)`），플러그인 설치는 `/ecc:` 접두사를 사용。

### 2. Skills（주요 워크플로 면）

Skills는 ECC의 주요 워크플로 호출 방식：

| Skill | 기능 |
|-------|------|
| `[tdd-workflow](../참조문서/skills/测试.md)` | 테스트 주도 개발 |
| `[e2e-testing](../참조문서/skills/测试.md)` | 엔드 투 엔드 테스트 |
| `security-review` | 보안 리뷰 |
| `verification-loop` | 검증 루프 |
| `search-first` | 연구 우선 워크플로 |

호출 방식：
```
[tdd-workflow](../참조문서/skills/测试.md) skill 사용
```

### 3. Agent 명령

Agent를 통해 서브태스크 실행：

| Agent | 용도 |
|-------|------|
| `planner` | 실시 계획 |
| `code-reviewer` | 코드 리뷰 |
| `build-error-resolver` | 빌드 에러 수리 |
| `security-reviewer` | 보안 리뷰 |
| `architect` | 아키텍처 설계 |

호출 방식：
```
@planner
```

## 자주 사용하는 워크플로

### 새 기능 개발

```
/ecc:plan "사용자 인증 기능 추가"
→ planner가 실시 블루프린트 작성
[tdd-workflow](../참조문서/skills/测试.md) skill 사용
→ tdd-guide가 테스트 우선을 강제
/ecc:code-review
→ code-reviewer가 코드 검사
```

### Bug 수리

```
[tdd-workflow](../참조문서/skills/测试.md) skill 사용
→ tdd-guide: 재현 테스트 작성
→ 수리 실시, 테스트 통과 검증
/ecc:code-review
→ code-reviewer: 회귀 검사
```

### 런칭 준비

```
/ecc:security-scan
→ security-reviewer: OWASP Top 10 감사
[e2e-testing](../참조문서/skills/测试.md) skill 사용
→ e2e-runner: 중요 사용자 플로 테스트
/ecc:test-coverage
→ 80%+ 커버리지 검증
```

## MCP 명령

|------|------|
| `/mcp` | MCP 서버 관리 |
| `/ecc:status` | 상태 확인 |
