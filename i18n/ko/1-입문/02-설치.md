# 02 - 설치 설정

## 환경 요건

- Node.js >= 18
- npm / pnpm / yarn / bun 중 하나
- Claude Code CLI v2.1.0+ 설치됨

## 설치 방법

### 방법 1: 플러그인 설치（권장）

#### 1. 플러그인 마켓플레이스 추가

```bash
/plugin marketplace add https://github.com/affaan-m/ECC
```

#### 2. 플러그인 설치

```bash
/plugin install ecc@ecc
```

#### 3. Rules 설치（플러그인은 자동 설치되지 않음）

```bash
# 리포지토리 클론
git clone https://github.com/affaan-m/ECC.git
cd ECC

# 규칙을 사용자 디렉토리에 복사
mkdir -p ~/.claude/rules/ecc
cp -r rules/common ~/.claude/rules/ecc/
# 기술 스택 선택
cp -r rules/typescript ~/.claude/rules/ecc/   # 또는 python, golang, swift, php
```

### 방법 2: 수동 설치

より精细に控制了いたい場合：

```bash
# 리포지토리 클론
git clone https://github.com/affaan-m/ECC.git
cd ECC

# 의존성 설치
npm install        # 또는 pnpm install | yarn install | bun install

# agents 설치
cp agents/*.md ~/.claude/agents/

# skills 설치（주요 워크플로 면）
mkdir -p ~/.claude/skills/ecc
cp -r .agents/skills/* ~/.claude/skills/ecc/

# rules 설치
mkdir -p ~/.claude/rules/ecc
cp -r rules/common ~/.claude/rules/ecc/
cp -r rules/typescript ~/.claude/rules/ecc/

# commands 설치（선택사항, ECC는 skills로移行中）
mkdir -p ~/.claude/commands
cp commands/*.md ~/.claude/commands/
```

## 설치 검증

```bash
# 플러그인이 설치되었는지 확인
/plugin list ecc@ecc

# 사용 가능한 명령 확인
/ecc:plan
/ecc:code-review
```

## 기본 설정

### 패키지 관리자 설정

ECC는 자동으로 선호하는 패키지 관리자를 감지하지만, 수동 설정도 가능：

```bash
# 환경 변수 방식
export CLAUDE_PACKAGE_MANAGER=pnpm

# 또는 명령 사용
/setup-pm
```

우선순위: 환경 변수 > 프로젝트 설정 > package.json > lock 파일 > 전역 설정

### Hook 런타임 제어

```bash
# Hook 엄격도（기본값: standard）
export ECC_HOOK_PROFILE=standard

# 특정 Hooks 비활성화
export ECC_DISABLED_HOOKS="pre:bash:tmux-reminder,post:edit:typecheck"

# SessionStart 컨텍스트 상한（기본값: 8000문자）
export ECC_SESSION_START_MAX_CHARS=4000
```

### 모델 설정

`~/.claude/settings.json`での設定 권장：

```json
{
  "model": "sonnet",
  "env": {
    "MAX_THINKING_TOKENS": "10000",
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "50"
  }
}
```
