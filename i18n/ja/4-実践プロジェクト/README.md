# 第4段階：実践プロジェクト

実際のプロジェクトを通じて所学の知識を巩固します。

## 学習目標

本段階を完了すると、以下ができる：
- ゼロから完全な CLI ツールを構築
- 自動化コードレビュープロセスを実装
- エンドツーエンド自動化ワークフローを設計
- マルチ Agent 協調プラットフォームを構築

## 予想所要時間

10-15時間

## 前提要件

- [第2段階：コア能力](../2-コア能力/README.md) を完了
- 少なくとも1つのスクリプト言語をマスター
- プロジェクト開発経験

## 学習内容

| プロジェクト | 内容 | 難易度 |
|------|------|------|
| [プロジェクト1：CLIツール](./project-1-cli.md) | コマンドラインツールを構築 | 基礎 |
| [プロジェクト2：コードレビュー](./project-2-review.md) | 自動化レビュープロセス | 上級 |
| [プロジェクト3：自動化ワークフロー](./project-3-auto.md) | エンドツーエンド自動化 | 上級 |
| [プロジェクト4：Agentシステム](./project-4-agent.md) | マルチ Agent 協調プラットフォーム | 高級 |

## 提案

感兴趣的项目深入完成，而非蜻蜓点水式地做所有项目。
建议按顺序完成：CLI工具 → 代码审查 → 工作流自动化 → Agent系统（难度递增）

## 次へ

実践プロジェクトを完了すると、[第5段階：クイックリファレンス](../5-クイックリファレンス/README.md)に進んで常用コマンドとパターンを参照します。
