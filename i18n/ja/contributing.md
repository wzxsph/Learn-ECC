# Learn-ECC 寄稿ガイド

Learn-ECCへの寄稿へようこそ！このガイドでは、新しいコンテンツの追加、既存のコンテンツの改善、変更の送信方法について説明します。

## 寄稿の種類

### 1. 新しいコンテンツの追加

寄稿できるコンテンツの種類：
- 新しいチャプターやモジュール
- 新しい演習
- 新しい実践プロジェクト
- 早見表の追加
- 翻訳の改善

### 2. 既存のコンテンツの改善

- エラーの修正
- 説明の明確化
- さらなる例を追加
- 演習の回答を改善
- 時代遅れの情報を更新

### 3. イシューの報告

- タイプミスが文法エラー
- 動作しないコード例
- 壊れたリンク
- 欠落しているコンテンツ

## コンテンツ形式基準

### Markdown形式

すべてのコンテンツはMarkdown形式を使用し、以下のセクションを含む必要があります：

```markdown
# タイトル

## 学習目標

このセクションを完了すると、以下ができるようになります：
- 目標1
- 目標2

---

## メインコンテンツ

### サブタイトル

コンテンツ...

---

## 演習

### 演習1

タスクの説明...

### 検証方法

1. 検証ステップ1
2. 検証ステップ2

---

## 次のステップ

- 次のチャプター: [リンク](./next.md)
- 目次: [テーブルに戻る](../README.md)
```

### ファイル命名

- 日本語命名を使用
- ハイフン `-` で単語を区切る
- チャプターファイルの命名: `01-章名.md`、`02-章名.md`
- 演習ファイルの命名: `演習-名前.md`

### リンク基準

- 参照リンク: `./参考ドキュメント/fileName.md`
- コース内リンク: `./章名.md`
- 相対パス: 現在のファイル的位置から計算

## 変更の送信

### 手順

1. リポジトリをフォーク
2. ブランチを作成: `git checkout -b feature/new-content-description`
3. 変更を加える
4. コミット: `git commit -m "feat: add new content"`
5. リモートにプッシュ: `git push origin feature/new-content-description`
6. Pull Requestを作成

### コミットメッセージ基準

```
<タイプ>: <説明>

<任意のボディ>
```

タイプの種類：
- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメント更新
- `test`: テスト更新
- `refactor`: リファクタリング

## 品質チェック

送信する前に、以下を確認してください：

- [ ] Markdown形式が正しい
- [ ] すべてのリンクが有効
- [ ] コード例が実行可能
- [ ] 演習に検証方法がある
- [ ] 日本語のタイプミスがない
- [ ] 形式基準を満たしている

## イシューのフィードバック

問題を発見したり提案がある場合は、以下を含むイシューを作成してください：
- 問題の説明
- ファイルの場所
- 推奨される改善方法
