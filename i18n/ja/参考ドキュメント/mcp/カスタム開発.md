# 自定義 MCP サーバー開発

## 概要

MCP サーバー基づいて Model Context Protocol 実装，使用拡張 Claude Code 的能力。本ガイド介紹如何開発和統合自定義 MCP サーバー。

## サーバータイプ

### プロセスパターンサーバー

通過本地プロセス起動，适合必要本地実行或訪問本地資源的サービス。

```json
{
  "command": "node",
  "args": ["/path/to/your/server.js"],
  "env": {
    "YOUR_API_KEY": "value"
  }
}
```

### HTTP パターンサーバー

通過 HTTP/HTTPS 訪問远程サーバー，适合微サービスアーキテクチャ或云サービス。

```json
{
  "type": "http",
  "url": "https://your-mcp-server.example.com/mcp",
  "headers": {
    "Authorization": "Bearer YOUR_TOKEN"
  }
}
```

## 開発流程

### 1. 使用 MCP SDK

官方提供します多种言語的 SDK：

```bash
# Node.js
npm install @modelcontextprotocol/sdk

# Python
pip install mcp
```

### 2. 実装サーバー

**Node.js サンプル：**

```javascript
const { Server } = require('@modelcontextprotocol/sdk');
const { StdioServerTransport } = require('@modelcontextprotocol/sdk');

const server = new Server(
  {
    name: "my-custom-server",
    version: "1.0.0"
  },
  {
    capabilities: {
      tools: {},
      resources: {}
    }
  }
);

// 登録ツール
server.setRequestHandler('tools/list', async () => {
  return {
    tools: [
      {
        name: "my_tool",
        description: "実行自定義操作",
        inputSchema: {
          type: "object",
          properties: {
            param: { type: "string" }
          }
        }
      }
    ]
  };
});

// 處理ツール調用
server.setRequestHandler('tools/call', async (request) => {
  const { name, arguments: args } = request.params;

  if (name === "my_tool") {
    // 実行操作
    return { content: [{ type: "text", text: "結果" }] };
  }
});

// 起動サーバー
async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
}

main();
```

**Python サンプル：**

```python
from mcp.server import Server
from mcp.server.stdio import stdio_server

server = Server("my-custom-server")

@server.list_tools()
async def list_tools():
    return [
        Tool(
            name="my_tool",
            description="実行自定義操作",
            inputSchema={
                "type": "object",
                "properties": {
                    "param": {"type": "string"}
                }
            }
        )
    ]

@server.call_tool()
async def call_tool(name, arguments):
    if name == "my_tool":
        # 実行操作
        return [TextContent(type="text", text="結果")]

async def main():
    async with stdio_server() as (read_stream, write_stream):
        await server.run(read_stream, write_stream, server.create_initialization_options())
```

## 設定與統合

### 1. 本地テスト

```json
{
  "mcpServers": {
    "my-custom": {
      "command": "node",
      "args": ["/absolute/path/to/server.js"],
      "env": {
        "DEBUG": "true"
      }
    }
  }
}
```

### 2. 追加到 ECC

將設定追加到 `mcp-configs/mcp-servers.json`：

```json
{
  "mcpServers": {
    "my-custom": {
      "command": "npx",
      "args": ["-y", "/path/to/your/server"],
      "env": {
        "YOUR_API_KEY": "YOUR_KEY_HERE"
      },
      "description": "你的自定義サーバー記述"
    }
  }
}
```

## ベストプラクティス

1. **エラー処理**：始终戻る構造化的エラー応答
2. **输入検証**：検証すべて输入參數，拒否なし效リクエスト
3. **タイムアウト控制**：設定合理的タイムアウト時間
4. **ログ記録**：記録重要操作便於デバッグ
5. **資源クリーンアップ**：確保接続和資源正确释放
6. **バージョン管理**：遵循语义化バージョン

## 常用ツールタイプ

| タイプ | 用途 |
|------|------|
| `tools/list` | 列出可用ツール |
| `tools/call` | 調用ツール |
| `resources/list` | 列出資源 |
| `resources/read` | 読み取り資源 |
| `prompts/list` | 列出提示模板 |
| `prompts/get` | 取得提示 |
