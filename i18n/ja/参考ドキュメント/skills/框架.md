## Django

### 用途説明
Django開発パターン提供了可拡張、可维护的生产级Django应用程序アーキテクチャパターン，パッケージ括REST API設計、ORMベストプラクティス、キャッシュ、信号和ミドルウェア。

### 使用タイミング
- ビルドDjango Web应用程序
- 設計Django REST Framework API
- 使用Django ORM和モデル
- 設定Djangoプロジェクト構造
- 実装キャッシュ、信号、ミドルウェア

### コア概念

**1. プロジェクト構造**
推奨使用`config/`ディレクトリ进行設定分离。

**2. モデル設計**
使用自定义QuerySet和Manager実装可复用查询逻辑。

**3. Django REST Frameworkパターン**
使用シリアライズ器、视图集和ルーティング组织API。

**4. サービス层**
将业务逻辑从视图中分离到サービス层。

**5. キャッシュ戦略**
使用视图级キャッシュ、模板片段キャッシュ和低级キャッシュ。

### 使用サンプル

```python
# モデルベストプラクティス
class Product(models.Model):
    name = models.CharField(max_length=200)
    slug = models.SlugField(unique=True)
    price = models.DecimalField(max_digits=10, decimal_places=2)
    is_active = models.BooleanField(default=True)

    objects = ProductQuerySet.as_manager()

    class Meta:
        indexes = [
            models.Index(fields=['slug']),
            models.Index(fields=['-created_at']),
        ]

# 自定义QuerySet
class ProductQuerySet(models.QuerySet):
    def active(self):
        return self.filter(is_active=True)

    def in_stock(self):
        return self.filter(stock__gt=0)

# サービス层
class OrderService:
    @staticmethod
    @transaction.atomic
    def create_order(user, cart: Cart) -> Order:
        order = Order.objects.create(user=user, total_price=cart.total_price)
        for item in cart.items.all():
            OrderItem.objects.create(order=order, product=item.product, quantity=item.quantity)
        cart.items.all().delete()
        return order
```

## Laravel

### 用途説明
Laravel開発パターン提供了可拡張、可维护的生产级Laravel应用程序アーキテクチャパターン，パッケージ括ルーティング、控制器、サービス层、キュー、事件、キャッシュ和API资源。

### 使用タイミング
- ビルドLaravel Web应用程序或API
- 组织控制器、サービス和领域逻辑
- 使用Eloquentモデル和关系
- 使用资源和分页設計API
- 追加キュー、事件、キャッシュ和后台ジョブ

### コア概念

**1. 控制器 -> サービス -> Actions**
保持控制器薄，将协调逻辑放在サービス中，单一用途逻辑放在actions中。

**2. ルーティングモデル绑定**
使用作用域绑定保持ルーティング可预测。

**3. Eloquentモデル**
使用型ヒント、casts和scopes保持领域逻辑一致。

**4. 表单リクエスト検証**
将検証逻辑放在表单リクエスト中，转换为DTO。

**5. API资源**
使用API资源保持応答一致。

### 使用サンプル

```php
// Action保持单一职责
final class CreateOrderAction
{
    public function __construct(private OrderRepository $orders) {}

    public function handle(CreateOrderData $data): Order
    {
        return $this->orders->create($data);
    }
}

// 控制器使用Action
final class OrdersController extends Controller
{
    public function __construct(private CreateOrderAction $createOrder) {}

    public function store(StoreOrderRequest $request): JsonResponse
    {
        $order = $this->createOrder->handle($request->toDto());
        return response()->json(['success' => true, 'data' => OrderResource::make($order)], 201);
    }
}

// Eloquentモデル
final class Project extends Model
{
    use HasFactory;

    protected $fillable = ['name', 'owner_id', 'status'];
    protected $casts = ['status' => ProjectStatus::class, 'archived_at' => 'datetime'];

    public function scopeActive(Builder $query): Builder
    {
        return $query->whereNull('archived_at');
    }
}
```

## NestJS

### 用途説明
NestJS開発パターン提供了モジュール化TypeScriptバックエンド的生产级パターン，パッケージ括モジュール、控制器、提供者、DTO検証、守卫、拦截器和設定。

### 使用タイミング
- ビルドNestJS API或サービス
- 组织モジュール、控制器和提供者
- 追加DTO検証、守卫、拦截器或异常过滤器
- 設定环境感知設定和データベース統合
- テストNestJS单元或HTTP端点

### コア概念

**1. モジュール化構造**
将领域コード放在機能モジュール中，跨领域关注点放在`common/`中。

**2. DTO検証**
使用`class-validator`検証各リクエストDTO。

**3. 依赖注入**
NestJS使用依赖注入管理依赖。

**4. 守卫和拦截器**
用于認証、認可和ログ記録。

**5. 异常过滤器**
集中处理エラー并保持応答フォーマット一致。

### 使用サンプル

```typescript
// モジュール構造
@Module({
  controllers: [UsersController],
  providers: [UsersService],
  exports: [UsersService],
})
export class UsersModule {}

// DTO検証
export class CreateUserDto {
  @IsEmail()
  email!: string;

  @IsString()
  @Length(2, 80)
  name!: string;

  @IsOptional()
  @IsEnum(UserRole)
  role?: UserRole;
}

// 控制器
@Controller('users')
export class UsersController {
  constructor(private readonly usersService: UsersService) {}

  @Get(':id')
  getById(@Param('id', ParseUUIDPipe) id: string) {
    return this.usersService.getById(id);
  }
}

// 全局検証管道
app.useGlobalPipes(
  new ValidationPipe({
    whitelist: true,
    forbidNonWhitelisted: true,
    transform: true,
  }),
);
```

## FastAPI

### 用途説明
FastAPIパターン提供了生产级非同期APIパターン，パッケージ括依赖注入、Pydanticリクエスト/応答モデル、OpenAPIドキュメント、テスト、セキュリティ和生产准备。

### 使用タイミング
- ビルド或レビューFastAPI应用
- 拆分ルーティング、schema、依赖和データベース访问
- 编写调用データベース或外部サービス的非同期端点
- 追加認証、認可、OpenAPIドキュメント、テスト或デプロイ設定

### コア概念

**1. 应用工厂**
使用工厂パターン使テスト和worker可能使用受控設定ビルド应用。

**2. Pydantic Schema**
保持リクエスト、更新和応答モデル分离。

**3. 依赖注入**
使用依赖注入管理リクエスト作用域资源。

**4. 非同期端点**
非同期ルーティング处理器中进行I/O操作时使用非同期库。

**5. エラー処理**
集中化领域异常，保持応答形状稳定。

### 使用サンプル

```python
# 应用工厂
@asynccontextmanager
async def lifespan(app: FastAPI):
    await init_db()
    yield
    await close_db()

def create_app() -> FastAPI:
    app = FastAPI(title=settings.api_title, version=settings.api_version, lifespan=lifespan)
    app.include_router(users.router, prefix="/api/v1/users", tags=["users"])
    return app

# Pydantic Schema
class UserBase(BaseModel):
    email: EmailStr
    full_name: Annotated[str, Field(min_length=1, max_length=100)]

class UserCreate(UserBase):
    password: Annotated[str, Field(min_length=12, max_length=128)]

class UserResponse(UserBase):
    model_config = ConfigDict(from_attributes=True)
    id: UUID
    created_at: datetime

# 依赖注入
async def get_current_user(
    token: str = Depends(oauth2_scheme),
    db: AsyncSession = Depends(get_db),
) -> User:
    payload = decode_token(token)
    user = await db.get(User, UUID(payload["sub"]))
    if user is None:
        raise HTTPException(status_code=401, detail="Invalid token")
    return user
```

## Spring Boot

### 用途説明
Spring Boot開発パターン提供了可拡張、生产级サービス的アーキテクチャ和APIパターン，パッケージ括REST API設計、分层サービス、データ访问、キャッシュ和非同期处理。

### 使用タイミング
- 使用Spring MVC或WebFluxビルドREST API
- 组织控制器 -> サービス -> 仓库层
- 設定Spring Data JPA、キャッシュ或非同期处理
- 追加検証、异常处理或分页
- 設定dev/staging/production环境設定ファイル

### コア概念

**1. REST API構造**
使用`@RestController`和`@RequestMapping`组织端点。

**2. 仓储パターン**
使用Spring Data JPA定义データ访问インターフェース。

**3. サービス层事务**
使用`@Transactional`管理事务边界。

**4. DTO和検証**
使用記録タイプ和Bean検証注解。

**5. 异常处理**
使用`@ControllerAdvice`集中处理异常。

### 使用サンプル

```java
// REST控制器
@RestController
@RequestMapping("/api/markets")
@Validated
class MarketController {
  private final MarketService marketService;

  @GetMapping
  ResponseEntity<Page<MarketResponse>> list(
      @RequestParam(defaultValue = "0") int page,
      @RequestParam(defaultValue = "20") int size) {
    Page<Market> markets = marketService.list(PageRequest.of(page, size));
    return ResponseEntity.ok(markets.map(MarketResponse::from));
  }

  @PostMapping
  ResponseEntity<MarketResponse> create(@Valid @RequestBody CreateMarketRequest request) {
    Market market = marketService.create(request);
    return ResponseEntity.status(HttpStatus.CREATED).body(MarketResponse.from(market));
  }
}

// サービス层事务
@Service
public class MarketService {
  private final MarketRepository repo;

  @Transactional
  public Market create(CreateMarketRequest request) {
    MarketEntity entity = MarketEntity.from(request);
    return Market.from(repo.save(entity));
  }
}

// DTO和検証
public record CreateMarketRequest(
    @NotBlank @Size(max = 200) String name,
    @NotBlank @Size(max = 2000) String description,
    @NotNull @FutureOrPresent Instant endDate,
    @NotEmpty List<@NotBlank String> categories) {}

// 全局异常处理
@ControllerAdvice
class GlobalExceptionHandler {
  @ExceptionHandler(MethodArgumentNotValidException.class)
  ResponseEntity<ApiError> handleValidation(MethodArgumentNotValidException ex) {
    String message = ex.getBindingResult().getFieldErrors().stream()
        .map(e -> e.getField() + ": " + e.getDefaultMessage())
        .collect(Collectors.joining(", "));
    return ResponseEntity.badRequest().body(ApiError.validation(message));
  }
}
```
