本一部涵盖コンテナ化、 orchestration、インフラ即コード和クラウドネイティブアーキテクチャ。

## コンテナ化

### docker-patterns

**用途**: Docker コンテナベストプラクティス

**コア概念**:
- 多フェーズビルド
- 最小化镜像サイズ
- 健康検査設定
- ログ管理
- 資源限制 (CPU/メモリ)

**サンプル**:
```dockerfile
# 多フェーズビルド
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build

FROM node:18-alpine AS runner
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
CMD ["node", "dist/index.js"]
```

### container-security

**用途**: コンテナセキュリティ

**チェック項目**:
- 最小化基础镜像
- 不使用 root ユーザー実行
- 密鑰管理
- 镜像扫描
- 網路隔离

- Provider 開発
- 状態管理
- モジュール化設計
- 远程バックエンド

**サンプル**:
```hcl

  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.micro"
  subnet_id     = aws_subnet.public.id
```

## 監視とオブザーバビリティ

### prometheus

**用途**: Prometheus 監視

**コア概念**:
- 指標采集
- PromQL 查詢
- アラートルール
- サービス发现

**サンプル**:
```yaml
- alert: HighErrorRate
  expr: rate(http_errors_total[5m]) > 0.05
  for: 5m
  labels:
    severity: critical
```

### observability

**用途**: 可観測性アーキテクチャ
