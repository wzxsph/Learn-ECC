## Python Patterns (python-patterns)

### 用途説明
提供しますPythonic惯用パターン、PEP 8標準、型ヒント和ベストプラクティス，使用ビルド健壮、高效和可維護的Pythonアプリケーション程序。

### 使用タイミング
- 编写新的Pythonコード
- レビューPythonコード
- リファクタリング既存のPythonコード
- 設計Python含む/モジュール

### コア概念
1. **可读性优先** - コードすべき清晰明显
2. **显式优於隐式** - 避免魔法操作
3. **EAFP风格** - 更容易リクエスト原谅而ではない许可
4. **型ヒント** - 使用タイプ注解提高コードセキュリティ
5. **上下文管理器** - 使用`with`进行資源管理
6. **一覧推导式** - 簡單的データ轉換
7. **生成器** - 惰性求值和大データ集

### 使用サンプル

**型ヒント：**
```python
from typing import Optional, List, Dict, Any

def process_user(
    user_id: str,
    data: Dict[str, Any],
    active: bool = True
) -> Optional[User]:
    if not active:
        return None
    return User(user_id, data)
```

**上下文管理器：**
```python
from contextlib import contextmanager

@contextmanager
def timer(name: str):
    start = time.perf_counter()
    yield
    elapsed = time.perf_counter() - start
    print(f"{name} took {elapsed:.4f} seconds")

with timer("data processing"):
    process_large_dataset()
```

**エラー処理：**
```python
try:
    parsed = json.loads(data)
except json.JSONDecodeError as e:
    raise ValueError(f"Failed to parse data: {data}") from e
```

## Go Patterns (golang-patterns)

### 用途説明
提供します惯用的Goパターン、ベストプラクティス和约定，使用ビルド健壮、高效和可維護的Goアプリケーション程序。

### 使用タイミング
- 编写新的Goコード
- レビューGoコード
- リファクタリング既存のGoコード
- 設計Go含む/モジュール

### コア概念
1. **简洁和清晰** - Go偏好简洁而ではない技巧
2. **零值あり用** - 設計タイプ使其零值可直接使用
3. **受け入れインターフェース，戻る構造体** - 関数受け入れインターフェース參數，戻る具體タイプ
4. **エラー処理** - エラーは值，遵循`error`戻る值パターン
5. **並行処理パターン** - 使用goroutine和channel进行並行処理
6. **インターフェース設計** - 小而專注的インターフェース
7. **含む組織** - 標準プロジェクト布局

### 使用サンプル

**エラー含む装：**
```go
func LoadConfig(path string) (*Config, error) {
    data, err := os.ReadFile(path)
    if err != nil {
        return nil, fmt.Errorf("load config %s: %w", path, err)
    }
    // ...
}
```

**Worker Pool：**
```go
func WorkerPool(jobs <-chan Job, results chan<- Result, numWorkers int) {
    var wg sync.WaitGroup
    for i := 0; i < numWorkers; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            for job := range jobs {
                results <- process(job)
            }
        }()
    }
    wg.Wait()
}
```

**機能选项パターン：**
```go
type Option func(*Server)

func WithTimeout(d time.Duration) Option {
    return func(s *Server) { s.timeout = d }
}

func NewServer(addr string, opts ...Option) *Server {
    s := &Server{addr: addr, timeout: 30 * time.Second}
    for _, opt := range opts {
        opt(s)
    }
    return s
}
```

## Rust Patterns (rust-patterns)

### 用途説明
提供します惯用的Rustパターン、すべて権システム、エラー処理、trait、並行処理和ベストプラクティス，使用ビルドセキュリティ、高效的アプリケーション程序。

### 使用タイミング
- 编写新的Rustコード
- レビューRustコード
- リファクタリング既存のRustコード
- 設計crate構造和モジュール布局

### コア概念
1. **すべて権和借用** - コンパイル时防止データ競爭和メモリエラー
2. **Result和?操作符** - 使用`?`进行エラー传播
3. **枚举和穷尽匹配** - 使用枚举使不可能的狀態できない表示
4. **Trait和泛型** - 零成本抽象
5. **セキュリティ並行処理** - 使用`Arc<Mutex<T>>`、channels和async/await
6. **最小pub表面** - 按領域組織モジュール

### 使用サンプル

**エラー処理：**
```rust
use anyhow::{Context, Result};

fn load_config(path: &str) -> Result<Config> {
    let content = std::fs::read_to_string(path)
        .with_context(|| format!("failed to read config from {path}"))?;
    let config: Config = toml::from_str(&content)
        .with_context(|| format!("failed to parse config from {path}"))?;
    Ok(config)
}
```

**使用Cow灵活處理すべて権：**
```rust
use std::borrow::Cow;

fn normalize(input: &str) -> Cow<'_, str> {
    if input.contains(' ') {
        Cow::Owned(input.replace(' ', "_"))
    } else {
        Cow::Borrowed(input) // なし需修改时零成本
    }
}
```

**Arc<Mutex<T>>共享可变狀態：**
```rust
use std::sync::{Arc, Mutex};

let counter = Arc::new(Mutex::new(0));
let handles: Vec<_> = (0..10).map(|_| {
    let counter = Arc::clone(&counter);
    std::thread::spawn(move || {
        let mut num = counter.lock().expect("mutex poisoned");
        *num += 1;
    })
}).collect();
```

## Kotlin Patterns (kotlin-patterns)

### 用途説明
提供します惯用的Kotlinパターン、ベストプラクティス和约定，使用ビルド健壮、高效和可維護的Kotlinアプリケーション程序，含む括コルーチン、空セキュリティ和DSLビルド器。

### 使用タイミング
- 编写新的Kotlinコード
- レビューKotlinコード
- リファクタリング既存のKotlinコード
- 設計Kotlinモジュール或庫
- 設定Gradle Kotlin DSLビルド

### コア概念
1. **空セキュリティ** - 使用タイプシステム和セキュリティ調用操作符
2. **デフォルト不可变** - 优先使用`val`而ではない`var`
3. **データクラス** - 使用主要持ありデータ的タイプ
4. **密封クラス** - 穷尽タイプ层次構造
5. **コルーチン** - 構造化並行処理
6. **拡張関数** - なし需繼承追加機能
7. **DSLビルド器** - 使用`@DslMarker`和lambda受信器

### 使用サンプル

**空セキュリティ：**
```kotlin
fun getUserEmail(userId: String): String {
    val user = userRepository.findById(userId)
    return user?.email ?: "unknown@example.com"
}
```

**密封クラス使用穷尽結果：**
```kotlin
sealed class Result<out T> {
    data class Success<T>(val data: T) : Result<T>()
    data class Failure(val error: AppError) : Result<Nothing>()
    data object Loading : Result<Nothing>()
}
```

**構造化並行処理：**
```kotlin
suspend fun fetchUserWithPosts(userId: String): UserProfile =
    coroutineScope {
        val user = async { userService.getUser(userId) }
        val posts = async { postService.getUserPosts(userId) }
        UserProfile(user = user.await(), posts = posts.await())
    }
```

## C++ Coding Standards (cpp-coding-standards)

### 用途説明
基づいてC++コアガイド的现代C++（C++17/20/23）エンコード標準。强制型セキュリティ、資源セキュリティ、不可变性和清晰度。

### 使用タイミング
- 编写新的C++コード
- レビュー或リファクタリングC++コード
- 做出アーキテクチャ决策
- 强制一致的コーディングスタイル
- 选择言語特性

### コア概念
1. **RAIIなし处不在** - 绑定資源ライフタイム到物件ライフタイム
2. **デフォルト不可变** - 從`const`/`constexpr`開始
3. **型セキュリティ** - 使用タイプシステム防止コンパイル时エラー
4. **表达意圖** - 名前、タイプ和概念应传达目的
5. **最小化複雜度** - 簡單コードは正确的コード
6. **值语义优先於指標语义** - 优先戻る值和作用域物件

### 使用サンプル

**智能指標使用：**
```cpp
// R.11 + R.20 + R.21: RAII和智能指標
auto widget = std::make_unique<Widget>("config");  // 唯一すべて権
auto cache = std::make_shared<Cache>(1024);         // 共享すべて権

// R.3: 原始指標は非拥あり的觀察者
void render(const Widget* w) {  // 不拥ありw
    if (w) w->draw();
}
```

**Rule of Zero：**
```cpp
// C.20: 讓コンパイル器生成特殊成员
struct Employee {
    std::string name;
    std::string department;
    int id;
    // なし需定義析构関数、コピー/移動构造関数或赋值运算符
};
```

## Java Coding Standards (java-coding-standards)

### 用途説明
Spring Boot和Quarkusサービス的Javaエンコード標準，含む括命名、不可变性、Optional使用、流處理、例外、泛型和プロジェクト布局。

### 使用タイミング
- 在Spring Boot或Quarkusプロジェクト中编写或レビューJavaコード
- 强制命名、不可变性或例外處理约定
- 使用records、密封クラス或パターンマッチング（Java 17+）
- レビューOptional、流或泛型的使用

### コア概念
1. **フレームワーク检测** - 根據ビルドファイル检测Spring Boot或Quarkus
2. **命名约定** - クラス/Records用PascalCase，メソッド/欄位用camelCase
3. **不可变性** - 优先使用records和final欄位
4. **Optional使用** - 從find*メソッド戻るOptional
5. **依賴注入** - 构造関数注入优於欄位注入
6. **応答式パターン（Quarkus）** - 戻るUni/Multi

### 使用サンプル

**Spring Boot构造関数注入：**
```java
@Service
public class MarketService {
    private final MarketRepository marketRepository;

    public MarketService(MarketRepository marketRepository) {
        this.marketRepository = marketRepository;
    }
}
```

**Optional使用：**
```java
return market
    .map(MarketResponse::from)
    .orElseThrow(() -> new EntityNotFoundException("Market not found"));
```

## Swift 6.2 Approachable Concurrency (swift-concurrency-6-2)

### 用途説明
Swift 6.2的"可接近並行処理"パターン——デフォルト单スレッド，使用`@concurrent`显式后台アンインストール，通過隔离一致性為主actorタイプ提供しますプロトコルサポート。

### 使用タイミング
- 將Swift 5.x或6.0/6.1プロジェクト移行到Swift 6.2
- 解決データ競爭セキュリティコンパイル器エラー
- 設計MainActor-basedアプリケーションアーキテクチャ
- 実装プロトコル一致性

### コア概念
1. **单スレッドデフォルト** - 大多数コードはデータ競爭セキュリティ的；並行処理は显式 opt-in
2. **Async保持在調用actor** - 消除导致データ競爭エラー的隐式アンインストール
3. **隔离一致性** - MainActorタイプ可能セキュリティ地実装非隔离プロトコル
4. **@concurrent显式opt-in** - 后台実行はあり意的パフォーマンス选择

### 使用サンプル

**隔离一致性：**
```swift
extension StickerModel: @MainActor Exportable {
    func export() {
        photoProcessor.exportAsPNG()
    }
}
```

**@concurrent使用后台作業：**
```swift
nonisolated final class PhotoProcessor {
    @concurrent
    static func extractSubject(from data: Data) async -> Sticker {
        // 后台実行的昂贵作業
    }
}
```

## Dart/Flutter Patterns (dart-flutter-patterns)

### 用途説明
本番準備完了的Dart和Flutterパターン，涵盖空セキュリティ、不可变狀態、非同期組合、コンポーネントアーキテクチャ、流行状態管理フレームワーク（BLoC、Riverpod、Provider）、GoRouter导航、Dio網路リクエスト、Freezedコード生成和清洁アーキテクチャ。

### 使用タイミング
- 起動新的Flutter機能並必要状態管理、导航或データ訪問的パターン
- レビュー或编写Dartコード並必要空セキュリティ、密封タイプ或非同期組合的指导
- 設定新的Flutterプロジェクト並选择BLoC、Riverpod或Provider
- 実装セキュリティ的HTTPクライアント、WebView統合或本地存儲
- 编写Flutterコンポーネント、Cubit或Riverpod provider的テスト

### コア概念
1. **空セキュリティ** - 避免`!`，优先使用`?.`/`??`/パターンマッチング
2. **不可变狀態** - 密封クラス、`freezed`、`copyWith`
3. **非同期組合** - 並行処理`Future.wait`、await后セキュリティ使用BuildContext
4. **コンポーネントアーキテクチャ** - 提取為クラス（ではないメソッド）、`const`传播、作用域重建
5. **状態管理** - BLoC/Cubitイベント、Riverpod notifiers和派生providers

### 使用サンプル

**密封狀態クラス：**
```dart
sealed class AsyncState<T> {}
final class Loading<T> extends AsyncState<T> {}
final class Success<T> extends AsyncState<T> { final T data; const Success(this.data); }
final class Failure<T> extends AsyncState<T> { final Object error; const Failure(this.error); }

Widget buildFrom(AsyncState<User> state) => switch (state) {
    UserInitial() => const SizedBox.shrink(),
    UserLoading() => const CircularProgressIndicator(),
    UserLoaded(:final user) => UserCard(user: user),
    UserError(:final message) => ErrorText(message),
};
```

**Riverpod派生provider：**
```dart
@riverpod
double cartTotal(Ref ref) {
    final cart = ref.watch(cartNotifierProvider);
    final products = ref.watch(productsProvider).valueOrNull ?? [];
    return cart.fold(0.0, (total, item) {
        final product = products.firstWhereOrNull((p) => p.id == item.productId);
        return total + (product?.price ?? 0) * item.quantity;
    });
}
```

使用xUnit、FsUnit、Unquote、FsCheck屬性テスト的F#テストパターン，統合テスト和テスト組織ベストプラクティス。

- 為F#コード编写新テスト
- レビューテスト品質和カバレッジ
- 為F#プロジェクト設定テストインフラ
- デバッグ不穩定或慢速テスト

### コア概念
1. **xUnit** - 標準.NET生态システムテストフレームワーク
2. **FsUnit.xUnit** - xUnit的F#友好断言语法
3. **Unquote** - 使用F#引用的断言庫，失敗消息清晰
4. **FsCheck.xUnit** - 統合到xUnit的屬性テスト

### 使用サンプル

**Unquote断言：**
```fsharp
[<Fact>]
let ``order total sums item prices`` () =
    let items = [ { Sku = "A"; Quantity = 2; Price = 10m }
                  { Sku = "B"; Quantity = 1; Price = 5m } ]
    let total = Order.calculateTotal items
    test <@ total = 25m @>
```

**FsCheck屬性テスト：**
```fsharp

let ``order total is always non-negative`` (items: NonEmptyList<PositiveInt * decimal>) =
    let orderItems = items.Get |> List.map (fun (qty, price) ->
        { Sku = "SKU"; Quantity = qty.Get; Price = abs price })
    let total = Order.calculateTotal orderItems
```
