# セキュリティ

本一部涵盖セキュリティレビュー、漏洞扫描和各フレームワーク的セキュリティベストプラクティス。

---

## セキュリティレビュー

### 用途説明
セキュリティレビュー技能確保すべてコード遵循セキュリティベストプラクティス，识别潜在漏洞。

### 使用タイミング
- 実装身份検証或認可
- 處理ユーザー输入或ファイル上传
- 作成新API端點
- 使用密鑰或凭证
- 実装支付機能
- 存儲或传输敏感データ
- 統合第三方API

### コア概念

**1. 密鑰管理**
永远不在源コード中硬エンコード密鑰。

**2. 输入検証**
すべてユーザー输入must検証。

**3. SQL注入防护**
使用參數化查詢。

**4. XSS防护**
クリーンアップユーザー提供します的HTML。

**5. CSRF保护**
在狀態改变操作上使用CSRF令牌。

**6. 速率限制**
在すべてAPI端點上あり効化速率限制。

### セキュリティ検査清单

| チェック項目 | 記述 |
|--------|------|
| 密鑰 | なし硬エンコード密鑰，すべて密鑰在環境変数中 |
| 输入検証 | すべてユーザー输入使用schema検証 |
| SQL注入 | すべて查詢使用參數化查詢 |
| XSS | ユーザー內容被クリーンアップ |
| CSRF | あり効化保护 |
| 身份検証 | 正确的令牌處理 |
| 認可 | 角色検査到位 |
| 速率限制 | すべて端點あり効化 |
| HTTPS | 本番環境强制使用 |
| セキュリティ头 | CSP、X-Frame-Options設定 |
| エラー処理 | エラーメッセージ不暴露敏感データ |
| ログ | 不記録敏感データ |
| 依賴 | 更新，なし漏洞 |

### 使用サンプル

```typescript
// 密鑰管理 - エラー做法
const apiKey = "sk-proj-xxxxx"  // 绝對不要这样做

// 密鑰管理 - 正确做法
const apiKey = process.env.OPENAI_API_KEY
if (!apiKey) {
  throw new Error('OPENAI_API_KEY not configured')
}

// 输入検証
import { z } from 'zod'

const CreateUserSchema = z.object({
  email: z.string().email(),
  name: z.string().min(1).max(100),
})

// SQL注入防护 - エラー做法
const query = `SELECT * FROM users WHERE email = '${userEmail}'`  // 危險！

// SQL注入防护 - 正确做法
const { data } = await supabase
  .from('users')
  .select('*')
  .eq('email', userEmail)

// XSS防护
import DOMPurify from 'isomorphic-dompurify'

function renderUserContent(html: string) {
  const clean = DOMPurify.sanitize(html, {
    ALLOWED_TAGS: ['b', 'i', 'em', 'strong', 'p'],
    ALLOWED_ATTR: []
  })
  return <div dangerouslySetInnerHTML={{ __html: clean }} />
}
```

## セキュリティ扫描

### 用途説明
使用AgentShield扫描Claude Code設定（`.claude/`ディレクトリ）中的セキュリティ漏洞、設定エラー和注入風險。

### 使用タイミング
- 設定新的Claude Codeプロジェクト
- 修改`.claude/settings.json`、`CLAUDE.md`或MCP設定后
- コミット設定更改前
- 加入已ありClaude Code設定的新仓庫
- 定期セキュリティ検査

### 扫描內容

| ファイル | チェック項目 |
|------|--------|
| `CLAUDE.md` | 硬エンコード密鑰、自動実行指令、提示注入パターン |
| `settings.json` | 过度宽松的許可一覧、缺少拒否一覧、危險绕过标志 |
| `mcp.json` | 風險MCPサーバー、硬エンコードenv密鑰、npx供应链風險 |
| `hooks/` | 通過插值的コマンド注入、データ泄露、静默エラー抑制 |
| `agents/*.md` | 不受限的ツール訪問、提示注入表面、缺少モデル规格 |

### 使用サンプル

```bash
# 基本扫描
npx ecc-agentshield scan

# 指定パス扫描
npx ecc-agentshield scan --path /path/to/.claude

# 最低重大性过滤
npx ecc-agentshield scan --min-severity medium

# JSONフォーマット（使用CI/CD）
npx ecc-agentshield scan --format json

# 自動修正
npx ecc-agentshield scan --fix

# Opus深さ分析（必要ANTHROPIC_API_KEY）
export ANTHROPIC_API_KEY=your-key
npx ecc-agentshield scan --opus --stream
```

### 重大性级别

| 等级 | 分数 | 含义 |
|------|------|------|
| A | 90-100 | セキュリティ設定 |
| B | 75-89 | 小問題 |
| C | 60-74 | 必要关注 |
| D | 40-59 | 重大風險 |
| F | 0-39 | 重大漏洞 |

## Djangoセキュリティ

### 用途説明
Djangoセキュリティベストプラクティス，保护アプリケーション程序免受常见漏洞攻击。

### 使用タイミング
- 設定Django身份検証和認可
- 実装ユーザー權限和角色
- 設定本番セキュリティ設定
- レビューDjangoアプリケーション程序セキュリティ問題
- デプロイDjangoアプリケーション程序到本番環境

### コアセキュリティ設定

```python
# settings/production.py
DEBUG = False  # 重要：本番環境绝不使用True

ALLOWED_HOSTS = os.environ.get('ALLOWED_HOSTS', '').split(',')

# セキュリティ头
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True
SECURE_HSTS_SECONDS = 31536000  # 1年
SECURE_HSTS_INCLUDE_SUBDOMAINS = True
SECURE_HSTS_PRELOAD = True
SECURE_CONTENT_TYPE_NOSNIFF = True
SECURE_BROWSER_XSS_FILTER = True
X_FRAME_OPTIONS = 'DENY'

# Cookieセキュリティ
SESSION_COOKIE_HTTPONLY = True
CSRF_COOKIE_HTTPONLY = True
SESSION_COOKIE_SAMESITE = 'Lax'

# 密碼検証
AUTH_PASSWORD_VALIDATORS = [
    {'NAME': 'django.contrib.auth.password_validation.MinimumLengthValidator', 'OPTIONS': {'min_length': 12}},
    {'NAME': 'django.contrib.auth.password_validation.CommonPasswordValidator'},
]
```

### SQL注入防护

```python
# セキュリティ：Django ORM自動转义參數
def get_user(username):
    return User.objects.get(username=username)  # セキュリティ

# セキュリティ：使用參數的raw()
def search_users(query):
    return User.objects.raw('SELECT * FROM users WHERE username = %s', [query])

# 危險：永远不要直接插值
def get_user_bad(username):
    return User.objects.raw(f'SELECT * FROM users WHERE username = {username}')  # 漏洞！
```

### XSS防护

```django
{# Djangoデフォルト自動转义変数 - セキュリティ #}
{{ user_input }}  {# HTML会被转义 #}

{# 显式标记為セキュリティ仅使用可信內容 #}
{{ trusted_html|safe }}  {# 不转义 #}

{# 使用模板過濾器セキュリティ處理HTML #}
{{ user_input|striptags }}  {# 移除すべてHTMLタグ #}
```

### CSRF保护

```django
{# 模板中使用 #}
<form method="post">
    {% csrf_token %}
    {{ form.as_p }}
    <button type="submit">コミット</button>
</form>

{# AJAXリクエスト #}
function getCookie(name) {
    let cookieValue = null;
    if (document.cookie && document.cookie !== '') {
        const cookies = document.cookie.split(';');
        for (let i = 0; i < cookies.length; i++) {
            const cookie = cookies[i].trim();
            if (cookie.substring(0, name.length + 1) === (name + '=')) {
                cookieValue = decodeURIComponent(cookie.substring(name.length + 1));
                break;
            }
        }
    }
    return cookieValue;
}

fetch('/api/endpoint/', {
    method: 'POST',
    headers: {
        'X-CSRFToken': getCookie('csrftoken'),
        'Content-Type': 'application/json',
    },
    body: JSON.stringify(data)
});
```

### 快速セキュリティ検査清单

| チェック項目 | 記述 |
|--------|------|
| `DEBUG = False` | 本番環境绝不使用DEBUG実行 |
| 仅HTTPS | 强制SSL，セキュリティcookie |
| 强密鑰 | 使用環境変数的SECRET_KEY |
| 密碼検証 | あり効化すべて密碼検証器 |
| CSRF保护 | デフォルトあり効化，不要無効化 |
| XSS防护 | Djangoデフォルト转义，不要對ユーザー输入使用`\|safe` |
| SQL注入 | 使用ORM，永远不要在查詢中拼接字符串 |
| ファイル上传 | 検証ファイルタイプ和サイズ |
| 速率限制 | API端點限流 |
| セキュリティ头 | CSP、X-Frame-Options、HSTS |
| ログ | 記録セキュリティイベント |
| 更新 | 保持Django和依賴更新 |
