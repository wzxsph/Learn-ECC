# パフォーマンス最適化

## モデル选择戦略

**Haiku 4.5**（Sonnet 90% 能力，1/3 成本）：
- 频繁調用的轻量级 agent
- 结對编程和コード生成
- 多 agent システム中的 worker agent

**Sonnet 4.6**（最佳エンコードモデル）：
- 主要開発作業
- 编排多 agent ワークフロー
- 複雜エンコードタスク

**Opus 4.5**（最深さ推論）：
- 複雜アーキテクチャ决策
- 最大推論需求
- 研究和分析タスク

## 上下文視窗管理

避免使用上下文視窗的最後 20%：
- 大规模リファクタリング
- 跨多ファイル的機能実装
- デバッグ複雜交互

低上下文敏感度タスク：
- 单ファイル編集
- 独立ツール作成
- ドキュメント更新
- 簡單 bug 修正

## 拡張思考 + 計画パターン

拡張思考デフォルトあり効化，為内部推論保留最多 31,999 tokens。

控制拡張思考：
- **切换**: Option+T (macOS) / Alt+T (Windows/Linux)
- **設定**: 在 `~/.claude/settings.json` 中設定 `alwaysThinkingEnabled`
- **預算上限**: `export MAX_THINKING_TOKENS=10000`
- **詳細パターン**: Ctrl+O 查看思考输出

必要深さ推論的複雜タスク：
1. 確保拡張思考已あり効化（デフォルト）
2. あり効化**計画パターン**以获得構造化メソッド
3. 使用多轮批评进行彻底分析
4. 使用分角色 sub-agents 获得異なる视角

## ビルド故障排除

如果ビルド失敗：
1. 使用 **build-error-resolver** agent
2. 分析エラーメッセージ
3. 增量修正
4. 每次修正后検証
