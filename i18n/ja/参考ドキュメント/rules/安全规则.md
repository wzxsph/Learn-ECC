# セキュリティルール

## ルール概要

ECC Rules セキュリティルールは强制性的セキュリティ検査体系，旨在确保コード在コミット前符合最高セキュリティ標準。该ルール覆盖了从Secrets管理到MCPサーバーセキュリティ的各个方面，はすべてコード变更must遵守的底线。

## コア要件

### コミット前强制検査清单

在コミット任意コード前，must完了以下セキュリティ検査：

| チェック項目 | 説明 |
|--------|------|
| なし硬エンコードSecrets | API密钥、密码、令牌等不得硬エンコード在源コード中 |
| 用户输入検証 | すべて用户输入must经过検証 |
| SQL注入防护 | 使用参数化查询防止SQL注入 |
| XSS防护 | 对用户输入进行HTML转义 |
| CSRF保护 | 确保CSRF保护已あり効化 |
| 認証認可検証 | 確認身份検証和認可逻辑正确 |
| 限流机制 | すべて端点mustあり速率限制 |
| エラーメッセージセキュリティ | エラーメッセージ不得泄露敏感データ |
