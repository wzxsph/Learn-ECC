# Agentオーケストレーション

## 可用 Agent

位於 `~/.claude/agents/`:

| Agent | 用途 | 使用タイミング |
|-------|------|----------|
| planner | 實施計画 | 複雜機能、リファクタリング |
| architect | システム設計 | アーキテクチャ决策 |
| tdd-guide | テスト驱动開発 | 新機能、bug 修正 |
| code-reviewer | コードレビュー | 编写コード后 |
| security-reviewer | セキュリティ分析 | コミット前 |
| build-error-resolver | 修正ビルドエラー | ビルド失敗时 |
| e2e-runner | E2Eテスト | 重要ユーザーフロー |
| refactor-cleaner | 死コードクリーンアップ | コード維護 |
| doc-updater | ドキュメント更新 | 更新ドキュメント时 |
| rust-reviewer | Rust コードレビュー | Rust プロジェクト |
| harmonyos-app-resolver | HarmonyOS アプリケーション開発 | HarmonyOS/ArkTS プロジェクト |

## 立即使用 Agent

なし需ユーザー提示：
1. 複雜機能リクエスト → 使用 **planner** agent
2. コード刚编写/修改 → 使用 **code-reviewer** agent
3. Bug 修正或新機能 → 使用 **tdd-guide** agent
4. アーキテクチャ决策 → 使用 **architect** agent

## 並列タスク実行

對於独立操作，始终使用並列 Task 実行：

```markdown
# 好：並列実行
起動 3 个 agent 並列：
1. Agent 1: 認証モジュールセキュリティ分析
2. Agent 2: キャッシュシステムパフォーマンスレビュー
3. Agent 3: ツールタイプ検査

# 坏：不必要的順序実行
先 agent 1，然後 agent 2，然後 agent 3
```

## 多视角分析

對於複雜問題，使用分角色 sub-agents：
- 事实レビュー员
- 高级工程师
- セキュリティ专家
- 一致性レビュー员
- 冗余検査员
