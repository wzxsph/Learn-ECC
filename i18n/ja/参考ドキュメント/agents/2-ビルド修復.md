# ビルド修正類 Agent

ビルド修正類 Agent は、さまざまなプログラミング言語のビルドエラー、コンパイルエラー、依存関係の問題の診断と修正を担当します。

## Agent 一覧

| Agent 名 | 用途 | 使用モデル | コアツール |
|------------|------|----------|----------|
| build-error-resolver | TypeScript/汎用ビルドエラー修正 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| go-build-resolver | Go ビルド/コンパイルエラー修正 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| kotlin-build-resolver | Kotlin/Gradle ビルドエラー修正 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| rust-build-resolver | Rust ビルド/借用チェッカーエラー修正 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| cpp-build-resolver | C++/CMake ビルドエラー修正 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| java-build-resolver | Java/Maven/Gradle ビルドエラー修正 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| swift-build-resolver | Swift/Xcode ビルドエラー修正 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| dart-build-resolver | Dart/Flutter ビルドエラー修正 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| django-build-resolver | Django/Python ビルド問題修正 | sonnet | Read, Write, Edit, Bash, Grep, Glob |
| pytorch-build-resolver | PyTorch/CUDA ビルド問題修正 | sonnet | Read, Write, Edit, Bash, Grep, Glob |

## build-error-resolver

### 名前と用途
TypeScript ビルドと型エラー解決専門家。ビルド失敗や型エラーを修正し、ビルドを通ることに焦点を当て、アーキテクチャの変更は行いません。

### 能力説明
- TypeScript エラー解決
- ビルドエラー修正
- 依存関係の問題修正
- 設定エラー解決
- 最小化の diff 修正

### 適用シナリオ
- ビルド失敗時
- TypeScript 型エラー発生時
- モジュール解決の問題時

### 使用ツール一覧
- Read: ファイル内容を読み取り
- Write: 修正を書き込み
- Edit: ファイルを編集
- Bash: tsc, npm build, eslint を実行
- Grep: エラーパターンを検索
- Glob: 関連ファイルを検索

### 他の Agent との連携
- リファクタリングは行わない → refactor-cleaner を使用
- アーキテクチャ変更は行わない → architect を使用
- 新機能追加は行わない → planner を使用
- テスト修正は行わない → tdd-guide を使用
- セキュリティ問題の処理は行わない → security-reviewer を使用

### コア原則
- エラーの修正のみ、リファクタリングはしない
- 変更を最小化
- 修正ごとにビルドが通ることを検証

- 借用チェッカーとライフタイムエラー修正
- トレイト実装不一致解決
- Cargo 依存関係とトレイト問題処理
- cargo clippy 警告修正

### 適用シナリオ
- Rust ビルド失敗時
- 借用チェッカーエラー発生時
- Cargo 依存関係衝突時

### 使用ツール一覧
- Read: ファイル内容を読み取り
- Write: 修正を書き込み
- Edit: ファイルを編集

| エラータイプ | 原因 | 修正メソッド |
|----------|------|----------|

## java-build-resolver

### 名前と用途
Java/Maven/Gradle ビルド、コンパイル、依存関係エラー解決専門家。Spring Boot または Quarkus を自動検出してお適切なフレームワーク固有の修正を適用します。

### 能力説明
- Java コンパイルエラー診断
- Maven と Gradle ビルド設定問題修正
- 依存関係衝突とバージョン不一致解決
- 注釈プロセッサ エラー処理（Lombok、MapStruct、Spring、Quarkus）
- Checkstyle と SpotBugs 違反修正

### 適用シナリオ
- Java ビルド失敗時
- Maven/Gradle 依存関係衝突時
- フレームワーク固有のビルド問題時

### 使用ツール一覧
- Read: ファイル内容を読み取り
- Write: 修正を書き込み
- Edit: ファイルを編集
- Bash: ./mvnw compile, ./gradlew build, dependency:tree を実行
- Grep: エラーパターンを検索
- Glob: 関連ファイルを検索

### 他の Agent との連携
- java-reviewer がコード品質をレビュー
- security-reviewer がセキュリティ関連問題を処理

### フレームワーク検出
プロジェクトのフレームワークを自動検出：
- モジュールに `quarkus` を含む場合 → [QUARKUS] ルールを適用
- モジュールに `spring-boot` を含む場合 → [SPRING] ルールを適用
- 両方を含む場合 → 両方のルールセットを適用

### 主なエラー修正パターン

| エラータイプ | 原因 | 修正メソッド |
|----------|------|----------|
| cannot find symbol | インポート欠落、スペルエラー | インポートまたは依存関係を追加 |
| incompatible types | 型不一致 | 明示的変換を追加 |
| package X does not exist | 依存関係欠落 | pom.xml/build.gradle に追加 |
| No qualifying bean of type X | @Component またはコンポーネント スキャン欠落 | 注釈を追加またはスキャンパスケージを修正 |
