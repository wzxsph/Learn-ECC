コードレビュー担当 Agent は、コード品質、セキュリティ、保守性のレビューを担当します。

## code-reviewer

### 名前と用途
汎用コードレビュー担当。コードの品質、セキュリティ、保守性を能動的にレビューします。すべてのコード変更後に使用する必要があります。

### 能力説明
- コード品質検査（関数サイズ、ネスト深さ、コード重複）
- セキュリティ脆弱性検出（SQL インジェクション、XSS、平文凭证）
- React/Next.js パターン検査
- Node.js/バックエンドパターン検査
- パフォーマンス問題識別
- ベストプラクティス提案

### 適用シナリオ
- コード記述または変更後
- Pull Request レビュー
- 共有ブランチへのマージ前

### 使用ツール一覧
- Read: ファイル内容を読み取り
- Grep: コードパターンを検索
- Glob: ファイルを検索
- Bash: git diff と lint コマンドを実行

### 他の Agent との連携
- セキュリティ問題を検出したら security-reviewer にエスカレーション
- ビルドエラーを検出したら build-error-resolver を使用
- リファクタリングが必要になったら refactor-cleaner を使用

## python-reviewer

### 名前と用途
Python コードレビュー専門家。PEP 8 準拠、Pythonic 慣用句、型ヒント、セキュリティ、パフォーマンスに焦点を当てます。

### 能力説明
- PEP 8 スタイル検査
- Pythonic 慣用句の普及（リスト内包表記、Enum、with 文）
- 型ヒント検証
- SQL インジェクション検出
- コルーチンと非同期パターン検査
- フレームワーク specific 検査（Django、FastAPI、Flask）

### 適用シナリオ
- Python プロジェクト内のすべてのコード変更
- Django/FastAPI/Flask 特定プロジェクト

### 使用ツール一覧
- Read: ファイル内容を読み取り
- Grep: コードパターンを検索
- Glob: Python ファイルを検索
- Bash: mypy, ruff, black, bandit, pytest を実行

### 他の Agent との連携
- security-reviewer とセキュリティ検査ルールを共有
- tdd-guide を使用してテストカバレッジを確保
