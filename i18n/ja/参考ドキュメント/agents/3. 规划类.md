# 計画類 Agent

計画類 Agent 专门负责機能計画、技術アーキテクチャ設計和实施ステップ分解。

## Agent 一覧

| Agent 名 | 用途 | 使用モデル | コアツール |
|------------|------|----------|----------|
| planner | 复杂機能实施計画专家 | opus | Read, Grep, Glob |
| code-architect | 機能アーキテクチャ設計专家 | sonnet | Read, Grep, Glob, Bash |
| pr-test-analyzer | PR テスト覆盖分析 | sonnet | Read, Grep, Glob, Bash |
| type-design-analyzer | タイプ設計分析 | sonnet | Read, Grep, Glob |
| comment-analyzer | コードコメント分析 | sonnet | Read, Grep, Glob |

---

## planner

### 名称と用途
复杂機能实施和リファクタリング的計画专家。在用户リクエスト機能实施、アーキテクチャ变更或复杂リファクタリング时主动使用。

### 能力説明
- 需求分析和详细实施計画作成
- 将复杂機能分解为可管理的ステップ
- 识别依赖关系和潜在风险
- 推奨最优实施順序
- 検討边缘情况和エラー场景

### 適用シナリオ
- 新機能实施計画
- アーキテクチャ变更計画
- 复杂リファクタリング計画
- 技術债务クリーンアップ計画

### 使用ツール一覧
- Read: 読み取り现ありコード和ドキュメント
- Grep: 搜索相关パターン和现あり実装
- Glob: 検索相关ファイル

### 他の Agent との連携
- 生成的計画供 code-architect 进一步细化
- 生成的計画供 tdd-guide 编写テスト
- 生成的計画供 code-reviewer レビュー

### 計画流程

1. **需求分析**
   - 完全理解機能リクエスト
   - 如あり必要提出澄清問題
   - 识别成功標準
   - 列出假设和约束

2. **アーキテクチャレビュー**
   - 分析现ありコード库構造
   - 识别受影响的コンポーネント
   - レビュークラス似実装
   - 検討可复用パターン

3. **ステップ分解**
   - 作成详细ステップ，パッケージ含：
     - 清晰、具体的操作
     - ファイル路径和位置
     - ステップ之间的依赖关系
     - 预估复杂度
     - 潜在风险

4. **实施順序**
   - 按依赖关系排序
   - 将相关变更分组
   - 最小化上下文切换
   - サポート增量テスト

### 計画输出フォーマット

```markdown
# 实施計画: [機能名称]

## 概要
[2-3 句话总结]

## 需求
- [需求 1]
- [需求 2]

## アーキテクチャ变更
- [变更 1: ファイル路径和描述]
- [变更 2: ファイル路径和描述]

## 实施ステップ

### フェーズ 1: [フェーズ名称]
1. **[ステップ名称]** (ファイル: path/to/file.ts)
   - 操作: 要采取的具体操作
   - 原因: 此ステップ的原因
   - 依赖: なし / 必要ステップ X
   - 风险: 低/中/高

2. **[ステップ名称]** (ファイル: path/to/file.ts)
   ...

### フェーズ 2: [フェーズ名称]
...

## テスト戦略
- ユニットテスト: [要テスト的ファイル]
- 統合テスト: [要テスト的流程]
- E2E テスト: [用户旅程要テスト]

## 风险与缓解
- **风险**: [描述]
  - 缓解: [如何处理]

## 成功標準
- [ ] 標準 1
- [ ] 標準 2
```

## code-architect

### 名称と用途
基于对现ありコード库的深入理解設計機能アーキテクチャ。提供具体ファイル、インターフェース、データ流和ビルド順序的实施蓝图。

### 能力説明
- 研究现ありコード组织和命名约定
- 识别已あり的アーキテクチャパターン
- 記録テストパターン和现あり边界
- 理解依赖关系图后再提出新抽象
- 設計适合当前パターン的機能アーキテクチャ

### 適用シナリオ
- 新機能アーキテクチャ設計
- 现あり機能リファクタリング設計
- コード组织最適化

### 使用ツール一覧
- Read: 読み取り现ありコード
- Grep: 搜索パターン和约定
- Glob: 検索相关ファイル
- Bash: 実行診断コマンド

### 他の Agent との連携
- 基于 planner 的計画进行アーキテクチャ設計
- 为コード実装提供蓝图
- 与 build-error-resolver 协作修正ビルド問題

### 設計流程

1. **パターン分析**
   - 研究现ありコード组织和命名约定
   - 识别已あり的アーキテクチャパターン
   - 注意テストパターン和现あり边界
   - 在提出新抽象前理解依赖关系

2. **アーキテクチャ設計**
   - 設計機能以自然契合当前パターン
   - 选择满足需求的最简单アーキテクチャ
   - 避免投机性抽象（除非コード库已使用）

3. **实施蓝图**
   为各重要コンポーネント提供：
   - ファイル路径
   - 目的
   - 重要インターフェース
   - 依赖关系
   - データ流角色

4. **ビルド順序**
   按依赖关系排序实施：
   1. タイプ和インターフェース
   2. コア逻辑
   3. 統合层
   4. UI
   5. テスト
   6. ドキュメント

### 输出フォーマット

```markdown
## アーキテクチャ: [機能名称]

### 設計决策
- 决策 1: [理由]
- 决策 2: [理由]

### 要作成的ファイル
| ファイル | 目的 | 优先级 |
|------|------|--------|
| | | |

### 要修改的ファイル
| ファイル | 变更 | 优先级 |
|------|------|--------|
| | | |

### データ流
[描述]

### ビルド順序
1. ステップ 1
2. ステップ 2
```

## type-design-analyzer

### 名称と用途
分析タイプ設計は否使非法状态难以或不可能表示。

### 能力説明
- 封装评估
- 不变性表达分析
- 不变性あり用性评估
- 强制実行検証

### 適用シナリオ
- タイプシステム設計レビュー
- API 設計レビュー
- 领域建模レビュー

### 使用ツール一覧
- Read: 読み取りタイプ定义
- Grep: 搜索タイプ使用
- Glob: 検索タイプファイル

### 他の Agent との連携
- 与 code-reviewer 协作进行コードレビュー
- 与 code-architect 协作进行アーキテクチャ設計

### 评估標準

1. **封装**
   - 内部细节は否非表示
   - 不変数は否可能从外部违反

2. **不变性表达**
   - タイプは否エンコード业务ルール
   - 不可能状态は否在タイプ级别被阻止

3. **不变性あり用性**
   - 这些不変数は否防止真实 bug
   - は否与领域对齐

4. **强制実行**
   - 不変数は否由タイプシステム强制実行
   - は否あり简单的逃逸舱

### 输出フォーマット

对各レビュー的タイプ：
- タイプ名和位置
- 四个维度的评分
- 整体评估
- 具体改善推奨

## comment-analyzer

### 名称と用途
分析コードコメント的准确性、完整性、保守性和コメント腐坏风险。

### 能力説明
- 事实准确性検証
- 完整性検査
- 长期价值评估
- 误导性元素识别

### 適用シナリオ
- コードレビュー时
- ドキュメント品質评估时
- 识别コメント技術债务时

### 使用ツール一覧
- Read: 読み取りコード和コメント
- Grep: 搜索コメント和コード
- Glob: 検索源ファイル

### 他の Agent との連携
- 与 code-reviewer 协作进行コードレビュー
- 与 doc-updater 协作更新ドキュメント

### 分析フレームワーク

1. **事实准确性**
   - 検証声明与コード对比
   - 検査参数和戻る描述与実装对比
   - 标记过时引用

2. **完整性**
   - 検査复杂逻辑は否あり足够解释
   - 検証重要副作用和边缘用例は否已記録
   - 确保公共 API あり足够コメント

3. **长期价值**
   - 标记只重述コード的コメント
   - 识别会快速腐坏的脆弱コメント
   - 发现 TODO / FIXME / HACK 债务

4. **误导性元素**
   - 与コード矛盾的コメント
   - 对已移除行为的过时引用
   - 过度承诺或描述不足的行为

### 输出フォーマット

按重大性分组的咨询发现：
- Inaccurate（不准确）
- Stale（过时）
- Incomplete（不完整）
- Low-value（低价值）
