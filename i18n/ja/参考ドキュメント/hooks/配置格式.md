# 設定フォーマット

## hooks.json 構造概要

ECC Hooksシステム的設定ファイル采用JSONフォーマット，主要構造如下：

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "hooks": {
    "PreToolUse": [...],
    "PostToolUse": [...],
    "Stop": [...],
    "SessionStart": [...],
    "SessionEnd": [...],
    "PreCompact": [...],
    "PostToolUseFailure": [...]
  }
}
```

## 顶层構造

| 字段 | タイプ | 説明 |
|------|------|------|
| `$schema` | string | JSON Schema URL，用于検証設定 |
| `hooks` | object | パッケージ含すべて钩子タイプ的对象 |

---

## 钩子数组構造

各事件タイプ（PreToolUse、PostToolUse等）パッケージ含一个钩子数组，各钩子对象を持つ以下構造：

```json
{
  "matcher": "Bash|Edit|Write",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/my-hook.js",
      "async": false,
      "timeout": 30
    }
  ],
  "description": "钩子描述",
  "id": "unique-hook-id"
}
```

### matcher 字段

#### タイプ説明
matcher字段用于指定触发钩子的ツールタイプ，サポート单个ツール或多个ツール（用`|`分隔）。

#### 可用值
| matcher值 | 匹配的ツールあり哪些 |
|-----------|-----------------|
| `*` | すべてツール |
| `Bash` | Bashツール |
| `Edit` | Editツール |
| `Write` | Writeツール |
| `Read` | Readツール |
| `MultiEdit` | MultiEditツール |
| `Bash\|Edit\|Write` | Bash、Edit或Write |
| `Edit\|Write\|MultiEdit` | Edit、Write或MultiEdit |

#### 使用サンプル
```json
// 匹配すべてツール
"matcher": "*"

// 仅匹配Bashツール
"matcher": "Bash"

// 匹配多个ツール
"matcher": "Edit|Write|MultiEdit"
```

---

## hooks 数组構造

hooks数组パッケージ含一个或多个钩子コマンド对象：

### type 字段

| type值 | 説明 |
|--------|------|
| `command` | 実行Node.jsコマンド |

### command 字段

要実行的コマンド，通常はNode.jsスクリプト路径。

### async 字段（オプション）

```json
"async": true
```

- `true`: 非同期実行，不阻塞ツール実行
- `false`或省略: 同期実行，阻塞ツール実行

### timeout 字段（オプション）

```json
"timeout": 30
```

最大実行时间（秒）。推奨：
- 同期钩子: <200ms
- 非同期钩子: ≤30秒

## 完整設定サンプル

### PreToolUse 钩子サンプル

```json
{
  "matcher": "Bash",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/pre-bash-dispatcher.js"
    }
  ],
  "description": "Bash预检分发器，用于品質、tmux、プッシュ和GateGuard検査",
  "id": "pre:bash:dispatcher"
}
```

### PostToolUse 钩子サンプル

```json
{
  "matcher": "Edit|Write|MultiEdit",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/run-with-flags.js post:quality-gate scripts/hooks/quality-gate.js standard,strict",
      "async": true,
      "timeout": 30
    }
  ],
  "description": "ファイル編集后実行品質门検査",
  "id": "post:quality-gate"
}
```

### Stop 钩子サンプル

```json
{
  "matcher": "*",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/run-with-flags.js stop:session-end scripts/hooks/session-end.js minimal,standard,strict",
      "async": true,
      "timeout": 10
    }
  ],
  "description": "每次応答后保存セッション状态",
  "id": "stop:session-end"
}
```

### SessionStart 钩子サンプル

```json
{
  "matcher": "*",
  "hooks": [
    {
      "type": "command",
      "command": "node scripts/hooks/session-start-bootstrap.js"
    }
  ],
  "description": "加载先前上下文并检测新セッション的パッケージマネージャー",
  "id": "session:start"
}
```

## ライフタイム事件設定フォーマット

对于SessionStart、SessionEnd、PreCompact等ライフタイム事件，使用不同的設定構造：

```json
{
  "description": "メモリ持久化的ライフタイム钩子定义",
  "events": [
    {
      "event": "SessionStart",
      "id": "session:start",
      "script": "scripts/hooks/session-start-bootstrap.js",
      "purpose": "在セッション開始时加载あり界限的先前上下文并检测プロジェクト状态",
      "blocking": false
    },
    {
      "event": "PreCompact",
      "id": "pre:compact",
      "script": "scripts/hooks/pre-compact.js",
      "purpose": "在上下文圧縮前持久化セッション状态",
      "blocking": false
    }
  ]
}
```

### ライフタイム事件字段

| 字段 | 説明 |
|------|------|
| `event` | 事件タイプ（SessionStart、PreCompact、SessionEnd等） |
| `id` | 唯一标识符 |
| `script` | スクリプト路径 |
| `purpose` | 用途描述 |
| `blocking` | は否阻塞（通常为false） |

## 無効化钩子

### 通過編集設定無効化

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write",
        "hooks": [],
        "description": "覆盖：許可すべて.mdファイル作成"
      }
    ]
  }
}
```

### 通過环境変数無効化

```bash
# 無効化特定钩子（逗号分隔）
export ECC_DISABLED_HOOKS="pre:bash:tmux-reminder,post:edit:typecheck"

# 無効化GateGuard
export ECC_GATEGUARD=off
```

## 钩子ID命名仕様

ECC使用冒号分隔的命名约定：

| 前缀 | 用途 | サンプル |
|------|------|------|
| `pre:` | PreToolUse钩子 | `pre:bash:dispatcher` |
| `post:` | PostToolUse钩子 | `post:quality-gate` |
| `stop:` | Stop钩子 | `stop:session-end` |
| `session:` | セッションライフタイム钩子 | `session:start` |

## run-with-flags.js 封装器

ECC使用`run-with-flags.js`封装器来実行钩子，サポート設定ファイル的実行时门控：

```
node scripts/hooks/run-with-flags.js <hook-id> <script-path> <profiles>
```

### 参数説明
| 参数 | 説明 |
|------|------|
| `hook-id` | 钩子的唯一标识符 |
| `script-path` | 实际要実行的スクリプト路径 |
| `profiles` | 逗号分隔的設定ファイル列表（minimal, standard, strict） |

### 作業原理
1. 読み取り`ECC_HOOK_PROFILE`环境変数（デフォルト: standard）
2. 検査钩子IDは否在`ECC_DISABLED_HOOKS`中
3. 如果許可，则実行スクリプト

## 跨平台路径处理

ECC的hookコマンド使用Node.jsスクリプト実装跨平台行为：

```javascript
const path = require('path');
const homedir = require('os').homedir();

// 跨平台路径解析
const configDir = path.join(homedir, '.claude');
const hookScript = path.join(configDir, 'scripts', 'hooks', 'my-hook.js');
```
