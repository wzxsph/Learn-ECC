# カスタム開発

## 概要

本ガイド介紹如何為ECC Hooksシステム開発自定義钩子，含む括基础構造、ベストプラクティス和exit 0要件。

## 基础構造

### 最小钩子模板

```javascript
// my-custom-hook.js
let data = '';
process.stdin.on('data', chunk => data += chunk);
process.stdin.on('end', () => {
  const input = JSON.parse(data);

  // 訪問ツール情報
  const toolName = input.tool_name;        // "Edit", "Bash", "Write"等
  const toolInput = input.tool_input;      // ツール特定參數
  const toolOutput = input.tool_output;    // 仅在PostToolUse可用

  // 警告（非阻塞）：書き込みstderr
  console.error('[Hook] 警告情報');

  // 阻止（仅PreToolUse）：exit code 2
  // process.exit(2);

  // 始终將原始データ输出到stdout
  console.log(data);
});
```

---

## 終了码要件

### 重要ルール：exit 0 on 非重要エラー

**重要**: すべて钩子must在非重要エラー时exit 0，避免意外阻塞ツール実行。

| 終了码 | 含义 | 使用场景 |
|--------|------|----------|
| 0 | 成功 | 継続実行，或非重要警告 |
| 2 | 阻止 | 仅使用PreToolUse的重要阻塞 |
| その他非零 | エラー | 仅記録ログ，**不要使用** |

### 為什么非零終了码不好？

如果钩子以非零終了码終了（除2外），Claude Code会將整个ツール調用标记為失敗，这可能导致：
- ツール実行被意外中断
- ユーザー体验受损
- 难以デバッグ的問題

### 正确處理エラー

```javascript
// エラーサンプル - 不要这样做
process.stdin.on('end', () => {
  try {
    const input = JSON.parse(data);
    // 處理逻辑
  } catch (e) {
    console.error('[Hook] Error:', e.message);
    process.exit(1);  // エラー：会阻塞ツール
  }
  console.log(data);
});

// 正确サンプル - 这样做
process.stdin.on('end', () => {
  try {
    const input = JSON.parse(data);
    // 處理逻辑
  } catch (e) {
    console.error('[Hook] Error:', e.message);
    // 非重要エラー，exit 0継続実行
    console.log(data);
    return;
  }
  console.log(data);
});
```

## 钩子输入パターン

### HookInput インターフェース

```typescript
interface HookInput {
  tool_name: string;          // ツール名前
  tool_input: {               // ツール输入參數
    command?: string;         // Bash: コマンド
    file_path?: string;        // Edit/Write/Read: ファイルパス
    old_string?: string;       // Edit: 被置換的文本
    new_string?: string;       // Edit: 置換文本
    content?: string;          // Write: ファイル內容
  };
  tool_output?: {             // PostToolUse only
    output?: string;          // コマンド/ツール输出
  };
}
```

### 訪問ツール情報

```javascript
process.stdin.on('end', () => {
  const input = JSON.parse(data);

  // 取得ツール名前
  console.log('Tool:', input.tool_name);

  // 根據ツールタイプ處理
  if (input.tool_name === 'Bash') {
    console.log('Command:', input.tool_input.command);
  }

  if (input.tool_name === 'Edit' || input.tool_name === 'Write') {
    console.log('File:', input.tool_input.file_path);
  }

  // PostToolUse可訪問输出
  if (input.tool_output) {
    console.log('Output:', input.tool_output.output);
  }
});
```

## 非同期钩子

對於不应阻塞主流程的钩子（如后台分析）：

```json
{
  "matcher": "Edit|Write|MultiEdit",
  "hooks": [{
    "type": "command",
    "command": "node my-slow-hook.js",
    "async": true,
    "timeout": 30
  }]
}
```

### 非同期钩子注意事项

- 非同期钩子実行在后台
- できない阻塞ツール実行
- すべき在30秒内完了
- 适使用：ログ、分析、遥测

---

## 阻塞钩子

對於must阻止ツール実行的情况（PreToolUse only）：

```javascript
// 在PreToolUse中阻止
process.exit(2);  // 終了码2表示阻止
```

### 何时使用阻止

- セキュリティ検査失敗（如检测到密鑰）
- 违反硬性戦略
- 可能导致データ丢失的操作

### 何时不要使用阻止

- コーディングスタイル問題（使用警告代替）
- 非重要検査
- 推奨クラス検査

## パフォーマンスベストプラクティス

### 保持快速

- PreToolUse钩子：<200ms
- PostToolUse同期钩子：<1秒
- 非同期钩子：<30秒

### 避免阻塞操作

```javascript
// 不好：同期ファイル読み取り
const content = fs.readFileSync('large-file.txt', 'utf8');

// 好：非同期或遅延載入
fs.readFile('large-file.txt', 'utf8', (err, content) => {
  // 處理
});
```

### キャッシュ結果

```javascript
// キャッシュ昂贵操作
let cachedResult = null;
let cacheTime = 0;
const CACHE_TTL = 60000; // 1分钟

function getCachedResult() {
  const now = Date.now();
  if (!cachedResult || now - cacheTime > CACHE_TTL) {
    cachedResult = expensiveOperation();
    cacheTime = now;
  }
  return cachedResult;
}
```

### 常见問題

| 問題 | 原因 | 解決方案 |
|------|------|----------|
| ツール被意外阻止 | 钩子exit非零（除2外） | 改用exit 0和stderr警告 |
| 中断 | 钩子ない終了 | 確保总は输出到stdout |
| データ损坏 | 输出到stdout非JSON | 仅输出原始data |
| パフォーマンス問題 | 钩子太慢 | 最適化或改為非同期 |

## ベストプラクティス总结

1. **始终exit 0 on 非重要エラー** - 不要用非零終了码（除2外）
2. **使用console.error输出警告** - 不要用console.log
3. **保持快速** - PreToolUse <200ms
4. **总は输出原始data到stdout** - 不要改变データフロー
5. **使用非同期處理长時間操作** - 設定async: true
6. **跨平台パス處理** - 使用path.join
7. **テスト输出フォーマット** - 確保stdoutは原始JSON
8. **文書化你的钩子** - 追加清晰的description
