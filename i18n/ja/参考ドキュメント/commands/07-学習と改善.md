# 学習と改善コマンド

## 概要

学習と改善コマンド使用從セッション中提取パターン、トレース instinct 和管理組織知識。

## コマンド一覧

### /learn

**用途**: 從當前セッション中提取可复用パターン並儲存為候选 skills

**記述**: 分析當前セッション，提取任意值得儲存為 skills 的パターン。実行 `/learn` 可在セッション期间任意时候使用。

**提取內容タイプ**:

| タイプ | 記述 |
|---|---|
| **エラー解決方案パターン** | 什么エラー？根本原因は什么？什么修正了？クラス似エラー可复用吗？ |
| **デバッグ技術** | 非显而易见的デバッグステップ、あり效的ツール組合、診断パターン |
| **变通メソッド** | 庫怪癖、API 限制、特定バージョン修正 |
| **プロジェクト特定パターン** | 发现的コードベース约定、アーキテクチャ决策、統合パターン |

**输出フォーマット**: 儲存到 `~/.claude/skills/learned/[pattern-name].md`

```markdown
# [Descriptive Pattern Name]

**Extracted:** [Date]
**Context:** [Brief description of when this applies]

## Problem
[What problem this solves - be specific]

## Solution
[The pattern/technique/workaround]

## Example
[Code example if applicable]

## When to Use
[Trigger conditions - what should activate this skill]
```

**ワークフロー**:
1. レビューセッション中可提取的パターン
2. 识别最あり价值/可复用的见解
3. 起草 skill ファイル
4. リクエストユーザー確認后再儲存
5. 儲存到 `~/.claude/skills/learned/`

**ベストプラクティス**:
- 不要提取trivial修正（拼写エラー、簡單语法エラー）
- 不要提取一次性問題（特定 API 中断等）
- 关注未来セッション可节省時間的パターン
- 保持 skills 專注 - 1パターン1 skill

### /learn-eval

**記述**: 在提取パターン的同时进行品質評估。

### /evolve

**用途**: 分析 instinct + 推奨进化構造

**記述**: 分析已学習的 instinct，提供します構造演进推奨。

---

### /promote

**用途**: 將プロジェクト instinct 提升到全局范围

**記述**: 將プロジェクト特定的 instinct 提升為全局可用的知識。

---

### /instinct-status

**用途**: 表示すべて学習的 instinct

**記述**: 表示當前すべて instinct 的狀態和置信度。

---

### /instinct-export

**用途**: エクスポート instinct 到ファイル

**記述**: 將 instinct エクスポート為可分享的ファイルフォーマット。

---

### /instinct-import

**用途**: 從ファイル/URL インポート instinct

**記述**: 從ファイル或 URL インポート instinct 到システム中。

---

### /skill-create

**用途**: 分析 git 历史+生成 skill ファイル

**記述**: 分析プロジェクト git 历史，提取可复用的パターン生成 skill ファイル。

**ワークフロー**:
1. 分析 git コミット历史
2. 识别重复パターン
3. 生成 skill ファイル
4. 検証並儲存

---

### /skill-health

**用途**: Skill 組合健康仪表板

**記述**: 表示 skill 組合的健康狀態和使用統計。

---

### /rules-distill

**用途**: 扫描 skills + 提取跨領域原則

**記述**: 扫描すべて skills，提取跨領域的通用原則。
