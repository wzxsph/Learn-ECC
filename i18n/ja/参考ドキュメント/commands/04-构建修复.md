# ビルド修正コマンド

本ドキュメント介绍 ECC 中用于修正各言語ビルドエラー的专门コマンド。

---

## /go-build

**用途説明**: 逐步修正 Go ビルドエラー、go vet 警告和 linter 問題。

**使用メソッド**:
```
/go-build
```

**使用场景**:
- `go build ./...` 失敗
- `go vet ./...` レポート問題
- `golangci-lint run` 表示警告
- モジュール依赖损坏
- プル更改后ビルド失敗

**ワークフロー**:
1. **実行診断** - 実行 `go build`、`go vet`、`staticcheck`
2. **解析エラー** - 按ファイル分组并按重大程度排序
3. **逐步修正** - 一次一个エラー
4. **検証修正** - 每次更改后重新実行ビルド
5. **レポート总结** - 表示已修正和剩余問題

**常见エラー修正**:

| エラー | 典型修正 |
|------|----------|
| `undefined: X` | 追加インポート或修正拼写 |
| `cannot use X as Y` | タイプ转换或修正赋值 |
| `missing return` | 追加 return 语句 |
| `X does not implement Y` | 追加缺失メソッド |
| `import cycle` | リファクタリングパッケージ構造 |
| `declared but not used` | 移除或使用変数 |
| `cannot find package` | `go get` 或 `go mod tidy` |

**診断コマンド**:
```bash
go build ./...                # 主ビルド検査
go vet ./...                  # 静态分析
staticcheck ./...             # 高级 lint（如果可用）
golangci-lint run             # Linting
go mod verify                # モジュール検証
go mod tidy -v               # 整理依赖
```

## /kotlin-build

**用途説明**: 逐步修正 Kotlin/Gradle ビルドエラー、コンパイル器警告和依赖問題。

**使用メソッド**:
```
/kotlin-build
```

**使用场景**:
- `./gradlew build` 失敗
- Kotlin コンパイル器レポートエラー
- `./gradlew detekt` レポート违规
- Gradle 依赖解析失敗
- プル更改后ビルド失敗

**常见エラー修正**:

| エラー | 典型修正 |
|------|----------|
| `Unresolved reference: X` | 追加インポート或依赖 |
| `Type mismatch` | 修正タイプ转换或赋值 |
| `'when' must be exhaustive` | 追加缺失的密封クラスブランチ |
| `Suspend function can only be called from coroutine` | 追加 `suspend` 修饰符 |
| `Smart cast impossible` | 使用局部 `val` 或 `let` |
| `Could not resolve dependency` | 修正バージョン或追加仓库 |

**診断コマンド**:
```bash
./gradlew build 2>&1                      # 主ビルド検査
./gradlew detekt 2>&1                      # 静态分析（如果設定）
./gradlew ktlintCheck 2>&1                # フォーマット検査（如果設定）
./gradlew dependencies --configuration runtimeClasspath | head -100  # 依赖問題
./gradlew build --refresh-dependencies     # 深さ更新（キャッシュ或依赖元データ可疑时）
```

## /rust-build

**用途説明**: 逐步修正 Rust ビルドエラー、借用検査器問題和依赖問題。

**使用メソッド**:
```
/rust-build
```

**使用场景**:
- `cargo build` 或 `cargo check` 失敗
- `cargo clippy` レポート警告
- 借用検査器或ライフタイムエラー阻止コンパイル
- Cargo 依赖解析失敗
- プル更改后ビルド失敗

**常见エラー修正**:

| エラー | 典型修正 |
|------|----------|
| `cannot borrow as mutable` | リファクタリング以在使用可变访问之前終了不可变借用 |
| `does not live long enough` | 使用拥ありタイプ或追加ライフタイム标注 |
| `cannot move out of` | リファクタリング以取得すべて権；仅作为最后手段克隆 |
| `mismatched types` | 追加 `.into()`、`as` 或显式转换 |
| `trait X not implemented` | 追加 `#[derive(Trait)]` 或手动実装 |
| `unresolved import` | 追加到 Cargo.toml 或修正 `use` 路径 |
| `cannot find value` | 追加インポート或修正路径 |

**診断コマンド**:
```bash
cargo check 2>&1                               # 主ビルド検査
cargo clippy -- -D warnings 2>&1               # Lints
cargo fmt --check 2>&1                         # フォーマット検査
cargo tree --duplicates                        # 重复依赖
cargo audit                                   # セキュリティ审计（如果可用）
```

## /cpp-build

**用途説明**: 逐步修正 C++ ビルドエラー、CMake 問題和链接器問題。

**使用メソッド**:
```
/cpp-build
```

**使用场景**:
- `cmake --build build` 失敗
- 链接器エラー（未定义引用、多次定义）
- 模板インスタンス化失敗
- パッケージ含/依赖問題
- プル更改后ビルド失敗

**常见エラー修正**:

| エラー | 典型修正 |
|------|----------|
| `undeclared identifier` | 追加 `#include` 或修正拼写 |
| `no matching function` | 修正参数タイプ或追加重载 |
| `undefined reference` | 链接库或追加実装 |
| `multiple definition` | 使用 `inline` 或移到 .cpp |
| `incomplete type` | 用 `#include` 置換前向声明 |
| `no member named X` | 修正成员名称或追加 include |
| `cannot convert X to Y` | 追加适当转换 |
| `CMake Error` | 修正 CMakeLists.txt 設定 |

**診断コマンド**:
```bash
cmake -B build -S .                        # CMake 設定
cmake --build build 2>&1 | head -100       # ビルド
clang-tidy src/*.cpp -- -std=c++17         # 静态分析（如果可用）
cppcheck --enable=all src/                 # 额外分析（如果可用）
```

## /gradle-build

**用途説明**: 修正 Android 和 Kotlin Multiplatform (KMP) プロジェクト的 Gradle ビルドエラー。

**使用メソッド**:
```
/gradle-build
```

**使用场景**:
- Android プロジェクトビルド失敗
- KMP プロジェクトコンパイルエラー
- Gradle 同期失敗
- 依赖冲突

**プロジェクトタイプ检测**:

| 指示器 | ビルドコマンド |
|--------|----------|
| `build.gradle.kts` + `composeApp/` (KMP) | `./gradlew composeApp:compileKotlinMetadata` |
| `build.gradle.kts` + `app/` (Android) | `./gradlew app:compileDebugKotlin` |
| `settings.gradle.kts` 带モジュール | `./gradlew assemble` |
| 設定了 detekt | `./gradlew detekt` |

**常见エラー修正**:

| エラー | 修正 |
|------|------|
| `commonMain` 中未解析的引用 | 検査依赖は否在 `commonMain.dependencies {}` 中 |
| Expect 声明なし actual | 在各平台源集中追加 `actual` 実装 |
| Compose コンパイル器バージョン不匹配 | 在 `libs.versions.toml` 中对齐 Kotlin 和 Compose コンパイル器バージョン |
| 重复クラス | 用 `./gradlew dependencies` 検査冲突依赖 |
| KSP エラー | 実行 `./gradlew kspCommonMainKotlinMetadata` 重新生成 |
| 設定キャッシュ問題 | 検査非可シリアライズ的タスク输入 |

## /flutter-build

**用途説明**: 逐步修正 Dart 分析器エラー和 Flutter ビルド失敗。

**使用メソッド**:
```
/flutter-build
```

**使用场景**:
- `flutter analyze` レポートエラー
- `flutter build` 任意平台失敗
- `flutter pub get` バージョン冲突
- `build_runner` 生成コード失敗
- プル更改后ビルド失敗

**常见エラー修正**:

| エラー | 典型修正 |
|------|----------|
| `A value of type 'X?' can't be assigned to 'X'` | 追加 `?? default` 或空值保护 |
| `The name 'X' isn't defined` | 追加インポート或修正拼写 |
| `Non-nullable instance field must be initialized` | 追加初期化器或 `late` |
| `Version solving failed` | 调整 pubspec.yaml 中的バージョン约束 |
| `Missing concrete implementation of 'X'` | 実装缺失的インターフェースメソッド |
| `build_runner: Part of X expected` | 削除过时的 `.g.dart` 并重新ビルド |

**診断コマンド**:
```bash
flutter analyze 2>&1                  # 分析
flutter pub get 2>&1                  # 依赖
dart run build_runner build --delete-conflicting-outputs 2>&1  # コード生成（如果使用 build_runner）
flutter build apk 2>&1                # 平台ビルド
flutter build web 2>&1                # Web ビルド
```

## ビルド修正コマンド对比表

| コマンド | 言語/平台 | 主要ツール | 常见問題 |
|------|----------|----------|----------|
| `/go-build` | Go | go build, go vet | タイプエラー、インポートループ |
| `/kotlin-build` | Kotlin | Gradle, detekt | タイプ不匹配、when 非穷举 |
| `/rust-build` | Rust | cargo check, clippy | 借用エラー、ライフタイム |
| `/cpp-build` | C++ | CMake, clang-tidy | 链接器エラー、模板問題 |
| `/gradle-build` | Android/KMP | Gradle | 依赖冲突、設定エラー |
| `/flutter-build` | Flutter/Dart | Flutter analyze | 空セキュリティ、分析エラー |

## 通用修正戦略

すべてビルド修正コマンド遵循相同的基本戦略：

1. **先修正ビルドエラー** - コードmustコンパイル
2. **其次修正 lint 警告** - 修正可疑构造
3. **然后修正フォーマット警告** - 样式和ベストプラクティス
4. **一次修正一个** - 検証每次更改
5. **最小更改** - 不要リファクタリング，只は修正

**停止条件**:
如果发生以下情况，プロキシ将停止并レポート：
- 相同エラー在 3 次尝试后仍然存在
- 修正引入了更多エラー
- 必要アーキテクチャ更改
- 缺少外部依赖
