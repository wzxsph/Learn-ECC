# 計画とアーキテクチャコマンド

## 概要

計画とアーキテクチャコマンド用于产品計画、機能設計和システムアーキテクチャ决策。这些コマンド确保在写コード之前あり清晰的計画，减少返工和提高コード品質。

## コマンド列表

### /plan

**用途**: 通用实施計画 - 复述需求、评估风险、作成分ステップ实施計画

**描述**: 在触碰任意コード之前，等待用户確認后作成全面的实施計画。受け入れ自由形式需求或 PRD markdown ファイル。

**输入パターン**:

| 输入 | パターン | 行为 |
|---|---|---|
| `path/to/name.prd.md` | PRD artifact パターン | 読み取り PRD，选择下一个待交付里程碑，生成 `.claude/plans/{name}.plan.md` |
| その他 markdown 路径 | 引用パターン | 読み取りファイル作为上下文，产生内联計画 |
| 自由形式文本 | 对话パターン | 产生内联計画 |
| 空输入 | 澄清パターン | 询问要計画什么 |

**重要参数**:
- `$ARGUMENTS`: `[feature description | path/to/*.prd.md]` - 機能描述或 PRD ファイル路径

**ワークフロー**:
1. **复述需求** - 澄清必要ビルド什么
2. **识别风险** - 列出潜在問題和阻碍
3. **作成ステップ計画** - 分解为フェーズ性实施
4. **等待確認** - must收到用户承認后才能継続

**输出フォーマット** (PRD artifact パターン):
```markdown
# Plan: {Feature Name}

**Source PRD**: {path}
**Selected Milestone**: {milestone or phase name}
**Complexity**: {Small | Medium | Large}

## Summary
{2-3 sentences}

## Patterns to Mirror
| Category | Source | Pattern |
|---|---|---|
| Naming | `path:line` | {short description} |
| Errors | `path:line` | {short description} |
| Tests | `path:line` | {short description} |

## Files to Change
| File | Action | Why |
|---|---|---|
| `path` | CREATE / UPDATE / DELETE | {reason} |

## Tasks
### Task 1: {name}
- **Action**: {what to do}
- **Mirror**: {pattern to follow}
- **Validate**: {command that proves correctness}

## Validation
```bash
{project-specific validation commands}
```

## Risks
| Risk | Likelihood | Mitigation |
|---|---|---|
```

**ベストプラクティス**:
- 使用前先研究コード库中的现ありパターン
- 在 PRD artifact パターン中作成 `.claude/plans/` ディレクトリ（如不存在）
- 如果 PRD パッケージ含 `Delivery Milestones` 表格，只更新所选行状态

**常见陷阱**:
- 空输入时不询问澄清就尝试計画
- 不等待用户確認就開始写コード
- 跳过 Pattern Grounding ステップ

**与その他のコマンド統合**:
- 計画后使用 `tdd-workflow` skill 进行テスト驱动開発
- 使用 `/build-fix` 修正ビルドエラー
- 使用 `/code-review` レビュー完了実装
- 使用 `/pr` 或 `/prp-pr` 開く Pull Request

### /plan-prd

**用途**: 交互式 PRD 生成器

**描述**: 生成以問題为中心的产品需求ドキュメント，パッケージ含問題定义、用户画像、成功指標和 MVP 范围。

**使用场景**: 
- 确定要ビルド的機能
- 必要明确产品需求
- 从零開始新機能計画

**ワークフロー**:
1. FRAME - 了解用户和問題
2. GROUND - 收集证据
3. DECIDE - 确定范围和假设
4. GENERATE - 生成 PRD ファイル

---

### /prp-plan

**用途**: 全面的機能計画

**描述**: 完整的プロジェクト計画，涵盖コード库分析、风险评估和分ステップ实施計画。

**使用场景**:
- 复杂機能的详细計画
- 必要コード库上下文分析
- 多フェーズ实施プロジェクト

---

### /prp-prd

**用途**: PRP ワークフロー PRD 生成器

**描述**: 使用 PRP（Plan-Record-Produce）ワークフロー生成产品需求ドキュメント。

---

### /prp-implement

**用途**: 実行 PRP 計画+検証ループ

**描述**: 按照 PRP ワークフロー実行計画，パッケージ含持续的検証和検査。

---

### /prp-pr

**用途**: 从 PRP ワークフロー作成 PR

**描述**: 从 PRP ワークフロー的结果作成 GitHub Pull Request。

---

### /prp-commit

**用途**: PRP 検証コミット

**描述**: 在 PRP ワークフロー中実行検証并コミットコード。

---

### /multi-plan

**用途**: 多モデル协作計画

**描述**: 结合 Codex 和 Gemini 的双モデル分析，生成综合实施計画。

**使用场景**:
- 必要多角度分析
- 复杂跨领域プロジェクト
- 必要平衡フロントエンド和バックエンド考量
