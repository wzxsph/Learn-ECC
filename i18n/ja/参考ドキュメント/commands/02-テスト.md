# テスト関連コマンド

本ドキュメント介紹 ECC 中使用テスト的專門コマンド，サポート多种プログラミング言語和テストフレームワーク。

---

## /go-test

**用途説明**: Go 言語的 TDD ワークフロー。使用表驱动テスト，先写テスト再実装，検証 80%+ カバレッジ。

**使用メソッド**:
```
/go-test [機能記述]
```

**使用场景**:
- 実装新的 Go 関数
- 為既存の Go コード追加テスト覆寫
- 修正 bug（先写失敗的テスト）
- 学習 Go TDD メソッド论

**サンプルワークフロー**:
```go
// Step 1: 定義インターフェース
func ValidateEmail(email string) error {
    panic("not implemented")
}

// Step 2: 写表驱动テスト (RED)
tests := []struct {
    name    string
    email   string
    wantErr bool
}{
    {"simple email", "user@example.com", false},
    {"empty string", "", true},
    {"no at sign", "userexample.com", true},
}

// Step 3: 実行テスト - 検証 FAIL

// Step 4: 実装最小コード (GREEN)
func ValidateEmail(email string) error {
    if email == "" {
        return ErrEmailEmpty
    }
    if !emailRegex.MatchString(email) {
        return ErrEmailInvalid
    }
    return nil
}

// Step 5: 検証 PASS

// Step 6: 検査カバレッジ
go test -cover ./...
// coverage: 100.0%
```

**カバレッジコマンド**:
```bash
go test -cover ./...                    # 基础カバレッジ
go test -coverprofile=coverage.out      # 生成カバレッジファイル
go tool cover -html=coverage.out        # 浏览器查看
go test -race -cover ./...              # 竞态检测
```

**使用メソッド**:
```
/kotlin-test [機能記述]
```

**使用场景**:
- 実装新的 Kotlin 関数或クラス
- 為 Android/KMP プロジェクト追加テスト覆寫
- 修正 bug（先写失敗的テスト）
- 学習 Kotlin TDD メソッド论

**Kotest テスト风格**:
```kotlin
// StringSpec (最簡單)
class CalculatorTest : StringSpec({
    "add two positive numbers" {
        Calculator.add(2, 3) shouldBe 5
    }
})

// FunSpec (標準ユニットテスト)
class RegistrationValidatorTest : FunSpec({
    test("valid registration returns Valid") {
        val request = RegistrationRequest(
            name = "Alice",
            email = "alice@example.com",
            password = "SecureP@ss1"
        )
        val result = validateRegistration(request)
        result.shouldBeInstanceOf<ValidationResult.Valid>()
    }
})

// BehaviorSpec (BDD 风格)
class OrderServiceTest : BehaviorSpec({
    Given("a valid order") {
        When("placed") {
            Then("should be confirmed") { /* ... */ }
        }
    }
})
```

**コルーチンテスト**:
```kotlin
test("concurrent fetch completes") {
    runTest {
        val result = service.fetchAll()
        result.shouldNotBeEmpty()
    }
}
```

**カバレッジコマンド**:
```bash
./gradlew koverHtmlReport     # HTML レポート
./gradlew koverVerify         # 検証カバレッジしきい値
./gradlew koverXmlReport      # CI 用 XML レポート
./gradlew test                # 実行テスト
```

## /rust-test

**用途説明**: Rust 言語的 TDD ワークフロー。使用 `#[test]`、rstest、proptest 和 mockall，先写テスト再実装，検証 80%+ カバレッジ（cargo-llvm-cov）。

**使用メソッド**:
```
/rust-test [機能記述]
```

**使用场景**:
- 実装新的 Rust 関数、メソッド或 trait
- 為 Rust プロジェクト追加テスト覆寫
- 修正 bug（先写失敗的テスト）
- 学習 Rust TDD メソッド论

**テストパターン**:
```rust
// ユニットテスト
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn adds_two_numbers() {
        assert_eq!(add(2, 3), 5);
    }

    #[test]
    fn handles_error() -> Result<(), Box<dyn std::error::Error>> {
        let result = parse_config(r#"port = 8080"#)?;
        assert_eq!(result.port, 8080);
        Ok(())
    }
}

// 參數化テスト (rstest)
#[rstest]
#[case("hello", 5)]
#[case("", 0)]
#[case("rust", 4)]
fn test_string_length(#[case] input: &str, #[case] expected: usize) {
    assert_eq!(input.len(), expected);
}

// 非同期テスト
#[tokio::test]
async fn fetches_data_successfully() {
    let client = TestClient::new().await;
    let result = client.get("/data").await;
    assert!(result.is_ok());
}

// 屬性テスト (proptest)
proptest! {
    #[test]
    fn encode_decode_roundtrip(input in ".*") {
        let encoded = encode(&input);
        let decoded = decode(&encoded).unwrap();
        assert_eq!(input, decoded);
    }
}
```

**カバレッジコマンド**:
```bash
cargo llvm-cov                    # カバレッジ摘要
cargo llvm-cov --html             # HTML レポート
cargo llvm-cov --fail-under-lines 80  # 低於しきい値則失敗
cargo test                        # 実行テスト
```

## /cpp-test

**用途説明**: C++ 言語的 TDD ワークフロー。使用 GoogleTest/GoogleMock 和 CMake/CTest，先写テスト再実装，検証カバレッジ（gcov/lcov）。

**使用メソッド**:
```
/cpp-test [機能記述]
```

**使用场景**:
- 実装新的 C++ クラス或関数
- 追加テスト覆寫
- 修正 bug（先写失敗的テスト）
- 学習 C++ TDD メソッド论

**GoogleTest パターン**:
```cpp
// 基本テスト
TEST(SuiteName, TestName) {
    EXPECT_EQ(add(2, 3), 5);
    EXPECT_NE(result, nullptr);
    EXPECT_TRUE(is_valid);
    EXPECT_THROW(func(), std::invalid_argument);
}

// Fixture
class DatabaseTest : public ::testing::Test {
protected:
    void SetUp() override { db_ = create_test_db(); }
    void TearDown() override { db_.reset(); }
    std::unique_ptr<Database> db_;
};

TEST_F(DatabaseTest, InsertsRecord) {
    db_->insert("key", "value");
    EXPECT_EQ(db_->get("key"), "value");
}

// 參數化テスト
class PrimeTest : public ::testing::TestWithParam<std::pair<int, bool>> {};

TEST_P(PrimeTest, ChecksPrimality) {
    auto [input, expected] = GetParam();
    EXPECT_EQ(is_prime(input), expected);
}

INSTANTIATE_TEST_SUITE_P(Primes, PrimeTest,
    ::testing::Values(
        std::make_pair(2, true),
        std::make_pair(4, false),
        std::make_pair(7, true)
    ));
```

**カバレッジコマンド**:
```bash
cmake -DCMAKE_CXX_FLAGS="--coverage" -B build
cmake --build build
ctest --test-dir build
lcov --capture --directory build --output-file coverage.info
genhtml coverage.info --output-directory coverage_html
```

## /flutter-test

**用途説明**: Flutter/Dart テスト。レポート失敗並逐步修正テスト問題，涵盖单元、コンポーネント、金色和統合テスト。

**使用メソッド**:
```
/flutter-test                # 実行すべてテスト
/flutter-test --coverage     # 带カバレッジ
/flutter-test test/file.dart # 実行特定テストファイル
```

**使用场景**:
- 検証機能実装后ない破坏
- 検査新コード的テストカバレッジ
- 修正特定テストファイル
- コミット PR 之前

**常见テスト失敗和修正**:

| 失敗タイプ | 典型修正 |
|----------|----------|
| `Expected: <X> Actual: <Y>` | 更新断言或修正実装 |
| `Widget not found` | 修正 finder 选择器或更新 widget 名前 |
| `Golden file not found` | 実行 `flutter test --update-goldens` 生成 |
| `MissingPluginException` | 在テスト設定中 mock 平台通道 |
| `pumpAndSettle timed out` | 用显式 `pump(Duration)` 置換 |

## /test-coverage

**用途説明**: 分析テストカバレッジ，识别差距，並生成缺失テスト以达到目標しきい値（デフォルト 80%）。

**使用メソッド**:
```
/test-coverage
```

**ワークフロー**:
1. **检测テストフレームワーク** - 根據プロジェクトタイプ选择覆寫コマンド
2. **分析カバレッジレポート** - 列出低於 80% 的ファイル
3. **生成缺失テスト** - 按优先级生成テスト
4. **検証** - 重新実行カバレッジ確認改善

**テストフレームワーク检测**:

| 指示器 | 覆寫コマンド |
|--------|----------|
| `jest.config.*` | `npx jest --coverage` |
| `vitest.config.*` | `npx vitest run --coverage` |
| `pytest.ini` | `pytest --cov=src --cov-report=json` |
| `Cargo.toml` | `cargo llvm-cov --json` |
| `go.mod` | `go test -coverprofile=coverage.out` |

## /python-testing

**用途説明**: Python テストワークフロー。使用 pytest，サポートカバレッジ分析。

**使用メソッド**:
```
/python-testing [機能記述]
```

**使用场景**:
- 実装新的 Python 機能
- 追加テスト覆寫
- 修正 bug
- FastAPI/Django/Flask プロジェクトテスト

**Pytest 特性**:
```python
# 參數化テスト
@pytest.mark.parametrize("input,expected", [
    ("hello", 5),
    ("", 0),
    ("world", 5),
])
def test_string_length(input, expected):
    assert len(input) == expected

# Fixtures
@pytest.fixture
def client():
    with TestClient(app) as c:
        yield c

# 非同期テスト
@pytest.mark.asyncio
async def test_async_fetch():
    result = await fetch_data()
    assert result is not None
```
