# ループと自動化コマンド

## 概要

ループと自動化コマンド用于起動和管理自主ループ作業パターン。

## コマンド列表

### /loop-start

**用途**: 起動托管自主ループパターン，带セキュリティデフォルト設定和明确停止条件

**描述**: 起動带セキュリティデフォルト設定和明确停止条件的托管自主ループパターン。

**用法**: `/loop-start [pattern] [--mode safe|fast]`

**参数**:

| 参数 | 值 | 説明 |
|---|---|---|
| `pattern` | `sequential` | 順序実行タスク，一个接一个 |
| | `continuous-pr` | 持续 PR 作成和マージループ |
| | `rfc-dag` | 基于 RFC 的あり向なし环图ワークフロー |
| | `infinite` | なし限ループ（需明确停止条件） |
| `--mode` | `safe`（デフォルト） | 严格品質门禁和検査点 |
| | `fast` | 为速度减少门禁 |

**ワークフロー**:
1. 確認仓库状态和ブランチ戦略
2. 选择ループパターン和相关モデルレイヤー戦略
3. 为所选パターンあり効化所需 hooks/profile
4. 作成ループ計画并在 `.claude/plans/` 下写 runbook
5. 印刷起動和監視コマンド

**必需セキュリティ検査**:
- 検証テスト在首次ループ前通過
- 确保 `ECC_HOOK_PROFILE` 未全局無効化
- 确保ループあり明确停止条件

**参数传递**:
- `$ARGUMENTS`: `[pattern]` 和オプション的 `[--mode safe|fast]`
- パターンはオプション的（デフォルト为 sequential）
- mode 标志はオプション的（デフォルト为 safe）

**ベストプラクティス**:
- 在 infinite パターン之前确保設定明确的停止条件
- fast パターン仅在充分テスト的コード上使用
- 監視ループ进度，使用 `/loop-status` 検査状态

**常见陷阱**:
- 在未検証テスト的情况下起動ループ
- 使用 infinite パターン而なし明确停止条件
- 在全局無効化 hooks 时尝试使用セキュリティパターン

**与その他のコマンド統合**:
- 使用 `/loop-status` 監視実行中ループ
- 使用 `/santa-loop` 进行 Santa 风格ループ
- 使用 `/checkpoint` 作成重要ノード検査点

### /loop-status

**用途**: 検査実行中ループ的状态

**描述**: 查看当前実行中ループ的状态和进度。

---

### /santa-loop

**用途**: Santa 风格自主ループ

**描述**: 一种特殊风格的自主ループパターン。
