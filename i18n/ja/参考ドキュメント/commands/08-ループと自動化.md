# ループと自動化コマンド

## 概要

ループと自動化コマンド使用起動和管理自主ループ作業パターン。

## コマンド一覧

### /loop-start

**用途**: 起動托管自主ループパターン，带セキュリティデフォルト設定和明确停止條件

**記述**: 起動带セキュリティデフォルト設定和明确停止條件的托管自主ループパターン。

**用法**: `/loop-start [pattern] [--mode safe|fast]`

**參數**:

| 參數 | 值 | 説明 |
|---|---|---|
| `pattern` | `sequential` | 順序実行タスク，1接1 |
| | `continuous-pr` | 継続 PR 作成和マージループ |
| | `rfc-dag` | 基づいて RFC 的あり向なし环圖ワークフロー |
| | `infinite` | なし限ループ（需明确停止條件） |
| `--mode` | `safe`（デフォルト） | 严格品質门禁和検査点 |
| | `fast` | 為速度减少门禁 |

**ワークフロー**:
1. 確認仓庫狀態和ブランチ戦略
2. 选择ループパターン和相關モデルレイヤー戦略
3. 為所选パターンあり効化所需 hooks/profile
4. 作成ループ計画並在 `.claude/plans/` 下写 runbook
5. 印刷起動和監視コマンド

**必需セキュリティ検査**:
- 検証テスト在首次ループ前通過
- 確保 `ECC_HOOK_PROFILE` 未全局無効化
- 確保ループあり明确停止條件

**參數传递**:
- `$ARGUMENTS`: `[pattern]` 和オプション的 `[--mode safe|fast]`
- パターンはオプション的（デフォルト為 sequential）
- mode 标志はオプション的（デフォルト為 safe）

**ベストプラクティス**:
- 在 infinite パターン之前確保設定明确的停止條件
- fast パターン仅在充分テスト的コード上使用
- 監視ループ进度，使用 `/loop-status` 検査狀態

**常见陷阱**:
- 在未検証テスト的情况下起動ループ
- 使用 infinite パターン而なし明确停止條件
- 在全局無効化 hooks 时尝试使用セキュリティパターン

**與その他のコマンド統合**:
- 使用 `/loop-status` 監視実行中ループ
- 使用 `/santa-loop` 进行 Santa 风格ループ
- 使用 `/checkpoint` 作成重要ノード検査点

### /loop-status

**用途**: 検査実行中ループ的狀態

**記述**: 查看當前実行中ループ的狀態和进度。

---

### /santa-loop

**用途**: Santa 风格自主ループ

**記述**: 一種特殊风格的自主ループパターン。
