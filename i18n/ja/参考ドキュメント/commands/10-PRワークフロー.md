# Pull Request ワークフローコマンド

## 概要

PR ワークフローコマンド使用作成、管理和レビュー GitHub Pull Requests。

## コマンド一覧

### /pr

**用途**: 從當前ブランチ作成 GitHub PR，带 unpushed コミット

**記述**: 发现模板、分析变更、プッシュブランチ並作成 PR。分析完整コミット历史，使用 `git diff [base-branch]...HEAD` 查看すべて变更。

**输入**: `$ARGUMENTS` - オプション，可含む含基础ブランチ名和/或标志（如 `--draft`）

**解析ルール**:
- 提取任意识别的标志（如 `--draft`）
- 將剩余的非标志文本作為基础ブランチ名
- 如果未指定，デフォルト為 `main`

**ワークフロー**:

| フェーズ | 説明 |
|---|---|
| **Phase 1: VALIDATE** | 検査前置條件（不在baseブランチ、なし未コミット变更、ありaheadコミット、なし已ありPR） |
| **Phase 2: DISCOVER** | 搜索PR模板、分析コミット、分析ファイル变更、検査相關計画产物 |
| **Phase 3: PUSH** | プッシュブランチ（如果必要則先rebase） |
| **Phase 4: CREATE** | 使用模板或デフォルトフォーマット作成PR |
| **Phase 5: VERIFY** | 検証PR作成成功 |
| **Phase 6: OUTPUT** | レポートPR URL和下一步 |

**PR 作成検証検査**:

| 検査 | 條件 | 失敗时动作 |
|---|---|---|
| 不在baseブランチ | 當前ブランチ ≠ base | 停止："先切换到機能ブランチ" |
| 作業区干净 | なし未コミット变更 | 警告："先コミット或stash" |
| ありaheadコミット | `git log origin/<base>..HEAD` 非空 | 停止："ないaheadコミット" |
| なし已ありPR | `gh pr list` 為空 | 停止："PR已存在" |

**模板搜索順序**:
1. `.github/PULL_REQUEST_TEMPLATE/` ディレクトリ（如存在）
2. `.github/PULL_REQUEST_TEMPLATE.md`
3. `.github/pull_request_template.md`
4. `docs/pull_request_template.md`

**デフォルト PR フォーマット**（なし模板时）:
```markdown
## Summary
<1-2 sentence description>

## Changes
<bulleted list grouped by area>

## Files Changed
<table of changed files>

## Testing
<how changes were tested, or "Needs testing">

## Related Issues
<linked issues or "None">
```

**边缘情况處理**:
- **なし `gh` CLI**: 停止並提示インストール
- **未認証**: 提示実行 `gh auth login`
- **ブランチ分叉**: 使用 `git fetch && git rebase` 然後プッシュ
- **大型 PR (>20 ファイル)**: 警告並推奨拆分

**與その他のコマンド統合**:
- 使用 `/code-review <number>` レビュー PR
- 使用 `gh pr merge <number>` マージ準備完了的 PR
- 使用 `/plan-prd` 或 `/plan` 作成相關計画产物

### /review-pr

**用途**: レビュー GitHub PR 或本地未コミット变更

**記述**: レビュー GitHub 上的 Pull Request 或本地未コミット变更。PR レビューパターン适配自 PRPs-agentic-eng。

**输入**: `$ARGUMENTS` - 可為 PR 编号、PR URL 或空白（本地レビュー）

---

#### パターン选择

| 输入 | 动作 |
|---|---|
| PR 编号（如 `42`） | 作為 PR 编号使用 |
| URL（如 `github.com/.../pull/42`） | 提取 PR 编号 |
| ブランチ名 | 通過 `gh pr list --head <branch>` 検索 PR |
| 空白 | 使用本地レビューパターン |

---

#### 本地レビューパターン

**ワークフロー**:
1. **GATHER** - 收集变更: `git diff --name-only HEAD`
2. **REVIEW** - レビュー各ファイル（セキュリティ問題、品質問題、ベストプラクティス）
3. **REPORT** - 生成带重大级别和修正推奨的レポート

**レビュー清单**:

| クラス别 | チェック項目 |
|---|---|
| **セキュリティ問題 (CRITICAL)** | 平文凭证、SQL注入、XSS、缺少输入検証、不セキュリティ依賴、パス遍历 |
| **コード品質 (HIGH)** | 関数>50行、ファイル>800行、嵌套>4层、缺少エラー処理、console.log、TODO/FIXME |
| **ベストプラクティス (MEDIUM)** | 变更パターン、emoji使用、新コード缺少テスト、可訪問性問題 |

**决策**:

| 條件 | 决策 |
|---|---|
| 存在 CRITICAL 或 HIGH 問題 | **BLOCK** - 阻止コミット |
| 仅 MEDIUM/LOW 問題 | 通過但带评论 |
| なし問題且検証通過 | **APPROVE** |

---

#### PR レビューパターン

**ワークフロー**:
1. **FETCH** - 取得 PR 元データ和 diff
2. **CONTEXT** - ビルドレビュー上下文（プロジェクトルール、計画产物、变更ファイル）
3. **REVIEW** - 読み取り完整ファイル，アプリケーション 7 クラスレビュー清单
4. **VALIDATE** - 実行プロジェクトタイプ的検証コマンド
5. **DECIDE** - 根據发现形成推奨
6. **REPORT** - 作成レビュー产物
7. **PUBLISH** - 發布レビュー到 GitHub
8. **OUTPUT** - レポート給ユーザー

**7 クラスレビュー清单**:

| クラス别 | チェック項目 |
|---|---|
| **Correctness** | 逻辑エラー、边界情况、空處理、竞态條件 |
| **Type Safety** | タイプ不匹配、不セキュリティ類型轉換、`any` 使用、缺少泛型 |
| **Pattern Compliance** | 符合プロジェクト约定（命名、ファイル構造、エラー処理、インポート） |
| **Security** | 注入、認証缺口、secret暴露、SSRF、パス遍历、XSS |
| **Performance** | N+1查詢、缺少インデックス、なし界ループ、メモリ泄漏、大payload |
| **Completeness** | 缺少テスト、缺少エラー処理、不完整移行、缺少ドキュメント |
| **Maintainability** | 死コード、魔法数字、深嵌套、命名不清、缺少タイプ |

**検証コマンド检测**:

| プロジェクトタイプ | 检测ファイル | 検証コマンド |
|---|---|---|
| Node.js/TypeScript | `package.json` | `npm run typecheck`, `npm run lint`, `npm test`, `npm run build` |
| Rust | `Cargo.toml` | `cargo clippy`, `cargo test`, `cargo build` |
| Go | `go.mod` | `go vet ./...`, `go test ./...`, `go build ./...` |
| Python | `pyproject.toml` | `pytest` |

**决策级别**:

| 级别 | 含义 | 动作 |
|---|---|---|
| CRITICAL | セキュリティ漏洞或データ丢失風險 | must修正才能マージ |
| HIGH | 可能导致問題的 bug 或逻辑エラー | マージ前应修正 |
| MEDIUM | コード品質或ベストプラクティス缺失 | 推奨修正 |
| LOW | 样式問題或小推奨 | オプション |

**發布レビュー到 GitHub**:
```bash
# APPROVE
gh pr review <NUMBER> --approve --body "<summary>"

# REQUEST_CHANGES
gh pr review <NUMBER> --request-changes --body "<summary with required fixes>"

# COMMENT only (draft PR 或情報性)
gh pr review <NUMBER> --comment --body "<summary>"
```
