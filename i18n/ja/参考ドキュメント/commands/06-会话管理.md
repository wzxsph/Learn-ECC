# セッション管理コマンド

## 概要

セッション管理コマンド用于保存、再開和管理 Claude Code セッション状态。这些コマンド对于在长セッション中保持上下文、在セッション之间交接作業、以及トレース进度至关重要。

## コマンド列表

### /save-session

**用途**: 保存セッション状态 - 将当前セッション的完整上下文書き込み日期ファイル

**描述**: 将当前セッション的すべて上下文保存到 `~/.claude/session-data/` ディレクトリ下的日期ファイル中，パッケージ括：ビルド了什么、什么作業了、什么失敗了、剩余作業、ファイル状态、决策記録和待解決 blockers。

**セッションファイル命名仕様**:
- フォーマット: `YYYY-MM-DD-<short-id>-session.tmp`
- あり效字符: 字母 `a-z`/`A-Z`、数字 `0-9`、连字符 `-`、下划线 `_`
- 最小長さ: 1 字符（但不推奨过短）
- 推奨: 8+ 字符的小写字母/数字/连字符组合避免同日冲突

**使用タイミング**:
- 作業終了时（セッション終了前）
- 接近上下文限制前（先保存，再开新セッション）
- 复杂問題解決后（保存成功パターン）
- 必要交接给未来セッション时

**ワークフロー**:
1. **收集上下文** - 読み取りすべて修改过的ファイル，レビュー讨论内容
2. **作成ディレクトリ** - `mkdir -p ~/.claude/session-data`
3. **書き込みファイル** - 作成带时间戳的セッションファイル
4. **展示给用户** - 表示内容并リクエスト確認

**セッションファイルフォーマット**:

```markdown
# Session: YYYY-MM-DD

**Started:** [approximate time if known]
**Last Updated:** [current time]
**Project:** [project name or path]
**Topic:** [one-line summary of what this session was about]

---

## What We Are Building
[1-3 paragraphs describing the feature, bug fix, or task]

## What WORKED (with evidence)
- **[thing that works]** — confirmed by: [specific evidence]

## What Did NOT Work (and why)
- **[approach tried]** — failed because: [exact reason / error message]

## What Has NOT Been Tried Yet
- [approach / idea]

## Current State of Files
| File              | Status         | Notes                      |
| ----------------- | -------------- | -------------------------- |
| `path/to/file.ts` | PASS: Complete | [what it does]             |

## Decisions Made
- **[decision]** — reason: [why this was chosen over alternatives]

## Blockers & Open Questions
- [blocker / open question]

## Exact Next Step
[The single most important thing to do when resuming]
```

**重要原則**:
- "What Did NOT Work" 一部は最重要的 - 未来セッション会盲目再試行失敗的メソッド
- 各セッション独立ファイル - 绝不追加到之前セッション
- ファイル供下次セッション通過 `/resume-session` 読み取り

**ベストプラクティス**:
- 中途保存时（ではない終了时），清楚标记進行中プロジェクト
- パッケージ含具体エラー情報和失敗原因（"threw X error because Y"）
- 下一具体ステップ要足够精确，なし需思考就知道从哪開始

### /resume-session

**用途**: 再開保存的セッション

**描述**: 从 `~/.claude/session-data/` 加载并再開之前保存的セッション上下文。

**使用场景**:
- 開始新セッション时
- 必要継続之前的作業
- 必要回顾之前的問題解決过程

---

### /sessions

**用途**: 浏览+搜索+管理セッション历史

**描述**: 浏览、搜索和管理すべて已保存的セッション历史。

**機能**:
- 列出すべてセッション
- 按日期搜索
- 管理セッション别名
- クリーンアップ旧セッション

---

### /checkpoint

**用途**: 作成/検証ワークフロー検査点

**描述**: 在ワークフロー重要ノード作成検査点，确保进度可追溯。

**使用场景**:
- 重要ステップ完了时
- 必要検証重要里程碑
- 多フェーズタスク中

---

### /aside

**用途**: 回答侧问而不丢失上下文

**描述**: 切换到侧向对话パターン，处理次要問題后なし缝戻る主タスク。

**使用场景**:
- 必要问一个快速問題
- 必要查阅ドキュメント
- 必要実行辅助タスク

---

## 相关コマンド

- `/save-session` - 保存当前セッション
- `/resume-session` - 再開セッション
- `/sessions` - 管理セッション历史
