# 学習と改善コマンド

## 概要

学習と改善コマンド用于从セッション中提取パターン、トレース instinct 和管理组织知识。

## コマンド列表

### /learn

**用途**: 从当前セッション中提取可复用パターン并保存为候选 skills

**描述**: 分析当前セッション，提取任意值得保存为 skills 的パターン。実行 `/learn` 可在セッション期间任意时候使用。

**提取内容タイプ**:

| タイプ | 描述 |
|---|---|
| **エラー解決方案パターン** | 什么エラー？根本原因は什么？什么修正了？クラス似エラー可复用吗？ |
| **デバッグ技術** | 非显而易见的デバッグステップ、あり效的ツール组合、診断パターン |
| **变通メソッド** | 库怪癖、API 限制、特定バージョン修正 |
| **プロジェクト特定パターン** | 发现的コード库约定、アーキテクチャ决策、統合パターン |

**输出フォーマット**: 保存到 `~/.claude/skills/learned/[pattern-name].md`

```markdown
# [Descriptive Pattern Name]

**Extracted:** [Date]
**Context:** [Brief description of when this applies]

## Problem
[What problem this solves - be specific]

## Solution
[The pattern/technique/workaround]

## Example
[Code example if applicable]

## When to Use
[Trigger conditions - what should activate this skill]
```

**ワークフロー**:
1. レビューセッション中可提取的パターン
2. 识别最あり价值/可复用的见解
3. 起草 skill ファイル
4. リクエスト用户確認后再保存
5. 保存到 `~/.claude/skills/learned/`

**ベストプラクティス**:
- 不要提取trivial修正（拼写エラー、简单语法エラー）
- 不要提取一次性問題（特定 API 中断等）
- 关注未来セッション可节省时间的パターン
- 保持 skills 专注 - 一个パターン一个 skill

### /learn-eval

**用途**: 提取パターン + 自我评估品質

**描述**: 在提取パターン的同时进行品質评估。

---

### /evolve

**用途**: 分析 instinct + 推奨进化構造

**描述**: 分析已学習的 instinct，提供構造演进推奨。

---

### /promote

**用途**: 将プロジェクト instinct 提升到全局范围

**描述**: 将プロジェクト特定的 instinct 提升为全局可用的知识。

---

### /instinct-status

**用途**: 表示すべて学習的 instinct

**描述**: 表示当前すべて instinct 的状态和置信度。

---

### /instinct-export

**用途**: エクスポート instinct 到ファイル

**描述**: 将 instinct エクスポート为可分享的ファイルフォーマット。

---

### /instinct-import

**用途**: 从ファイル/URL インポート instinct

**描述**: 从ファイル或 URL インポート instinct 到システム中。

---

### /skill-create

**用途**: 分析 git 历史+生成 skill ファイル

**描述**: 分析プロジェクト git 历史，提取可复用的パターン生成 skill ファイル。

**ワークフロー**:
1. 分析 git コミット历史
2. 识别重复パターン
3. 生成 skill ファイル
4. 検証并保存

---

### /skill-health

**用途**: Skill 组合健康仪表板

**描述**: 表示 skill 组合的健康状态和使用統計。

---

### /rules-distill

**用途**: 扫描 skills + 提取跨领域原則

**描述**: 扫描すべて skills，提取跨领域的通用原則。

---

## 相关コマンド

- `/learn` - 提取パターン
- `/skill-create` - 从 git 历史生成 skill
- `/instinct-status` - 查看 instinct 状态
