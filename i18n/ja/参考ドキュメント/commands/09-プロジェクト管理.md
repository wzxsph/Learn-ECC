# プロジェクト管理コマンド

## 概要

プロジェクト管理コマンド使用プロジェクト設定、インフラ設定和ツール链管理。

## コマンド一覧

### /projects

**用途**: 列出已知プロジェクト + instinct 統計

**記述**: 列出ユーザーすべて已知プロジェクト及其相關的 instinct 統計情報。

---

### /harness-audit

**用途**: 審計 agent harness 設定

**記述**: 検査和審計 Claude Code 的 harness 設定。

**検査內容**:
- ツール链設定
- プロキシ設定
- MCP サーバー狀態
- ルール設定

---

### /model-route

**用途**: ルーティングタスク到合适モデル

**記述**: 根據タスクタイプ选择最合适的モデル（Haiku/Sonnet/Opus）。

**モデル选择ガイド**:
- **Haiku**: 轻量级タスク、コード生成、频繁調用
- **Sonnet**: 主要開発作業、複雜エンコードタスク
- **Opus**: アーキテクチャ决策、深さ推論、研究分析

---

### /pm2

**用途**: PM2 プロセス管理器初期化

**記述**: 初期化 PM2 プロセス管理器設定。

---

### /setup-pm

**用途**: 設定含むマネージャー

**記述**: 設定プロジェクト使用的含むマネージャー。

**サポート的含むマネージャー**:
- npm
- pnpm
- yarn
- bun

**可通過** `CLAUDE_PACKAGE_MANAGER` **環境変数設定**

---

### /project-init

**用途**: プロジェクト初期化

**記述**: 初期化新プロジェクト的標準構造和設定。

---

## 相關コマンド

- `/projects` - 列出プロジェクト
- `/harness-audit` - 審計設定
- `/model-route` - モデルルーティング
