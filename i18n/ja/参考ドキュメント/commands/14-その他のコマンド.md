# その他のコマンド

## 概要

その他杂项コマンド，パッケージ括 Jira 統合、GAN 操作、セキュリティ扫描等機能。

## コマンド列表

### /jira

**用途**: 与 Jira 工单交互 - 取得、分析、评论、转换状态、搜索

**描述**: 与 Jira プロジェクト管理ツール統合，サポート直接ワークフロー中使用 Jira tickets。

**用法**:

| コマンド | 説明 |
|---|---|
| `/jira get <TICKET-KEY>` | 取得并分析 Jira 工单 |
| `/jira comment <TICKET-KEY>` | 追加进度评论 |
| `/jira transition <TICKET-KEY>` | 更改工单状态 |
| `/jira search <JQL>` | 使用 JQL 搜索問題 |

**`/jira get` ワークフロー**:
1. 从 Jira 取得工单（通過 MCP 或 REST API）
2. 提取すべて字段：摘要、描述、验收標準、优先级、タグ、关联問題
3. オプション取得评论以取得额外上下文
4. 生成構造化分析

**先决条件** - Jira 凭证設定（二选一）:

**选项 A — MCP Server（推奨）**:
将 `jira` 追加到 `mcpServers` 設定。

**选项 B — 环境変数**:
```bash
export JIRA_URL="https://yourorg.atlassian.net"
export JIRA_EMAIL="your.email@example.com"
export JIRA_API_TOKEN="your-api-token"
```

**与その他のコマンド統合**:
- 分析工单后使用 `/plan` 作成实施計画
- 使用 `tdd-workflow` skill 进行テスト驱动開発
- 使用 `/code-review` レビュー実装
- 使用 `/jira comment` 或 `/jira transition` 更新工单状态

### /prune

**用途**: 削除超过 30 天未提升的陈旧 pending instinct

**描述**: クリーンアップ超过 30 天的陈旧 instinct，这些 instinct は自動生成但从未被レビュー或提升的。

**用法**:
```
/prune                    # 削除超过 30 天的 instinct
/prune --max-age 60      # 自定义天数しきい値
/prune --dry-run         # 预览而不削除
```

**実装**:
```bash
python3 "${CLAUDE_PLUGIN_ROOT}/skills/continuous-learning-v2/scripts/instinct-cli.py" prune
```

或手动インストール时（`CLAUDE_PLUGIN_ROOT` 未設定）:
```bash
python3 ~/.claude/skills/continuous-learning-v2/scripts/instinct-cli.py prune
```

**参数**:

| 参数 | 説明 |
|---|---|
| なし | 削除超过 30 天的 instinct |
| `--max-age <days>` | 自定义时间しきい値（天） |
| `--dry-run` | 预览将削除的内容，不实际削除 |

**ベストプラクティス**:
- 使用 `--dry-run` 先预览要削除的内容
- 定期実行 prune 保持 instinct 库存整洁
- 注意：只会削除 pending 状态的 instinct，不会削除已提升的
