# 言語レビューコマンド

本ドキュメント介绍 ECC 中用于各プログラミング言語コードレビュー的专门コマンド。

---

## /python-review

**用途説明**: Python コード全面レビュー。検査 PEP 8 合规性、型ヒント、セキュリティ和 Python 习惯用法。

**使用メソッド**:
```
/python-review
```

**使用场景**:
- 编写或修改 Python コード之后
- コミット Python 变更之前
- レビューパッケージ含 Python コード的 PR
- 学習 Python 习惯用法和ベストプラクティス

**レビュークラス别**:

### CRITICAL (must修正)
- SQL/コマンド注入漏洞
- 不セキュリティ的 eval/exec 使用
- Pickle 不セキュリティデシリアライズ
- 平文凭证
- YAML 不セキュリティ加载
- 非表示エラー的裸 except 子句

### HIGH (すべき修正)
- 公共関数缺少型ヒント
- 可变デフォルト参数
- 静默吞没异常
- 资源管理未使用上下文管理器
- C 风格ループ而非推导式
- 使用 type() 而非 isinstance()

### MEDIUM (推奨検討)
- PEP 8 フォーマット违规
- 公共関数缺少ドキュメント字符串
- Print 语句而非ログ
- 低效的字符串操作
- 魔术数字未使用命名定数
- 未使用 f-strings

**自動検査コマンド**:
```bash
mypy .                              # タイプ検査
ruff check .                        # Linting
black --check .                     # フォーマット化検査
isort --check-only .                # インポート排序
bandit -r .                         # セキュリティ扫描
pip-audit                           # 依赖审计
pytest --cov=app --cov-report=term-missing  # テスト覆盖
```

**常见問題修正**:

```python
# SQL インジェクション修正
# エラー
query = f"SELECT * FROM users WHERE id = {user_id}"

# 正确 - 使用参数化查询
query = "SELECT * FROM users WHERE id = %s"
cursor.execute(query, (user_id,))

# 可变デフォルト参数修正
# エラー
def process_items(items=[]):
    items.append("new")
    return items

# 正确
def process_items(items=None):
    if items is None:
        items = []
    items.append("new")
    return items

# 使用上下文管理器
# エラー
f = open("config.json")
data = f.read()
f.close()

# 正确
with open("config.json") as f:
    data = f.read()
```

## /go-review

**用途説明**: Go コード全面レビュー。検査惯用パターン、並行処理セキュリティ、エラー処理和セキュリティ。

**使用メソッド**:
```
/go-review
```

**使用场景**:
- 编写或修改 Go コード之后
- コミット Go 变更之前
- レビューパッケージ含 Go コード的 PR
- 学習 Go 惯用パターン

**レビュークラス别**:

### CRITICAL (must修正)
- SQL/コマンド注入漏洞
- なし同期的竞态条件
- Goroutine 泄漏
- 平文凭证
- 不セキュリティ的指针使用
- 重要路径無視エラー

### HIGH (すべき修正)
- 缺少エラー的上下文パッケージ装
- Panic 而非エラー戻る
- Context 未传播
- 未缓冲通道导致死锁
- インターフェース未実装エラー
- 缺少互斥锁保护

### MEDIUM (推奨検討)
- 非惯用コードパターン
- エクスポート関数缺少 godoc コメント
- 低效的字符串拼接
- Slice 未预分配
- 未使用表驱动テスト

**自動検査コマンド**:
```bash
go vet ./...                         # 静态分析
staticcheck ./...                    # 高级検査（如果インストール）
golangci-lint run                   # Linting
go build -race ./...                # 竞态检测
govulncheck ./...                   # セキュリティ漏洞
```

## /kotlin-review

**用途説明**: Kotlin コード全面レビュー。検査惯用パターン、空セキュリティ、コルーチンセキュリティ和セキュリティ。

**使用メソッド**:
```
/kotlin-review
```

**使用场景**:
- 编写或修改 Kotlin コード之后
- コミット Kotlin 变更之前
- レビュー Android/KMP プロジェクト的 PR
- 学習 Kotlin 惯用パターン

**レビュークラス别**:

### CRITICAL (must修正)
- SQL/コマンド注入漏洞
- なし充分理由的强制解パッケージ `!!`
- 平台タイプ空セキュリティ违规
- GlobalScope 使用（構造化並行処理违规）
- 平文凭证
- 不セキュリティデシリアライズ

### HIGH (すべき修正)
- 可变状态（可用不可变时）
- コルーチン上下文内的阻塞调用
- 长ループ中缺少取消検査
- 密封クラス `when` 非穷举
- 大関数（>50行）
- 深嵌套（>4层）

### MEDIUM (推奨検討)
- 非惯用 Kotlin（Java 风格パターン）
- 缺少尾随逗号
- Scope 関数误用或嵌套
- 大集合链缺少 Sequence
- 冗余显式タイプ

**自動検査コマンド**:
```bash
./gradlew build                      # ビルド検査
./gradlew detekt                     # 静态分析
./gradlew ktlintCheck                # フォーマット検査
./gradlew test                       # テスト
```

## /rust-review

**用途説明**: Rust コード全面レビュー。検査すべて権ライフタイム、エラー処理、不セキュリティ使用和惯用パターン。

**使用メソッド**:
```
/rust-review
```

**使用场景**:
- 编写或修改 Rust コード之后
- コミット Rust 变更之前
- レビュー Rust プロジェクト的 PR
- 学習 Rust 惯用パターン

**レビュークラス别**:

### CRITICAL (must修正)
- 生产コード路径中未検査的 `unwrap()`/`expect()`
- 不带 `// SAFETY:` コメント的 `unsafe`
- 字符串插值的 SQL インジェクション
- 未検証输入的コマンド注入
- 平文凭证
- 原始指针的释放后使用

### HIGH (すべき修正)
- 不必要的 `.clone()` 以满足借用検査器
- `String` 参数应用 `&str` 或 `impl AsRef<str>`
- 非同期上下文中的阻塞（`std::thread::sleep`）
- 共享タイプ缺少 `Send`/`Sync` 约束
- 重要枚举上的通配符 `_ =>` 匹配
- 大関数（>50行）

### MEDIUM (推奨検討)
- 热路径中的不必要分配
- 已知サイズ时缺少 `with_capacity`
- なし充分理由抑制 clippy 警告
- 公共 API 缺少 `///` ドキュメント
- 検討对可能無視值的非 `must_use` 戻るタイプ使用 `#[must_use]`

**自動検査コマンド**:
```bash
cargo check                           # ビルド検査
cargo clippy -- -D warnings            # Lints
cargo fmt --check                      # フォーマット検査
cargo test                            # テスト
cargo audit                           # セキュリティ审计（如果可用）
```

## /cpp-review

**用途説明**: C++ コード全面レビュー。検査メモリセキュリティ、现代 C++ 习惯用法、並行処理和セキュリティ。

**使用メソッド**:
```
/cpp-review
```

**使用场景**:
- 编写或修改 C++ コード之后
- コミット C++ 变更之前
- レビュー C++ プロジェクト的 PR
- 検査メモリセキュリティ問題

**レビュークラス别**:

### CRITICAL (must修正)
- なし RAII 的裸 `new`/`delete`
- 缓冲区溢出和释放后使用
- なし同期的データ竞态
- 通過 `system()` 的コマンド注入
- 未初期化変数読み取り
- 空指针解引用

### HIGH (すべき修正)
- 五法则违规
- 缺少 `std::lock_guard` / `std::scoped_lock`
- detach スレッド的寿命管理不当
- C 风格强制转换而非 `static_cast`/`dynamic_cast`
- 缺少 `const` 正确性

### MEDIUM (推奨検討)
- 不必要的拷贝（传值而非 `const&`）
- 已知サイズ时缺少 `reserve()`
- 头ファイル中 `using namespace std;`
- 重要戻る值缺少 `[[nodiscard]]`
- 过于复杂的模板元编程

**自動検査コマンド**:
```bash
clang-tidy --checks='*,-llvmlibc-*' src/*.cpp -- -std=c++17
cppcheck --enable=all --suppress=missingIncludeSystem src/
cmake --build build -- -Wall -Wextra -Wpedantic
```

## /flutter-review

**用途説明**: Flutter/Dart コードレビュー。検査惯用パターン、コンポーネントベストプラクティス、状態管理、パフォーマンス、可访问性和セキュリティ。

**使用メソッド**:
```
/flutter-review
```

**使用场景**:
- コミットパッケージ含 Flutter/Dart 变更的 PR 之前（ビルド和テスト通過后）
- 実装新機能后早期发现問題
- レビュー他人的 Flutter コード
- 审核コンポーネント、状態管理コンポーネント或サービスクラス
- 生产发布之前

**レビュー区域**:

| 区域 | 重大级别 |
|------|----------|
| 硬エンコード密钥、明文 HTTP | CRITICAL |
| アーキテクチャ违规、状態管理反パターン | CRITICAL |
| コンポーネント重建問題、资源泄漏 | HIGH |
| 缺少 `dispose()`、`await` 后的 BuildContext | HIGH |
| Dart 空セキュリティ、缺少エラー/加载状态 | HIGH |
| Const 传播、コンポーネント组合 | HIGH |
| `build()` 中的昂贵作業 | HIGH |
| 可访问性、语义タグ | MEDIUM |
| 状态转换缺少テスト | HIGH |
| 硬エンコード字符串（l10n） | MEDIUM |

## /fastapi-review

**用途説明**: FastAPI 应用レビュー。検査アーキテクチャ、非同期正确性、依赖注入、Pydantic パターン、セキュリティ、パフォーマンス和可テスト性。

**使用メソッド**:
```
/fastapi-review [ファイル或ディレクトリ]
```

**レビュー区域**:
- App factory、ルーティング边界、ミドルウェア和异常处理器
- Pydantic リクエスト和応答パターン分离
- データベースセッション、認証、分页和設定的依赖注入
- 非同期データベース和外部 HTTP パターン
- CORS、認証、限流、ログ和密钥处理
- OpenAPI 元データ和文書化応答モデル
- テストクライアント設定和依赖覆盖
