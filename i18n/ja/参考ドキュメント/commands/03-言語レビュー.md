# 言語レビューコマンド

本ドキュメントでは、ECC で使用される各プログラミング言語のコードレビュー専門のコマンドについて説明します。

---

## /python-review

**用途説明**: Python コード全面レビュー。PEP 8 準拠、型ヒント、セキュリティと Python 慣用法を検査。

**使用メソッド**:
```
/python-review
```

**使用シーン**:
- Python コードを作成または変更した後
- Python 変更をコミットする前に
- Python コードを含む PR をレビュー
- Python 慣用法とベストプラクティスを学習

**レビューグelsche**:

### CRITICAL (must修正)
- SQL/コマンド注入漏洞
- 安全でない eval/exec 使用
- Pickle 安全でないデシリアライズ
- 平文凭证
- YAML 安全でないロード
- 非表示エラーの裸 except 子句

### HIGH (すべき修正)
- 公共関数に型ヒントがない
- 可変デフォルトパラメータ
- サイレント例外
- 資源管理でコンテキストマネージャーを不使用
- C スタイルループ而非推导式
- type() の使用而非 isinstance()

### MEDIUM (推奨検討)
- PEP 8 フォーマット違反
- 公共関数にドキュメント文字列がない
- Print 文而非ログ
- 非能率的な文字列操作
- 魔术数字が命名定数を使用していない
- f-strings の未使用

**自動検査コマンド**:
```bash
mypy .                              # タイプ検査
ruff check .                        # Linting
black --check .                     # フォーマット化検査
isort --check-only .                # インポート整列
bandit -r .                         # セキュリティスキャン
pip-audit                           # 依存関係監査
pytest --cov=app --cov-report=term-missing  # テスト覆寫
```

**常见問題修正**:

```python
# SQL インジェクション修正
# エラー
query = f"SELECT * FROM users WHERE id = {user_id}"

# 正しい - パラメータ化クエリを使用
query = "SELECT * FROM users WHERE id = %s"
cursor.execute(query, (user_id,))

# 可変デフォルトパラメータ修正
# エラー
def process_items(items=[]):
    items.append("new")
    return items

# 正しい
def process_items(items=None):
    if items is None:
        items = []
    items.append("new")
    return items

# コンテキストマネージャーの使用
# エラー
f = open("config.json")
data = f.read()
f.close()

# 正しい
with open("config.json") as f:
    data = f.read()
```

## /go-review

**用途説明**: Go コード全面レビュー。慣用パターン、並行処理セキュリティ、エラー処理とセキュリティを検査。

**使用メソッド**:
```
/go-review
```

**使用シーン**:
- Go コードを作成または変更した後
- Go 変更をコミットする前に
- Go コードを含む PR をレビュー
- Go 慣用パターンを学習

**レビューグelsche**:

### CRITICAL (must修正)
- SQL/コマンド注入漏洞
- 同期的な竞態條件なし
- Goroutine 泄漏
- 平文凭证
- 安全でない指標使用
- 重要パスが無視エラー

### HIGH (すべき修正)
- エラーのコンテキスト包装がない
- Panic 而非エラー返回
- Context が传播しない
- バッファなし通道によるデッドロック
- インターフェース未実装エラー
- 互斥鎖保護がない

### MEDIUM (推奨検討)
- 非慣用コードパターン
- エクスポート関数に godoc コメントがない
- 非能率的な文字列連結
- Slice の未予分配
- 表駆動テストの未使用

**自動検査コマンド**:
```bash
go vet ./...                         # 静的分析
staticcheck ./...                    # 高度な検査（インストールされている場合）
golangci-lint run                   # Linting
go build -race ./...                # 竞態検出
govulncheck ./...                   # セキュリティ脆弱性
```

## /kotlin-review

**用途説明**: Kotlin コード全面レビュー。慣用パターン、空セキュリティ、コルーチンセキュリティとセキュリティを検査。

**使用メソッド**:
```
/kotlin-review
```

**使用シーン**:
- Kotlin コードを作成または変更した後
- Kotlin 変更をコミットする前に
- Android/KMP プロジェクトの PR をレビュー
- Kotlin 慣用パターンを学習

**レビューグelsche**:

### CRITICAL (must修正)
- SQL/コマンド注入漏洞
- 十分な理由のない強制分解 `!!`
- プラットフォームタイプの空セキュリティ違反
- GlobalScope 使用（構造化並行処理違反）
- 平文凭证
- 安全でないデシリアライズ

### HIGH (すべき修正)
- 可変状態（使用不可変時）
- コルーチンコンテキスト内の阻塞呼び出し
- 長いループ中にキャンセル検査がない
- 密封クラス `when` の非窮举
- 大きい関数（>50行）
- 深嵌套（>4層）

### MEDIUM (推奨検討)
- 非慣用 Kotlin（Java 風パター）
- 尾随カンマがない
- Scope 関数の誤用またはネスト
- 大きい集合鎖に Sequence がない
- 冗長な明示的タイプ

**自動検査コマンド**:
```bash
./gradlew build                      # ビルド検査
./gradlew detekt                     # 静的分析
./gradlew ktlintCheck                # フォーマット検査
./gradlew test                       # テスト
```

## /rust-review

**用途説明**: Rust コード全面レビュー。所有権、ライフタイム、安全でない使用と慣用パターンをすべて検査。

**使用メソッド**:
```
/rust-review
```

**使用シーン**:
- Rust コードを作成または変更した後
- Rust 変更をコミットする前に
- Rust プロジェクトの PR をレビュー
- Rust 慣用パターンを学習

**レビューグelsche**:

### CRITICAL (must修正)
- 本番コードパス中の未検査の `unwrap()`/`expect()`
- `// SAFETY:` コメントなしの `unsafe`
- 文字列補間の SQL インジェクション
- 未検証入力のコマンド注入
- 平文凭证
- 原始指標の解放後使用

### HIGH (すべき修正)
- 借用検査器を満たすための不必要な `.clone()`
- `String` パラメータへの `&str` または `impl AsRef<str>` の application
- 非同期コンテキスト内の阻塞（`std::thread::sleep`）
- 共有タイプに `Send`/`Sync` 制約がない
- 重要な列挙型のワイルドカード `_ =>` マッチ
- 大きい関数（>50行）

### MEDIUM (推奨検討)
- 熱パス内の不必要な割り当て
- 既知サイズ時の `with_capacity` がない
- 十分な理由のない clippy 警告抑制
- 公共 API に `///` ドキュメントがない
- 可能性のある無視値に非 `must_use` 返回タイプでの `#[must_use]` の使用を検討

**自動検査コマンド**:
```bash
cargo check                           # ビルド検査
cargo clippy -- -D warnings            # Lints
cargo fmt --check                      # フォーマット検査
cargo test                            # テスト
cargo audit                           # セキュリティ監査（使用可能な場合）
```

## /cpp-review

**用途説明**: C++ コード全面レビュー。メモリセキュリティ、現代 C++ 慣用法、並行処理とセキュリティを検査。

**使用メソッド**:
```
/cpp-review
```

**使用シーン**:
- C++ コードを作成または変更した後
- C++ 変更をコミットする前に
- C++ プロジェクトの PR をレビュー
- メモリセキュリティ問題を検査

**レビューグelsche**:

### CRITICAL (must修正)
- RAII のない裸の `new`/`delete`
- バッファオーバーフローと解放後使用
- 同期的なデータ竞態
- `system()` によるコマンド注入
- 未初期化変数読み取り
- 空指標逆参照

### HIGH (すべき修正)
- 五則違反
- `std::lock_guard` / `std::scoped_lock` がない
- detach スレッドの寿命管理不当
- C スタイルの強制変換而非 `static_cast`/`dynamic_cast`
- `const` 正確性がない

### MEDIUM (推奨検討)
- 不必要なコピ（値而非 `const&`）
- 既知サイズ時の `reserve()` がない
- ヘッダーファイル内の `using namespace std;`
- 重要な返回値に `[[nodiscard]]` がない
- 過度に複雑なテンプレートメタプログラミング

**自動検査コマンド**:
```bash
clang-tidy --checks='*,-llvmlibc-*' src/*.cpp -- -std=c++17
cppcheck --enable=all --suppress=missingIncludeSystem src/
cmake --build build -- -Wall -Wextra -Wpedantic
```

## /flutter-review

**用途説明**: Flutter/Dart コードレビュー。慣用パターン、コンポーネントベストプラクティス、状態管理、パフォーマンス、可アクセス性とセキュリティを検査。

**使用メソッド**:
```
/flutter-review
```

**使用シーン**:
- Flutter/Dart 変更を含む PR をコミットする前に（ビルドとテスト通過後）
- 新機能を実装した後の早期の問題発見
- 他人の Flutter コードをレビュー
- コンポーネント、状態管理コンポーネントまたはサービスクラスを監査
- 本番リリースの前に

**レビュー区域**:

| 区域 | 重大レベル |
|------|----------|
| ハードコードされた密鑰、平文 HTTP | CRITICAL |
| アーキテクチャ違反、状態管理反パターン | CRITICAL |
| コンポーネント再構築問題、資源泄漏 | HIGH |
| `dispose()`、`await` 後の BuildContext がない | HIGH |
| Dart 空セキュリティ、エラー/読み込み状態がない | HIGH |
| Const 伝播、コンポーネント組合 | HIGH |
| `build()` 内の昂贵的作業 | HIGH |
| 可アクセス性、セマンティックタグ | MEDIUM |
| 状態変換にテストがない | HIGH |
| ハードコードされた文字列（l10n） | MEDIUM |

## /fastapi-review

**用途説明**: FastAPI アプリケーションレビュー。アーキテクチャ、非同期正確性、依存関係注入、Pydantic パターン、セキュリティ、パフォーマンスとテスト可能性を検査。

**使用メソッド**:
```
/fastapi-review [ファイルまたはディレクトリ]
```

**レビュー区域**:
- App factory、ルーティング境界、ミドルウェアと例外処理器
- Pydantic リクエストと応答パターンの分離
- データベースセッション、認証、ページネーションと設定の依存関係注入
- 非同期データベースと外部 HTTP パターン
- CORS、認証、制限、ログと密鑰処理
- OpenAPI メタデータと文書化された応答モデル
- テストクライアント設定と依存関係覆寫
