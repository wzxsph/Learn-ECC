# ビルドスクリプト

## release.sh

**パス**: `scripts/release.sh`

プラグインバージョンアップグレード和發布スクリプト。

### 機能

1. **バージョン検証**
   - 確認提供します了 semver フォーマットバージョン号 (X.Y.Z 或 X.Y.Z-prerelease)
   - 確認當前ブランチは main
   - 確認作業区干净

2. **ファイル更新**
   - 更新すべて含む/プラグイン清单中的バージョン号
   - 更新ドキュメント中的バージョン引用
   - 更新 agent 設定

3. **検証ビルド**
   - 実行 OpenCode ビルド
   - 実行ビルドテスト
   - 実行プラグイン清单テスト

4. **Git 操作**
   - 暂存すべて更改
   - 作成コミット
   - 作成並プッシュタグ

### 使用メソッド

```bash
# 發布新バージョン
./scripts/release.sh 1.5.0

# 预發布バージョン
./scripts/release.sh 2.0.0-rc.1
```

- Git 作業区must干净
- must位於 main ブランチ
- すべて清单ファイルmust存在

## build-opencode.js

**パス**: `scripts/build-opencode.js`

ビルド OpenCode プラグイン的 TypeScript コード。

### ビルド流程

1. クリーンアップ `dist` ディレクトリ
2. 検索 TypeScript コンパイル器 (`typescript/bin/tsc`)
3. 使用プロジェクト根ディレクトリ的 tsconfig.json コンパイル `.opencode` ディレクトリ

### 使用メソッド

```bash
# ビルド OpenCode
node scripts/build-opencode.js

# 作為 release 的一一部自動実行
./scripts/release.sh 1.5.0
```

### 前提條件

必要已インストール根ディレクトリ的開発依賴,確保 TypeScript コンパイル器可用。

## 發布ワークフロー

```
1. 確保在 main ブランチ且作業区干净
2. 実行發布スクリプト並提供しますバージョン号
3. スクリプト自動完了:
   ├── 検証前提條件
   ├── 更新すべてバージョン引用
   ├── ビルド OpenCode
   ├── 実行テスト検証
   ├── 作成 Git コミット
   └── プッシュタグ
4. GitHub Actions 检测到タグ自動發布
```
