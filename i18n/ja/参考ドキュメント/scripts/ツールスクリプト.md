## ecc.js

**パス**: `scripts/ecc.js`

ECC 选择性インストール的 CLI 主入口点,统一調度すべて子コマンド。

### コマンド一覧

| コマンド | スクリプト | 説明 |
|------|------|------|
| `install` | install-apply.js | インストール ECC 內容到目標 |
| `plan` | install-plan.js | 検査インストール清单和解析計画 |
| `catalog` | catalog.js | 发现インストール設定ファイル和コンポーネント ID |
| `consult` | consult.js | 根據自然言語查詢推奨コンポーネント |
| `list-installed` | list-installed.js | 検査當前上下文的インストール狀態 |
| `doctor` | doctor.js | 診断缺失或漂移的 ECC 管理ファイル |
| `repair` | repair.js | 再開漂移或缺失的 ECC 管理ファイル |
| `auto-update` | auto-update.js | プル最新 ECC 更改並重新インストール |
| `status` | status.js | 查詢 SQLite 狀態存儲摘要 |
| `platform-audit` | platform-audit.js | 審計 GitHub キュー、讨论、路线圖等 |
| `security-ioc-scan` | ci/scan-supply-chain-iocs.js | 扫描依賴和 AI ツール持久化面的供应链 IOC |
| `sessions` | sessions-cli.js | 列出或検査 SQLite 狀態存儲中的セッション |
| `work-items` | work-items.js | トレース链接的 Linear、GitHub 作業项 |
| `session-inspect` | session-inspect.js | 從 dmux 或 Claude 历史目標发出仕様セッション快照 |
| `loop-status` | loop-status.js | 検査 Claude スクリプト中的过时ループ唤醒和保留中ツール結果 |
| `uninstall` | uninstall.js | 削除記録在 install-state 中的 ECC 管理ファイル |

### 使用サンプル

```bash
# 直接インストール言語
ecc typescript

# 带設定ファイル的インストール
ecc install --profile developer --target claude

# 查看インストール計画
ecc plan --profile core --target cursor

# 列出可用コンポーネント
ecc catalog profiles
ecc catalog components --family language

# 表示コンポーネント详情
ecc catalog show framework:nextjs

# 咨询推奨
ecc consult "security reviews"

# 検査已インストール
ecc list-installed --json

# 診断問題
ecc doctor --target cursor
ecc repair --dry-run

# 自動更新
ecc auto-update --dry-run

# 狀態查詢
ecc status --json
ecc status --exit-code
ecc status --markdown --write status.md

# 平台審計
ecc platform-audit --json --allow-untracked docs/drafts/

# セキュリティ扫描
ecc security-ioc-scan --home

# セッション管理
ecc sessions
ecc sessions session-active --json

# 作業项トレース
ecc work-items upsert linear-ecc-20 --source linear --source-id ECC-20 --title "Review control-plane contract" --status blocked
ecc work-items sync-github --repo affaan-m/ECC

# 検査ループ狀態
ecc loop-status --json

# アンインストール
ecc uninstall --target antigravity --dry-run
```

## install-apply.js

**パス**: `scripts/install-apply.js`

ECC インストール実行时,保持传统言語インストール入口点完整,同时將目標特定的变更逻辑移入可テスト的 Node コード。

### サポート的目標

| 目標 | 説明 |
|------|------|
| `claude` | インストール到 ~/.claude/,ルール/技能放在 rules/ecc 和 skills/ecc |
| `claude-project` | インストール到 ./.claude/ (プロジェクト级) |
| `cursor` | インストールルール、钩子、光标設定到 ./.cursor/ |
| `antigravity` | インストールルール、ワークフロー、技能、プロキシ到 ./.agent/ |
| `codex` | インストール共享プロキシ/設定到 ~/.codex/ |
| `gemini` | インストールプロジェクト级 Gemini 設定到 ./.gemini/ |
| `opencode` | インストール共享コマンド/钩子/設定到 ~/.opencode/ |
| `codebuddy` | インストールコマンド、プロキシ、技能到 ./.codebuddy/ |
| `joycode` | インストールコマンド、プロキシ、技能到 ./.joycode/ |
| `qwen` | インストール到 ~/.qwen/ |
| `zed` | インストール到 ./.zed/ |

### インストール方式

1. **传统言語パターン**: `install.sh <language>` (如 `ecc-install typescript`)
2. **Profile パターン**: `install.sh --profile <name> [--with <component>]... [--without <component>]...`
3. **モジュールパターン**: `install.sh --modules <id,id,...>`
4. **技能パターン**: `install.sh --skills <skill-id[,skill-id...]>`
5. **本地化パターン**: `install.sh --target claude|claude-project --locale <locale-code>`

### 选项

- `--dry-run` - 表示インストール計画但不コピーファイル
- `--json` - 发出机器可读的 JSON フォーマット
- `--config <path>` - 從ファイル載入インストール意圖

## claw.js

**パス**: `scripts/claw.js`

NanoClaw v2 - 面向 Everything Claude Code 的零外部依賴、セッション感知的 REPL,基づいて `claude -p` ビルド。

### 機能

- セッション历史持久化存儲在 `~/.claude/claw/`
- サポート动态載入技能作為上下文
- セッション圧縮以管理上下文長さ
- セッションブランチ和エクスポート
- 跨セッション搜索

### REPL コマンド

| コマンド | 説明 |
|------|------|
| `/help` | 表示帮助 |
| `/clear` | 清除當前セッション历史 |
| `/history` | 印刷完整對话历史 |
| `/sessions` | 列出已儲存的セッション |
| `/model [name]` | 表示/設定モデル |
| `/load <skill-name>` | 載入技能到活動上下文 |
| `/branch <session-name>` | 將當前セッションブランチ到新セッション |
| `/search <query>` | 跨セッション搜索 |
| `/compact` | 保留最近的轮次,圧縮旧上下文 |
| `/export <md\|json\|txt> [path]` | エクスポートセッション |
| `/metrics` | 表示セッション指標 |
| `exit` | 終了 REPL |

### 環境変数

| 変数 | デフォルト值 | 説明 |
|------|--------|------|
| `CLAW_SESSION` | `default` | 初始セッション名前 |
| `CLAW_MODEL` | `sonnet` | デフォルトモデル |
| `CLAW_SKILLS` | 空 | 載入的技能一覧 (逗号分隔) |

### 使用サンプル

```bash
# 使用デフォルトセッション起動
node scripts/claw.js

# 使用指定セッション起動
CLAW_SESSION=my-session node scripts/claw.js

# 載入技能起動
CLAW_SKILLS="javascript-patterns,python-testing" node scripts/claw.js
```

## harness-audit.js

**パス**: `scripts/harness-audit.js`

確定性ツール链審計ツール,基づいて显式ファイル/ルール検査。自動检测 ECC 仓庫パターン vs 消费者プロジェクトパターン。

### 審計クラス别

| クラス别 | 説明 |
|------|------|
| Tool Coverage | ツール覆寫完整性 |
| Context Efficiency | 上下文效率 |
| Quality Gates | 品質门禁 |
| Memory Persistence | 记忆持久化 |
| Eval Coverage | 評估覆寫 |
| Security Guardrails | セキュリティ护栏 |
| Cost Efficiency | 成本效率 |
| GitHub Integration | GitHub 統合 |
| Vercel/Netlify/Cloudflare/Fly Integration | 各平台統合 |

### 使用サンプル

```bash
# 審計當前ディレクトリ
node scripts/harness-audit.js

# 指定作用域
node scripts/harness-audit.js --scope repo
node scripts/harness-audit.js --scope hooks
node scripts/harness-audit.js --scope skills

# 指定根ディレクトリ
node scripts/harness-audit.js --root /path/to/project

# JSON フォーマット输出
node scripts/harness-audit.js --format json

# 組合使用
node scripts/harness-audit.js repo --format json --root .
```

## catalog.js

**パス**: `scripts/catalog.js`

发现 ECC インストールコンポーネント和設定ファイル的ツール。

### コマンド

#### profiles

列出すべてインストール設定ファイル。

```bash
node scripts/catalog.js profiles
node scripts/catalog.js profiles --json
```

#### components

列出インストールコンポーネント,可按家族和目標筛选。

```bash
node scripts/catalog.js components
node scripts/catalog.js components --family language
node scripts/catalog.js components --family framework
node scripts/catalog.js components --target claude
node scripts/catalog.js components --json
```

#### show

表示特定コンポーネント的詳細情報。

```bash
node scripts/catalog.js show <component-id>
node scripts/catalog.js show framework:nextjs --json
```

### コンポーネント家族

- `baseline` - 基础コンポーネント
- `language` - 言語コンポーネント (javascript, typescript, python, etc.)
- `framework` - フレームワークコンポーネント (nextjs, react, vue, etc.)
- `capability` - 能力コンポーネント
- `agent` - プロキシコンポーネント
- `skill` - 技能コンポーネント

## build-opencode.js

**パス**: `scripts/build-opencode.js`

ビルド OpenCode プラグイン的 TypeScript コード。

### 機能

1. クリーンアップ `dist` ディレクトリ
2. 使用 TypeScript コンパイル器コンパイル `.opencode` ディレクトリ下的コード
3. 输出到 `.opencode/dist`

### 前提條件

必要已インストール根ディレクトリ的開発依賴,確保 TypeScript コンパイル器可用。

### 使用

```bash
node scripts/build-opencode.js
```
