# テストランナー

本ドキュメント介紹 ECC プロジェクト的テスト構造和実行方式。

## テスト入口

**パス**: `tests/run-all.js`

集中式テストランナー,自動发现並実行すべてテストファイル。

### 使用メソッド

```bash
# 実行すべてテスト
node tests/run-all.js

# 実行单个テストファイル
node tests/lib/utils.test.js
node tests/hooks/hooks.test.js
```

### テストファイル发现

テストランナー通過 glob パターン `tests/**/*.test.js` 自動发现テストファイル。

ルール:
- ファイルmust位於 `tests/` ディレクトリ下
- ファイル名must以 `.test.js` 结尾
- ディレクトリ構造会被保留使用表示

### テスト統計

テストランナー会解析各テストファイル的输出来汇总統計:
- `Passed: <數量>`
- `Failed: <數量>`

## テスト構造

### ディレクトリ布局

```
tests/
├── run-all.js              # 主テストランナー
├── lib/                    # 庫ツールテスト
│   ├── utils.test.js
│   ├── package-manager.test.js
│   └── ...
├── hooks/                  # Hook 統合テスト
│   └── hooks.test.js
├── integration/            # 統合テスト
├── scripts/                # スクリプトテスト
├── commands/               # コマンドテスト
├── docs/                   # ドキュメントテスト
├── opencode-config.test.js
├── plugin-manifest.test.js
├── test_*.py               # Python テスト
└── conftest.py             # pytest 設定
```

### テスト類型

1. **ユニットテスト** (`*.test.js`)
   - テスト独立関数和ツール
   - 使用 Node.js 原生 assert 或 Jest 风格

2. **統合テスト** (`tests/integration/`)
   - テスト複数コンポーネント协作
   - テスト钩子完整流程

3. **Python テスト** (`test_*.py`)
   - pytest フォーマット的 Python テスト
   - 使用 `conftest.py` 設定

## テスト設定

### Node.js テスト

テストファイル使用標準的 Node.js assert モジュール:

```javascript
const assert = require('assert');

test('example test', () => {
  assert.strictEqual(actual, expected, 'Optional message');
});
```

### Python テスト (pytest)

位置: `tests/conftest.py`

pytest 設定ファイル,提供します共享的 fixtures 和設定。

```bash
# 実行 Python テスト
python -m pytest tests/

# 実行特定テスト
python -m pytest tests/test_builder.py
```

## テストベストプラクティス

### AAA パターン

```
test('テスト記述', () => {
  // Arrange - 准备テストデータ
  const input = 'test data';

  // Act - 実行被测操作
  const result = myFunction(input);

  // Assert - 検証結果
  assert.strictEqual(result, expected);
});
```

### テスト命名

使用記述性名前,清晰説明テスト的行為:

```javascript
test('returns empty array when no markets match query', () => {});
test('throws error when API key is missing', () => {});
test('falls back to substring search when Redis is unavailable', () => {});
```

### テスト隔离

各テストすべき:
- 独立実行,不依賴その他テスト
- クリーンアップ自己的テストデータ
- 使用 mock 隔离外部依賴

## CI 統合

在 `package.json` 中設定テストスクリプト:

```json
{
  "scripts": {
    "test": "validate-commands.js && tests/run-all.js"
  }
}
```

テストランナー設計為可在 CI 環境中作業:
- 非零終了码表示失敗
- JSON 输出使用自動化解析
- 清晰的エラーメッセージ使用デバッグ
