# 02 Agent コラボレーション

> 6類Agentの責務を理解し、組み合わせて复杂な開発任務を完了

## モジュール目標

本モジュールを完了すると、以下ができる：

- Claude Codeの6類Agent及各責務を識別
- タスクタイプに応じて適切なAgentを選択
- 複数のAgentを組み合わせて複雑なタスクを完了
- Agentコラボレーションモードとベストプラクティスを理解

## Agent 分類体系

Claude CodeのAgentは6大類に分類：

| Agent タイプ | コア責務 | 適用シーン |
|------------|----------|----------|
| **レビュー類** | コード品質、パターン規範、セキュリティ脆弱性 | コードレビュー、技術債清理 |
| **ビルド類** | コンパイルビルド、エラー修復、依存関係管理 | ビルド失敗、コンパイルエラー |
| **計画類** | タスク分解、アーキテクチャ設計、技術方案 | 新機能、アーキテクチャ変更 |
| **テスト類** | テスト作成、カバレッジ分析、テスト戦略 | TDD、テスト補完 |
| **セキュリティ類** | セキュリティ監査、脆弱性スキャン、コンプライアンス検査 | セキュリティ敏感的コード、決済モジュール |
| **アーキテクチャ類** | システム設計、パターン選択、技術選定 | 大規模リファクタリング、アーキテクチャ進化 |

## 6類Agent 详解

### 1. レビュー類 Agent

**責務**: コード品質レビュー、パターン規範検査

利用可能なAgent：
- `code-reviewer` - 通用コードレビュー
- `python-reviewer` - Python 專門レビュー
- `go-reviewer` - Go言語レビュー
- `rust-reviewer` - Rust セキュリティレビュー
- `security-reviewer` - セキュリティ脆弱性レビュー

**使用シーン**:
- コード提交前の品質検査
- 潜在的なコードの异臭を発見
- サードパーティコードのレビュー

### 2. ビルド類 Agent

**責務**: コンパイルビルド、エラー修復、依存関係管理

利用可能なAgent：
- `build-error-resolver` - ビルドエラー解析
- `go-build` - Go ビルド
- `rust-build` - Rust ビルド
- `kotlin-build` - Kotlin ビルド
- `flutter-build` - Flutter ビルド

**使用シーン**:
- ビルド失敗時のエラー修復
- 依存関係衝突解決
- クロスプラットフォームビルド問題

### 3. 計画類 Agent

**責務**: タスク分解、アーキテクチャ設計、実施計画

利用可能なAgent：
- `planner` - 実施計画生成
- `architect` - アーキテクチャ設計
- `tdd-guide` - TDD ワークフローガイド

**使用シーン**:
- 新機能開発計画
- 技術方案評価
- 複雑なタスク分解

### 4. テスト類 Agent

**責務**: テスト作成、カバレッジ向上、テスト戦略

利用可能なAgent：
- `tdd-guide` - テスト駆動開発
- `e2e-runner` - エンドツーエンドテスト
- `test-coverage` - カバレッジ分析

**使用シーン**:
- 新機能テストの作成
- テストカバレッジの向上
- E2E テストシーン設計

### 5. セキュリティ類 Agent

**責務**: セキュリティ監査、脆弱性スキャン、コンプライアンス検査

利用可能なAgent：
- `security-reviewer` - セキュリティ脆弱性レビュー
- `security-scan` - 自動化セキュリティスキャン

**使用シーン**:
- 決済と認証コードのレビュー
- 外部データ処理検査
- コンプライアンス検証

### 6. アーキテクチャ類 Agent

**責務**: システム設計、パターン選択、技術選定

利用可能なAgent：
- `architect` - アーキテクチャ設計
- `refactor-cleaner` - リファクタリングガイド
- `api-design-reviewer` - API デザインレビュー

**使用シーン**:
- マイクロサービス分割
- データベース設計
- API アーキテクチャ進化

## Agent コラボレーションモード

### モード 1: 直列コラボレーション

```
タスク → Planner → 実施 → Reviewer → テスト → 完了
```

適用：線形フロー、ステップ間に依存関係がある

### モード 2: 並列コラボレーション

```
            → Reviewer A ─→
タスク → 分解 ─→ Reviewer B ─→ 汇总 → 完了
            → Reviewer C ─→
```

適用：多角度レビュー、独立モジュールレビュー

### モード 3: カスケードコラボレーション

```
重大タスク → Architect → Planner → 実施 → 各级Reviewer → 合併
```

適用：複雑プロジェクト、大型機能開発

## Agent 組み合わせ例

### シーン: 新機能開発

```
1. /plan を使用して機能を計画
2. tdd-guide を使用して TDD 開発を行う
3. code-reviewer を使用してコードをレビュー
4. security-reviewer を使用してセキュリティを検査
5. verify を使用して機能を検証
```

### シーン: セキュリティ脆弱性の修復

```
1. security-reviewer を使用して脆弱性をスキャン
2. planner を使用して修復計画を制定
3. build-fix を使用して修復を實施
4. security-reviewer を使用して再検証
5. verify を使用して修復を確認
```

## Agent 選択ガイド

| タスクタイプ | 推奨 Agent | 理由 |
|----------|------------|------|
| 新機能開発 | planner + tdd-guide | 計画と TDD が必要 |
| コードレビュー | code-reviewer + security-reviewer | 品質とセキュリティ双重検査 |
| ビルド失敗 | build-error-resolver | エラー修復に专注 |
| アーキテクチャリファクタリング | architect + refactor-cleaner | アーキテクチャレベルのガイド |
| セキュリティ敏感 | security-reviewer (優先) | セキュリティ过关必须 |
| テスト作成 | tdd-guide + test-coverage | テスト戦略とカバレッジ |
