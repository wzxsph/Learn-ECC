# 演習：Agent コラボレーション

## 演習 1: コードレビュー類 Agent の使用

### 目標
コードレビュー Agent を呼び出し、そのフィードバックメカニズムを理解する。

### シーン
以下の Python コードを作成したと假设（実際の実行は不要、脑袋の中で考えるだけ）：

```python
def get_user_data(user_id, include_password=False):
    user = db.query("SELECT * FROM users WHERE id = " + user_id)
    if include_password:
        return user
    return {"name": user.name, "email": user.email}
```

### 手順

1. **このコードを分析**
   潜在的な問題を思考：
   - SQL 注入脆弱性
   - パスワード泄露リスク
   - エラー処理の欠落

2. **適切な Agent を選択**
   このコードには以下を使用すべき：
   - `security-reviewer` - セキュリティ脆弱性をチェック
   - `python-reviewer` - Python ベストプラクティスをチェック

3. **Agent フィードバックをシミュレート**
   Agent が提供すると思うフィードバックを記録：

   ```markdown
   ## コードレビューフィードバック

   ### セキュリティ問題 (CRITICAL)
   - SQL 注入：user_id を SQL 文に直接連結
   - パスワード泄露：include_password オプションがパスワードを露出可能性

   ### 品質問題 (HIGH)
   - エラー処理が欠落
   - 入力検証が欠落

   ### 改善提案
   - パラメータ化クエリを使用
   - パスワードフィールドを返さない
   - 例外処理を追加
   ```

4. **修復計画を制定**
   `/plan` の思路を使用して、修復計画を制定：

   ```markdown
   # 修復計画

   1. SQL 注入を修復：パラメータ化クエリを使用
   2. パスワード返送を移除：決してパスワードを返さない
   3. 入力検証を追加：user_id 形式を検証
   4. エラー処理を追加：ユーザーが存在しない情况的を処理
   ```

### 完了基準
- [ ] コードのセキュリティ問題を識別できる
- [ ] レビュー Agent のフィードバック構造を理解
- [ ] 修復計画を制定できる

## 演習 2: ビルドリペア類 Agent の使用

### 目標
ビルド Agent を使用してビルドエラーを診断・修復する方法を学習。

### シーン
Go プロジェクトに以下のビルドエラーがあると假设：

```
# github.com/myproject/pkg
./main.go:15: undefined: " fmt"
./main.go:23: undefined: calculateTotal
./main.go:31: too many arguments to function call
```

### 手順

1. **エラータイプを分析**
   エラーを分類

2. **修復フローを設計**
   Agent コラボレーションを使用

3. **各エラーの修復方案を設計**

4. **修復経験を記録**
   エラーパターンサマリーを作成

### 完了基準
- [ ] ビルドエラーのタイプとPriorityを分析できる
- [ ] 完全な修復フローを設計
- [ ] 一般的なエラーパターンを記録
