# 01 コマンドシステム精通

> Claude Codeの75+コマンドを掌握し、灵活组合で高效開発を実現

## モジュール目標

本モジュールを完了すると、以下ができる：

- すべてのコマンド分類下のコマンドを識別し、使用できる
- シーンに応じて最も適切なコマンドを選択できる
- 複数のコマンドを組み合わせてカスタムワークフローを作成できる
- コマンドドキュメントを使用してコマンドの使用法を快速検索できる

## コマンド概要

Claude Codeは75+コマンドを提供し、以下の分類をカバー：

| 分類 | 命令数 | 例コマンド |
|------|--------|----------|
| 開発フロー | 15+ | `/plan`, `/tdd`, `/build-fix`, `/verify` |
| コードレビュー | 10+ | `/code-review`, `/security-review`, `/python-review` |
| ビルドテスト | 15+ | `/go-build`, `/rust-build`, `/flutter-build` |
| プロジェクト管理 | 10+ | `/project-init`, `/projects`, `/santa-loop` |
| スキル管理 | 10+ | `/skill-create`, `/skill-stocktake`, `/skill-scout` |
| 学習研究 | 10+ | `/learn`, `/learn-eval`, `/deep-research` |
| 辅助ツール | 15+ | `/config`, `/jira`, `/pr`, `/e2e` |

## コマンドドキュメント構造

各コマンドは標準的なドキュメント構造を持つ：

```
コマンド名称
├── コマンド説明
├── 使用シーン
├── パラメータ説明
├── 使用例
└── 注意事項
```

## 常用コマンド快速参照

### 開発フロー
- `/plan` - 実施計画を作成
- `/tdd` - テスト駆動開発
- `/build-fix` - ビルドエラーを修復
- `/verify` - コード変更を検証
- `/review` - コードレビュー

### ビルド関連
- `/go-build`, `/rust-build`, `/kotlin-build` - 各种言語ビルド
- `/cpp-build`, `/flutter-build` - コンパイル型言語ビルド
- `/docker-build` - Dockerイメージビルド

### プロジェクト管理
- `/project-init` - 新プロジェクトを初期化
- `/projects` - すべてのプロジェクトを一覧表示
- `/santa-loop` - 自動化タスクループ

### スキル開発
- `/skill-create` - git歴史からSkillを作成
- `/skill-scout` - 利用可能なSkillsを検索
- `/learn` - セッションからパターンを抽出

## コマンドドキュメントの使用方法

配套の完全コマンドドキュメント：`../../参考ドキュメント/commands/01-核心工作流.md`

ドキュメントに含まれるもの：
- 各コマンドの詳細説明
- パラメータとオプションの説明
- 使用例と出力
- ベストプラクティス提案

## コマンドの組み合わせて使用

コマンドは組み合わせて使用でき、复杂ワークフローを実現：

```
/plan → /tdd → /build-fix → /code-review → /verify
```

各コマンドの出力は次のコマンドに渡され、自動化パイプラ인을形成。
