# 演習：コマンドシステム精通

## 演習 1: すべてのコマンド分類を探索

### 目標
Claude Codeのすべてのコマンド分類を理解し、各分類の主要なコマンドを識別する。

### 手順

1. **すべての利用可能なコマンドを一覧表示**
   - Claude Codeで `/` を入力してすべてのコマンドを表示
   - コマンドの分類オーガナイズ方式に注意

2. **各分類を探索**
   - 各分類で `/help` または `?` を使用してヘルプを取得
   - 各分類のコアコマンドと用途を記録

3. **コマンド清单を記録**
   自分のコマンド快速参照表を作成：

   | 分類 | コマンド | 用途 | 使用頻度 |
   |------|------|------|----------|
   | 開発フロー | /plan | 実施計画を作成 | ★★★ |
   | ... | ... | ... | ... |

### 完了基準
- [ ] 少なくとも5つの分類のコマンドを一覧表示できる
- [ ] 20+個の常用コマンドを記録
- [ ] コマンドの分類ロジックを理解

## 演習 2: 複数のコマンドを組み合わせて使用

### 目標
複数のコマンドを組み合わせて、连贯の開発ワークフローを作成する方法を学習。

### シーン
假设のビルドエラーを修復し、コードレビューを行う。

### 手順

1. **シーンシミュレーション**
   Goプロジェクトを開発中で、ビルドエラーが発生：

   ```
   # シーン設定（実際の実行は不要）
   修復が必要な以下のビルドエラーがあると假设：
   - import cycle detected
   - undefined: someFunction
   ```

2. **コマンド組み合わせ演習**
   次のコマンドの組み合わせを順番に思考し記録：

   a. まず `/plan` を使用して修復手順を計画
   b. `/go-build` を使用してビルドを試行
   c. `/build-fix` を使用してエラーを修復
   d. `/code-review` を使用してレビュー
   e. `/verify` を使用して修復を検証

3. **ワークフローを記録**
   修復ワークフロードキュメントを作成：

   ```markdown
   ## ビルド修復ワークフロー

   1. /plan - 問題を分析し、修復計画を制定
   2. /build-fix - 修復を実行
   3. /code-review - 修復品質を検査
   4. /verify - 機能が正常であることを検証
   ```

### 完了基準
- [ ] コマンド組み合わせのロジックを理解
- [ ] 少なくとも3つのカスタムワークフローを作成
- [ ] コマンド組み合わせのベストプラクティスを記録

## 演習 3: カスタムワークフローを作成

### 目標
プロジェクト需求に基づいて、自動化されたワークフローコマンド組み合わせを作成。

### シーン
假设のFlutterプロジェクトの開発ワークフローを作成。

### 手順

1. **ワークフロー目標を定義**
   Flutterプロジェクトには以下のワークフローが必要：

   - 日常開発：コーディング → テスト → ビルド
   - コードレビュー：自検 → レビュー → 修復
   - リリース準備：テスト → ビルド → 検証

2. **コマンド組み合わせを設計**

   **日常開発ワークフロー**:
   ```
   /flutter-build → /flutter-test → /verify
   ```

   **コードレビュワークフロー**:
   ```
   /flutter-review → /build-fix → /code-review
   ```

3. **ワークフロードキュメントを作成**
   プロジェクトで `CLAUDE_WORKFLOWS.md` を作成：

   ```markdown
   # Claude Code ワークフロー

   ## Flutter 開発ワークフロー

   ### 日常開発
   1. `/plan` を使用して任務を計画
   2. `/flutter-build` を使用してビルド
   3. `/flutter-test` を使用してテスト
   4. `/verify` を使用して検証

   ### コードレビュー
   1. `/flutter-review` を使用して自検
   2. `/build-fix` を使用して問題を修復
   3. `/code-review` を使用してレビューをリクエスト

   ### リリース準備
   1. 完全テストスイートを実行
   2. ビルド検証を実行
   3. `/verify` を使用して最終確認
   ```

4. **実践検証**
   実際のプロジェクトがある場合、このワークフローを実行试着。

### 完了基準
- [ ] 3つの完全なワークフローを設計
- [ ] ワークフロードキュメントを作成
- [ ] 実際のプロジェクトで少なくとも1つのワークフローをテスト

### 演習 3 ワークフローテンプレート

```markdown
# [プロジェクト名] Claude Code ワークフロー

## 開発フロー
- 日常: /plan → [コーディング] → /verify
- 修復: /build-fix → /code-review → /verify
- リリース: /test → /build → /verify

## 使用説明
1. 任何任務を開始するには `/plan` を使用
2. コーディング完了後 `/verify` を実行
3. 重要な変更には `/code-review` を使用
```
