# 01 - 命令システム精通

ECCのコマンドシステムを深く学习し、すべてのコマンドタイプと使用シーンを掌握する。

## 概述

ECCは **75個の[コマンド](../../参考ドキュメント/commands/01-コアワークフロー.md)** を提供し、3大类别に分類：斜杠命令（Slash Commands）、スクリプト命令、Agent命令。这些命令通过标准化的工作流大幅提升开发效率。

## 命令ファイル形式

各コマンドは `.md` ファイルであり、YAML frontmatter + Markdown 本文を含む：

```yaml
---
description: "コマンドの短い説明"
argument-hint: "[オプション参数提示]"
name: command-name          # オプション、デフォルトで文件名を使用
command: true              # オプション、CLI透伝コマンド
allowed_tools: ["Bash"]    # オプション、利用可能なツールを制限
---
```

### Frontmatter フィールド説明

| フィールド | 必填 | 説明 |
|------|------|------|
| description | はい | `/コマンドリストに表示 |
| argument-hint | いいえ | パラメータ提示 |
| name | いいえ | ファイル名をオーバーライドしてコマンド名として使用 |
| command | いいえ | CLI透伝コマンドとしてマーク |
| allowed_tools | いいえ | 利用可能なツールを制限 |

## 2. 命令链と組み合わせ

複数の命令を組み合わせして、复杂ワークフローを形成できる：

```markdown
# 典型開発フロー
/plan → /tdd → /code-review → /verify → /pr

# 問題修復フロー
/build-fix → /code-review → /test-coverage

# マルチモデル協力
/multi-plan → /multi-execute → /pr
```

## 3. カスタム命令作成

### 命令配置位置

| タイプ | パス |
|------|------|
| 命令ファイル | `commands/` |
| 登録メカニズム | `.claude-plugin/plugin.json` |

### 作成手順

1. `commands/` ディレクトリに `.md` ファイルを作成
2. YAML frontmatterを追加
3. 命令内容を記述（Markdown形式）
4. 命令は自動的にECCに登録

---

## 4. 最佳実践

### 命令選択ガイド

| シーン | 推奨命令 |
|------|----------|
| 新機能開発 | `/plan` → `/tdd` |
| コードレビュー | `/code-review` |
| Bug 修復 | `/build-fix` → `/verify` |
| テスト駆動 | `/tdd` → `/test-coverage` |
| PR作成 | `/pr` |
| パターン学習 | `/learn` → `/evolve` |

### 命令実行順序

ワークフロー原則に従う：
1. 計画：`/plan` または `/prp-plan`
2. 実装：`/tdd` または feature 命令
3. レビュー：`/code-review`
4. 検証：`/verify` または `/quality-gate`
5. 提交：`/pr`
