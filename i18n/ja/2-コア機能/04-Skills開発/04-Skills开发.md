# 04 - Skills 開発

再利用可能な Skills ワークフローを作成する方法を学習します。

## 概要

ECCは**234+の[Skills](../../参考ドキュメント/skills/最佳实践.md)**を提供し、分野別に分類され、ベストプラクティス、プログラミング言語、フレームワーク、テスト、セキュリティ、フロントエンドとデザイン、バックエンドとAPI、デプロイとDevOpsなどの16の分野をカバーします。

---

## 1. Skill 形式

### 1.1 基本構造

Skillsは Markdown 形式を使用し、明確な章構造を持ちます：

```markdown
# Skill 名称

## When to Use
## How It Works
## Examples
```

### 1.2 YAML Frontmatter

```yaml
---
name: skill-name
description: Skill 用途説明
category: 分類
---
```

### 1.3 章構造

| 章 | 説明 |
|------|------|
| **When to Use** | この Skill を使用する时机 |
| **How It Works** | 作動原理と手順 |
| **Examples** | 使用例 |

## 3. カスタム Skill の作成

### 3.1 作成手順

1. **用途を確定**：Skill が解決する問題を明確化
2. **構造を記述**：When to Use / How It Works / Examples 構造に従う
3. **YAML frontmatter を追加**：メタデータを提供
4. **skills/ ディレクトリに保存**：小文字 + ハイフンで命名

### 3.2 Skill の例

```markdown
---
name: my-custom-skill
description: 特定のタスクを実行するワークフロー
category: カスタム
---

# My Custom Skill

## When to Use

特定のタスク X を実行する必要がある場合、この Skill を使用。

## How It Works

1. ステップ1：データを準備
2. ステップ2：コア操作を実行
3. ステップ3：結果を検証
4. ステップ4：リソースを清理

## Examples

### 例 1：基本用法

```
My Custom Skill を実行して X を完了
```

### 例 2：高度な用法

```
カスタムパラメータで My Custom Skill を実行
```
```

### 3.3 命名規範

| タイプ | 規範 | 例 |
|------|------|------|
| ファイル名 | 小文字 + ハイフン | `code-review.md`, `tdd-workflow.md` |
| ディレクトリ | 分野別に組織 | `skills/プログラミング言語/` |
| 位置 | プロジェクト skills/ または 全般 ~/.claude/skills/ | |

## 4. ワークフロー設計

### 4.1 ステップ定義

各ステップを明確に定義：

```markdown
## How It Works

1. **需要分析**：ユーザー需要を理解
2. **方案設計**：実施方案を制定
3. **コード実装**：コードを記述
4. **テスト検証**：品質を確保
5. **ドキュメント更新**：変更を記録
```

### 4.2 条件分岐

異なるシーンを処理：

```markdown
## How It Works

1. 入力を分析
2. 条件に応じて分岐：
   - A の場合、ステップ X を実行
   - B の場合、ステップ Y を実行
   - それ以外の場合、ステップ Z を実行
3. 結果を統合
```

### 4.3 エラー処理

標準化されたエラー処理：

```markdown
## How It Works

1. 操作を実行
2. 結果をチェック
   - 成功：次のステップに進む
   - 失敗：错误を記録し修復提案を提供
3. 最終結果を返す
```

---

## 5. パラメータ化テンプレート

### 5.1 パラメータ定義

```markdown
## How It Works

1. 入力パラメータを受け取る：
   - `name`: 名前（必須）
   - `type`: タイプ（オプション、デフォルト：`standard`）
2. パラメータを処理
3.結果を返す
```

### 5.2 変数置換

```markdown
## Examples

### 基本用法

デフォルトパラメータを使用：
```
Skill を実行、名前は {{name}}
```
```

### カスタムパラメータ

カスタムタイプを指定：
```
Skill を実行、名前は {{name}}、タイプは {{type}}
```
```

## 6. 发布と共有

### 6.1 ローカル存储

| 位置 | 説明 |
|------|------|
| プロジェクト `skills/` | プロジェクト固有の Skills |
| `~/.claude/skills/` | 全般で使用可能な Skills |

### 6.2 インポートとエクスポート

```bash
# Skill をファイルにエクスポート
/skill-export my-skill

# ファイルから Skill をインポート
/skill-import path/to/skill.md
```

### 6.3 Skill マーケット

ECC はマーケットから共有 Skills をインストールするのをサポート。

## 7. Skill 作成ワークフロー

`/skill-create` コマンドを使用して git 歴史を分析し、Skill ファイルを生成：

```bash
# git 歴史を分析して Skill を生成
/skill-create

# 分析範囲を指定
/skill-create --days 30

# 出力パスを指定
/skill-create --output ./my-skills/
```

## 8. ベストプラクティス

### 8.1 設計原則

| 原則 | 説明 |
|------|------|
| **単一責務** | 各 Skill は1つのことだけを実行 |
| **組み合わせ可能** | Skill は組み合わせて使用可能 |
| **再利用性** | 汎用的に設計，而非特定 |
| **テスト可能** | 検証ステップを提供 |

### 8.2 ドキュメント品質

| 要件 | 説明 |
|------|------|
| 明確な When to Use | ユーザーに使用时机を知らせる |
| 詳細な How It Works | ステップが明確で実行可能 |
| 実用的な Examples | 実際のユースケースを提供 |
| エラー処理説明 | ユーザーにエラーの处理方法を通知 |

### 8.3 メンテナンス提案

- 最新の実践を反映するために定期的に更新
- ユーザーのフィードバックに基づいて改善
- シンプルで保ち、過度に複雑にならない
- バージョン番号を追加して追跡を容易に
