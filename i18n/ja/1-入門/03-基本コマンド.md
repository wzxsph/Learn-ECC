# 03 - 基本コマンド

## コマンドタイプ

ECCは3種類のコマンドを提供：

### 1. スラッシュコマンド（Slash Commands）

直接入力して使用：

| コマンド | 機能 | 例 |
|------|------|------|
| `/ecc:plan` | 実施計画を作成 | `/ecc:plan "ユーザー認証を追加"` |
| `/ecc:code-review` | コードレビュー | `/ecc:code-review` |
| `/ecc:build-fix` | ビルドエラーを修復 | `/ecc:build-fix` |
| `/ecc:learn` | セッションからパターンを抽出 | `/ecc:learn` |
| `/ecc:skill-create` | Git歴史からSkillを生成 | `/ecc:skill-create` |
| `/ecc:sessions` | セッション履歴管理 | `/ecc:sessions` |

注意：手動インストールバージョンはプレフィックスなしのコマンドを使用（如`[/plan](../参考ドキュメント/commands/01-核心工作流.md)`），プラグインインストールは`/ecc:`プレフィックスを使用。

### 2. Skills（主要ワークフロー面）

SkillsはECCの主要ワークフロー呼び出し方式：

| Skill | 機能 |
|-------|------|
| `[tdd-workflow](../参考ドキュメント/skills/测试.md)` | テスト駆動開発 |
| `[e2e-testing](../参考ドキュメント/skills/测试.md)` | エンドツーエンドテスト |
| `security-review` | セキュリティレビュー |
| `verification-loop` | 検証ループ |
| `search-first` | 研究優先ワークフロー |

呼び出し方式：
```
[tdd-workflow](../参考ドキュメント/skills/测试.md) skillを使用
```

### 3. Agent コマンド

Agentを通じてサブタスクを実行：

| Agent | 用途 |
|-------|------|
| `planner` | 実施計画 |
| `code-reviewer` | コードレビュー |
| `build-error-resolver` | ビルドエラー修復 |
| `security-reviewer` | セキュリティレビュー |
| `architect` | アーキテクチャ設計 |

呼び出し方式：
```
@planner
```

## 常用ワークフロー

### 新機能開発

```
/ecc:plan "ユーザー認証機能を追加"
→ plannerが実施ブループリントを作成
[tdd-workflow](../参考ドキュメント/skills/测试.md) skillを使用
→ tdd-guideがテストファーストを強制
/ecc:code-review
→ code-reviewerがコードを検査
```

### Bug修正

```
[tdd-workflow](../参考ドキュメント/skills/测试.md) skillを使用
→ tdd-guide: 再現テストを作成
→ 修正を實施し、テストが通過することを検証
/ecc:code-review
→ code-reviewer: 回帰を検査
```

### ローンチ準備

```
/ecc:security-scan
→ security-reviewer: OWASP Top 10監査
[e2e-testing](../参考ドキュメント/skills/测试.md) skillを使用
→ e2e-runner: 重要ユーザー flowテスト
/ecc:test-coverage
→ 80%+ カバレッジを検証
```

## MCP コマンド

| コマンド | 機能 |
|------|------|
| `/mcp` | MCPサーバーを管理 |
| `/ecc:status` | ステータスを確認 |
