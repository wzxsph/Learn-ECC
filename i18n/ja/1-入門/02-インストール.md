# 02 - インストール設定

## 環境要件

- Node.js >= 18
- npm / pnpm / yarn / bun のいずれか
- Claude Code CLI v2.1.0+ がインストール済み

## インストール方法

### 方法一：プラグインインストール（推奨）

#### 1. プラグイン市場を追加

```bash
/plugin marketplace add https://github.com/affaan-m/ECC
```

#### 2. プラグインをインストール

```bash
/plugin install ecc@ecc
```

#### 3. Rulesをインストール（プラグインは自動ではインストールされない）

```bash
# リポジトリをクローン
git clone https://github.com/affaan-m/ECC.git
cd ECC

# ルールをユーザーディレクトリにコピー
mkdir -p ~/.claude/rules/ecc
cp -r rules/common ~/.claude/rules/ecc/
# 技術スタックを選択
cp -r rules/typescript ~/.claude/rules/ecc/   # または python, golang, swift, php
```

### 方法二：手動インストール

より精细に控制了たい場合：

```bash
# リポジトリをクローン
git clone https://github.com/affaan-m/ECC.git
cd ECC

# 依存関係をインストール
npm install        # または pnpm install | yarn install | bun install

# agentsをインストール
cp agents/*.md ~/.claude/agents/

# skillsをインストール（主要ワークフロー面）
mkdir -p ~/.claude/skills/ecc
cp -r .agents/skills/* ~/.claude/skills/ecc/

# rulesをインストール
mkdir -p ~/.claude/rules/ecc
cp -r rules/common ~/.claude/rules/ecc/
cp -r rules/typescript ~/.claude/rules/ecc/

# commandsをインストール（オプション、ECCはskillsに移行中）
mkdir -p ~/.claude/commands
cp commands/*.md ~/.claude/commands/
```

## インストール確認

```bash
# プラグインがインストールされたか確認
/plugin list ecc@ecc

# 利用可能なコマンドを確認
/ecc:plan
/ecc:code-review
```

##  基本設定

### パッケージマネージャー設定

ECCは自動的に好みのパッケージマネージャーを検出しますが、手動設定も可能：

```bash
# 環境変数方式
export CLAUDE_PACKAGE_MANAGER=pnpm

# またはコマンドを使用
/setup-pm
```

優先順位：環境変数 > プロジェクト設定 > package.json > lockファイル > 全般設定

### Hook ランタイム制御

```bash
# Hook厳格度（デフォルト: standard）
export ECC_HOOK_PROFILE=standard

# 特定のHooksを無効化
export ECC_DISABLED_HOOKS="pre:bash:tmux-reminder,post:edit:typecheck"

# SessionStartコンテキスト上限（デフォルト: 8000文字）
export ECC_SESSION_START_MAX_CHARS=4000
```

### モデル設定

`~/.claude/settings.json`での設定推奨：

```json
{
  "model": "sonnet",
  "env": {
    "MAX_THINKING_TOKENS": "10000",
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "50"
  }
}
```
