# Backend-e-APIHabilidades

## Visão Geral

Backend-e-API habilidades são usadas para construção de aplicações server-side, design de API e operações de banco de dados.

## Habilidades Principais

### backend-patterns

**Propósito**: Melhores-Práticas de desenvolvimento backend

**Conceitos Centrais**:
- Arquitetura em camadas (Controller → Service → Repository)
- Injeção de dependência
- Estratégias de tratamento de erros
- Padrões de cache
- Integração de message queue

---

### api-design

**Propósito**: Design de API e padrões RESTful

**Conceitos Centrais**:
- Design de rotas RESTful
- Formato de request/response
- Autenticação e autorização
- Estratégias de versionamento
- Rate limiting e cache

---

### api-connector-builder

**Propósito**: Ferramenta de construção de conectores API

**Cenários de Uso**:
- Integração com APIs de terceiros
- Processamento de Webhooks
- Implementação de fluxos OAuth
- Mecanismos de retry de erros de API

---

## Habilidades de Banco de Dados

### database-migrations

**Propósito**: Gerenciamento de migrações de banco de dados

**Conceitos Centrais**:
- Organização de arquivos de migração
- Estratégias de rollback de dados
- Gerenciamento de seed data
- Deploy zero-downtime

---

### mysql-patterns

**Propósito**: Melhores-Práticas de banco de dados MySQL

**Conceitos Centrais**:
- Otimização de índices
- Otimização de queries
- Estratégias de sharding
- Replicação master-slave

---

### postgres-patterns

**Propósito**: Melhores-Práticas de banco de dados PostgreSQL

**Conceitos Centrais**:
- Operações JSONB
- Busca em texto completo
- Tabelas particionadas
- Estratégias de índice avançado

---

### redis-patterns

**Propósito**: Padrões de cache Redis

**Conceitos Centrais**:
- Estratégias de cache (Cache-Aside, Write-Through)
- Locks distribuídos
- Armazenamento de sessão
- Publicação/assinatura de mensagens

---

### clickhouse-io

**Propósito**: Armazenamento colunar ClickHouse

**Cenários de Uso**:
- Queries analíticas
- Armazenamento de logs
- Análise em tempo real
- Pipelines de big data

---

## Message Queue

### logistics-exception-management

**Propósito**: Gerenciamento de exceções logísticas

**Cenários de Uso**:
- Processamento de exceções de pedidos
- Fluxo de devoluções
- Sincronização de inventário

---
