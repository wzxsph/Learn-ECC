# 练习：04-DesenvolvimentoDeSkills

## 练习 1: 探索技能分类

### 目标
了解 Claude Code 的所有技能分类，并识别每个分类中的关键技能。

### 步骤

1. **列出所有可用技能**
   - 使用 `/skills` 命令查看所有可用技能
   - 注意技能的分类组织方式

2. **探索每个分类**
   - 对每个分类，尝试获取帮助信息
   - 记录每个分类的核心技能和用途

3. **记录技能清单**
   创建自己的技能速查表：

   | 分类 | 技能 | 用途 | 使用频率 |
   |------|------|------|----------|
   | 开发工具链 | /go-build | Go 构建 | ★★★ |
   | ... | ... | ... | ... |

### 验收标准
- [ ] 能够列出至少 5 个分类的技能
- [ ] 记录了 20+ 个常用技能
- [ ] 理解了技能的分类逻辑

## 练习 2: 创建自定义技能

### 目标
学习如何创建自定义技能来扩展 Claude Code 的能力。

### 场景
为一个假设的项目创建一个代码质量检查技能。

### 步骤

1. **定义技能需求**
   你的项目需要以下技能：
   - 代码风格检查
   - 复杂度检查
   - 命名规范检查

2. **设计技能结构**

   ```markdown
   # 自定义技能：代码质量检查

   ## 触发条件
   当用户使用 `/quality-check` 时触发

   ## 执行步骤
   1. 运行代码风格检查
   2. 检查圈复杂度
   3. 验证命名规范
   4. 生成报告

   ## 输出格式
   - 发现的问题列表
   - 严重程度评级
   - 修复建议
   ```

3. **实现技能**
   按照 Claude Code 技能创建流程实现技能。

### 验收标准
- [ ] 理解了技能的结构
- [ ] 创建了完整的技能文档
- [ ] 在实际项目中测试了技能
