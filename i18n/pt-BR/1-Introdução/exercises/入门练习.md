# Exercícios para Iniciantes

Complete os seguintes exercícios para consolidar as bases do ECC.

## Exercício 1: Verificação de Ambiente

**Objetivo**: Confirmar que ambiente ECC está corretamente instalado

**Passos**:
1. Executar suite de testes: `node tests/run-all.js`
2. Verificar que todos os testes passam
3. Listar estrutura do diretório `.claude/`

**Critérios de Aceitação**:
- Todos os testes passam
- Directórios agents, skills, hooks, rules são visíveis

---

## Exercício 2: Usar Comandos com Barra

**Objetivo**: Familiarizar-se com comandos com barra frequentes

**Passos**:
1. Iniciar Claude Code
2. Executar os seguintes comandos em sequência, observar saída:
   - `/help` ou `/ecc:plan`
   - `/ecc:code-review`
3. Registrar formato de saída de cada comando

**Critérios de Aceitação**:
- Entender função e formato de saída de cada comando

---

## Exercício 3: Criar Agent Simples

**Objetivo**: Entender formato de arquivo Agent

**Passos**:
1. Ver arquivos de Agent existentes no diretório `agents/`
2. Usar formato de `planner.md` como referência
3. Criar um `hello-agent.md` simples, com função de output de saudação

**Critérios de Aceitação**:
- Formato do novo arquivo Agent correto
- Contém YAML frontmatter (name, description, tools, model)

---

## Exercício 4: Executar Hook

**Objetivo**: Entender mecanismo de Hook

**Passos**:
1. Ver configuração de Hook no diretório `hooks/`
2. Executar `node scripts/hooks/run-with-flags.js --help` (se suportado)
3. Entender momento de disparo e lógica de execução do Hook

**Critérios de Aceitação**:
- Consegue descrever princípio de funcionamento de Hooks
- Entende diferença entre PreToolUse e PostToolUse

---

## Exercício 5: Tarefa Completa

**Objetivo**: Completar um pequeno projeto

**Tarefa**: Criar um formatador de nomes de pessoa

**Requisitos Funcionais**:
- `formatName(name)` - Formatar como "Sobrenome, Nome"
- `initials(name)` - Retornar sigla

**Fluxo de Execução**:
1. Usar `/ecc:plan` para planejar
2. Usar `/tdd` para desenvolver
3. Usar `/ecc:code-review` para revisar

**Critérios de Aceitação**:
- Implementar completamente ambas funções
- Incluir casos de teste
- Passar na revisão de código
