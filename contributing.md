# Learn-ECC 贡献指南

欢迎为 Learn-ECC 贡献内容！本指南说明如何添加新内容、改进现有内容或提交修改。

## 贡献类型

### 1. 添加新内容

可以贡献的内容类型：
- 新章节或模块
- 新的练习题
- 新的实战项目
- 速查表补充
- 翻译改进

### 2. 改进现有内容

- 修正错误
- 改进解释的清晰度
- 添加更多示例
- 完善练习答案
- 更新过时的信息

### 3. 报告问题

- 错别字或语法错误
- 代码示例不工作
- 链接失效
- 内容遗漏

## 内容格式标准

### Markdown 格式

所有内容必须使用 Markdown 格式，包含以下部分：

```markdown
# 标题

## 学习目标

完成本节学习后，你将能够：
- 目标1
- 目标2

---

## 正文内容

### 子标题

内容...

---

## 练习

### 练习1

任务描述...

### 验证方法

1. 验证步骤1
2. 验证步骤2

---

## 下一步

- 下一章：[链接](./next.md)
- 返回：[目录](../README.md)
```

### 文件命名

- 使用中文命名
- 使用连字符 `-` 分隔词语
- 章节文件命名：`01-章节名.md`、`02-章节名.md`
- 练习文件命名：`练习-名称.md`

### 链接规范

- 教科书链接：`./参考文档/文件名.md`
- 课程内链接：`./章节名.md`
- 相对路径：从当前文件位置计算

## 提交修改

### 步骤

1. Fork 仓库
2. 创建分支：`git checkout -b feature/新内容描述`
3. 进行修改
4. 提交：`git commit -m "feat: 添加新内容"`
5. 推送到远程：`git push origin feature/新内容描述`
6. 创建 Pull Request

### 提交信息规范

```
<type>: <description>

<optional body>
```

Type 类型：
- `feat`: 新功能
- `fix`: 错误修复
- `docs`: 文档更新
- `test`: 测试更新
- `refactor`: 重构

## 质量检查

提交前请检查：

- [ ] Markdown 格式正确
- [ ] 所有链接有效
- [ ] 代码示例可运行
- [ ] 练习有验证方法
- [ ] 中文无错别字
- [ ] 符合格式标准

## 问题反馈

如果发现问题或建议，请创建 Issue，包含：
- 问题描述
- 所在文件位置
- 建议的改进方式
