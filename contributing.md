# Learn-ECC 贡献指南

欢迎为 Learn-ECC 贡献内容！本指南说明如何添加新内容、改进现有内容或提交修改。

> **⚠️ 多语言说明**
> 
> 其他语言的翻译文档存在质量问题，目前**仅中文版本正常维护**。
> 如需帮助修正其他语言，请提交 PR 并标注语言。
> 
> 英文参考文档位于 `./i18n/en/` 目录下。

## 贡献类型

### 1. 添加新内容

可以贡献的内容类型：
- 新章节或模块
- 新的练习题
- 新的实战项目
- 速查表补充
- 翻译改进（仅中文）

### 2. 改进现有内容

- 修正错误
- 改进解释的清晰度
- 添加更多示例
- 完善练习答案
- 更新过时的信息

### 3. 报告问题

- 错别字或语法错误
- 代码示例不工作
- 链接失效
- 内容遗漏

## 内容格式标准

### Markdown 格式

所有内容必须使用 Markdown 格式，包含以下部分：

```markdown

## 学习目标

完成本节学习后，你将能够：
- 目标1
- 目标2

---

## 正文内容

### 子标题

内容...

---

## 练习

### 练习1

任务描述...

### 验证方法

1. 验证步骤1
2. 验证步骤2

---

## 下一步

- 下一章：[链接](./next.md)
- 返回：[目录](../README.md)
```

### 文件命名

- **主课程**：使用中文命名，如 `01-ECC简介.md`、`02-安装配置.md`
- **章节分隔**：使用连字符 `-`，如 `01-命令系统精通/`
- **参考文档**：使用中文，如 `参考文档/agents/代码审查类.md`
- **i18n 目录**：英文目录名 + 对应语言文件名，如 `i18n/en/`

## 提交修改

### 步骤

1. Fork 仓库
2. 创建分支：`git checkout -b feature/新内容描述`
3. 进行修改
4. 提交：`git commit -m "feat: 添加新内容"`
5. 推送到远程：`git push origin feature/新内容描述`
6. 创建 Pull Request

### 提交信息规范

```
<type>: <description>

<optional body>
```

Type 类型：
- `feat`: 新功能
- `fix`: 错误修复
- `docs`: 文档更新
- `test`: 测试更新
- `refactor`: 重构

### 分支策略

- `main`: 主分支，稳定版本
- `i18n/*`: 多语言分支（如有修改）

## 质量检查

提交前请检查：

- [ ] Markdown 格式正确
- [ ] 所有链接有效（指向正确文件）
- [ ] 代码示例可运行
- [ ] 练习有验证方法
- [ ] 中文无错别字
- [ ] 符合格式标准
- [ ] 如修改参考文档，同步更新英文原版

## 多语言贡献

如果你想帮助修正其他语言（en, ja, ko, de, ru, tr, pt-BR, vi, th），请注意：

1. **质量优先**：翻译后请确保内容准确、通顺
2. **文件名匹配**：使用目标语言的命名规范
3. **内部链接**：确保链接指向正确文件名
4. **测试验证**：提交前验证链接有效性
